第1章 认识OpenClaw ​

本章学习目标: 理解OpenClaw的核心价值、工作原理和适用场景 预计用时: 30-45分钟 前置要求: 无

1.1 背景与诞生 ​

从"对话"到"执行"的AI进化 ​

2022年，ChatGPT横空出世，让全世界看到了大语言模型的威力。但很快，人们发现了一些局限：

局限具体表现
只能对话"帮我整理桌面文件" → "我可以告诉你如何整理..."
无法访问本地不能读取你的文件、日历、邮件
没有记忆每次对话都是重新开始
数据上传敏感文件需要上传到第三方服务器

OpenClaw的创造者Peter Steinberger也有同样的困扰。他想要一个能够真正帮他做事的AI助手。

2025年11月，他将这个工具开源，命名为Clawdbot。没想到，这个项目迅速引爆GitHub：

  • 一周内获得100,000+ Stars
  • 超过Linux成为GitHub历史上增长最快的开源项目

经过短暂更名（Moltbot）后，最终定名为OpenClaw。

3. 浏览器控制 ​

1.3 工作原理 ​

架构概览 ​

用户层 → Gateway层 → Agent层 → 工具层 → 系统层

当用户在飞书发送消息时：

  1. 用户发送: "帮我整理下载文件夹"
  2. 飞书Bot接收: 转发到OpenClaw Gateway
  3. Gateway路由: 确定Agent处理
  4. Agent处理: LLM理解 → 任务规划 → 工具选择
  5. 执行操作: 调用read/exec工具
  6. 返回结果: 通过飞书Bot返回

核心配置文件 ​

openclaw.json 位于 ~/.openclaw/，定义：

  • 使用的AI模型和API密钥
  • 启用的聊天渠道
  • 安装的技能
  • 安全设置

1.5 硬件与软件要求 ​

最低配置:

  • CPU: 任意现代处理器
  • 内存: 2GB RAM
  • 存储: 1GB可用空间

推荐配置:

  • CPU: 4核+
  • 内存: 4GB+ RAM
  • 存储: 10GB+ SSD

软件要求:

  • Node.js >= 22.0
  • 操作系统: macOS / Linux / Windows (WSL)

本章小结 ​

  1. OpenClaw是一个开源AI智能体平台，能在你自己设备上运行
  2. 六大核心能力：多渠道通信、文件管理、浏览器控制、系统操作、技能系统、记忆系统
  3. 成本优势明显：相比ChatGPT Plus可节省73%-96%
  4. 数据完全私有
  5. 需要一定技术门槛，但本教程会手把手教你

常见问题 ​

Q1: OpenClaw和ChatGPT有什么区别？ A: ChatGPT只能回答问题；OpenClaw可以执行实际任务。

Q2: 我需要会编程才能用OpenClaw吗？ A: 基础使用不需要编程。深度定制需要JavaScript/Node.js基础。

Q3: OpenClaw安全吗？ A: 相对安全。数据保存在自己设备上，但要注意安全策略配置。

Q4: 没有服务器可以运行OpenClaw吗？ A: 可以！可以在自己电脑运行。需要24/7在线建议用云服务器。
