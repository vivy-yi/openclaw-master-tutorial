10.2 钉钉集成 ​

  • 掌握钉钉插件安装

2.1 插件选择 ​

推荐插件 ​

插件作者特点
openclaw-channel-dingtalksoimy官方推荐，Stream 模式，无需公网 IP
openclaw-chinaBytePioneer-AI统一包，包含所有中国平台

插件特点 ​

![钉钉集成]/assets/diagrams/67_dingtalk_integration.png)

钉钉插件核心功能：

✅ Stream 模式 - WebSocket 长连接，无需公网 IP
✅ 私聊支持 - 直接与机器人对话
✅ 群聊支持 - @机器人触发
✅ 多种消息类型 - 文本/图片/语音/视频/文件/卡片
✅ Markdown 回复 - 富文本格式支持
✅ 互动卡片 - 流式更新，AI 实时输出
✅ 引用消息 - 支持文字/图片/文件/视频引用
✅ 附件文本抽取 - PDF/DOCX 自动提取正文

2.2 安装 ​

方法一：npm 安装（推荐） ​

bash
# 安装钉钉插件
openclaw plugins install @soimy/dingtalk

方法二：统一安装（包含所有中国平台） ​

bash
# 安装中国平台统一包
openclaw plugins install @openclaw-china/channels

# 运行配置向导
openclaw china setup

方法三：本地源码安装 ​

bash
# 克隆仓库
git clone https://github.com/soimy/openclaw-channel-dingtalk.git
cd openclaw-channel-dingtalk

# 安装依赖
npm install

# 链接模式安装
openclaw plugins install -l .

国内网络环境安装 ​

bash
# 使用镜像源
NPM_CONFIG_REGISTRY=https://registry.npmmirror.com openclaw plugins install @soimy/dingtalk

2.3 钉钉开放平台配置 ​

创建应用 ​

  1. 访问 钉钉开发者后台
  2. 创建企业内部应用
  3. 添加「机器人」能力
  4. 配置消息接收模式为 Stream 模式
  5. 发布应用

配置权限 ​

在应用的权限管理页面，开启以下权限：

必需权限：
├── Card.Instance.Write - 创建和投放卡片实例
├── Card.Streaming.Write - 对卡片进行流式更新
├── im:message - 机器人消息发送接收
└── im:resource - 媒体文件上传下载

可选权限（引用群文件时需要）：
├── ConvFile.Space.Read - 群文件空间读权限
├── Storage.File.Read - 企业存储文件读权限
├── Storage.DownloadInfo.Read - 企业存储文件下载信息
└── Contact.User.Read - 通讯录用户信息读权限

获取凭证 ​

从开发者后台获取以下信息：

  • Client ID (AppKey) - 应用标识
  • Client Secret (AppSecret) - 应用密钥
  • Robot Code - 机器人代码
  • Corp ID - 企业 ID
  • Agent ID - 应用 ID

2.4 OpenClaw 配置 ​

交互式配置（推荐） ​

bash
# 启动配置向导
openclaw onboard

配置流程：

  1. 选择 dingtalk 或 DingTalk (钉钉)
  2. 输入 Client ID (AppKey)
  3. 输入 Client Secret (AppSecret)
  4. 可选配置 Robot Code、Corp ID、Agent ID
  5. 选择卡片模式
  6. 选择私聊/群聊策略

手动配置 ​

编辑 ~/.openclaw/openclaw.json：

json
{
  "plugins": {
    "enabled": true,
    "allow": ["dingtalk"]
  },
  "channels": {
    "dingtalk": {
      "enabled": true,
      "clientId": "dingxxxxxx",
      "clientSecret": "your-app-secret",
      "robotCode": "dingxxxxxx",
      "corpId": "dingxxxxxx",
      "agentId": "123456789",
      "dmPolicy": "open",
      "groupPolicy": "open",
      "messageType": "markdown"
    }
  }
}

配置选项 ​

选项类型默认值说明
enabledbooleantrue是否启用
clientIdstring必填应用 AppKey
clientSecretstring必填应用 AppSecret
robotCodestring-机器人代码
corpIdstring-企业 ID
agentIdstring-应用 ID
dmPolicystring"open"私聊策略：open/pairing/allowlist
groupPolicystring"open"群聊策略：open/allowlist
messageTypestring"markdown"消息类型：markdown/card
cardTemplateIdstring-AI 卡片模板 ID
cardTemplateKeystring"content"卡片内容字段键

私聊策略 (dmPolicy) ​

值说明
open任何人都可以私聊机器人
pairing新用户需要配对码验证
allowlist只有白名单用户可用

群聊策略 (groupPolicy) ​

值说明
open任何群都可以 @机器人
allowlist只有白名单群可用

2.6 AI 互动卡片 ​

json

2.8 故障排除 ​

常见问题 ​

Q: 收不到消息

  1. 确认应用已发布
  2. 确认消息接收模式是 Stream
  3. 检查 Gateway 日志：openclaw logs | grep dingtalk

Q: 群消息无响应

  1. 确认机器人已添加到群
  2. 确认正确 @机器人
  3. 确认是企业内部群

Q: 连接失败

bash
# 运行连接检查脚本
bash scripts/dingtalk-connection-check.sh --config ~/.openclaw/openclaw.json

关键设置检查清单 ​

  • [ ] 应用为企业内部应用，已发布版本
  • [ ] 可见范围为"全员员工"
  • [ ] 已开启机器人能力
  • [ ] 消息接收方式为"Stream 模式"

2.9 反馈学习功能 ​

插件支持本地反馈学习闭环：

常用命令 ​

bash
# 查看自己的 ID
我是谁
/learn whoami

# 查看当前会话 ID
这里是谁

# 注入规则到当前会话
/learn here #@# <规则>

# 注入规则到指定目标
/learn target <conversationId> #@# <规则>

# 注入全局规则
/learn global <规则>

# 查看/禁用/删除规则
/learn list
/learn disable <ruleId>
/learn delete <ruleId>

配置示例 ​

json
{
  "channels": {
    "dingtalk": {
      "learningEnabled": true,
      "learningAutoApply": false,
      "learningNoteTtlMs": 21600000
    }
  }
}
