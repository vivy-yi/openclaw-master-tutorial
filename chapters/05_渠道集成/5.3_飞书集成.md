## 8.2.2 安装

### npm 安装（推荐）

```bash
openclaw plugins install @m1heng-clawd/feishu
```

### 升级

```bash
openclaw plugins update feishu

# 查看版本
openclaw plugins list | rg -i feishu
```

### Windows 排错

如果安装失败，使用 tarball 手动安装：

```bash
# 方案 A：下载最新插件
npm pack @m1heng-clawd/feishu
openclaw plugins install ./m1heng-clawd-feishu-<version>.tgz

# 方案 B：使用镜像
TARBALL_URL="$(npm view @m1heng-clawd/feishu dist.tarball)"
curl -L -o feishu-latest.tgz "$TARBALL_URL"
openclaw plugins install ./feishu-latest.tgz
```

## 8.2.3 飞书开放平台配置

### 创建应用

1. 访问 [飞书开放平台](https://open.feishu.cn)
2. 点击「创建应用」
3. 填写应用信息
4. 获取 **App ID** 和 **App Secret**

### 配置权限

#### 必需权限

| 权限 | 范围 | 说明 |
|------|------|------|
| `im:message` | 消息 | 发送和接收消息 |
| `im:message.p2p_msg:readonly` | 私聊 | 读取发给机器人的私聊消息 |
| `im:message.group_at_msg:readonly` | 群聊 | 接收群内 @机器人的消息 |
| `im:message:send_as_bot` | 发送 | 以机器人身份发送消息 |
| `im:resource` | 媒体 | 上传和下载图片/文件 |

#### 可选权限

| 权限 | 范围 | 说明 |
|------|------|------|
| `contact:user.base:readonly` | 用户信息 | 获取用户基本信息 |
| `im:message.group_msg` | 群聊 | 读取所有群消息（敏感权限）|
| `im:message:readonly` | 读取 | 获取历史消息 |
| `im:message:update` | 编辑 | 更新已发送消息 |
| `im:message:recall` | 撤回 | 撤回已发送消息 |
| `im:message.reactions:read` | 表情 | 查看消息表情回复 |

## 8.2.5 事件订阅配置

> ⚠️ **这是最容易遗漏的配置！** 如果机器人能发消息但收不到消息，请检查此项。

### 配置步骤

1. 进入飞书应用的 **事件与回调** 页面
2. 根据 `connectionMode` 选择订阅方式：
   - **使用长连接接收事件** → 对应 `connectionMode: "websocket"`
   - **使用请求地址接收事件** → 对应 `connectionMode: "webhook"`

### 必需事件

| 事件 | 说明 |
|------|------|
| `im.message.receive_v1` | 接收消息（必需）|

### 可选事件

| 事件 | 说明 |
|------|------|
| `im.message.message_read_v1` | 消息已读回执 |
| `im.chat.member.bot.added_v1` | 机器人进群 |
| `im.chat.member.bot.deleted_v1` | 机器人被移出群 |

## 8.2.6 OpenClaw 配置

### 基础配置

```

![飞书集成](./.assets/diagrams/25_feishu_integration.png)

bash
openclaw config set channels.feishu.appId "cli_xxxxx"
openclaw config set channels.feishu.appSecret "your_app_secret"
openclaw config set channels.feishu.enabled true
```

### 完整配置

```json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_xxxxx",
      "appSecret": "secret",
      "domain": "feishu",
      "connectionMode": "websocket",
      "dmPolicy": "pairing",
      "allowFrom": [],
      "groupPolicy": "allowlist",
      "requireMention": true,
      "allowMentionlessInMultiBotGroup": false,
      "groupCommandMentionBypass": "single_bot",
      "mediaMaxMb": 30,
      "renderMode": "auto"
    }
  }
}
```

### 配置选项

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | boolean | true | 是否启用 |
| `appId` | string | 必填 | 飞书应用 App ID |
| `appSecret` | string | 必填 | 飞书应用密钥 |
| `domain` | string | "feishu" | 域名：feishu/lark 或自定义 |
| `connectionMode` | string | "websocket" | 连接模式：websocket/webhook |
| `dmPolicy` | string | "pairing" | 私聊策略 |
| `groupPolicy` | string | "allowlist" | 群聊策略 |
| `requireMention` | boolean | true | 群聊是否需要 @机器人 |
| `renderMode` | string | "auto" | 渲染模式：auto/raw/card |

### 连接模式

**WebSocket 模式**（推荐，无需公网地址）：

```yaml
connectionMode: "websocket"
```

飞书控制台：事件与回调 → 选择 **使用长连接接收事件**

**Webhook 模式**（需要公网地址）：

```

![飞书集成](./.assets/diagrams/25_feishu_integration.png)

yaml
connectionMode: "webhook"
webhookPort: 3000
webhookPath: "/feishu/events"
encryptKey: "your_encrypt_key"
verificationToken: "your_verify_token"
```

飞书控制台：事件与回调 → 选择 **使用请求地址接收事件**

### 私聊策略 (dmPolicy)

| dmPolicy | 谁能私聊 | 如何开通 |
|----------|----------|---------|
| `pairing` | 白名单用户或配对审批用户 | 用户先私聊拿到配对码，管理员执行 `openclaw pairing approve feishu <code>` |
| `open` | 所有人 | 配置 `allowFrom: ["*"]` |
| `allowlist` | 仅白名单用户 | 将用户 open_id 加入 allowFrom |

### 群聊策略

| groupPolicy | 说明 |
|-------------|------|
| `open` | 所有群可触发 |
| `allowlist` | 只有白名单群可用 |
| `disabled` | 禁用群聊 |

### 渲染模式

| 模式 | 说明 |
|------|------|
| `auto` | 自动检测：有代码块或表格时用卡片，否则纯文本 |
| `raw` | 始终纯文本 |
| `card` | 始终使用卡片 |

## 8.2.8 资源访问权限

### 云空间访问

> ⚠️ **重要**：机器人没有自己的"我的空间"，只能访问被分享的文件/文件夹。

**配置步骤**：
1. 在飞书云空间创建文件夹

> ⚠️ **重要**：仅有 API 权限不够，还需要将机器人添加到知识库空间。

### 多维表格访问

> ⚠️ **重要**：机器人只能访问被分享的多维表格。

**配置步骤**：
1. 打开多维表格 → **分享**
2. 搜索机器人名称，授予权限

## 8.2.9 动态 Agent

### 功能说明

启用后，每个私聊用户会自动获得独立的 Agent 实例和专属 Workspace。

### 配置

```

![飞书集成](./.assets/diagrams/25_feishu_integration.png)

yaml
channels:
  feishu:
    dmPolicy: "open"
    allowFrom: ["*"]
    dynamicAgentCreation:
      enabled: true
      workspaceTemplate: "~/workspaces/feishu-{agentId}"
      agentDirTemplate: "~/.openclaw/agents/{agentId}/agent"
      maxAgents: 100
```

### 工作原理

1. 新用户发送私聊时，系统创建新的 Agent 条目
2. 创建 Binding 将用户路由到专属 Agent
3. 自动创建 Workspace 和 Agent 目录
4. 后续消息路由到隔离的 Agent

## 8.2.10 @ 消息转发

### 功能说明

在消息中 @某人，机器人的回复会自动 @该用户。

### 使用示例

- **私聊**：`@张三 跟他说你好` → 机器人回复 `@张三 你好！`
- **群聊**：`@机器人 @张三 跟他说你好` → 机器人回复 `@张三 你好！`

## 8.2.11 常见问题

### 机器人收不到消息

检查：
1. 是否配置了事件订阅？
2. 事件订阅方式是否与 connectionMode 匹配？
   - websocket → 飞书控制台选择 **使用长连接接收事件**
   - webhook → 飞书控制台选择 **使用请求地址接收事件**
3. 是否添加了 `im.message.receive_v1` 事件？
4. 相关权限是否已审核通过？

### requireMention: false 但群里仍然必须 @

检查：
1. 是否已申请并审核通过 `im:message.group_msg`（敏感权限）
2. 群里若有多个 Bot，是否设置了 `allowMentionlessInMultiBotGroup: true`

### 返回消息时 403 错误

确保已申请 `im:message:send_as_bot` 权限。

### 如何清理历史会话

在聊天中发送 `/new` 命令。
