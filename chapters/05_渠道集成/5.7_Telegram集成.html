8.3 Telegram 集成 ​

本节目标 ​

  • 掌握 Telegram Bot 创建和配置
  • 理解 OpenClaw Telegram 频道功能
  • 完成 Telegram 接入实战
  • 掌握高级功能（群组、媒体、命令）

8.3.1 Telegram 插件概述 ​

OpenClaw 内置支持 ​

Telegram 是 OpenClaw 内置支持的渠道，无需额外安装插件，开箱即用。

核心功能 ​

Telegram 插件核心功能：

✅ 两种连接模式
   ├── Long Polling（默认）- 主动拉取消息
   └── Webhook - 服务器推送

✅ 消息类型支持
   ├── 文本消息
   ├── 图片/照片
   ├── 音频/语音
   ├── 视频/视频笔记
   ├── 文档/文件
   ├── 表情贴纸
   └── 位置分享

✅ 交互功能
   ├── Inline 键盘按钮
   ├── 命令菜单
   ├── 回调数据处理
   └── 表情响应

✅ 群组管理
   ├── 隐私模式
   ├── 群组白名单
   ├── 发言权限控制
   └── @提及触发

8.3.2 Bot 创建 ​

通过 BotFather 创建 ​

  1. 打开 Telegram，搜索 @BotFather
  2. 发送 /newbot 命令
  3. 输入 Bot 名称（如 My AI Assistant）
  4. 输入用户名（必须以 bot 结尾，如 my_ai_assistant_bot）
  5. 获取 API Token

Token 格式 ​

1234567890:ABCdefGHIjklMNOpqrsTUVwxyz

⚠️ 注意：

  • 妥善保管，不要泄露
  • 可以随时通过 BotFather 重置

配置 Bot 简介 ​

/setdescription - 设置 Bot 简介
/setabout - 设置关于信息
/setuserpic - 设置头像

8.3.3 OpenClaw 配置 ​

基础配置 ​

bash
# 方式一：环境变量
export TELEGRAM_BOT_TOKEN="your-bot-token"

# 方式二：配置文件
openclaw config set channels.telegram.enabled true
openclaw config set channels.telegram.botToken "your-bot-token"

完整配置 ​

json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz",
      "dmPolicy": "pairing",
      "allowFrom": [],
      "groupPolicy": "allowlist",
      "groupAllowFrom": []
    }
  }
}

配置选项 ​

选项类型默认值说明
enabledbooleanfalse是否启用
botTokenstring-Bot Father 获取的 Token
dmPolicystring"pairing"私聊策略
allowFromstring[][]允许的发送者
groupPolicystring"allowlist"群组策略
groupAllowFromstring[][]群组允许的发送者
groupsobject-群组详细配置

8.3.4 连接模式 ​

Long Polling（默认） ​

OpenClaw 默认使用 Long Polling 模式，主动拉取消息。

json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "your-token"
    }
  }
}

Webhook 模式 ​

需要公网可访问的 URL。

json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "your-token",
      "webhookUrl": "https://your-domain.com/telegram-webhook",
      "webhookSecret": "your-secret",
      "webhookPath": "/telegram-webhook",
      "webhookHost": "0.0.0.0",
      "webhookPort": 8787
    }
  }
}

Webhook 配置选项 ​

选项类型默认值说明
webhookUrlstring-公网可访问的 URL
webhookSecretstring-Webhook 密钥
webhookPathstring"/telegram-webhook"回调路径
webhookHoststring"127.0.0.1"本地监听地址
webhookPortnumber8787本地监听端口

设置 Webhook ​

bash
# 通过 Telegram API 设置
curl -X POST \
  https://api.telegram.org/bot<TOKEN>/setWebhook \
  -d '{"url": "https://your-domain.com/telegram-webhook"}'

# 删除 Webhook
curl -X POST \
  https://api.telegram.org/bot<TOKEN>/deleteWebhook

8.3.5 私聊策略配置 ​

dmPolicy 选项 ​

值说明
pairing默认模式，需要配对码
allowlist仅白名单用户
open完全开放

配置示例 ​

json
{
  "channels": {
    "telegram": {
      "dmPolicy": "pairing",
      "allowFrom": []
    }
  }
}

配对流程 ​

  1. 用户私聊发送消息
  2. 机器人返回配对码
  3. 管理员执行：openclaw pairing approve telegram <code>

8.3.6 群组配置 ​

隐私模式 ​

Telegram Bot 默认启用隐私模式，只能收到：

  • 命令消息
  • @ 机器人提及的消息

禁用隐私模式 ​

两种方式：

  1. 通过 BotFather：

    • 发送 /setprivacy
    • 选择 Disable

  2. 提升为管理员：

    • 在群组中将机器人设为管理员

⚠️ 注意：更改隐私模式后，需要将机器人移出群组并重新添加。

群组策略 ​

json
{
  "channels": {
    "telegram": {
      "groupPolicy": "allowlist",
      "groups": {
        "-1001234567890": {
          "requireMention": true,
          "allowFrom": ["8734062810", "745123456"]
        }
      }
    }
  }
}

群组配置选项 ​

选项类型说明
groupPolicystring群组策略：open/allowlist/disabled
groupAllowFromstring[]允许的发送者
groupsobject按群组ID的配置

群组级别配置 ​

json
{
  "channels": {
    "telegram": {
      "groups": {
        "-1001234567890": {
          "requireMention": true,
          "allowFrom": ["123456789"]
        },
        "-1009876543210": {
          "requireMention": false,
          "allowFrom": ["*"]
        }
      }
    }
  }
}

接收消息类型 ​

类型说明

json
{
  "action": "send",
  "channel": "telegram",

  "message": "Choose an option:",
  "buttons": [
    [
      { "text": "Yes", "callback_data": "yes" },
      { "text": "No", "callback_data": "no" }
    ],
    [{ "text": "Cancel", "callback_data": "cancel" }]
  ]
}

发送图片 ​

json

  "action": "send",
  "channel": "telegram",
  "to": "123456789",
  "media": "https://example.com/image.jpg"
}

发送音频/语音 ​

json
{
  "action": "send",
  "channel": "telegram",
  "to": "123456789",
  "media": "https://example.com/voice.ogg",
  "asVoice": true
}

发送视频 ​

json
{
  "action": "send",
  "channel": "telegram",
  "to": "123456789",
  "media": "https://example.com/video.mp4",
  "asVideoNote": true
}

发送文件 ​

json
{
  "action": "send",
  "channel": "telegram",
  "to": "123456789",
  "media": "/path/to/document.pdf"
}

媒体选项说明 ​

选项说明
media文件 URL 或本地路径
asVoice发送为语音消息
asVideoNote发送为视频笔记

⚠️ 注意：视频笔记不支持标题，如果有文本会作为单独消息发送。

8.3.11 TTS 语音回复 ​

功能说明 ​

OpenClaw 支持将文本转换为语音并发送到 Telegram。

支持格式 ​

格式说明
OpusTelegram 语音气泡（推荐）
MP3标准音频

生成语音 ​

在 Skill 中使用：

当用户请求语音回复时：
1. 使用 tts 工具生成音频
2. 在回复中包含 MEDIA:/path/to/audio.mp3
3. 系统会自动发送为语音消息

配置示例 ​

json
{
  "tts": {
    "provider": "elevenlabs",
    "voice": "rachel",
    "model": "eleven_multilingual_v2"
  }
}

8.3.13 常见问题 ​

Q1: Webhook 设置失败 ​

问题：无法设置 Webhook

解决：

  1. 确认 URL 可访问（公网）
  2. 检查 SSL 证书有效
  3. 验证 Token 正确
  4. 检查防火墙/端口

Q2: 群组消息收不到 ​

问题：在群组中收不到消息

解决：

  1. 确认已禁用隐私模式或机器人为管理员
  2. 检查 groupPolicy 配置
  3. 确认群组 ID 在白名单中

Q3: 消息发送失败 ​

问题：发送消息返回错误

解决：

  1. 检查 chat_id 是否正确
  2. 验证文件 URL 可访问
  3. 确认 Bot 有发送消息权限

Q4: 配对失败 ​

问题：用户无法完成配对

解决：

  1. 检查配对码是否正确
  2. 确认 dmPolicy 为 "pairing"
  3. 查看日志确认配对状态

Q5: Bot 启动报 404 Not Found ​

问题：Gateway 启动时报 404 Not Found，Bot 无法连接 Telegram

排查：

  1. 验证 Token：curl -s "https://api.telegram.org/bot<YOUR_TOKEN>/getMe" —— 返回 404 = Token 无效
  2. 检查 Webhook：curl -s "https://api.telegram.org/bot<YOUR_TOKEN>/getWebhookInfo" —— 有 URL 则删掉：curl -X POST "https://api.telegram.org/bot<YOUR_TOKEN>/deleteWebhook"
  3. 中国大陆需确认代理有效（HTTPS_PROXY 或 channels.telegram.proxyUrl）
  4. 参考 15.2.4 问题 1

8.3.14 国际化（i18n）：简体中文配置 ​

功能说明 ​

v2026.3.24 新增了 简体中文（zh-CN） 本地化支持。OpenClaw Telegram Bot 可以使用中文回复。

启用中文 ​

json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "your-bot-token",
      "language": "zh-CN"
    }
  }
}

支持的语言 ​

语言代码说明
enEnglish（默认）
zh-CN简体中文（v2026.3.24+ 新增）

效果 ​

语言设置前（英文）：
└── "Session expired, please re-authenticate."

语言设置后（中文）：
└── "会话已过期，请重新验证身份。"

动态切换 ​

用户也可以通过命令动态切换语言：

/language zh-CN   # 切换到中文
/language en      # 切换回英文

与系统提示词的关系 ​

⚠️ language 配置影响 OpenClaw 系统 UI 和错误提示的显示语言，不影响 Agent 的回复语言。Agent 的回复语言由系统提示词（SOUL.md/IDENTITY.md）中的 persona 配置决定。

8.3.15 ⚠️ 已知问题 ​

exec 审批气泡不消失（#56286，v2026.3.24） ​

问题：当 exec 工具需要审批时，Telegram 会弹出审批气泡。点击"批准"后气泡仍然显示，不消失。

影响：纯 UI 问题，不影响 exec 实际执行。

临时应对：

  1. 手动刷新：在 Telegram 对话中下拉刷新，气泡应消失
  2. 忽略即可：任务已正常执行，气泡不影响功能

官方修复跟踪：Issue #56286
