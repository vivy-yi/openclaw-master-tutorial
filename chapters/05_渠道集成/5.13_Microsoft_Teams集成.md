# Microsoft Teams 集成

> **v2026.3.24 重要更新**：从旧版自定义实现迁移至 Microsoft 官方 Bot Framework SDK，支持流式回复、欢迎卡片、反馈机制等新功能。

## v2026.3.24 新功能对比

| 功能 | 旧版 | v2026.3.24+ |
|------|------|-------------|
| 回复方式 | 非流式 | ✅ **流式回复**（Streaming） |
| 新用户体验 | 简陋 | ✅ **欢迎卡片**（Welcome Card） |
| 交互反馈 | 无 | ✅ **反馈机制** |
| SDK 来源 | 自定义 | ✅ **Microsoft 官方 Bot Framework** |

## 迁移步骤

### 第一步：Azure Portal 创建应用注册

1. 登录 [Azure Portal](https://portal.azure.com)
2. 进入 **Azure Active Directory** → **应用注册** → **新建注册**
3. 填写应用信息：

| 字段 | 值 |
|------|-----|
| 名称 | `OpenClaw-Teams-Bot` |
| 支持的账户类型 | **任何组织目录中的账户** |
| 重定向 URI | `https://api.botframework.com` |

4. 在 **概述** 页面记录：
   - **应用程序（客户端）ID** → 即 `botId`
   - **目录（租户）ID** → 即 `tenantId`

5. 进入 **证书和密码** → **新建客户端密码** → 记录密码值（即 `botPassword`）

### 第二步：配置 Bot Channels Registration

1. 在 Azure Portal 搜索 **Bot Channels Registration** → **创建**
2. 填写 Bot 句柄（需全局唯一）
3. 在 **Microsoft App ID 和密码** 中填入第一步的凭据
4. **消息传送终结点**：填入 OpenClaw Gateway 公网地址，如 `https://your-gateway.com/api/msteams/webhook`
5. 配置 **Channels**：勾选 **Microsoft Teams**，启用 **支持即时消息**

### 第三步：打包并安装 Teams 应用

创建以下文件结构：
```
teams-app/
├── manifest.json          # 应用清单
├── color.png              # 16x16 图标
└── outline.png           # 32x32 图标
```

**manifest.json 示例：**

```json
{
  "$schema": "https://developer.microsoft.com/en-us/json/schemas/teams/v1.17/MicrosoftTeams.schema.json",
  "manifestVersion": "1.17",
  "version": "1.0.0",
  "id": "{{YOUR_APP_ID}}",
  "packageName": "com.openclaw.teams",
  "developer": {
    "name": "OpenClaw",
    "websiteUrl": "https://openclaw.ai"
  },
  "name": { "short": "OpenClaw" },
  "description": { "short": "OpenClaw AI 助手" },
  "icons": { "color": "color.png", "outline": "outline.png" },
  "accentColor": "#0078D4",
  "bots": [{
    "botId": "{{YOUR_BOT_APP_ID}}",
    "scopes": ["team", "personal"],
    "commandLists": [{
      "scopes": ["team", "personal"],
      "commands": [
        { "title": "帮助", "description": "显示所有可用命令" },
        { "title": "日程", "description": "查看今日日程安排" }
      ]
    }]
  }],
  "permissions": ["identity", "messageTeamMembers"],
  "validDomains": ["*.openclaw.ai", "*.botframework.com"]
}
```

**安装方式 A — 侧载（开发测试）：**
1. 打开 Teams 桌面客户端 → **应用** → **管理应用** → **上传应用** → **上载自定义应用**

**安装方式 B — 管理员发布（生产）：**
1. 进入 [Teams 管理中心](https://admin.teams.microsoft.com) → **Teams 应用** → **管理应用** → **上传**

### 第四步：OpenClaw Gateway 配置

**最小配置（仅消息）：**

```json
{
  "channels": {
    "msteams": {
      "enabled": true,
      "botId": "your-azure-app-client-id",
      "botPassword": "your-azure-app-client-secret",
      "tenantId": "your-microsoft-365-tenant-id"
    }
  }
}
```

**生产配置（启用流式回复 + 欢迎卡片）：**

```json
{
  "channels": {
    "msteams": {
      "enabled": true,
      "botId": "your-azure-app-client-id",
      "botPassword": "your-azure-app-client-secret",
      "tenantId": "your-microsoft-365-tenant-id",
      "useStreaming": true,
      "streamingEndpoint": "wss://your-gateway.com/api/msteams/streaming",
      "welcomeCard": {
        "enabled": true,
        "title": "👋 OpenClaw 助手已就绪",
        "subtitle": "我可以帮你完成日程管理、代码开发、数据分析等任务",
```

## 流式回复原理

```
非流式（旧版）：用户发送 → Bot 完整处理 → 一次性返回（等待 30s+）
流式（v2026.3.24+）：用户发送 → Bot 边生成边显示 → 逐段落更新（首 Token < 2s）

实现：
1. Bot 接收消息，立即返回「占位」Activity
2. 开启流式推理，将 Token 逐个/逐句推送
3. 使用 Edit Activity 更新同一消息
4. 流结束时标记 [DONE]
```

## 常见问题

**Bot 频道中未 @mention 不响应** → 这是 Teams 设计行为，必须 @mention Bot。或设置 `respondToChannelMessagesWithoutMention: true`（存在消息风暴风险）。

**媒体文件发送失败** → 改用 Azure Blob Storage 直链分享，不要直接上传。

**Adaptive Card 按钮点击无反应** → 改用 `Action.Submit` 类型（详见上方代码示例）。
