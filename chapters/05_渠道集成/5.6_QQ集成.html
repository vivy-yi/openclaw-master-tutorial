4.2 NapCat 安装 ​

下载与安装 ​

  1. 下载 NapCat QQ
  2. 解压并运行 NapCat.exe

配置 WebSocket 服务器 ​

  1. 打开 NapCat WebUI: http://localhost:6099/webui/network
  2. 点击 新建 → Websocket 服务器
  3. 配置：
    • 名称: OpenClaw-WS-Server
    • Host: 127.0.0.1
    • Port: 3001
    • Token: (留空)
    • 消息格式: Array
    • 心跳间隔: 30000
    • 启用: ✅

配置 HTTP 服务器 ​

  1. 点击 新建 → HTTP 服务器
  2. 配置：
    • 名称: OpenClaw-HTTP-API
    • Host: 127.0.0.1
    • Port: 8080
    • Token: your-token (自定义，需与 OpenClaw 配置一致)
    • 消息格式: Array
    • 启用: ✅

4.3 安装 OpenClaw 插件 ​

方法一：npm 安装 ​

bash
# 安装 OneBot 插件
git clone https://github.com/wusiyu123/openclaw-qq-bot.git ~/.openclaw/extensions/openclaw-onebot
cd ~/.openclaw/extensions/openclaw-onebot
npm install
npm run build

方法二：统一安装 ​

bash
# 安装中国平台统一包
openclaw plugins install @openclaw-china/channels

# 运行配置向导
openclaw china setup

4.4 OpenClaw 配置 ​

配置 openclaw.json ​

编辑 ~/.openclaw/openclaw.json：

json
{
  "channels": {
    "onebot": {
      "enabled": true,
      "wsUrl": "ws://localhost:3001",
      "httpUrl": "http://localhost:8080",
      "accessToken": "your-token"
    }
  }
}

QQ 机器人配置（BytePioneer-AI） ​

bash
# 启用 QQ 渠道
openclaw config set channels.qqbot.enabled true
openclaw config set channels.qqbot.appId your-app-id
openclaw config set channels.qqbot.clientSecret your-app-secret

# 可选配置
openclaw config set channels.qqbot.markdownSupport true
openclaw config set channels.qqbot.c2cMarkdownDeliveryMode proactive-all
openclaw config set channels.qqbot.c2cMarkdownChunkStrategy markdown-block

完整配置示例 ​

json
{
  "channels": {
    "qqbot": {
      "enabled": true,
      "appId": "your-app-id",
      "clientSecret": "your-app-secret",
      "markdownSupport": true,
      "c2cMarkdownDeliveryMode": "proactive-all",
      "c2cMarkdownChunkStrategy": "markdown-block",
      "autoSendLocalPathMedia": false,
      "asr": {
        "enabled": true,
        "appId": "your-tencent-app-id",
        "secretId": "your-tencent-secret-id",
        "secretKey": "your-tencent-secret-key"
      }
    }
  }
}

配置选项 ​

选项类型默认值说明
enabledbooleantrue是否启用
appIdstring必填应用 ID
clientSecretstring必填应用密钥
markdownSupportbooleanfalse启用 Markdown 支持
c2cMarkdownDeliveryModestring"passive"私聊 Markdown 发送方式
c2cMarkdownChunkStrategystring"length"Markdown 切分策略

私聊 Markdown 模式 ​

模式说明
passive被动发送
proactive-table-only有表格时主动发送
proactive-all全部主动发送

Markdown 切分策略 ​

策略说明
markdown-block按标题/表格/引用等边界切分
length按长度切分

4.7 引用消息功能 ​

功能说明 ​

  • 支持"引用上一条消息"功能
  • 用户问"这个是什么""你刚才说的哪个文件"时，模型会参考被引用内容
  • 引用内容自动缓存在 ~/.openclaw/qqbot/data/ref-index.jsonl
  • 支持文本、图片、语音、视频、文件摘要

引用命令 ​

bash
# 引用上一条消息
[右键消息 → 引用]

4.8 主动发送功能 ​

已知目标 ​

  • 机器人见过的用户或群会被记录
  • 可直接给指定对象发送文字或媒体
  • 记录保存到 ~/.openclaw/qqbot/data/known-targets.json

发送代码示例 ​

typescript
import { sendProactiveQQBotMessage } from "@openclaw-china/qqbot";

await sendProactiveQQBotMessage({
  cfg: {
    channels: {
      qqbot: {
        appId: "your-app-id",
        clientSecret: "your-app-secret",
      }
    }
  },
  to: "user:your-openid",
  text: "这是一条主动发送的消息"
});

目标格式 ​

格式示例说明
user:<openid>user:u123456发送给用户
group:<groupid>group:g123456发送给群

4.9 快速中断功能 ​

功能说明 ​

新增 停止 / /stop 快速通道：

  • 当前任务执行时，中断命令会绕过本地排队
  • 立即发送给 OpenClaw
  • 丢弃同一会话里尚未处理的排队消息
  • 减少"停不下来还继续串消息"的情况

4.11 故障排除 ​

问题 1: WebSocket 连接失败 ​

现象: WebSocket closed: 1005 或 1006

解决方案:

  1. 检查 NapCat WebSocket 服务器是否启用
  2. 确保端口 3001 未被占用
  3. 检查防火墙设置

问题 2: 消息发送失败 (403 Forbidden) ​

现象: token verify failed!

解决方案:

  1. 确认 NapCat HTTP 服务器的 Token 与 OpenClaw 配置一致
  2. 检查 Token 是否正确传递

问题 3: 收到消息但无回复 ​

现象: NapCat 日志显示接收但无发送

解决方案:

  1. 检查 OpenClaw 日志中的 deliver(final) 记录
  2. 查看是否有 HTTP API 调用错误
  3. 确认 accessToken 配置正确
