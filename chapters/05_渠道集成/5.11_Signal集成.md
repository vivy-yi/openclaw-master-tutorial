# 8.7 Signal 集成

本节讲解如何将 Signal 与 OpenClaw 集成，使用 signal-cli 实现隐私通讯。

---

## 8.7.1 概述

Signal 是注重隐私的即时通讯工具。OpenClaw 通过 `signal-cli`（外部 CLI）与 Signal 服务通信。

```
★ Insight ─────────────────────────────────────
Signal 集成特点：
1. 通过 signal-cli 实现（不是内置 libsignal）
2. 确定性路由：回复始终返回 Signal
3. 需要独立号码
4. 支持 DM 和群组
─────────────────────────────────────────────────
```

## 8.7.2 前置要求

### 环境要求

- OpenClaw 已安装
- `signal-cli` 可用
- 可接收验证短信的电话号码
- 浏览器用于解决 Signal captcha

### 安装 signal-cli

```bash
# Ubuntu/Debian
sudo apt-get install signal-cli

# macOS
brew install signal-cli

# 或从源码编译
```

---

## 8.7.3 快速开始

### 方式 A：QR 链接配对

```bash
# 创建链接
signal-cli link -n "OpenClaw"

# 扫描 QR 码（用 Signal 手机客户端）
```

### 方式 B：短信注册

```bash
# 注册新号码（需要 captcha）
signal-cli register +15551234567

# 验证
signal-cli verify +15551234567 <验证码>
```

### 配置 OpenClaw

```json5
{
  channels: {
    signal: {
      enabled: true,
      account: "+15551234567",
      cliPath: "signal-cli",
      dmPolicy: "pairing",
      allowFrom: ["+15557654321"],
    },
  },
}
```

### 批准配对

```bash
# 发送第一条 DM 后批准
openclaw pairing approve signal <CODE>
```

---

## 8.7.4 配置详解

### 字段说明

| 字段 | 说明 | 必需 |
|------|------|------|
| `account` | Bot 电话号码（E.164 格式） | 是 |
| `cliPath` | signal-cli 路径 | 否（默认 PATH） |
| `dmPolicy` | DM 访问策略 | 否 |
| `allowFrom` | 允许的号码列表 | 否 |

### DM 策略

```json5
{
  channels: {
    signal: {
      dmPolicy: "pairing",  // pairing | allowlist | open | disabled
      allowFrom: ["+15557654321", "+15559876543"],
    },
  },
}
```

| 策略 | 说明 |
|------|------|
| `pairing` | 需要配对批准 |
| `allowlist` | 仅允许白名单号码 |
| `open` | 开放（需设置 allowFrom: ["*"]） |
| `disabled` | 禁用 DM |

### 群组策略

```json5
{
  channels: {
    signal: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["+15557654321"],
    },
  },
}
```

## 8.7.6 故障排除

### 常见问题

**Q: signal-cli 未找到**

```
解决：
1. 确认 signal-cli 已安装：which signal-cli
2. 配置 cliPath 为完整路径
3. 或将 signal-cli 添加到 PATH
```

**Q: 注册失败 (captcha)**

```
解决：
1. 访问 signalcaptchas.org 获取 captcha token
2. 使用 token 进行注册
3. 如果持续失败，尝试 QR 链接方式
```

**Q: 消息发送失败**

```
解决：
1. 检查配对状态
2. 查看 Gateway 日志
3. 确认号码格式正确（E.164）
```

```
★ Insight ─────────────────────────────────────
Signal 安全最佳实践：
1. 使用独立 Signal 号码（推荐）
2. 限制 dmPolicy 为 pairing
3. 定期轮换配对
4. 监控配对请求
─────────────────────────────────────────────────
```

## 本节小结

1. **连接方式**：通过 signal-cli 外部 CLI
2. **配对方式**：QR 链接或短信注册
3. **访问控制**：DM 策略和群组策略
4. **安全**：使用独立号码，配置访问控制
