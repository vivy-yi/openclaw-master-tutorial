12.6 Hook 事件完全指南 ​

本节目标 ​

  • 掌握 OpenClaw Hook 事件系统
  • 理解各类 Hook 的触发时机和用途
  • 学会在插件中注册和使用 Hook

6.1 Hook 事件概述 ​

什么是 Hook ​

Hook（钩子）是 OpenClaw 插件系统的核心扩展机制，允许插件在系统关键节点注入自定义逻辑，实现：

  • 消息处理拦截
  • 工具调用监控
  • Agent 生命周期控制
  • 会话管理
  • 系统级事件响应

Hook vs registerHook vs on ​

![Hook事件]/assets/diagrams/64_hook_events.png)

typescript
// 两种注册方式
api.registerHook("message_received", handler, opts)  // 注册特定事件
api.on("hook_name", handler, opts)                  // 生命周期钩子（通用）

6.3 Gateway 生命周期 Hook ​

gateway_start ​

Gateway 启动时触发，用于初始化插件资源。

typescript
api.on("gateway_start", async (ctx) => {
  // 初始化数据库连接
  await db.connect();
  // 加载缓存
  await cache.load();
  console.log("Plugin initialized");
});

gateway_stop ​

Gateway 停止时触发，用于清理资源。

typescript
api.on("gateway_stop", async (ctx) => {
  // 保存状态
  await state.save();
  // 关闭连接
  await db.disconnect();
  console.log("Plugin shutdown");
});

6.4 Agent 生命周期 Hook ​

before_model_resolve ​

模型解析前触发，可用于动态切换模型。

typescript
api.on("before_model_resolve", async (event, ctx) => {
  // 根据用户输入动态选择模型
  if (event.prompt.length > 10000) {
    return {
      modelOverride: "gpt-4-turbo",  // 使用更长上下文的模型
      providerOverride: "openai"
    };
  }
});

事件参数：

typescript
{
  prompt: string;  // 用户提示
}

返回结果：

typescript
{
  modelOverride?: string;      // 覆盖模型
  providerOverride?: string;   // 覆盖提供商
}

before_prompt_build ​

Prompt 构建前触发，可用于注入提示词。

typescript
api.on("before_prompt_build", async (event, ctx) => {
  // 添加自定义指令
  event.prompt += "\n\n[插件提示：用户使用的是 VIP 账户]";
});

before_agent_start ​

Agent 启动前触发，可用于最后调整。

typescript
api.on("before_agent_start", async (event, ctx) => {
  console.log(`Agent starting for session: ${ctx.sessionKey}`);
});

llm_input ​

LLM 输入前触发，可用于修改输入。

typescript
api.on("llm_input", async (event, ctx) => {
  // 过滤敏感信息
  event.messages = event.messages.map(msg => {
    if (msg.role === "user") {
      msg.content = msg.content.replace(/password:\s*\S+/gi, "password: ***");
    }
    return msg;
  });
});

llm_output ​

LLM 输出后触发，可用于后处理。

typescript
api.on("llm_output", async (event, ctx) => {
  // 记录 token 使用量
  console.log(`Tokens used: ${event.usage.total_tokens}`);
});

agent_end ​

Agent 结束后触发。

typescript
api.on("agent_end", async (event, ctx) => {
  // 清理临时文件
  await cleanupTempFiles(ctx.workspaceDir);
});

6.5 上下文管理 Hook ​

before_compaction ​

上下文压缩前触发，可用于保存重要信息。

typescript
api.on("before_compaction", async (event, ctx) => {
  // 提取关键信息到摘要
  const summary = extractKeyInfo(event.messages);
  await saveToMemory("session_summary", summary);
});

after_compaction ​

上下文压缩后触发。

typescript
api.on("after_compaction", async (event, ctx) => {
  console.log(`Compacted ${event.originalTokens} → ${event.compactedTokens} tokens`);
});

6.6 消息处理 Hook ​

message_received ​

消息接收后触发，可用于消息预处理。

typescript
api.on("message_received", async (event, ctx) => {
  console.log(`Received from ${event.from}: ${event.content}`);

  // 消息过滤
  if (event.content.startsWith("/")) {
    // 命令消息不进入 Agent 处理
    return { handled: true };
  }
});

事件参数：

typescript
{
  from: string;           // 发送者
  content: string;        // 消息内容
  timestamp?: number;     // 时间戳
  metadata?: Record<string, unknown>;  // 元数据
}

message_sending ​

消息发送前触发，可用于修改或拦截。

typescript
api.on("message_sending", async (event, ctx) => {
  // 添加水印
  event.content += "\n\n-- Sent via OpenClaw";

  // 消息审核
  if (containsSensitiveWords(event.content)) {
    return { blocked: true, reason: "内容包含敏感词" };
  }
});

事件参数：

typescript
{
  to: string;             // 接收者
  content: string;        // 消息内容
  metadata?: Record<string, unknown>;
}

返回结果：

typescript
{
  blocked?: boolean;      // 是否拦截
  content?: string;       // 修改后的内容
  reason?: string;       // 拦截原因
}

message_sent ​

消息发送后触发。

typescript
api.on("message_sent", async (event, ctx) => {
  // 记录发送日志
  await logMessage(event.to, event.content);
});

6.7 工具调用 Hook ​

before_tool_call ​

工具调用前触发，可用于参数验证或修改。

typescript
api.on("before_tool_call", async (event, ctx) => {
  console.log(`Tool call: ${event.toolName}`);

  // 参数验证
  if (event.toolName === "exec") {
    const cmd = event.params.command;
    if (isDangerousCommand(cmd)) {
      return { blocked: true, error: "禁止执行危险命令" };
    }
  }
});

事件参数：

typescript
{
  toolName: string;                    // 工具名称
  params: Record<string, unknown>;     // 工具参数
  runId?: string;                      // 运行 ID
  toolCallId?: string;                 // 工具调用 ID
}

返回结果：

typescript
{
  blocked?: boolean;         // 是否阻止调用
  params?: object;          // 修改后的参数
  error?: string;          // 错误信息
}

after_tool_call ​

工具调用后触发。

typescript
api.on("after_tool_call", async (event, ctx) => {
  console.log(`Tool ${event.toolName} completed in ${event.duration}ms`);
});

tool_result_persist ​

工具结果持久化前触发。

typescript
api.on("tool_result_persist", async (event, ctx) => {
  // 敏感结果过滤
  if (event.toolName === "read") {
    event.result = filterSensitiveData(event.result);
  }
});

6.10 Hook 注册示例 ​

完整插件示例 ​

typescript
import type { OpenClawPluginApi } from "openclaw/plugin-sdk";

export default function myPlugin(api: OpenClawPluginApi) {
  // 注册消息处理 Hook
  api.registerHook("message_received", async (event, ctx) => {
    console.log(`Received: ${event.content}`);
  }, { priority: 100 });

  // 注册工具调用 Hook
  api.registerHook("before_tool_call", async (event, ctx) => {
    if (event.toolName === "exec") {
      // 验证命令安全性
      const cmd = event.params.command as string;
      if (cmd.includes("rm -rf")) {
        return { blocked: true, error: "禁止删除操作" };
      }
    }
  }, { priority: 50 });

  // 注册生命周期 Hook（通用方式）
  api.on("agent_end", async (ctx) => {
    console.log("Agent run completed");
  });
}

Hook 优先级 ​

typescript
{
  // 优先级越高越早执行
  priority: 100   // 高优先级
  priority: 50    // 中优先级（默认）
  priority: 10    // 低优先级
}

6.11 最佳实践 ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         Hook 最佳实践                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ✅ 推荐做法：                                                                  │
│  1. 优先使用 before_* 类 Hook 进行拦截/修改                                  │
│  2. 注意 Hook 顺序，合理设置 priority                                         │
│  3. 异步 Hook 使用 async/await                                                │
│  4. 返回结果时只返回必要的字段                                                 │
│  5. 错误处理使用 try/catch                                                    │
│                                                                                 │
│  ❌ 避免做法：                                                                  │
│  1. 在 Hook 中执行耗时操作                                                     │
│  2. 修改 event 对象（使用返回结果）                                           │
│  3. 忘记处理 Promise                                                          │
│  4. 阻塞主流程太久                                                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
