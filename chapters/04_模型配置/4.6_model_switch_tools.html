4.6.1 工具概述 ​

什么是 cc-switch ​

cc-switch 是一个跨平台桌面应用程序，用于管理和切换 Claude Code、Codex、OpenCode、OpenClaw 和 Gemini 的模型配置。

4.6.2 安装指南 ​

Windows 安装 ​

bash
# 方式一：从 Release 页面下载
# 访问 https://github.com/farion1231/cc-switch/releases
# 下载 CC-Switch-v{version}-Windows.msi 或便携版

# 方式二：使用 winget
winget install CCSwitch

# 方式三：使用 Chocolatey
choco install cc-switch

macOS 安装 ​

bash
# 方式一：使用 Homebrew
brew tap farion1231/ccswitch
brew install --cask cc-switch

# 方式二：从 Release 页面下载 .dmg 文件
# 访问 https://github.com/farion1231/cc-switch/releases

Linux 安装 ​

bash
# 方式一：使用 apt (Debian/Ubuntu)
sudo apt install ./cc-switch_{version}_amd64.deb

# 方式二：使用 dnf (Fedora/RHEL)
sudo dnf install ./cc-switch-{version}.rpm

# 方式三：使用 AppImage
chmod +x cc-switch-{version}.AppImage
./cc-switch-{version}.AppImage

# 方式四：使用 Flatpak
flatpak install cc-switch.flatpak

4.6.4 添加模型提供商 ​

方式一：使用预设配置 ​

cc-switch 内置了常用提供商的预设配置，只需输入 API Key 即可：

bash
# 预设支持的提供商
- OpenAI
- Anthropic (Claude)
- Google Gemini
- DeepSeek
- 智谱 GLM
- Moonshot (Kimi)
- MiniMax
- 百度 ERNIE
- 阿里 Qwen
- 腾讯 Hunyuan
- OpenRouter
- Cohere
- Mistral
- xAI (Grok)

方式二：自定义配置 ​

对于自定义 API Endpoint 或代理配置：

添加自定义提供商步骤：
1. 点击 "+" 按钮
2. 选择 "Custom" 或 "Other"
3. 填写配置：
   - Name: 提供商名称
   - API Endpoint: API 地址
   - API Key: 密钥
   - Model: 默认模型
4. 保存并启用

常用 API Endpoint 参考 ​

4.6.5 切换模型 ​

基本切换 ​

切换步骤：
1. 在主界面点击目标提供商卡片
2. 点击 "Enable" 按钮启用
3. 重启终端或 Claude Code/Codex/Gemini 应用
4. 配置生效

通过系统托盘快速切换 ​

托盘菜单使用：
1. cc-switch 运行时会显示在系统托盘
2. 点击托盘图标显示菜单
3. 直接点击提供商名称
4. 即时切换（部分应用需要重启）

注意事项 ​

⚠️ 重要提示：
- 切换后需要重启终端或 Claude Code / Codex / Gemini 客户端才能应用更改
- 部分应用可能需要完全退出再重新启动
- 切换后首次使用可能需要等待配置加载

cc-switch 可以管理 MCP (Model Context Protocol) 服务器配置：

添加 MCP 服务器 ​

bash
# 方式一：使用内置模板
1. 点击 "MCP" 按钮
2. 选择 "Browse Templates"
3. 选择需要的 MCP 服务器模板
4. 点击 "Install"

# 方式二：自定义 MCP
1. 点击 "MCP" 按钮
2. 点击 "Add Custom"
3. 填写 MCP 服务器配置
4. 保存并启用

4.6.10 常见问题 ​

Q1: 切换后模型没有生效 ​

bash
# 解决步骤：
1. 确保已点击 "Enable" 启用提供商
2. 完全退出 Claude Code / Codex / Gemini 应用
3. 重新启动应用
4. 检查托盘图标是否显示正确

Q2: API Key 验证失败 ​

bash
# 检查：
1. API Key 是否正确
2. API Key 是否有足够额度
3. 网络是否可以访问 API Endpoint
4. 是否使用代理需要额外配置

Q3: MCP 服务器无法连接 ​

bash
# 排查：
1. 检查 MCP 服务器地址是否正确
2. 确认 MCP 服务器是否运行
3. 检查网络连通性
4. 查看 cc-switch 日志

Q4: Skills 安装失败 ​

bash
# 可能原因：
1. 网络问题无法访问 GitHub
2. 磁盘空间不足
3. ~/.claude/skills/ 目录权限问题

# 解决：
1. 检查网络连接
2. 清理磁盘空间
3. 检查目录权限
