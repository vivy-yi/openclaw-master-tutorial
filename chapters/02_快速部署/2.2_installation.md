# 2.2 安装步骤

## 本节目标
- 掌握 OpenClaw 的多种安装方式
- 完成首次安装和验证
- 了解目录结构

---

## 2.2.1 安装方式选择

OpenClaw 提供多种安装方式：

| 方式 | 优点 | 缺点 | 适用场景 |
|-----|-----|-----|---------|
| **官方一键安装** | 简单自动，包含环境检测 | 需要网络 | 个人用户（推荐） |
| **npm 安装** | 灵活可控 | 需要手动配置 | 开发者 |
| **Docker** | 环境隔离 | 需要 Docker | 开发者/生产环境 |
| **手动安装** | 可定制 | 需要手动配置 | 高级用户 |

---

## 2.2.2 官方一键安装（推荐）

### 为什么选择官方一键安装？

```
✅ 自动检测系统环境
✅ 自动安装依赖（Node.js等）
✅ 自动配置环境变量
✅ 自动创建目录结构
✅ 支持交互式配置向导
```

### 安装步骤

#### 步骤一：打开终端

**macOS**：
```
Command + 空格 → 输入 "Terminal" → 回车
```

![安装步骤](./.assets/diagrams/92_installation_steps.png)



**Linux**：
```
Ctrl + Alt + T
```

**Windows (WSL)**：
```
在 Ubuntu 终端中执行
```

#### 步骤二：执行安装命令

```bash
# 官方一键安装脚本
curl -fsSL https://openclaw.ai/install.sh | bash
```

> [!NOTE]
> 如果提示权限不足：
> ```bash
> curl -fsSL https://openclaw.ai/install.sh | sudo bash
> ```

#### 步骤三：安装过程

安装脚本会自动执行以下操作：

```
1. 检测系统环境
   - 操作系统类型
   - Node.js 版本
   - 可用端口

2. 安装/更新依赖
   - Node.js 22+ (如未安装)
   - 必要的系统包

3. 下载 OpenClaw
   - 从 npm registry 下载
   - 安装全局命令

4. 创建目录结构
   - ~/.openclaw/
   - workspace/, memory/, skills/
```

![包管理](./.assets/diagrams/171_package_management.png)



#### 步骤四：验证安装

```bash
# 查看版本
openclaw --version

# 查看帮助
openclaw --help
```

**预期输出**：
```
$ openclaw --version

$ openclaw --help

  -V, --version   output the version number

Commands:
  gateway         Manage gateway

  skill           Manage skills
  config          Configuration management

```

## 2.2.3 交互式配置向导

安装完成后，运行配置向导：

```bash
openclaw onboard
```

### 配置流程

#### 1. 选择启动模式

```
? Select a startup mode
  ❯ QuickStart (recommended)
    Manual Configuration
    Advanced
```

- **QuickStart**：快速启动，推荐新手
- **Manual**：手动配置
- **Advanced**：高级选项

#### 2. 选择 AI 模型

```
? Select AI provider
  ❯ DeepSeek (性价比高)
    Kimi (长文本)
    OpenAI
    Anthropic (Claude)
    智谱 GLM
    MiniMax
    Ollama (本地)
```

#### 3. 输入 API Key

```
? Enter your API key: (press Enter to skip)
```

> [!TIP]
> 没有 API key 可以先跳过，后续在配置文件中添加。

#### 4. 选择聊天工具

```
? Select chat platforms to enable
  ❯ ○ None (CLI only)
    ○ Feishu (飞书)
    ○ Telegram
    ○ Discord
    ○ WeCom (企业微信)
```

#### 5. 选择 Skills

```
? Select skills to install (Press space to select)
  ❯ ◉ file-management
    ◉ web-search
    ◉ browser
    ◉ code-runner
```

#### 6. 完成配置

配置完成后，系统会自动：
- 启动 Gateway 服务
- 打开 Web UI (`http://127.0.0.1:18789/chat`)

## 2.2.4 npm 安装（开发者）

如果你更倾向于手动安装：

### 步骤一：检查 Node.js

```bash
# 检查版本 (需要 >= 22.0)
node --version

# 如果版本过低，安装 nvm 后升级
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.zshrc
nvm install 22
nvm use 22
```

### 步骤二：安装 OpenClaw

```bash
# 全局安装
npm install -g openclaw

# 或使用 pnpm
pnpm add -g openclaw
```

### 步骤三：验证

```bash
openclaw --version
```

## 2.2.5 Docker 安装

适合开发者或生产环境：

### 基本运行

```bash
# 创建配置目录
mkdir -p ~/.openclaw

# 运行容器
docker run -d \
  --name openclaw \
  -v ~/.openclaw:/root/.openclaw \
  -p 18789:18789 \
  openclaw/openclaw:latest

# 查看日志
docker logs -f openclaw
```

### Docker Compose（推荐）

创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    ports:
      - "18789:18789"
    volumes:
      - ~/.openclaw:/root/.openclaw
    environment:
      - TZ=Asia/Shanghai
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:18789/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

```bash
# 启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止
docker-compose down
```

## 2.2.6 目录结构

安装完成后，会创建以下目录结构：

```
~/.openclaw/
├── openclaw.json          # 主配置文件
├── workspace/             # 工作目录
│   ├── downloads/        # 下载文件
│   ├── uploads/         # 上传文件
│   └── temp/            # 临时文件
├── memory/               # 记忆存储
│   ├── sessions/        # 会话历史
│   └── longterm/        # 长期记忆
├── skills/              # 本地 Skills
├── plugins/             # 本地 Plugins
└── logs/                # 日志文件
    └── openclaw.log
```

## 2.2.7 日常使用命令

### 基本操作

| 命令 | 功能 |
|-----|------|
| `openclaw gateway start` | 启动 Gateway |
| `openclaw gateway stop` | 停止 Gateway |
| `openclaw gateway status` | 查看状态 |
| `openclaw update` | 更新版本 |
| `openclaw uninstall` | 卸载 |

### 开机自启（macOS）

```bash
# 使用 launchd
openclaw gateway enable
```

### 开机自启（Linux）

```bash
# 使用 systemd
sudo systemctl enable openclaw
```

## 2.2.8 常见问题

### 问题一：权限错误

```bash
# macOS
sudo chown -R $(whoami) ~/.openclaw

# Linux
sudo chown -R $USER ~/.openclaw
```

### 问题二：端口占用

```bash
# 检查端口
lsof -i :18789

# 更改端口
openclaw gateway start --port 18889
```

### 问题三：安装失败

```bash
# 清理后重试
npm uninstall -g openclaw
npm cache clean --force
npm install -g openclaw
```

### 问题四：Node.js 版本过低

```bash
# 使用 nvm 安装最新版本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.zshrc
nvm install 22
nvm use 22
```
