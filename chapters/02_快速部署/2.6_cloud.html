2.6.1 为什么选择云端部署 ​

云端部署 vs 本地部署 ​

特性云端部署本地部署
硬件要求无需准备电脑需要 Mac/PC
运行环境24/7 在线需要保持开机
访问方式任意设备需在同一网络
成本20-40元/月电费为主
维护服务商负责自行维护
数据安全云服务商保障本地物理安全

适用场景 ​

✅ 适合云端部署：
├── 没有 Mac 电脑
├── 需要 24 小时运行
├── 需要手机远程访问
├── 多用户同时访问
└── 不想维护本地设备

❌ 适合本地部署：
├── 有 Mac 电脑
├── 注重数据隐私
├── 预算有限
└── 开发调试阶段

2.6.3 腾讯云轻量部署（推荐） ​

步骤一：购买服务器 ​

  1. 登录 腾讯云轻量应用服务器
  2. 选择 应用镜像 > OpenClaw 或 Docker
  3. 选择地域（建议选择离你最近的）
  4. 选择配置（推荐 2核2G）
  5. 设置登录密码
  6. 购买

步骤二：配置安全组 ​

控制台 → 安全组 → 配置规则

入站规则：
├── TCP 18789 (OpenClaw)    ✅ 必须
├── TCP 22 (SSH)            ✅ 必须
└── TCP 80/443 (HTTP/HTTPS)  可选

出站规则：
└── 全部允许

![VPS部署]/assets/diagrams/52_cloud_vps.png)

步骤三：SSH 连接 ​

bash
# 连接服务器
ssh root@你的服务器IP

# 首次连接需要确认，输入 yes

步骤四：配置 OpenClaw ​

bash
# 1. 创建配置目录
mkdir -p ~/.openclaw

# 2. 创建配置文件
vim ~/.openclaw/openclaw.json

编辑配置文件：

json
{
  "model": {
    "provider": "deepseek",
    "model": "deepseek-chat",
    "apiKey": "${DEEPSEEK_API_KEY}"
  },
  "gateway": {
    "port": 18789,
    "channels": {
      "cli": { "enabled": true },
      "feishu": { "enabled": true }
    }
  }
}

步骤五：设置环境变量 ​

bash
# 编辑环境变量
vim /etc/profile

# 添加以下内容
export DEEPSEEK_API_KEY="你的API Key"

# 生效
source /etc/profile

步骤六：启动 OpenClaw ​

bash
# 使用官方一键安装
curl -fsSL https://openclaw.ai/install.sh | bash

# 启动
openclaw gateway start

# 设置开机自启
systemctl enable openclaw

2.6.4 火山引擎部署 ​

步骤一：购买服务器 ​

  1. 登录 火山引擎 Lighthouse
  2. 选择 Docker 镜像
  3. 选择地域
  4. 选择配置（推荐 2核2G）
  5. 购买

步骤二：SSH 连接 ​

bash
ssh root@你的服务器IP

步骤三：安装 Docker（如果没有） ​

bash
# 安装 Docker
curl -fsSL https://get.docker.com | sh

# 启动 Docker
systemctl start docker
systemctl enable docker

步骤四：运行 OpenClaw ​

bash
# 创建配置目录
mkdir -p ~/.openclaw

# 创建配置文件
vim ~/.openclaw/openclaw.json

# 运行容器
docker run -d \
  --name openclaw \
  --restart unless-stopped \
  -p 18789:18789 \
  -v ~/.openclaw:/root/.openclaw \
  -e DEEPSEEK_API_KEY=$DEEPSEEK_API_KEY \
  openclaw/openclaw:latest

2.6.5 一键部署脚本 ​

通用部署脚本 ​

bash
#!/bin/bash

#============= 配置区域 =============
# 修改以下变量
OPENCLAW_PORT=18789
OPENCLAW_DIR=~/.openclaw
MODEL_PROVIDER="deepseek"
MODEL_NAME="deepseek-chat"
#=====================================

echo "=== OpenClaw 一键部署脚本 ==="
echo ""

# 1. 创建目录
echo "[1/5] 创建目录..."
mkdir -p $OPENCLAW_DIR

# 2. 创建配置文件
echo "[2/5] 创建配置文件..."
cat > $OPENCLAW_DIR/openclaw.json << EOF
{
  "model": {
    "provider": "$MODEL_PROVIDER",
    "model": "$MODEL_NAME",
    "apiKey": "\${${MODEL_PROVIDER^^}_API_KEY}"
  },
  "gateway": {
    "port": $OPENCLAW_PORT,
    "channels": {
      "cli": {"enabled": true}
    }
  }
}
EOF

# 3. 拉取镜像
echo "[3/5] 拉取 Docker 镜像..."
docker pull openclaw/openclaw:latest

# 4. 运行容器
echo "[4/5] 启动容器..."
docker stop openclaw 2>/dev/null
docker rm openclaw 2>/dev/null

docker run -d \
  --name openclaw \
  --restart unless-stopped \
  -p $OPENCLAW_PORT:$OPENCLAW_PORT \
  -v $OPENCLAW_DIR:/root/.openclaw \
  openclaw/openclaw:latest

# 5. 完成
echo "[5/5] 完成!"
echo ""
echo "=== 部署成功 ==="
echo "访问地址: http://你的服务器IP:$OPENCLAW_PORT"
echo ""
echo "请设置环境变量:"
echo "  export ${MODEL_PROVIDER^^}_API_KEY='你的API Key'"
echo ""
echo "然后重启容器:"
echo "  docker restart openclaw"

使用方法 ​

bash
# 1. 下载脚本
curl -O https://example.com/deploy-openclaw.sh

# 2. 修改权限
chmod +x deploy-openclaw.sh

# 3. 运行脚本
./deploy-openclaw.sh

2.6.6 域名与 HTTPS 配置 ​

购买域名（可选） ​

  1. 在阿里云/腾讯云购买域名
  2. 添加 DNS 解析到服务器 IP

Nginx 反向代理 ​

nginx
# /etc/nginx/sites-available/openclaw

server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

配置 HTTPS ​

方法一：使用 Certbot（推荐） ​

bash
# 安装 Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期
sudo certbot renew --dry-run

方法二：手动配置 ​

nginx
# HTTPS 配置
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/certificate.crt;
    ssl_certificate_key /path/to/private.key;

    location / {
        proxy_pass http://127.0.0.1:18789;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

# HTTP 重定向到 HTTPS
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

2.6.7 系统安全配置 ​

防火墙配置 ​

bash
# Ubuntu (UFW)
sudo ufw allow 22/tcp    # SSH
sudo ufw allow 80/tcp    # HTTP
sudo ufw allow 443/tcp    # HTTPS
sudo ufw allow 18789/tcp  # OpenClaw
sudo ufw enable

# CentOS (firewalld)
sudo firewall-cmd --permanent --add-port=22/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=18789/tcp
sudo firewall-cmd --reload

系统时间同步 ​

bash
# 安装 chrony
sudo apt install chrony

# 启动服务
sudo systemctl enable chronyd
sudo systemctl start chronyd

# 设置时区
sudo timedatectl set-timezone Asia/Shanghai

# 验证
timedatectl

SSH 安全加固 ​

bash
# 编辑 SSH 配置
sudo vim /etc/ssh/sshd_config

# 修改以下配置
Port 2222                    # 更改 SSH 端口
PermitRootLogin no          # 禁止 root 登录
PasswordAuthentication no    # 禁用密码登录
PubkeyAuthentication yes    # 启用密钥登录

# 重启 SSH
sudo systemctl restart sshd

2.6.8 监控与维护 ​

监控脚本 ​

bash
#!/bin/bash

# 检查 OpenClaw 是否运行
if ! docker ps | grep -q openclaw; then
    echo "OpenClaw 未运行，正在启动..."
    docker start openclaw
fi

# 检查端口
if ! netstat -tuln | grep -q 18789; then
    echo "端口 18789 未监听"
fi

# 检查内存使用
MEMORY=$(docker stats openclaw --no-stream --format "{{.MemUsage}}")
echo "内存使用: $MEMORY"

# 检查 CPU 使用
CPU=$(docker stats openclaw --no-stream --format "{{.CPUPerc}}")
echo "CPU 使用: $CPU"

自动重启 ​

bash
# 添加定时任务
crontab -e

# 每天凌晨 3 点检查并重启
0 3 * * * /path/to/check_openclaw.sh

2.6.10 常见问题 ​

问题一：无法连接服务器 ​

检查安全组：

  • 确认 22 端口已开放
  • 确认 18789 端口已开放

检查防火墙：

bash
sudo ufw status
sudo iptables -L -n

问题二：OpenClaw 启动失败 ​

bash
# 查看日志
docker logs openclaw

# 检查配置
docker exec openclaw cat /root/.openclaw/openclaw.json | jq .

# 检查环境变量
docker exec openclaw env

问题三：端口被占用 ​

bash
# 检查端口
netstat -tuln | grep 18789

# 更改端口
docker stop openclaw
docker rm openclaw
docker run -d -p 18889:18789 ...

问题四：域名无法访问 ​

检查 DNS 解析：

bash
nslookup your-domain.com
ping your-domain.com

检查 Nginx：

bash
sudo nginx -t
sudo systemctl status nginx
