1. 什么是 OpenClaw？ ​

OpenClaw 是一个可自托管的 AI Agent 运行框架，主打「实干型」路线：

特性说明
🔧 自托管完全私有，数据不出本机
🤖 多 Agent支持主 Agent + 子 Agent 协作
🧰 Skill 系统可插拔工具集，开箱即用
💬 多渠道接入支持飞书、Telegram、Discord 等
🔗 开源完全透明，社区驱动

OpenClaw 走红背后：Agent、AI Coding 与团队协作（36Kr 报道）

2. 快速部署（10 分钟） ​

2.1 环境要求 ​

要求版本
Node.js≥ 22.0
npm / yarn最新版
操作系统macOS / Linux / Windows (WSL)
内存≥ 4GB（推荐 8GB）
磁盘≥ 2GB 可用空间

2.2 官方一键安装 ​

优点： 自动检测环境、自动安装依赖、自动配置引导 ✅

bash
# 方式一：官方推荐（自动）
curl -fsSL https://get.openclaw.ai | bash

# 方式二：npm 全局安装
npm install -g openclaw

# 方式三：Docker（适合开发者）
docker pull openclaw/openclaw:latest
docker run -d -p 18789:18789 openclaw/openclaw:latest

来源：快速入门：10分钟完成本地安装与配置

2.3 安装验证 ​

bash
openclaw --version
openclaw gateway status

看到类似输出即成功：

✅ Gateway running at ws://127.0.0.1:18789
✅ Dashboard: http://127.0.0.1:18789/

2.4 目录结构 ​

~/.openclaw/
├── agents/          # Agent 配置
├── skills/          # Skill 工具集
├── shared/          # 共享知识库
├── sessions/        # 会话记录
└── openclaw.json    # 主配置文件

3. 配置你的第一个渠道 ​

3.1 飞书（推荐国内用户） ​

bash
openclaw channel add feishu

配置步骤：

  1. 登录 飞书开放平台
  2. 创建企业自建应用
  3. 获取 App ID 和 App Secret
  4. 配置机器人能力
  5. 在 openclaw.json 中填入凭证
json
{
  "channels": {
    "feishu": {
      "enabled": true,
      "appId": "cli_xxxxxx",
      "appSecret": "xxxxxxxx"
    }
  }
}

来源：OpenClaw 新手入门手册

3.2 Telegram ​

bash
openclaw channel add telegram
  1. 联系 @BotFather 创建机器人
  2. 获取 Bot Token
  3. 配置 Webhook 或 Long Polling
json
{
  "channels": {
    "telegram": {
      "enabled": true,
      "botToken": "123456:ABCdefGHIjklMNOpqrsTUVwxyz"
    }
  }
}

5. 常见问题与排错 ​

Q1：安装时报 node: command not found ​

原因： Node.js 未安装或版本过低。

解决：

bash
# macOS
brew install node@22

# Linux
curl -fsSL https://deb.nodesource.com/setup_22.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node --version  # 应显示 v22.x.x

Q2：Gateway 启动成功但 Web 界面打不开 ​

原因： 可能是端口占用或防火墙。

解决：

bash
# 检查端口
lsof -i :18789

# 如端口被占用，修改配置
openclaw config set gateway.port 18790

Q3：TUI 会话重连后丢失 ​

原因： Session 未持久化或网关重启。

解决： 确保 openclaw.json 中 session 持久化已开启，参考 章节 06：上下文与记忆。

Q4：渠道机器人无响应 ​

排查顺序：

  1. ✅ openclaw gateway status 确认网关运行
  2. ✅ 检查渠道配置凭证是否正确
  3. ✅ 查看日志 tail -f /tmp/openclaw/openclaw-$(date +%Y-%m-%d).log
  4. ✅ 确认机器人已被加入对话/群组
