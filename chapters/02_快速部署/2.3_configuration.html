2.3 配置文件详解 ​

本节目标 ​

  • 掌握 openclaw.json 配置结构
  • 理解各配置项的作用
  • 完成基础配置

2.3.1 配置文件位置 ​

OpenClaw 的配置文件位于：

![JSON配置]/assets/diagrams/76_json_config.png)

![资源限制]/assets/diagrams/95_resource_limits.png)

~/.openclaw/openclaw.json

首次运行时会自动创建默认配置文件。

手动创建 ​

bash
mkdir -p ~/.openclaw
touch ~/.openclaw/openclaw.json

2.3.2 配置层次结构 ​

openclaw.json
├── model          # 模型配置
├── gateway        # 网关配置
├── agent          # 智能体配置
├── memory         # 记忆配置
├── skills         # 技能配置
├── security       # 安全配置
└── channels       # 渠道配置

2.3.3 最小配置 ​

仅需配置模型即可运行：

json
{
  "model": {
    "provider": "deepseek",
    "model": "deepseek-chat",
    "apiKey": "${DEEPSEEK_API_KEY}"
  }
}

TIP

使用环境变量避免硬编码密钥：

bash
export DEEPSEEK_API_KEY="sk-..."

在配置文件中使用 "apiKey": "${DEEPSEEK_API_KEY}"

2.3.5 配置项详解 ​

model - 模型配置 ​

json
"model": {
  "provider": "deepseek",           // 模型提供商
  "model": "deepseek-chat",       // 模型名称
  "apiKey": "${DEEPSEEK_API_KEY}", // API 密钥（环境变量）
  "baseUrl": "https://api.deepseek.com",  // API 地址
  "maxTokens": 4096,              // 最大生成 token 数
  "temperature": 0.7,             // 创造性参数 (0-2)
  "topP": 0.9,                    // 采样参数
  "stream": true                   // 是否启用流式输出
}

支持的模型提供商 ​

provider 值模型示例特点
openaigpt-4o, gpt-4o-mini功能全面
anthropicclaude-3-5-sonnet推理能力强
deepseekdeepseek-chat性价比高
moonshotkimimoe长文本
minimaxabab6.5s-chat中文优化
glmglm-4国产优选
ollamallama2, qwen本地部署

gateway - 网关配置 ​

json
"gateway": {
  "port": 18789,                   // 端口
  "host": "0.0.0.0",             // 监听地址
  "baseUrl": "http://localhost:18789",
  "channels": {                    // 渠道配置
    "cli": { "enabled": true },
    "feishu": { "enabled": false },
    "telegram": { "enabled": false }
  },
  "rateLimit": {                   // 限流配置
    "enabled": true,
    "maxRequests": 100,
    "windowMs": 60000
  }
}

agent - 智能体配置 ​

json
"agent": {
  "name": "assistant",            // Agent 名称
  "description": "Your AI assistant",

  // 启用的工具
  "tools": [
    "read",        // 读取文件
    "write",       // 写入文件
    "exec",        // 执行命令
    "browser",     // 浏览器控制
    "web",         // 网络请求
    "calendar",    // 日历操作
    "email"        // 邮件操作
  ],

  "maxSteps": 100,               // 最大执行步数
  "timeout": 300000,             // 超时时间（毫秒）
  "maxRetries": 3,               // 最大重试次数
  "thinking": {                  // 思考配置
    "enabled": true,
    "depth": "medium"
  }
}

工具说明 ​

工具功能权限级别
read读取文件/目录basic
write写入/编辑文件basic
exec执行系统命令elevated
browser浏览器自动化elevated
webHTTP 请求basic
calendar日历操作basic
email邮件操作basic

memory - 记忆配置 ​

json
"memory": {
  "enabled": true,                // 是否启用
  "type": "filesystem",          // 存储类型

  // 文件系统存储
  "path": "~/.openclaw/memory",

  // 会话超时（秒）
  "sessionTimeout": 3600,

  // 上下文压缩
  "compaction": {
    "enabled": true,
    "threshold": 10000
  }
}

skills - 技能配置 ​

json
"skills": {
  "enabled": true,                // 是否启用
  "autoInstall": true,           // 自动安装 Skills

  // Skills 市场地址
  "registry": "https://.clawhub.dev",

  // 本地 Skills 目录
  "localPath": "~/.openclaw/skills",

  // 预装的 Skills
  "installed": [
    "file-management",
    "web-search",
    "browser"
  ]
}

security - 安全配置 ​

json
"security": {
  // 安全策略配置
  "profile": "default",

  // 执行审批（需要用户确认）
  "execApproval": false,

  // 允许的命令
  "allowedCommands": ["ls", "cat", "grep", "pwd"],

  // 禁止的命令（正则表达式）
  "deniedCommands": ["rm -rf", "sudo", "chmod 777"],

  // 文件访问限制
  "allowedPaths": [
    "~/.openclaw/workspace",
    "~/Documents"
  ],

  // 网络访问限制
  "allowedDomains": [
    "api.openai.com",
    "api.deepseek.com"
  ]
}

2.3.6 环境变量 ​

推荐使用环境变量管理敏感信息：

macOS / Linux ​

bash
# 在 ~/.zshrc 或 ~/.bashrc 中添加
export OPENAI_API_KEY="sk-..."
export DEEPSEEK_API_KEY="sk-..."
export ANTHROPIC_API_KEY="sk-ant-..."
export MOONSHOT_API_KEY="..."

# 重新加载
source ~/.zshrc

Windows (PowerShell) ​

powershell
# 在 $env:USERPROFILE\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1 中添加
$env:OPENAI_API_KEY = "sk-..."
$env:DEEPSEEK_API_KEY = "sk-..."

验证环境变量 ​

bash
# 检查是否设置成功
echo $OPENAI_API_KEY
echo $DEEPSEEK_API_KEY

2.3.7 多模型配置 ​

配置文件多模型 ​

json
{
  "model": {
    "provider": "deepseek",
    "model": "deepseek-chat"
  },

  // 备用模型
  "fallbackModels": [
    {
      "provider": "openai",
      "model": "gpt-4o-mini"
    },
    {
      "provider": "anthropic",
      "model": "claude-3-haiku"
    }
  ]
}

模型选择策略 ​

json
{
  "model": {
    "provider": "deepseek",
    "model": "deepseek-chat",

    // 根据任务自动选择模型
    "autoSelect": {
      "enabled": true,
      "rules": [
        { "task": "reasoning", "use": "claude-3-sonnet" },
        { "task": "coding", "use": "gpt-4o" },
        { "task": "simple", "use": "gpt-4o-mini" }
      ]
    }
  }
}

2.3.8 配置文件验证 ​

验证 JSON 格式 ​

bash
# 使用 jq
cat ~/.openclaw/openclaw.json | jq .

# 或
jq . ~/.openclaw/openclaw.json

OpenClaw 内置验证 ​

bash
# 验证配置
openclaw config validate

# 查看配置路径
openclaw config path

2.3.9 常见问题 ​

问题一：配置不生效 ​

bash
# 1. 检查配置文件路径
openclaw config path

# 2. 验证 JSON 格式
jq . ~/.openclaw/openclaw.json

# 3. 重启 Gateway
openclaw gateway restart

问题二：JSON 格式错误 ​

bash
# 格式化 JSON（修复格式问题）
jq . ~/.openclaw/openclaw.json > temp.json && mv temp.json ~/.openclaw/openclaw.json

问题三：环境变量未读取 ​

bash
# 1. 检查环境变量是否设置
echo $OPENAI_API_KEY

# 2. 如果使用 Docker，确保传递环境变量
docker run -e OPENAI_API_KEY=$OPENAI_API_KEY ...

问题四：权限被拒绝 ​

bash
# 修复目录权限
chmod 700 ~/.openclaw
chmod 600 ~/.openclaw/openclaw.json
