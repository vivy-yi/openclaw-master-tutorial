2.4.1 启动 OpenClaw ​

根据部署方式启动 ​

本地部署 ​

![备份恢复]/assets/diagrams/54_backup_recovery.png)

![首次运行]/assets/diagrams/93_first_command.png)

![环境配置]/assets/diagrams/172_environment_setup.png)

bash
# 启动 Gateway
openclaw gateway start

# 或使用快捷命令
openclaw start

Docker 部署 ​

bash
# 启动容器
docker start openclaw

# 或使用 docker-compose
docker-compose up -d

云端部署 ​

直接访问 Web UI：
http://你的服务器IP:18789/

查看状态 ​

bash
# 查看 Gateway 状态
openclaw gateway status

# 查看详细状态
openclaw status

# 查看日志
openclaw logs

# 实时查看日志
openclaw logs -f

预期输出：

✅ Gateway is running on port 18789
✅ Channel: CLI enabled
✅ Channel: Web UI enabled

2.4.3 发送第一条消息 ​

验证安装 ​

在 Web UI 或终端中发送：

你好，能听到我说话吗？

预期回复 ​

你好！我能听到你说话。我是你的 AI 助手，可以帮你处理各种任务。

我可以帮你：
📁 搜索和管理文件
🌐 浏览网页和搜索信息
📅 管理日历和提醒
💻 执行命令和代码
📝 创建和编辑文档

有什么我可以帮你的吗？

验证成功的标志 ​

  • ✅ 收到 AI 的回复
  • ✅ 右上角显示模型名称
  • ✅ 显示 Token 使用情况

2.4.4 理解 OpenClaw 的回复 ​

OpenClaw 的回复包含几个部分：

[AI 回复]
好的，我帮你搜索一下。

[工具调用]
🔍 正在搜索文件...
📁 找到 3 个相关文件

[Token 统计]
输入：120 tokens
输出：45 tokens

2.4.5 如何提出好问题 ​

❌ 不好的问题 ​

帮我

✅ 好的问题 ​

帮我找一下电脑上所有的PDF文件，按修改时间排序

提问技巧 ​

技巧说明示例
明确目标说清楚你想做什么"帮我找一下我电脑上的一张发票"
提供上下文给出必要的背景信息"报销用的，上个月买的跑步机"
具体细节说明具体要求"按日期排序，显示前10个"
分步骤复杂任务分步骤说明"首先...然后...最后..."

示例对比 ​

不好的问题好的问题
"找文件""帮我找一下我电脑上的一张发票，里面详情是买了一个跑步机"
"整理""把我下载文件夹里的所有图片，按日期分类到不同的文件夹"
"提醒我""明天上午10点提醒我开会，地点是会议室A"

2.4.8 工作区配置文件 ​

OpenClaw 的工作区包含多个配置文件，每个文件都有特定的作用。

工作区结构 ​

~/.openclaw/workspace/
├── SOUL.md         # AI的人格/语气
├── USER.md         # 你的偏好设置
├── AGENTS.md       # Agent的工作指令
├── MEMORY.md       # 长期记忆
├── HEARTBEAT.md    # 定期检查任务
├── IDENTITY.md     # AI的身份设定
└── BOOT.md         # 启动配置

核心配置文件说明 ​

文件作用说明
SOUL.md人格/语气AI的性格、说话风格、行为准则
USER.md偏好设置你的信息、习惯、偏好
AGENTS.md指令说明Agent的工作指令和任务说明
MEMORY.md长期记忆AI的长期记忆和学习内容
HEARTBEAT.md检查清单定期检查和维护任务

TIP

新手建议：刚开始只需要配置 SOUL.md 和 USER.md 这两个文件就够了。

2.4.9 人设配置（Persona） ​

什么是人设？ ​

人设（Persona）是给 AI 助手设定的性格、身份和行为方式。

为什么要设置人设？ ​

  • 🎭 个性化：让 AI 更符合你的喜好
  • 💬 沟通风格：调整回复的语气和风格
  • 🎯 专业化：针对特定场景优化
  • 😊 趣味性：让对话更有趣

SOUL.md 配置示例 ​

markdown
# SOUL.md - AI 性格配置

## 名字
叫我"小助手"就好

## 身份
你是一个专业、高效的AI助手，专门帮助用户处理日常工作中的任务。

## 性格特点
- 专业但友好
- 简洁明了，不说废话
- 善于总结和信息提取

## 说话风格
- 使用清晰的中文
- 适当使用emoji让回复更生动
- 长回复会分段说明

## 行为准则
- 理解用户意图后再执行
- 不确定时会询问确认
- 重视用户隐私和数据安全

USER.md 配置示例 ​

markdown
# USER.md - 用户偏好

## 基本信息
- 名字：张三
- 时区：Asia/Shanghai

## 工作习惯
- 工作时间：周一到周五 9:00-18:00
- 常用办公软件：Notion, Slack, VS Code

## 偏好
- 喜欢简洁直接的沟通
- 需要详细的技术文档
- 优先使用中文

## 忌讳
- 不要随便删除文件
- 执行危险操作前需要确认

2.4.11 常见问题 ​

问题一：启动失败 ​

bash
# 检查端口是否被占用
lsof -i :18789

# 查看详细错误
openclaw logs

问题二：API 错误 ​

Error: Invalid API key

解决：

  1. 检查 API 密钥是否正确
  2. 检查环境变量是否设置
  3. 检查 API 余额是否充足
  4. 验证 API 是否可以访问
bash
# 验证 API
curl -sS https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer $DEEPSEEK_API_KEY"

问题三：无响应 ​

可能原因：

  1. 网络问题
  2. 模型服务商的 API 不可用
  3. 请求超时

解决：

  1. 检查网络连接
  2. 查看日志
  3. 尝试其他模型
  4. 检查防火墙

问题四：权限被拒绝 ​

可能原因：

  1. 文件权限不足
  2. 未授权的操作

解决：

bash
# 修复权限
chmod -R 755 ~/.openclaw
