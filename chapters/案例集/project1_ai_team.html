项目一：AI 专家团队 - 完整实施指南 ​

本指南将帮助你构建一个由多个专业化 AI Agent 组成的团队，通过单一 Telegram 对话界面协调工作。

1.1 项目概述 ​

什么是 AI 专家团队 ​

AI 专家团队是一种多 Agent 协作模式，通过为每个 Agent 分配特定角色和专业领域，形成一个虚拟的专业团队。

核心优势 ​

★ Insight ─────────────────────────────────────
为什么需要多个 Agent：
1. 单 Agent 上下文有限，多角色易混淆
2. 专业模型做专业事，成本优化
3. 并行处理提高效率
4. 单一入口简化管理
─────────────────────────────────────────────────

1.6 Telegram 路由配置 ​

创建 Telegram 组 ​

  1. 创建新 Telegram 组
  2. 添加所有 Agent Bot
  3. 设置组名为 "Team"

配置路由 ​

yaml
# ~/openclaw-teams/founder-team/routing.yaml
telegram:
  group_name: "Team"
  routing:
    # 精确匹配
    "@milo": milo
    "@josh": josh
    "@marketing": marketing
    "@dev": dev

    # 广播
    "@all": broadcast

    # 默认
    default: milo

  triggers:
    # 关键词触发
    "代码": dev
    "开发": dev
    "营收": josh
    "增长": josh
    "内容": marketing
    "推广": marketing
    "规划": milo
    "战略": milo

1.7 定时任务配置 ​

cron 配置 ​

yaml
# ~/openclaw-teams/founder-team/schedules.yaml
schedules:
  # Milo 早间站会
  - name: morning_standup
    agent: milo
    cron: "0 8 * * 1-5"
    prompt: |
      汇总昨晚所有 Agent 的活动，生成早间站会报告。

      格式：
      ## 站会 - {日期}

      ### 昨日完成
      - [Agent]: 任务

      ### 今日计划
      - [Agent]: 任务

      ### 障碍
      - 问题描述

      请读取共享状态文件获取详细信息。

  # Josh 指标汇总
  - name: metrics_summary
    agent: josh
    cron: "0 9 * * 1-5"
    prompt: |
      拉取并汇总关键指标：
      - 用户注册数
      - 活跃用户数
      - 营收数据（如可用）

      生成简短报告，标注趋势。

  # Marketing 内容创意
  - name: content_ideas
    agent: marketing
    cron: "0 10 * * 1-5"
    prompt: |
      基于当前趋势，生成 3 个内容创意。

      参考：
      - HackerNews 热门
      - Reddit 相关讨论
      - 竞品动态

      格式：
      ## 内容创意

      ### 创意 1
      标题: xxx
      平台: xxx
      要点: xxx

  # Milo 晚间总结
  - name: end_of_day
    agent: milo
    cron: "0 18 * * 1-5"
    prompt: |
      生成今日总结报告。

      包含：
      - 完成的 OKR 进度
      - 各 Agent 工作汇总
      - 明日重点

      更新 GOALS.md 和 STATUS.md

启用定时任务 ​

bash
# 在 OpenClaw 中加载定时任务
openclaw cron load ~/openclaw-teams/founder-team/schedules.yaml

# 查看任务
openclaw cron list

# 启用
openclaw cron enable --all

1.9 最佳实践 ​

Agent 命名规范 ​

markdown
# SOUL.md 结构

## 身份
- 名字和角色
- 性格特点

## 职责
- 核心职责 (3-5条)
- 不涉及的职责（边界）

## 沟通风格
- 语气
- 格式偏好

## 上下文
- 需要读取的共享文件
- 需要维护的私人文件

避免的问题 ​

★ Insight ─────────────────────────────────────
常见错误：

1. ❌ Agent 职责重叠
   → 明确定义边界

2. ❌ 共享记忆混乱
   → 定期整理结构

3. ❌ 单一 Agent 过载
   → 分散任务到专业 Agent

4. ❌ 缺乏明确目标
   → 每周更新 OKR
─────────────────────────────────────────────────
