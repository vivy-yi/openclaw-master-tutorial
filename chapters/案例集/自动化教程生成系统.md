# OpenClaw 教程自动化生成系统

> 从理论到实践：搭建完全自动化的教程内容生成流程

## 概述

本文介绍如何搭建一个完全自动化的 OpenClaw 教程内容生成系统，实现从多渠道采集内容到自动发布的一条龙服务。

## 核心组件

### 1. 输入源采集

| 输入源 | 采集内容 | 采集方式 |
|--------|----------|----------|
| 用户交互 | 有效对话、解决方案、工作流 | 会话记录分析 |
| GitHub | Issues、PRs、Release Notes | GitHub API |
| 社媒平台 | Twitter、Reddit、YouTube、Bilibili | agent-reach |
| 全网搜索 | 技术博客、教程网站 | Web Search |

### 2. Agent 设计

```json
{
  "name": "墨客-主编",
  "description": "协调教程内容的采集、生成、审核和发布全流程",
  "instructions": "你是教程主编，协调整个自动化内容生成流程...",
  "triggers": ["cron:0 23 * * *", "telegram:群ID"]
}
```

### 3. 定时任务调度

| 任务 | 时间 | 功能 |
|------|------|------|
| GitHub同步 | 每日 06:00 | Issues/PRs/Release |
| 全网搜索 | 每日 10:00 | Web 搜索高质量内容 |
| 社媒采集 | 每日 12:00 | Twitter/Reddit/YouTube/B站 |
| 教程采集 | 每日 23:00 | 用户交互内容提取 |
| 深度调研 | 每日 23:30 | 深度分析报告 |
| 生成审核发布 | 每日 00:00 | 生成+审核+发布 |

## 实施步骤

### 步骤 1: 创建 Workspace

```bash
mkdir -p /Users/d/.openclaw/workspaces/openclaw-tutor
```

### 步骤 2: 创建 Agent 定义

在 workspace 内创建各子 Agent：

```
openclaw-tutor/
├── SOUL.md
└── agents/
    ├── 采集/agent.json
    ├── 同步/agent.json
    ├── 社媒采集/agent.json
    ├── 全网搜索/agent.json
    ├── 深度调研/agent.json
    ├── 生成/agent.json
    └── 审核/agent.json
```

### 步骤 3: 配置 Cron 任务

使用 OpenClaw 的 cron 功能创建定时任务：

```bash
# GitHub 同步
openclaw cron add --name "墨客-GitHub同步" \
  --schedule "0 6 * * *" \
  --message "执行 GitHub 内容同步..." \
  --deliver-to "-5015409771"
```

### 步骤 4: 绑定到群

在 openclaw.json 中添加 binding：

```json
{
  "agentId": "openclaw-tutor",
  "match": {
    "channel": "telegram",
    "peer": {
      "kind": "group",
      "id": "-5015409771"
    }
  }
}
```
