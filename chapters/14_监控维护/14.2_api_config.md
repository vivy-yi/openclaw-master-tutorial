# 10.2 API 配置

## 本节目标
- 掌握 API 端点配置
- 理解请求/响应格式
- 完成 API 集成

---

## 10.2.1 API 端点

### 端点列表

```
API 端点：

1. Agent 对话
   POST /api/v1/agent
   └── 发送消息，获取响应

2. 会话管理
   GET  /api/v1/sessions
   POST /api/v1/sessions
   GET  /api/v1/sessions/:id
   DELETE /api/v1/sessions/:id

3. 模型管理
   GET /api/v1/models

4. 健康检查
   GET /health

5. WebSocket
   WS /ws
```

---

## 10.2.2 请求格式

### Agent 请求

```json
{
  "message": "你好",
  "model": "deepseek-chat",
  "session_key": "user_123",

  "options": {
    "temperature": 0.7,
    "max_tokens": 2048,
    "stream": false
  },

  "context": {
    "custom_data": "value"
  }
}
```

### 流式请求

```json
{
  "message": "讲个故事",
  "stream": true
}
```

---

## 10.2.3 响应格式

### 普通响应

```json
{
  "msg_id": "msg_123",
  "message": "你好，我是 OpenClaw",
  "model": "deepseek-chat",
  "tokens": {
    "input": 100,
    "output": 50
  },
  "finish_reason": "stop"
}
```

### 流式响应

```
SSE 格式：

data: {"content": "你", "type": "chunk"}
data: {"content": "好", "type": "chunk"}
data: {"content": "！", "type": "chunk"}
data: {"content": "[DONE]", "type": "done}"}
```

## 10.2.4 路由配置

### 自定义路由

```yaml
gateway:
  routes:
    - path: /api/chat
      agent: default
      methods: [POST]

    - path: /api/code
      agent: coder
      methods: [POST]
      auth: required

    - path: /api/admin
      agent: admin
      methods: [POST]
      auth: required
      role: admin
```

### 重写规则

```yaml
gateway:
  rewrites:
    - source: /old/api/*
      target: /api/v1/*
      permanent: true
```

## 10.2.5 CORS 配置

### 基础配置

```yaml
gateway:
  cors:
    enabled: true
    origins:
      - "https://example.com"
      - "https://app.example.com"
    methods:
      - GET
      - POST
      - OPTIONS
    headers:
      - Content-Type
      - Authorization
```
