## 10.1.1 什么是 Gateway

### Gateway 定义

```
┌─────────────────────────────────────────────┐
│           Gateway 核心能力                    │
├─────────────────────────────────────────────┤
│                                             │
│  Gateway = API 网关                         │
│                                             │
│  核心功能：                                  │
│  ├── 统一入口 - 单一端点服务所有客户端        │
│  ├── 路由转发 - 请求分发到后端服务           │
│  ├── 认证授权 - 验证身份和权限               │
│  ├── 限流熔断 - 保护后端服务                │
│  ├── 日志监控 - 记录请求日志                 │
│  └── 协议转换 - 支持多种协议                 │
│                                             │
└─────────────────────────────────────────────┘
```

### 与 Agent 的关系

```
Gateway 和 Agent 的关系：

用户请求 → Gateway → 路由 → Agent Engine → 返回
              ↓
         认证/限流
              ↓
         日志/监控
```

## 10.1.3 启动 Gateway

### 启动命令

```bash
# 启动 Gateway
openclaw gateway start

# 指定端口
openclaw gateway start --port 8080

# 指定配置
openclaw gateway start --config /path/to/config.yaml

# 后台运行
openclaw gateway start --daemon
```

### 验证启动

```bash
# 检查状态
openclaw gateway status

# 测试连接
curl http://127.0.0.1:18789/health
```

## 10.1.4 默认配置

### 基础配置

```yaml
gateway:
  host: 0.0.0.0
  port: 18789

  # 启用功能
  enabled:
    - api
    - webui
    - websocket

  # 跨域配置
  cors:
    enabled: true
    origins: ["*"]
```
