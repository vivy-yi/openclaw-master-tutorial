# 15.4 模型异常排查

## 本节目标

掌握模型调用失败、超时、限流等问题的排查方法。

---

## 15.4.1 排查检查项

### 1. 主模型状态检查

主模型是否限流、超时或认证失败：

```bash
# 查看模型状态
openclaw models status

# 测试模型调用
openclaw models test deepseek-chat
```

**典型报错**：

```
ProviderError: 429 Too Many Requests (OpenAI)
ProviderError: 401 Invalid API Key
ProviderError: 403 Rate Limit Exceeded
```

**原因**：
- API 调用超出限制
- API Key 无效或过期
- 账户余额不足

### 2. 回退链路检查

是否触发回退链路，回退是否有证据链：

**典型表现**：

```
[WARN] Model gpt-5 timed out. Falling back to claude-sonnet-4-6
[INFO] Fallback triggered: primary_model: gpt-4o -> fallback_model: gpt-4o-mini
```

**检查**：
```bash
# 查看回退配置
openclaw models fallback show

# 查看回退日志
openclaw logs | grep -i fallback
```

### 3. 冷却状态检查

冷却或禁用状态是否生效，是否出现重试放大：

```bash
# 查看模型冷却状态
openclaw models cooldown list
```

### 4. 认证检查

认证档案与密钥是否过期或被吊销：

```bash
# 查看密钥状态
openclaw keys list

# 测试密钥有效性
openclaw keys test <key_name>
```

## 15.4.2 常见模型问题

### 问题 1：API 限流

```
错误：429 Too Many Requests
```

**原因**：
- 超过每分钟请求数限制
- 超过每日消费限制

**排查**：
1. 检查账户余额和限制
2. 减少请求频率
3. 配置回退模型

### 问题 2：API Key 无效

```
错误：401 Invalid API Key
```

**原因**：
- Key 过期
- Key 被吊销
- Key 拼写错误

**排查**：
1. 检查 Key 是否正确
2. 在提供商控制台验证 Key
3. 重新生成 Key

### 问题 3：模型超时

```
错误：Request timeout
```

**原因**：
- 网络延迟
- 模型负载高
- 请求内容太长

**排查**：
1. 增加超时时间
2. 减少上下文长度
3. 检查网络连接

### 问题 4：上下文超限

```
错误：context length exceeded
```

**原因**：
- 消息历史太长
- 上下文超出模型限制

**排查**：
1. 启用上下文压缩
2. 减少保留消息数
3. 使用支持更长上下文的模型

## 15.4.3 诊断命令

```bash
# 1. 统计错误分类占比
cat runtime.log | jq -r 'select(.type=="log") | .log | select(.stage=="model_call_end" and .err_type!="") | .err_type' | sort | uniq -c | sort -nr | head

# 2. 查看模型调用日志
openclaw logs --type model --limit 200

# 3. 查看 Token 使用
openclaw models usage --period day

# 4. 测试模型响应时间
openclaw models benchmark
```
