6.5.2 向量存储 ​

什么是向量 ​

向量 = 文本的数学表示

文本 → 向量化模型 → 向量

示例：
"Python" → [0.12, -0.34, 0.56, ...]
"编程语言" → [0.11, -0.33, 0.58, ...]

相似度计算：
"Python" 和 "编程语言" 相似度高
→ 向量距离近

向量数据库 ​

支持的向量数据库：

1. Pinecone
   ├── 云服务
   ├── 无需自建
   └── 免费额度

2. Weaviate
   ├── 开源
   ├── 自部署
   └── 支持本地

3. Qdrant
   ├── 开源
   ├── 高性能
   └── Rust 开发

4. Milvus
   ├── 开源
   ├── 大规模
   └── 百度开源

5. 本地向量
   ├── FAISS
   └── 支持离线

配置向量存储 ​

json
{
  "memory": {
    "vector": {
      "enabled": true,
      "provider": "qdrant",
      "url": "http://localhost:6333",
      "collection": "openclaw_memory",
      "dimensions": 1536,
      "metric": "cosine"
    }
  }
}

6.5.3 知识库构建 ​

构建流程 ​

步骤：

1. 准备知识文档
   → 收集资料
   → 整理格式

2. 文档分块
   → 按章节/段落分割
   → 保持语义完整

3. 向量化
   → 转换为向量
   → 存储到数据库

4. 建立索引
   → 优化检索
   → 加速查询

代码实现 ​

python
# 构建知识库
from openclawsdk import Memory

memory = Memory()

# 文档分块
chunks = [
    "Python 编码规范：使用 4 空格缩进",
    "函数命名：小写下划线风格",
    "类命名：大驼峰风格",
    # ...
]

# 存储
for chunk in chunks:
    await memory.save(
        content=chunk,
        metadata={"source": "coding_standards"}
    )

# 检索
results = await memory.search(
    query="如何命名函数",
    top_k=3
)

6.5.5 记忆检索策略 ​

RAG 流程 ​

RAG = Retrieval Augmented Generation

┌─────────────────────────────────────────────┐
│           RAG 流程                           │
├─────────────────────────────────────────────┤
│                                             │
│  用户问题                                    │
│      │                                       │
│      ▼                                       │
│  ┌─────────────┐                            │
│  │  检索       │ → 从记忆/知识库检索         │
│  └──────┬──────┘                            │
│         │                                    │
│         ▼                                    │
│  ┌─────────────┐                            │
│  │  增强       │ → 组合上下文               │
│  └──────┬──────┘                            │
│         │                                    │
│         ▼                                    │
│  ┌─────────────┐                            │
│  │  生成       │ → LLM 生成答案             │
│  └─────────────┘                            │
│                                             │
└─────────────────────────────────────────────┘

检索优化 ​

优化策略：

1. 重排序
   ├── 粗筛：向量相似度
   └── 精排：交叉编码器

2. 混合检索
   ├── 向量搜索
   └── 关键词搜索

3. 缓存
   ├── 常用查询缓存
   └── 结果复用

6.5.6 自动学习 ​

学习机制 ​

自动学习流程：

1. 信息提取
   → 从对话中提取关键信息

2. 模式识别
   → 发现重复模式

3. 知识更新
   → 更新记忆存储

4. 主动建议
   → 基于学习提供建议

配置自动学习 ​

json
{
  "memory": {
    "learning": {
      "enabled": true,
      "auto_extract": true,
      "patterns": {
        "user_preference": {
          "enabled": true,
          "triggers": ["我偏好", "我喜欢"]
        },
        "project_knowledge": {
          "enabled": true,
          "triggers": ["项目用", "我们用"]
        }
      }
    }
  }
}

6.5.7 常见问题 ​

Q1: 记忆不同步 ​

问题：不同设备记忆不同步

解决：
1. 使用共享存储
2. 使用云端向量数据库
3. 手动导出导入

Q2: 检索不准确 ​

问题：检索结果不相关

解决：
1. 优化分块策略
2. 调整向量模型
3. 混合关键词搜索

Q3: 存储空间不足 ​

问题：向量数据太大

解决：
1. 定期清理
2. 压缩存储
3. 分级存储
