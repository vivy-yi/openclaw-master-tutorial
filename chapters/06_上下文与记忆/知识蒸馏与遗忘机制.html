知识蒸馏与遗忘机制：让 Agent 真正学会 ​

本教程基于 OpenClaw 实战经验，介绍如何构建三层记忆架构，实现"提取→蒸馏→遗忘"的完整闭环。

背景：为什么需要这套机制？ ​

大多数 Agent 系统只会记录，不会学习。典型表现：

  • 对话历史无限堆积 → 检索质量下降
  • 记忆文件中重复内容越来越多 → 知识冗余
  • 没有置信度概念 → 正确和错误的知识混在一起
  • 没有遗忘机制 → 记忆文件持续膨胀，最终失控

核心问题是：记录 ≠ 学习。真正的学习需要筛选、提炼、更新和淘汰。

三层记忆架构 ​

┌─────────────────────────────────────────────────────────────┐
│  L1 提取层（每次任务后）                                     │
│  └── extract.py — 从任务输出中提取 3-5 条关键信息            │
│      → 写入 memory/YYYY-MM-DD.md                           │
├─────────────────────────────────────────────────────────────┤
│  L2 蒸馏层（每周五 17:00）                                  │
│  └── distill.py — 读取 7 天日记忆 → LLM 分析               │
│      → 写入 MEMORY.md（分类、去重、置信度更新、标签生成）    │
├─────────────────────────────────────────────────────────────┤
│  L3 遗忘层（每月最后一天）                                   │
│  └── forget.py — 扫描所有条目 → 删除低置信度/长期未使用      │
│      → 整理结构，输出删除报告                                │
└─────────────────────────────────────────────────────────────┘

L1：提取（每次任务后） ​

目标：从每个任务的输出中自动提取关键信息。

提取规则（简单版，无需 LLM）：

  • 包含关键词的行：记住、规则、重要、决策、待办
  • 每条记忆限制 200 字符
  • 最多提取 5 条
bash
python3 extract.py \
  --task-id "task-2026-04-06-001" \
  --task-output "任务输出摘要" \
  --workspace /path/to/workspace \
  --num-items 5

输出格式：写入 memory/YYYY-MM-DD.md

markdown
### [分类] 主题

- 内容描述
- 来源：task-id
- 置信度：0.7
- 使用次数：0
- 最后使用：2026-04-06
- 标签：#tag1 #tag2

L2：蒸馏（每周五 17:00） ​

目标：将一周的日记忆合并到长期记忆，LLM 负责去重、归类、更新置信度。

关键问题：谁来执行蒸馏？

方案优点缺点
主 Agent 执行简单阻塞主流程
Python 脚本调用 LLM API不阻塞需要配置 API Key
派生子 Agent 执行不阻塞 + LLM 能力强需额外调度

推荐：派生子 Agent 执行蒸馏，命令示例：

bash
# 在 Cron 中触发
sessions_spawn --task "执行蒸馏" --runtime subagent

蒸馏后的知识条目格式：

markdown
### [技术知识] 知识蒸馏三层架构

- 架构：L1提取 → L2蒸馏 → L3遗忘
- 置信度机制：初始0.7，被验证正确+0.05，被纠正-0.3
- 来源：知识蒸馏Skill开发
- 置信度：0.75
- 使用次数：1
- 最后使用：2026-04-06
- 标签：#记忆系统 #skill #OpenClaw

L3：遗忘（每月最后一天） ​

触发条件（满足任一即删除）：

  1. 置信度 < 阈值（默认 0.3）
  2. 未使用天数 > 阈值（默认 365 天）
bash
python3 forget.py \
  --memory-file MEMORY.md \
  --confidence-threshold 0.3 \
  --unused-days 365

输出示例：

遗忘报告：
  总条目: 15
  保留: 12
  遗忘: 3

将被遗忘的条目:
  - [规则] 过时配置：置信度 0.2 < 阈值 0.3
  - [项目] 已结束项目X：未使用 400 天 > 365 天

置信度机制详解 ​

每条记忆都有一个置信度字段（0.0 ~ 1.0），动态调整：

事件调整
新条目0.7（初始值）
被用户确认正确+0.1
在多个任务中被验证+0.2
被用户纠正-0.3
与新获取的信息矛盾-0.2
超过一年未使用每月 -0.01

置信度等级参考：

置信度范围含义行动
0.9 ~ 1.0高度确认，从未出错保留，可作为规则
0.7 ~ 0.9基本正确保留，继续使用验证
0.5 ~ 0.7存疑，需要验证谨慎使用，标记待验证
0.3 ~ 0.5可能错误降权，不作为决策依据
< 0.3大概率错误遗忘阈值，触发删除

Cron 配置 ​

yaml
# 每周五 17:00 蒸馏
cron:
  name: weekly-distill
  schedule: "0 17 * * 5"
  session: isolated
  action:
    kind: agentTurn
    message: |
      cd /path/to/workspace
      python3 /path/to/distill.py --workspace . --days 7

# 每月最后一天 0:00 遗忘
cron:
  name: monthly-forget
  schedule: "0 0 L * *"  # L = 月份最后一天
  session: isolated
  action:
    kind: agentTurn
    message: |
      python3 /path/to/forget.py \
        --memory-file /path/to/MEMORY.md \
        --confidence-threshold 0.3 \
        --unused-days 365

常见问题 ​

Q：日记忆文件太多怎么办？ A：蒸馏时自动合并，遗忘时自动清理，历史文件超过 30 天可归档。

Q：蒸馏会不会丢失重要信息？ A：不会。蒸馏是合并到 MEMORY.md，不是删除日记忆。日记忆保留原文件。

Q：置信度如何初始设定？ A：新条目默认 0.7；来自用户明确告知的规则设为 0.95；来自外部信息的设为 0.6。

Q：遗忘机制会不会删掉有用的知识？ A：有两个保护：1）365 天未使用才删除，给长周期知识缓冲；2）只有置信度 < 0.3 才触发，初期大部分知识不会触及。
