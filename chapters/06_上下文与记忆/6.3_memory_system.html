本节目标 ​

  • 理解记忆系统的架构

6.3.1 记忆系统概述 ​

记忆类型 ​

┌─────────────────────────────────────────────┐
│           OpenClaw 记忆系统                   │
├─────────────────────────────────────────────┤
│                                             │
│  短期记忆 (Working Memory)                  │
│  ├── 当前会话信息                           │
│  ├── 工作进度                               │
│  └── 临时变量                               │
│                                             │
│  长期记忆 (Long-term Memory)               │
│  ├── 用户偏好                               │
│  ├── 项目知识                               │
│  └── 跨会话学习                             │
│                                             │
│  语义记忆 (Semantic Memory)                │
│  ├── 事实知识                               │
│  ├── 技能知识                               │
│  └── 世界知识                               │
│                                             │
└─────────────────────────────────────────────┘

![记忆层次]/assets/diagrams/09_memory_hierarchy.png)

记忆 vs 上下文 ​

区别：

上下文 (Context)
├── 每次请求发送给 LLM
├── 有 Token 限制
├── 会话级别
└── 临时性

记忆 (Memory)
├── 按需加载到上下文
├── 可持久化存储
├── 跨会话
└── 可索引搜索

![记忆层次]/assets/diagrams/09_memory_hierarchy.png)

6.3.2 短期记忆 ​

工作原理 ​

短期记忆流程：

用户：记住我的项目叫 "OpenClaw"

═══════════════════════════════════════════════

Step 1: 提取记忆
  → "项目名称 = OpenClaw"
  → 类型：user_preference

Step 2: 存储到工作内存
  → session[project_name] = "OpenClaw"

Step 3: 后续使用
  用户：构建项目
  → 自动使用："OpenClaw"

═══════════════════════════════════════════════

![记忆层次]/assets/diagrams/09_memory_hierarchy.png)

配置短期记忆 ​

json
{
  "memory": {
    "working": {
      "enabled": true,
      "max_items": 50,
      "ttl": 3600,
      "auto_recall": true
    }
  }
}

6.3.3 长期记忆 ​

功能 ​

长期记忆能力：

1. 用户偏好记忆
   ├── 编程语言偏好
   ├── 文档风格
   └── 交互习惯

2. 项目知识记忆
   ├── 项目结构
   ├── 常用命令
   └── 代码规范

3. 学习能力
   ├── 从交互中学习
   ├── 总结规律
   └── 主动建议

配置长期记忆 ​

json
{
  "memory": {
    "longterm": {
      "enabled": true,
      "storage": "file",
      "path": "~/.openclaw/memory",
      "index": "vector",
      "auto_save": true,
      "save_interval": 300
    }
  }
}

6.3.4 记忆存储 ​

存储后端 ​

支持的后端：

1. 文件存储 (File)
   ├── 简单易用
   ├── 适合个人
   └── ~/.openclaw/memory/

2. SQLite
   ├── 轻量级数据库
   ├── 支持查询
   └── 适合中小规模

3. 向量数据库 (Vector)
   ├── 语义搜索
   ├── 支持相似度
   └── Pinecone/Weaviate/Qdrant

配置示例 ​

json
{
  "memory": {
    "storage": {
      "backend": "file",
      "path": "~/.openclaw/memory",
      "encryption": false
    }
  }
}
json
{
  "memory": {
    "storage": {
      "backend": "vector",
      "provider": "qdrant",
      "url": "http://localhost:6333",
      "collection": "openclaw_memory"
    }
  }
}

6.3.5 记忆检索 ​

检索方式 ​

检索类型：

1. 精确匹配
   → 关键词完全匹配
   → 速度快

2. 语义搜索
   → 理解意图
   → 支持相似度
   → 使用向量索引

3. 混合搜索
   → 结合关键词和语义
   → 效果最好

检索配置 ​

json
{
  "memory": {
    "retrieval": {
      "method": "hybrid",
      "top_k": 5,
      "threshold": 0.7,
      "boost_recent": true
    }
  }
}

6.3.6 记忆操作 ​

手动操作 ​

记忆命令：

1. 记住信息
   用户：记住我的 API Key 是 xxx
   → 保存到长期记忆

2. 回忆信息
   用户：之前记住的 API Key 是什么？
   → 从记忆检索

3. 列出记忆
   用户：记住我哪些信息？
   → 列出所有记忆

4. 删除记忆
   用户：删除所有记忆
   → 清理记忆存储

编程操作 ​

python
# 保存记忆
await memory.save(
    key="user_preference",
    value={"language": "python"},
    ttl=86400 * 30  # 30天
)

# 检索记忆
results = await memory.search(
    query="编程偏好",
    top_k=5
)

# 删除记忆
await memory.delete("user_preference")

6.3.7 ⚠️ memory_search / memory_get 在 Subagent 中被禁用（v2026.3.24 #55385） ​

问题：在 Subagent（子 Agent）环境中调用 memory_search 或 memory_get 工具时，请求被硬编码拒绝，工具不可用。

根因：memory_search 和 memory_get 被列入 SUBAGENT_TOOL_DENY_ALWAYS 黑名单，Subagent 永远无法调用这两个工具。

影响范围：

  • Subagent 无法主动检索长期记忆
  • Subagent 无法读取 MEMORY.md 或 daily memory 文件
  • 在 isolated session 中运行的 Cron Job 任务也无法使用记忆检索

临时解决方案：目前无有效 workaround，Subagent 如需记忆只能通过以下方式绕行：

  1. 主 Agent 中转：在主 session 先检索记忆，再通过 sessions_send 将结果传给 Subagent
  2. 文件注入：在 Subagent 启动时通过 task 参数直接传入关键记忆内容

示例 — 主 Agent 中转：

json
{
  "task": "基于以下记忆内容回答用户问题：\n${MEMORY_CONTENT}\n\n用户问题：分析今日运营数据",
  "runtime": "subagent",
  "mode": "run"
}

官方修复跟踪：Issue #55385

6.3.8 常见问题 ​

Q1: 记忆不生效 ​

问题：记住的信息没有被使用

解决：
1. 检查是否保存成功
2. 检查检索关键词
3. 确认 auto_recall 启用

Q2: 隐私问题 ​

问题：担心敏感信息泄露

解决：
1. 启用加密存储
2. 排除敏感字段
3. 使用本地存储

Q3: 记忆过多 ​

问题：检索变慢

解决：
1. 定期清理
2. 使用向量索引
3. 分类存储

  4. 检索：精确、语义、混合
