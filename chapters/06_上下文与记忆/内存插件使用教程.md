## 什么是内存插件？

OpenClaw 的内存（Memory）系统让你的 AI 助手能够**跨会话记住重要信息**。默认使用内置的 `memory-core` 插件（SQLite 后端），也支持 QMD、Honcho 等高级引擎。

### 核心能力

- **自动记忆写入** — 每次会话结束前，Agent 会自动将重要上下文刷新到内存文件
- **语义搜索** — 用自然语言搜索历史记忆，无需精确关键词
- **多后端支持** — 内置 SQLite、QMD（本地优先）、Honcho（AI 原生跨会话）
- **自动索引** — 支持 OpenAI / Gemini / Ollama / 本地 GGUF 等 Embedding 提供商

## 快速开始

### 查看内存状态

```bash
openclaw memory status
```

带详细信息：
```bash
openclaw memory status --deep
```

### 手动触发索引

```bash
openclaw memory index --force
```

查看索引详情：
```bash
openclaw memory index --verbose
```

### 搜索记忆

```bash
openclaw memory search "会议记录"
```

指定结果数量：
```bash
openclaw memory search --query "项目计划" --max-results 20
```

### 特定 Agent 的记忆

```bash
openclaw memory status --agent main
openclaw memory index --agent main --verbose
```

## 配置内存插件

配置文件位置：`~/.openclaw/openclaw.json`

### 基本配置示例

```json
{
  "memory": {
    "provider": "openai",
    "model": "text-embedding-3-small"
  }
}
```

### 支持的 Embedding 提供商

| 提供商 | ID | 自动检测 | 备注 |
|--------|-----|---------|------|
| OpenAI | `openai` | ✅ | 默认使用 `text-embedding-3-small` |
| Gemini | `gemini` | ✅ | 支持多模态（图片+音频） |
| Ollama | `ollama` | ❌ | 本地部署，需显式设置 |
| 本地 GGUF | `local` | ✅ | 约 0.6GB 自动下载 |

### 进阶配置（参考 [memory-config](https://docs.openclaw.ai/reference/memory-config)）

```json
{
  "memory": {
    "provider": "gemini",
    "model": "gemini-embedding-exp-03-07",
    "remote": {
      "apiKey": "your-api-key"
    },
    "store": {
      "path": "~/.openclaw/memory/{agentId}.sqlite"
    },
    "sessions": {
      "enabled": true
    }
  }
}
```

**关键配置项说明：**

- `sessions.enabled: true` — 开启会话转录索引（实验性功能）
- `remote.batch.enabled: true` — 启用批量 Embedding API
- `local.modelPath` — 本地 GGUF 模型路径

### 禁用内存插件

如需完全禁用：

```json
{
  "plugins": {
    "slots": {
      "memory": "none"
    }
  }
}
```

## 内存工具（Agent 侧）

在你的 Agent 中，可以通过以下工具与内存交互：

### memory_search

语义搜索记忆内容。**每次对话前自动调用**，确保 Agent 能获取相关历史上下文。

```markdown
# 使用方式（由 Agent 自动调用）
memory_search(query="用户之前的项目偏好", maxResults=5)
```

### memory_get

安全读取 MEMORY.md 或 memory/*.md 的指定片段，常在收到用户提问后主动获取相关记忆。

```markdown
memory_get(path="MEMORY.md", from=1, lines=20)
```

## 自动内存刷新机制

OpenClaw 在**压缩（Compaction）**之前会运行一个静默轮次，提醒 Agent 将重要上下文保存到内存文件。

```
会话进行 → Compaction 触发 → 静默提醒 → Agent 写入 memory/ → 对话摘要生成
```

这确保即使长时间对话，关键信息也不会丢失。

## 常见问题

### Q: 内存索引占用空间大怎么办？

使用 `memory status --deep` 查看索引大小。定期清理：
```bash
# 删除旧索引文件
rm -rf ~/.openclaw/memory/*.sqlite
# 重新索引
openclaw memory index --force
```

### Q: 如何只搜索特定 Agent 的记忆？

```bash
openclaw memory search --query "关键词" --agent <agent-id>
```

### Q: 内存搜索结果不准确？

- 检查 Embedding 提供商配置（推荐 OpenAI 或 Gemini）
- 调高 `--max-results` 增加候选数量
- 确保记忆文件中有相关内容

### Q: API Key 如何安全配置？

使用 SecretRef 引用环境变量，避免明文写入配置：
```json
{
  "memory": {
    "remote": {
      "apiKey": "${OPENAI_API_KEY}"
    }
  }
}
```
