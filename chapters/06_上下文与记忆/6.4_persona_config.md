## 6.4.1 什么是人设

### 人设的概念

```
┌─────────────────────────────────────────────┐
│              人设 (Persona)                   │
├─────────────────────────────────────────────┤
│                                             │
│  人设 = Agent 的身份、性格、行为模式          │
│                                             │
│  包含内容：                                  │
│  ├── 角色定义：做什么工作                    │
│  ├── 性格特点：怎么说话                      │
│  ├── 专业领域：擅长什么                      │
│  └── 交互风格：如何回复                      │
│                                             │
│  类似：游戏中的 NPC 设定                      │
│                                             │
└─────────────────────────────────────────────┘
```

![Agent Persona](./.assets/diagrams/31_agent_persona.png)



### SOUL.md vs USER.md

```
两个核心文件：

1. SOUL.md
   └── Agent 的灵魂
   └── 定义 Agent 的身份和能力

2. USER.md
   └── 用户画像
   └── 告诉 Agent 用户是谁
```

---

## 6.4.2 SOUL.md 配置

### 文件位置

```
位置：
~/.openclaw/SOUL.md

或项目内：
./openclaw/SOUL.md
```

### 配置示例

```markdown
# SOUL.md - Agent 灵魂

## 身份
- 名称：CodeMaster
- 角色：专业程序员
- 经验：10年开发经验

## 能力
- 精通：Python, JavaScript, Go
- 擅长：代码审查、性能优化、架构设计
- 工具：Git, Docker, K8s

## 性格
- 回答简洁、专业
- 注重代码质量
- 乐于分享知识

## 行为
- 主动提供最佳实践
- 给出具体代码示例
- 解释背后的原理

## 约束
- 不写有安全漏洞的代码
- 遵循项目代码规范
- 考虑性能影响
```

## 6.4.3 USER.md 配置

### 文件位置

```
位置：
~/.openclaw/USER.md
```

### 配置示例

```markdown
# USER.md - 用户画像

## 基本信息
- 身份：全栈开发者
- 经验：3年

## 技术栈
- 主要：Python, React
- 了解：Go, Rust
- 工具：VSCode, Git

## 偏好
- 代码风格：简洁、注释完善
- 文档：详细
- 沟通：直接

## 项目背景
- 当前项目：电商平台
- 技术：Vue3 + Node.js
- 规模：中型项目
```

## 6.4.4 动态人设

### 按会话切换

```json
{
  "persona": {
    "default": "coder",
    "profiles": {
      "coder": {
        "soul": "./personas/coder.md",
        "user": "./personas/user.md"
      },
      "writer": {
        "soul": "./personas/writer.md",
        "user": "./personas/user.md"
      }
    }
  }
}
```

### 使用示例

```
切换人设：
@switch-persona writer

回复风格会立即切换到作家模式
```

## 6.4.5 提示词注入

### 注入方式

```
注入位置：System Prompt

═══════════════════════════════════════════════

[基础系统提示]
[SOUL.md 内容]
[USER.md 内容]
[当前任务]

═══════════════════════════════════════════════
```

### 条件注入

```json
{
  "persona": {
    "dynamic_injection": {
      "enabled": true,
      "triggers": {
        "code_task": "coding_persona.md",
        "write_task": "writing_persona.md"
      }
    }
  }
}
```

## 6.4.6 高级配置

### 多语言支持

```markdown
# 多语言 Persona

## 中文回复
- 使用简体中文
- 使用中文标点

## English Reply
- Use English punctuation
- Keep it concise
```

### 专业领域

```markdown
# 医疗领域 Persona

## 专业背景
- 医学知识：资深
- 术语：使用标准医学术语
- 免责声明：提供健康建议需谨慎

## 回答规范
- 建议用户咨询专业医生
- 不提供具体诊疗建议
- 引用权威来源
```

## 6.4.7 常见问题

### Q1: Persona 不生效

```
问题：修改后没有变化

解决：
1. 重启 OpenClaw
2. 检查文件格式
3. 确认路径正确
```

### Q2: Persona 冲突

```
问题：多个 Persona 冲突

解决：
1. 明确优先级
2. 使用动态切换
3. 清理冲突定义
```

### Q3: Token 占用过多

```
问题：Persona 内容太长

解决：
1. 精简内容
2. 使用引用
3. 动态加载
```
