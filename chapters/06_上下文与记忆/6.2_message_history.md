## 6.2.1 消息结构

### 消息类型

```
消息类型：

1. user - 用户消息
2. assistant - AI 回复
3. system - 系统消息
4. tool - 工具调用
5. tool_result - 工具结果
```

### 完整结构

```json
{
  "id": "msg_123456",
  "type": "user",
  "role": "user",
  "content": "帮我读取这个文件",
  "timestamp": "2024-01-15T10:30:00Z",
  "attachments": [
    {
      "type": "file",
      "path": "/path/to/file"
    }
  ],
  "metadata": {
    "channel": "cli",
    "user_id": "user_123"
  }
}
```

## 6.2.2 历史管理策略

### 保留策略

```
历史消息保留方式：

1. 全部保留
   ├── 保留完整历史
   ├── 最耗 Token
   └── 最完整上下文

2. 保留最近 N 条
   ├── 保留最近消息
   ├── 简单有效
   └── 可能丢失早期关键信息

3. 摘要压缩
   ├── 压缩为摘要
   ├── 平衡信息和 Token
   └── 需要好的摘要模型

4. 重要性筛选
   ├── 只保留重要消息
   ├── 需要标记重要性
   └── 智能筛选
```

### 配置示例

```json
{
  "history": {
    "strategy": "compress",
    "max_messages": 50,
    "max_tokens": 20000,
    "compression": {
      "enabled": true,
      "threshold": 15000,
      "summary_model": "gpt-4o-mini"
    }
  }
}
```

## 6.2.3 消息压缩

### 压缩触发条件

```
触发条件：

1. Token 超限
   → 当前 Token > 阈值
   → 触发压缩

2. 手动触发
   → 用户命令
   → @compress

3. 定期触发
   → 每 N 条消息
   → 定时压缩
```

### 压缩流程

```
原始消息（100条）

═══════════════════════════════════════════════

Step 1: 分段
  → 分为 5 组，每组 20 条

Step 2: 摘要每组
  → 组1: "用户询问了文件操作..."
  → 组2: "AI 读取并分析了文件..."
  → 组3: "用户追问细节..."
  → ...

Step 3: 合并摘要
  → 生成整体摘要

Step 4: 替换原消息
  → 删除原始消息
  → 保留摘要 + 最近 N 条

═══════════════════════════════════════════════

压缩后：摘要 + 10条原始消息
```

### 压缩提示词

```markdown
# 消息摘要提示词

请将以下消息历史压缩为简洁的摘要：

## 要求
1. 保留关键信息
2. 保留用户意图
3. 保留重要结论
4. 删除重复内容
5. 保持时间线清晰

## 格式
用 3-5 句话概括：
- 用户的核心需求
- AI 的主要回复
- 关键结论或操作
```

## 6.2.4 上下文窗口

### 窗口管理

```
上下文窗口概念：

┌─────────────────────────────────────────────┐
│              上下文窗口                       │
├─────────────────────────────────────────────┤
│                                             │
│  [窗口内消息] ← 发送给 LLM                  │
│  ─────────────                             │
│  用户: 你好                                 │

│  用户: 帮我查天气                          │

│  [历史消息] ← 已处理，不重复发送            │
│                                             │
└─────────────────────────────────────────────┘
```

## 6.2.5 会话隔离

### 按会话隔离

```
会话隔离机制：

┌─────────────────────────────────────────────┐
│           会话隔离                             │
├─────────────────────────────────────────────┤
│                                             │
│  Session A                                   │
│  ├── 用户 A1                                │
│  ├── AI A1                                  │
│  └── 独立上下文                             │
│                                             │
│  Session B                                   │
│  ├── 用户 B1                                │
│  ├── AI B1                                  │
│  └── 独立上下文                             │
│                                             │
│  Global (可选)                              │
│  └── 跨会话共享信息                         │
│                                             │
└─────────────────────────────────────────────┘
```

### 配置会话

```json
{
  "session": {
    "default_isolation": "session",
    "global_session": "global_shared",
    "cross_session_memory": false
  }
}
```

## 6.2.6 常见问题

### Q1: 历史消息丢失

```
问题：切换会话后历史丢失

解决：
1. 使用相同 sessionKey
2. 启用跨会话记忆
3. 手动传递上下文
```

### Q2: 压缩后信息不完整

```
问题：摘要质量差

解决：
1. 优化摘要提示词
2. 使用更好的模型
3. 增加原始消息保留数量
```

### Q3: Token 仍然超限

```
问题：压缩后仍然超限

解决：
1. 减少保留消息数
2. 使用更强压缩
3. 分段处理长任务
```
