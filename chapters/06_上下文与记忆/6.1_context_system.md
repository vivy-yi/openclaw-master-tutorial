## 6.1.1 什么是上下文

### 上下文的定义

```
┌─────────────────────────────────────────────┐
│              上下文 (Context)                 │
├─────────────────────────────────────────────┤
│                                             │
│  上下文 = 发送给 LLM 的完整信息              │
│                                             │
│  包含内容：                                  │
│  ├── 系统提示 (System Prompt)               │
│  ├── 用户消息 (User Message)                │
│  ├── 历史消息 (History Messages)            │
│  ├── 工具结果 (Tool Results)                │
│  ├── 记忆内容 (Memory)                     │
│  └── 额外上下文 (Extra Context)            │
│                                             │
│  限制：受模型 max_tokens 限制               │
│                                             │
└─────────────────────────────────────────────┘
```

![上下文管理](./.assets/diagrams/103_context_management.png)



![上下文窗口](./.assets/diagrams/33_context_window.png)




### 上下文的作用

```
为什么需要上下文：

1. 保持对话连贯
   ├── 记住之前的对话内容
   └── 理解用户意图

2. 提供背景信息
   ├── 项目信息
   └── 用户偏好

3. 引用历史数据
   ├── 之前的分析
   └── 生成的代码
```

## 6.1.3 上下文组装流程

### 组装步骤

```
用户消息：帮我审查这段代码

═══════════════════════════════════════════════

Step 1: 加载系统提示
  → OpenClaw 基础指令
  → "你是一个专业的代码审查助手"

Step 2: 加载 Persona
  → 用户配置的风格
  → "回答要简洁、专业"

Step 3: 加载 Skills
  → 激活的 Skill 提示词
  → "代码审查标准..."

Step 4: 加载用户消息
  → "帮我审查这段代码"
  → 附带的代码内容

Step 5: 加载历史消息
  → 最近 10 条对话
  → "之前的审查结果..."

Step 6: 加载工具结果
  → 之前的文件读取结果
  → "代码内容：..."

Step 7: 加载记忆
  → 用户偏好
  → 项目背景

Step 8: Token 预算检查
  → 计算总 Token
  → 触发压缩（如超限）

═══════════════════════════════════════════════

组装完成 → 发送给 LLM
```

## 6.1.4 上下文限制

### 模型限制

```
常见模型上下文限制：

| 模型 | 上下文限制 |
|-----|----------|
| GPT-4o | 128K |
| GPT-4o-mini | 128K |
| Claude 3.5 | 200K |
| Claude 3 Haiku | 200K |
| DeepSeek V3 | 64K |
| Kimi | 200万字 |

注意：实际可用 = 限制 - 输出预留
```

### 超出限制的处理

```
处理策略：

1. 消息压缩
   → 合并历史消息为摘要
   → 保留关键信息

2. 截断策略
   → 保留最近消息
   → 优先保留用户消息

3. 渐进摘要
   → 每次只摘要一部分
   → 平衡历史和现在

4. 人工介入
   → 请求确认关键信息
   → 清理不相关内容
```

## 6.1.5 上下文优化

### 优化技巧

```
优化建议：

1. 精简系统提示
   ├── 删除冗余指令
   └── 使用占位符

2. 压缩历史
   ├── 启用自动摘要
   └── 保留关键转折

3. 精简工具结果
   ├── 提取关键信息
   └── 避免重复内容

4. 分段处理
   ├── 长文档分段输入
   └── 多次交互处理
```


### 配置示例

```json
{
  "context": {
    "max_tokens": 80000,
    "system_prompt": {
      "max_tokens": 8000,
      "compress_on_limit": true
    },
    "history": {
      "max_messages": 20,
      "compression": {
        "enabled": true,
        "threshold": 10000
      }
    },
    "tool_results": {
      "max_tokens": 10000,
      "truncate": true
    }
  }
}
```

## 6.1.6 常见问题

### Q1: 上下文超出限制

```
错误：Context length exceeded

解决：
1. 启用压缩
2. 减少历史消息
3. 精简工具结果
4. 使用更大模型
```

### Q2: 重要信息丢失

```
问题：压缩后关键信息丢失

解决：
1. 设置关键信息标记
2. 手动保存关键结论
3. 调整压缩策略
```

### Q3: 响应变慢

```
问题：上下文太长导致响应慢

解决：
1. 减少上下文长度
2. 使用更快的模型
3. 启用缓存
```
