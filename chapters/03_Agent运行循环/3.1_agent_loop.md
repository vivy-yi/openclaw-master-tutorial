## 3.1.1 什么是 Agent Loop

**Agent Loop（智能体循环）** 是 OpenClaw 处理用户请求的完整流程：接收消息 → 理解意图 → 规划任务 → 执行工具 → 返回结果。

```
┌─────────────────────────────────────────────────────────────────────┐
│                        Agent Loop 生命周期                           │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   用户消息                                                            │
│      │                                                               │
│      ▼                                                               │
│  ┌─────────────┐                                                    │
│  │  1. 入口   │  Gateway 接收消息，验证参数                        │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│  ┌─────────────┐                                                    │
│  │ 2. 会话解析 │  加载/创建会话，验证权限                           │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│  ┌─────────────┐                                                    │
│  │ 3. 上下文   │  组装系统提示，加载 Skills                        │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│  ┌─────────────┐                                                    │
│  │ 4. 模型推理 │  LLM 分析意图，规划任务                           │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│  ┌─────────────┐                                                    │
│  │ 5. 工具执行 │  调用工具，获取结果                               │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│  ┌─────────────┐                                                    │
│  │ 6. 流式输出 │  实时返回生成内容                                 │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│  ┌─────────────┐                                                    │
│  │ 7. 持久化   │  保存会话状态，更新记忆                           │
│  └──────┬──────┘                                                    │
│         │                                                            │
│         ▼                                                            │
│      返回结果                                                         │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 3.1.2 详细流程解析

### 步骤一：入口接收

```
用户消息 → Gateway RPC

支持的入口：
├── agent RPC (异步，立即返回)
├── agent.wait RPC (同步，等待完成)
└── CLI 命令
```


**参数验证**：
- sessionKey / sessionId
- 模型配置
- 超时设置

### 步骤二：会话解析

```
┌─────────────────────────────────────────────┐
│           会话管理                           │
├─────────────────────────────────────────────┤
│                                             │
│  1. 根据 sessionKey 查找会话                │
│     - 存在：加载历史上下文                   │
│     - 不存在：创建新会话                    │
│                                             │
│  2. 验证权限 Profile                       │
│     - 检查用户权限                          │
│     - 加载安全策略                          │
│                                             │
│  3. 准备工作区                             │
│     - 加载 workspace                       │
│     - 初始化沙箱（如需要）                  │
│                                             │
└─────────────────────────────────────────────┘
```


### 步骤三：上下文组装

```
┌─────────────────────────────────────────────┐
│              上下文组装                       │
├─────────────────────────────────────────────┤
│                                             │
│  1. 系统提示 (System Prompt)               │
│     ├── OpenClaw 基础提示                   │
│     ├── Skills 提示                        │
│     └── Bootstrap 上下文                    │
│                                             │
│  2. 用户提示 (User Prompt)                │
│     ├── 当前消息                            │
│     └── 历史消息                            │
│                                             │
│  3. Token 预算                            │
│     ├── 模型限制                            │
│     └── 压缩预留                           │
│                                             │
└─────────────────────────────────────────────┘
```


### 步骤四：模型推理

```
LLM 处理流程：

┌─────────────────────────────────────────────┐
│  输入：组装好的提示                          │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│  LLM 分析：                                  │
│  1. 理解用户意图                            │
│  2. 判断是否需要工具                        │
│  3. 如果需要，选择工具和参数                  │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│  输出：                                      │
│  • 纯文本回复                               │
│  • 工具调用请求                              │
│  • 或两者混合                               │
└─────────────────────────────────────────────┘
```


### 步骤五：工具执行

```
工具调用流程：

LLM 返回工具调用
       │
       ▼
┌─────────────────────────────────────────────┐
│  1. 参数验证                                │
│     - 检查工具是否存在                       │
│     - 验证参数格式                          │
└─────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────┐
│  2. 安全检查                                │
│     - Profile 权限验证                      │
│     - 危险命令拦截                          │
└─────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────┐
│  3. 执行工具                                │
│     - 调用对应工具                          │
│     - 收集执行结果                          │
└─────────────────────────────────────────────┘
       │
       ▼
┌─────────────────────────────────────────────┐
│  4. 返回结果                                 │
│     - 格式化工具输出                        │
│     - 添加到对话上下文                       │
└─────────────────────────────────────────────┘
       │
       ▼
    回到步骤四：再次调用 LLM
```


### 步骤六：流式输出

```
┌─────────────────────────────────────────────┐
│            流式输出                          │
├─────────────────────────────────────────────┤
│                                             │
│  事件类型：                                  │
│  ├── assistant: 助手消息片段                 │
│  ├── tool: 工具调用信息                    │
│  └── lifecycle: 生命周期事件               │
│      ├── start: 开始                         │
│      ├── end: 正常结束                      │
│      └── error: 错误                        │
│                                             │
└─────────────────────────────────────────────┘
```


### 步骤七：持久化

```
保存内容：
├── 会话消息历史
├── 工具调用记录
├── Token 使用统计
└── 长期记忆（如有）
```

## 3.1.3 并发与队列

### 会话队列

```
┌─────────────────────────────────────────────┐
│              队列机制                        │
├─────────────────────────────────────────────┤
│                                             │
│  每个会话 (sessionKey)                      │
│     └── 串行执行，确保历史一致                │
│                                             │
│  全局队列（可选）                           │
│     └── 跨会话串行                           │
│                                             │
└─────────────────────────────────────────────┘
```


### 队列模式

| 模式 | 说明 |
|-----|------|
| collect | 收集多条消息后批量处理 |
| steer | 实时控制 Agent 行为 |
| followup | 跟踪并响应执行过程 |

## 3.1.4 Hook 钩子机制

OpenClaw 提供两种钩子机制：

### Gateway 钩子

```
┌─────────────────────────────────────────────┐
│           Gateway 钩子                       │
├─────────────────────────────────────────────┤
│                                             │
│  • agent:bootstrap                         │
│    └── 构建 bootstrap 文件前运行             │
│                                             │
│  • /new, /reset, /stop 等命令             │
│    └── 命令事件触发                          │
│                                             │
└─────────────────────────────────────────────┘
```


### Plugin 钩子

```
┌─────────────────────────────────────────────┐
│            Plugin 钩子                       │
├─────────────────────────────────────────────┤
│                                             │
│  会话阶段：                                  │
│  ├── before_model_resolve                  │
│  ├── before_prompt_build                   │
│  ├── agent_end                             │
│                                             │
│  压缩阶段：                                  │
│  ├── before_compaction                     │
│  └── after_compaction                      │
│                                             │
│  工具阶段：                                  │
│  ├── before_tool_call                      │
│  ├── after_tool_call                       │
│  └── tool_result_persist                  │
│                                             │
│  消息阶段：                                  │
│  ├── message_received                      │
│  ├── message_sending                       │
│  └── message_sent                          │
│                                             │
│  会话阶段：                                  │
│  ├── session_start                          │
│  └── session_end                           │
│                                             │
└─────────────────────────────────────────────┘
```

## 3.1.5 一个完整的例子

```
用户输入: "帮我搜索最新的AI新闻"

═══════════════════════════════════════════════

Step 1: 入口接收
  ✓ 消息验证通过
  ✓ sessionKey: user_123

Step 2: 会话解析
  ✓ 找到会话: session_abc
  ✓ 加载历史: 5条消息
  ✓ 权限: read + web

Step 3: 上下文组装
  ✓ 系统提示: 1024 tokens
  ✓ 用户消息: "帮我搜索最新的AI新闻"
  ✓ 历史上下文: 500 tokens

Step 4: 模型推理
  → 调用 LLM (deepseek-chat)
  ← 需要调用 web search 工具

Step 5: 工具执行
  → 调用 web 工具
  → 搜索: "AI news 2024"
  ← 返回: 10条新闻摘要

Step 4: 模型推理 (第二轮)
  → LLM 处理结果
  ← 返回: 格式化后的新闻摘要

Step 6: 流式输出
  📤 正在搜索...
  📤 找到10条新闻...
  📤 整理中...

Step 7: 持久化
  ✓ 保存会话消息
  ✓ 更新 token 统计
  ✓ 保存工具调用记录

═══════════════════════════════════════════════

最终回复:
"以下是最新的AI新闻：
1. OpenAI 发布 GPT-5...
2. Claude 3.5 发布...
..."
```


---

## 本节小结

1. **Agent Loop**：接收 → 理解 → 规划 → 执行 → 返回
2. **七步流程**：入口→会话→上下文→推理→工具→输出→持久化
3. **队列机制**：会话内串行，保持历史一致性
4. **Hook 机制**：可在任意阶段拦截和扩展
