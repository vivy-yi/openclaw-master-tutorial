3.11.1 上下文核心概念 ​

什么是上下文？ ​

上下文（Context） 是 OpenClaw 发送给模型的所有内容，受模型的**上下文窗口（context window）**限制。

┌─────────────────────────────────────────────────────────────────────────────────┐
│                           上下文组成                                            │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  上下文 = 系统提示词 + 对话历史 + 工具调用/结果 + 附件                         │
│                                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │                    系统提示词 (System Prompt)                         │      │
│  │  • 工具列表 + 描述                                                    │      │
│  │  • 技能列表（元数据）                                                │      │
│  │  • 工作区位置                                                        │      │
│  │  • 时间信息                                                          │      │
│  │  • 运行时元数据                                                      │      │
│  │  • 注入的工作区文件                                                  │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│                                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │                    对话历史 (Conversation History)                     │      │
│  │  • 用户消息                                                          │      │
│  │  • Assistant 消息                                                     │      │
│  │  • 工具调用和结果                                                    │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│                                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │                    工具调用 (Tool Calls)                              │      │
│  │  • 工具调用                                                          │      │
│  │  • 工具结果                                                          │      │
│  │  • 附件/图片/音频                                                    │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

上下文 vs 记忆 ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                    上下文 vs 记忆                                             │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  上下文 (Context)                                                              │
│  • 当前模型窗口内的内容                                                        │
│  • 每次请求发送给模型                                                         │
│  • 受上下文窗口限制                                                           │
│                                                                                 │
│  记忆 (Memory)                                                                 │
│  • 存储在磁盘上的 Markdown 文件                                               │
│  • 可以稍后重新加载                                                          │
│  • 长期存储重要信息                                                           │
│                                                                                 │
│  📌 记忆文件写入磁盘后，需要通过工具调用才能加载到上下文中                    │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

系统提示词组成 ​

│                                                                                 │
│  3. 工作区信息                                                                │

│                                                                                 │
│  5. 运行时元数据                                                              │
│     • 主机/操作系统/模型信息                                                  │
│     • Thinking 模式                                                          │
│                                                                                 │
│  6. 项目上下文（Project Context）                                             │

注入的工作区文件 ​

默认情况下，OpenClaw 会注入以下工作区文件：

文件说明用途
AGENTS.mdAgent 规则定义 Agent 行为
SOUL.mdAgent 灵魂定义 Agent 个性
TOOLS.md工具说明补充工具文档
IDENTITY.md身份信息Agent 身份定义
USER.md用户信息用户偏好设置
HEARTBEAT.md心跳配置定时任务配置
BOOTSTRAP.md启动引导首次运行引导

文件截断配置 ​

json5
{
  agents: {
    defaults: {
      // 每个文件最大字符数（默认 20000）
      bootstrapMaxChars: 20000,
      // 所有文件总字符数上限（默认 150000）
      bootstrapTotalMaxChars: 150000,
      // 截断警告
      bootstrapPromptTruncationWarning: "once"  // off | once | always
    }
  }
}

3.11.4 上下文压缩（Compaction） ​

什么是压缩？ ​

压缩将较旧的对话历史总结为紧凑的摘要，保留最近的消息。摘要存储在会话历史中。

┌─────────────────────────────────────────────────────────────────────────────────┐
│                           压缩前后对比                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  压缩前：                                                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │ [消息1] → [消息2] → [消息3] → [消息4] → [消息5] → ... → [消息N] │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│                                                                                 │
│  压缩后：                                                                     │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │ [摘要] → [最近的消息] → ... → [消息N]                              │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│                                                                                 │
│  📌 摘要持久化到会话的 JSONL 历史文件中                                      │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

自动压缩（默认开启） ​

当会话接近或超过模型的上下文窗口时，OpenClaw 会触发自动压缩：

json5
{
  agents: {
    defaults: {
      compaction: {
        // 压缩目标（保留多少 Token）
        targetTokens: 16000,
        // 软阈值
        softThresholdTokens: 4000,
        // 保留 Token 下限
        reserveTokensFloor: 20000
      }
    }
  }
}

手动压缩 ​

/compact
/compact 关注决策和开放问题

压缩模型配置 ​

可以指定不同的模型进行压缩摘要：

json5
{
  agents: {
    defaults: {
      compaction: {
        // 使用更便宜的模型进行压缩
        model: "ollama/llama3.1:8b"
        // 或使用远程模型
        // model: "openrouter/anthropic/claude-sonnet-4-5"
      }
    }
  }
}

压缩 vs 修剪 ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                      压缩 vs 修剪                                             │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  压缩 (Compaction)                                                            │
│  • 总结旧对话为摘要                                                          │
│  • 持久化到 JSONL 文件                                                       │
│  • 摘要保留在上下文中                                                        │
│                                                                                 │
│  修剪 (Pruning)                                                               │
│  • 仅删除旧的工具结果                                                        │
│  • 内存中执行，不修改磁盘文件                                                │
│  • 临时性的，每次请求重新计算                                                │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

3.11.5 会话修剪（Session Pruning） ​

什么是修剪？ ​

修剪在每次 LLM 调用前从内存中删除旧的工具结果，减少上下文膨胀。

┌─────────────────────────────────────────────────────────────────────────────────┐
│                           修剪前后对比                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  修剪前：                                                                     │
│  [用户消息] → [工具调用1] → [工具结果1] → [工具调用2] → [工具结果2] → ...  │
│                                                                                 │
│  修剪后：                                                                     │
│  [用户消息] → [工具调用1] → [...截断...] → [工具调用2] → [工具结果2] → ...  │
│                                                                                 │
│  📌 修剪仅影响内存中的上下文，不修改磁盘上的 JSONL 文件                       │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

修剪配置 ​

json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",     // 模式: off | cache-ttl
        ttl: "5m",             // 缓存过期时间
        keepLastAssistants: 3, // 保留最近 N 条助手消息
        softTrimRatio: 0.3,    // 软删除比例
        hardClearRatio: 0.5    // 硬删除比例
      }
    }
  }
}

软删除 vs 硬删除 ​

类型说明
软删除 (Soft-trim)保留头部和尾部，中间显示 ...，标注原始大小
硬删除 (Hard-clear)将整个工具结果替换为占位符

工具选择 ​

json5
{
  agents: {
    defaults: {
      contextPruning: {
        mode: "cache-ttl",
        tools: {
          allow: ["exec", "read"],  // 只修剪这些工具
          deny: ["*image*"]         // 保留图片相关工具结果
        }
      }
    }
  }
}

3.11.6 记忆与上下文 ​

记忆文件布局 ​

~/.openclaw/workspace/
├── memory/
│   ├── 2024-01-01.md
│   ├── 2024-01-02.md
│   └── ...
├── MEMORY.md              # 长期记忆
└── AGENTS.md             # Agent 规则

记忆文件说明 ​

文件用途加载时机
memory/YYYY-MM-DD.md每日记录加载今天 + 昨天的内容
MEMORY.md长期记忆仅加载到主会话

什么时候写记忆？ ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         记忆写入时机                                           │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  写入 MEMORY.md：                                                              │
│  • 重要决策                                                                    │
│  • 用户偏好                                                                    │
│  • 持久化的事实                                                                │
│                                                                                 │
│  写入 memory/YYYY-MM-DD.md：                                                 │
│  • 日常笔记                                                                    │
│  • 正在进行的任务上下文                                                        │
│                                                                                 │
│  📌 如果用户说"记住这个"，一定要写下来                                         │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

自动记忆刷新（Pre-compaction） ​

当会话接近压缩时，OpenClaw 会触发静默的记忆刷新：

json5
{
  agents: {
    defaults: {
      compaction: {
        reserveTokensFloor: 20000,
        memoryFlush: {
          enabled: true,
          softThresholdTokens: 4000,
          systemPrompt: "会话即将压缩。现在存储持久记忆。",
          prompt: "将任何持久的笔记写入 memory/YYYY-MM-DD.md；如无需存储则回复 NO_REPLY。"
        }
      }
    }
  }
}

3.11.7 向量记忆搜索 ​

概述 ​

OpenClaw 可以在 MEMORY.md 和 memory/*.md 上构建向量索引，支持语义搜索。

配置 ​

json5
{
  agents: {
    defaults: {
      memorySearch: {
        enabled: true,
        // 使用远程嵌入（默认）
        provider: "openai",  // openai | gemini | voyage | mistral | local | ollama
        // 或使用本地
        local: {
          modelPath: "/path/to/model"
        }
      }
    }
  }
}

后端选择 ​

后端说明
SQLite (默认)内置向量搜索
QMD实验性：BM25 + 向量 + 重排序
json5
{
  memory: {
    backend: "qmd"  // 使用 QMD 后端
  }
}

3.11.8 上下文优化最佳实践 ​

优化技巧 ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                       上下文优化技巧                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  1. 使用 /compact 定期清理                                                    │
│     • 会话感觉陈旧或膨胀时使用                                                │
│                                                                                 │
│  2. 重要信息写入记忆                                                          │
│     • 不要依赖上下文记住重要信息                                              │

│                                                                                 │
│  3. 配置合适的修剪策略                                                         │
│     • 启用 contextPruning 减少工具结果堆积                                    │
│                                                                                 │
│  4. 控制工作区文件大小                                                         │
│     • 使用 bootstrapMaxChars 限制注入大小                                    │
│     • 避免过大的 TOOLS.md                                                     │
│                                                                                 │
│  5. 使用更便宜的模型处理简单任务                                              │
│     • 配置 subagent 使用轻量模型                                              │
│                                                                                 │
│  6. 需要全新开始时使用 /new                                                  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

json5
{
  agents: {
    defaults: {
      // 压缩配置
      compaction: {

3.11.9 完整配置示例 ​

json5
{
  agents: {
    defaults: {
      // 工作区
      workspace: "~/.openclaw/workspace",
      // 模型
      model: "anthropic/claude-sonnet-4-5",

      // 压缩配置
      compaction: {
        // 压缩目标 Token 数
        targetTokens: 16000,
        // 软阈值
        softThresholdTokens: 4000,
        // 保留 Token 下限
        reserveTokensFloor: 20000,
        // 压缩使用的模型
        model: "anthropic/claude-haiku-3-5",
        // 标识符策略
        identifierPolicy: "strict",
        // 自动记忆刷新
        memoryFlush: {
          enabled: true,
          softThresholdTokens: 4000,
          systemPrompt: "会话即将压缩。现在存储持久记忆。",
          prompt: "将重要信息写入 MEMORY.md 或 memory/今天.md。"
        }
      },

      // 上下文修剪
      contextPruning: {
        mode: "cache-ttl",
        ttl: "5m",
        keepLastAssistants: 3,
        softTrimRatio: 0.3,
        hardClearRatio: 0.5,
        minPrunableToolChars: 50000,
        tools: {
          allow: ["exec", "read", "browser"],
          deny: ["*image*", "*screenshot*"]
        }
      },

      // 引导文件配置
      bootstrapMaxChars: 15000,
      bootstrapTotalMaxChars: 100000,
      bootstrapPromptTruncationWarning: "once",

      // 记忆搜索
      memorySearch: {
        enabled: true,
        provider: "openai"
      }
    }
  },

  memory: {
    backend: "default"  // default | qmd
  }
}

本节小结 ​

  1. 上下文组成：系统提示词 + 对话历史 + 工具调用结果
  2. 系统提示词：工具列表、技能列表、工作区信息、项目上下文
  3. 检查命令：/status、/context list、/context detail、/usage tokens
  4. 压缩：总结旧对话为摘要，持久化到 JSONL
  5. 修剪：内存中删除旧工具结果，不修改磁盘文件
  6. 记忆：Markdown 文件，磁盘存储，按需加载
  7. 向量搜索：语义搜索记忆内容
  8. 优化：定期压缩、启用修剪、写入记忆
