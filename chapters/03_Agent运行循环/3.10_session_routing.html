3.10 Session 路由完全指南 ​

本节目标 ​

  • 理解消息如何路由到 Session
  • 掌握 Binding 配置规则
  • 理解 Session Key 构建逻辑
  • 学会配置不同渠道的路由

3.10.1 路由核心概念 ​

什么是 Session 路由？ ​

Session 路由确定哪条消息进入哪个 Session，以及哪个 Agent 处理该消息。

┌─────────────────────────────────────────────────────────────────────────────────┐
│                           消息路由流程                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  用户消息                                                                      │
│      │                                                                       │
│      ▼                                                                       │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │                    消息解析                                           │      │
│  │  • 渠道 (channel)                                                   │      │
│  │  • 发送者 (peer)                                                    │      │
│  │  • 会话类型 (DM/群组/话题)                                          │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│      │                                                                       │
│      ▼                                                                       │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │                    Binding 匹配                                       │      │
│  │  优先级: peer > parentPeer > guildId > accountId > channel > default│      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│      │                                                                       │
│      ▼                                                                       │
│  ┌─────────────────────────────────────────────────────────────────────┐      │
│  │                    Session Key 构建                                   │      │
│  │  agent:<agentId>:<channel>:<type>:<id>                              │      │
│  └─────────────────────────────────────────────────────────────────────┘      │
│      │                                                                       │
│      ▼                                                                       │
│  Agent 处理 → 响应发送回原渠道                                               │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

3.10.2 Binding 路由规则 ​

路由优先级 ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         Binding 优先级顺序                                     │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  1. peer (精确匹配)                                                          │
│     • peer.kind + peer.id                                                    │
│     • 例如: { kind: "group", id: "-1001234567890" }                         │
│                                                                                 │
│  2. parentPeer (话题继承)                                                    │
│     • Telegram 话题继承父话题                                                 │
│                                                                                 │
│  3. guildId + roles (Discord 角色路由)                                      │
│     • 指定服务器 + 角色匹配                                                  │
│                                                                                 │
│  4. guildId (Discord)                                                       │
│     • 指定服务器                                                              │
│                                                                                 │
│  5. teamId (Slack)                                                           │
│     • 指定工作区                                                              │
│                                                                                 │
│  6. accountId (账户匹配)                                                     │
│     • 渠道账户 ID                                                            │
│                                                                                 │
│  7. channel (渠道级匹配)                                                     │
│     • accountId: "*" 匹配该渠道所有账户                                      │
│                                                                                 │
│  8. default (默认 Agent)                                                      │
│     • agents.list[].default                                                  │
│                                                                                 │
│  📌 多个同级别 Binding 匹配时，按配置顺序优先                                │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

![Session路由]/assets/diagrams/79_session_routing.png)

Binding 配置示例 ​

json5
{
  bindings: [
    // 1. 精确 peer 匹配 - 最高优先级
    {
      agentId: "work",
      match: {
        channel: "telegram",
        peer: { kind: "group", id: "-1001234567890" }
      }
    },
    // 2. Discord 角色路由
    {
      agentId: "developer",
      match: {
        channel: "discord",
        guildId: "123456789",
        roles: ["987654321"]
      }
    },
    // 3. 账户匹配
    {
      agentId: "support",
      match: {
        channel: "whatsapp",
        accountId: "support-account"
      }
    },
    // 4. 渠道级匹配
    {
      agentId: "main",
      match: {
        channel: "telegram"
      }
    },
    // 5. 默认 Agent
    {
      agentId: "main"
    }
  ]
}

3.10.3 Session Key 格式详解 ​

格式对照表 ​

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         Session Key 格式                                        │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  直接消息 (DM):                                                               │
│  ┌─────────────────────────────────────────────────────────────────────┐     │
│  │ dmScope: "main"                                                    │     │
│  │ agent:<agentId>:main                                               │     │
│  │ 例: agent:main:main                                                │     │
│  └─────────────────────────────────────────────────────────────────────┘     │
│  ┌─────────────────────────────────────────────────────────────────────┐     │
│  │ dmScope: "per-peer"                                               │     │
│  │ agent:<agentId>:direct:<peerId>                                    │     │
│  │ 例: agent:main:direct:123456789                                   │     │
│  └─────────────────────────────────────────────────────────────────────┘     │
│  ┌─────────────────────────────────────────────────────────────────────┐     │
│  │ dmScope: "per-channel-peer"                                        │     │
│  │ agent:<agentId>:<channel>:direct:<peerId>                         │     │
│  │ 例: agent:main:telegram:direct:123456789                           │     │
│  └─────────────────────────────────────────────────────────────────────┘     │
│                                                                                 │
│  群组消息:                                                                   │
│  agent:<agentId>:<channel>:group:<id>                                       │
│  例: agent:main:telegram:group:-1001234567890                               │
│                                                                                 │
│  话题/线程:                                                                  │
│  • Telegram 话题: agent:<agentId>:telegram:group:<id>:topic:<threadId>     │
│  • Discord 线程: agent:<agentId>:discord:channel:<id>:thread:<threadId>   │
│  • Slack 线程: agent:<agentId>:slack:channel:<id>:thread:<threadId>       │
│                                                                                 │
│  其他来源:                                                                    │
│  • Cron: cron:<job.id>                                                     │
│  • Webhook: hook:<uuid>                                                    │
│  • Node: node-<nodeId>                                                    │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

3.10.5 DM 路由详解 ​

dmScope 选项对路由的影响 ​

dmScopeSession Key 格式适用场景
mainagent:<agentId>:main单用户连续对话
per-peeragent:<agentId>:direct:<peerId>简单多用户
per-channel-peeragent:<agentId>:<channel>:direct:<peerId>推荐多用户多渠道
per-account-channel-peeragent:<agentId>:<channel>:<accountId>:direct:<peerId>多账户场景

主会话固定 (Main DM Route Pinning) ​

当 dmScope 设置为 main 时，直接消息共享主会话。为防止非所有者 DM 覆盖 lastRoute，OpenClaw 会推断固定所有者：

┌─────────────────────────────────────────────────────────────────────────────────┐
│                         主会话固定逻辑                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  当满足以下条件时，固定所有者：                                                 │
│  • allowFrom 只有一个非通配符条目                                              │
│  • 该条目可以规范化为该渠道的具体发送者 ID                                      │
│  • 入站 DM 发送者与固定所有者不匹配                                            │
│                                                                                 │
│  在不匹配情况下：                                                              │
│  • 仍记录入站会话元数据                                                        │
│  • 但跳过更新主会话的 lastRoute                                               │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

3.10.6 多账户路由 ​

多账户配置 ​

json5
{
  agents: {
    list: [
      { id: "personal", workspace: "~/workspace-personal" },
      { id: "work", workspace: "~/workspace-work" }
    ]
  },
  channels: {
    whatsapp: {
      accounts: {
        personal: {},
        work: {}
      }
    }
  },
  bindings: [
    // 路由到个人账户
    {
      agentId: "personal",
      match: {
        channel: "whatsapp",
        accountId: "personal"
      }
    },
    // 路由到工作账户
    {
      agentId: "work",
      match: {
        channel: "whatsapp",
        accountId: "work"
      }
    }
  ]
}

默认账户配置 ​

json5
{
  channels: {
    whatsapp: {
      defaultAccount: "personal"  // 设置默认账户
    }
  }
}

3.10.7 路由调试命令 ​

查看会话列表 ​

bash
# 查看所有会话
openclaw sessions --json

# 查看活跃会话（最近 60 分钟）
openclaw sessions --json --active 60

# 查看特定渠道的会话
openclaw sessions --json | jq '.[] | select(.channel == "telegram")'

调试路由 ​

bash
# 查看 Gateway 状态
openclaw status

# 查看渠道状态
openclaw channels status

# 查看特定会话
openclaw sessions history <session-key>

查看上下文 ​

在聊天中发送：

  • /status - 查看会话状态、上下文使用量
  • /context list - 查看系统提示和注入的工作区文件
  • /context detail - 查看上下文详情

3.10.8 完整配置示例 ​

json5
{
  // Agent 定义
  agents: {
    list: [
      {
        id: "main",
        name: "Main Assistant",
        workspace: "~/.openclaw/workspace-main",
        default: true
      },
      {
        id: "work",
        name: "Work Assistant",
        workspace: "~/.openclaw/workspace-work"
      },
      {
        id: "support",
        name: "Support Agent",
        workspace: "~/.openclaw/workspace-support"
      }
    ]
  },

  // Session 配置
  session: {
    dmScope: "per-channel-peer",
    mainKey: "main"
  },

  // 渠道配置
  channels: {
    whatsapp: {
      dmPolicy: "allowlist",
      allowFrom: ["+15551230001", "+15551230002"],
      accounts: {
        personal: {},
        work: {}
      }
    },
    telegram: {
      dmPolicy: "pairing",
      groups: {
        "-1001234567890": { allow: true },
        "-1009876543210": { allow: true }
      }
    },
    discord: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["123456789"]
    }
  },

  // 路由绑定
  bindings: [
    // WhatsApp 个人账户 → main
    {
      agentId: "main",
      match: {
        channel: "whatsapp",
        accountId: "personal"
      }
    },
    // WhatsApp 工作账户 → work
    {
      agentId: "work",
      match: {
        channel: "whatsapp",
        accountId: "work"
      }
    },
    // Telegram 群组 → support
    {
      agentId: "support",
      match: {
        channel: "telegram",
        peer: { kind: "group", id: "-1001234567890" }
      }
    },
    // Discord 服务器 → main
    {
      agentId: "main",
      match: {
        channel: "discord",
        guildId: "123456789"
      }
    }
  ],

  // 广播组
  broadcast: {
    strategy: "parallel",
    "120363000000000001@g.us": ["code-reviewer", "security", "docs"]
  }
}

本节小结 ​

  1. 路由优先级：peer > parentPeer > guildId > accountId > channel > default
  2. Session Key 格式：agent:<agentId>:<channel>:<type>:<id>
  3. DM 作用域：dmScope 控制 DM 会话隔离方式
  4. 多账户：通过 accountId 区分不同账户
  5. 广播组：多个 Agent 处理同一消息
  6. 调试命令：sessions list、status、context list
