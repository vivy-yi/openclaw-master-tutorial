## 9.4.2 定义工作流

### YAML 定义

```yaml
name: code_review_pipeline
version: 1.0
description: 自动化代码审查流程

triggers:
  - type: webhook
    event: push
  - type: schedule
    cron: "0 */6 * * *"

agents:
  scanner:
    role: security_scan
  reviewer:
    role: code_review
  notifier:
    role: notification

stages:
  - name: scan
    agent: scanner
    input: code
    output: scan_result

  - name: review
    agent: reviewer
    input: [code, scan_result]
    output: review_result

  - name: notify
    agent: notifier
    input: review_result
    condition: review_result.has_issues
```

## 9.4.3 编排逻辑

### 串行编排

```yaml
stages:
  - name: step1
    agent: agent1

  - name: step2
    agent: agent2
    input: step1.output

  - name: step3
    agent: agent3
    input: step2.output
```

### 并行编排

```yaml
stages:
  - name: parallel
    parallel:
      - agent: agent1
      - agent: agent2
      - agent: agent3
    output: combined_results
```

### 条件编排

```yaml
stages:
  - name: decision
    agent: analyzer

  - name: process_a
    condition: decision.result == "A"
    agent: agent_a

  - name: process_b
    condition: decision.result == "B"
    agent: agent_b
```

## 9.4.4 错误处理

### 重试机制

```yaml
error_handling:
  retry:
    enabled: true
    max_attempts: 3
    backoff: exponential
    initial_interval: 5
    multiplier: 2

  fallback:
    enabled: true
    action: notify_admin
```

### 错误通知

```yaml
notifications:
  on_error:
    channels:
      - type: email
        to: admin@example.com
      - type: slack
        channel: "#alerts"
    template: |
      工作流 {{workflow_name}} 执行失败

      错误：{{error}}
      阶段：{{stage}}
      时间：{{timestamp}}
```

## 9.4.5 监控配置

### 状态追踪

```yaml
monitoring:
  track:
    - execution_time
    - success_rate
    - token_usage
    - cost

  logging:
    level: info
    retention: 30d

  metrics:
    - name: workflow_duration
      type: histogram
      buckets: [60, 300, 900, 1800]
```

### 告警规则

```yaml
alerts:
  - name: slow_execution
    condition: duration > 300
    severity: warning
    channels: [slack]

  - name: high_failure_rate
    condition: failure_rate > 0.1
    severity: critical
    channels: [slack, email]
```

## 9.4.6 实战案例

### CI/CD 自动化

```yaml
name: ci_cd_pipeline

triggers:
  - type: webhook
    event: pull_request

stages:
  - name: build
    agent: builder

  - name: test
    parallel:
      - agent: unit_test
      - agent: integration_test

  - name: security_scan
    agent: security_scanner

  - name: deploy
    condition: all_passed
    agent: deployer
```
