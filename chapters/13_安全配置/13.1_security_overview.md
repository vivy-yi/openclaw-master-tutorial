## 11.1.2 安全原则

```
核心原则：

1. 最小权限
   └── 只授予必要权限

2. 纵深防御
   └── 多层安全保护

3. 零信任
   └── 每次请求都验证

4. 可审计
   └── 记录所有操作
```

## 11.1.3 安全配置

### 基础配置

```yaml
security:
  # 认证
  auth:
    enabled: true
    default_policy: deny

  # 加密
  encryption:
    enabled: true
    algorithm: AES-256-GCM

  # 审计
  audit:
    enabled: true
    retention: 90d
```

## 本节小结

1. **分层**：网络、认证、权限、数据、审计
2. **原则**：最小权限、纵深防御、零信任
3. **配置**：认证、加密、审计
