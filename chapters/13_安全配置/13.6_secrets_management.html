11.6 密钥管理：SecretRef 契约与安全实践 ​

本节讲解 OpenClaw 的密钥管理系统，包括 SecretRef 契约、运行时模型、密钥提供商配置等。

11.6.1 密钥管理目标 ​

OpenClaw 支持可增量的 SecretRef，使凭据无需以明文存储在配置中。

核心目标 ​

┌─────────────────────────────────────────────────────────────┐
│                      密钥管理核心目标                          │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 运行时快照                                            │
│     └── 密钥在运行时解析为内存快照，非懒加载                  │
│                                                             │
│  2. 启动快速失败                                           │
│     └── 无法解析的有效活跃 SecretRef 会阻止启动              │
│                                                             │
│  3. 原子重载                                              │
│     └── 重载使用原子交换：全成功，或保留最后已知好的快照     │
│                                                             │
│  4. 活跃表面过滤                                           │
│     └── 仅在有效活跃表面上验证 SecretRef                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘

运行时模型 ​

密钥在激活时急切解析（非请求路径上的懒加载）：

  • 启动失败快速返回
  • 重载使用原子交换
  • 运行时请求仅读取活跃内存快照

11.6.2 SecretRef 契约 ​

通用格式 ​

json5
{ source: "env" | "file" | "exec", provider: "default", id: "..." }

source: "env" ​

json5
{ source: "env", provider: "default", id: "OPENAI_API_KEY" }

验证规则：

  • provider 必须匹配 ^[a-z][a-z0-9_-]{0,63}$
  • id 必须匹配 ^[A-Z][A-Z0-9_]{0,127}$

source: "file" ​

json5
{ source: "file", provider: "filemain", id: "/providers/openai/apiKey" }

验证规则：

  • provider 必须匹配 ^[a-z][a-z0-9_-]{0,63}$
  • id 必须是绝对 JSON 指针 (/...)

source: "exec" ​

json5
{ source: "exec", provider: "vault", id: "providers/openai/apiKey" }

验证规则：

  • provider 必须匹配 ^[a-z][a-z0-9_-]{0,63}$
  • id 必须匹配 ^[A-Za-z0-9][A-Za-z0-9._:/-]{0,255}$

11.6.3 密钥提供商配置 ​

配置结构 ​

json5
{
  secrets: {
    providers: {
      default: { source: "env" },
      filemain: {
        source: "file",
        path: "~/.openclaw/secrets.json",
        mode: "json",  // 或 "singleValue"
      },
      vault: {
        source: "exec",
        command: "/usr/local/bin/openclaw-vault-resolver",
        args: ["--profile", "prod"],
        passEnv: ["PATH", "VAULT_ADDR"],
        jsonOnly: true,
      },
    },
    defaults: {
      env: "default",
      file: "filemain",
      exec: "vault",
    },
  },
}

Env 提供商 ​

  • 可选允许列表 via allowlist
  • 缺失/空环境变量会导致解析失败

File 提供商 ​

  • 从 path 读取本地文件
  • mode: "json" 期望 JSON 对象负载，用 id 作为指针解析
  • mode: "singleValue" 期望 ref id "value" 并返回文件内容
  • 必须通过权限检查

Exec 提供商 ​

  • 运行配置的绝对路径，无 shell
  • 默认 command 必须指向常规文件（非符号链接）
  • 支持超时、输出字节限制、环境变量白名单

11.6.4 集成示例 ​

1Password CLI ​

json5
{
  secrets: {
    providers: {
      onepassword_openai: {
        source: "exec",
        command: "/opt/homebrew/bin/op",
        allowSymlinkCommand: true,
        trustedDirs: ["/opt/homebrew"],
        args: ["read", "op://Personal/OpenClaw QA API Key/password"],
        passEnv: ["HOME"],
        jsonOnly: false,
      },
    },
  },
  models: {
    providers: {
      openai: {
        apiKey: { source: "exec", provider: "onepassword_openai", id: "value" },
      },
    },
  },
}

HashiCorp Vault CLI ​

json5
{
  secrets: {
    providers: {
      vault_openai: {
        source: "exec",
        command: "/opt/homebrew/bin/vault",
        allowSymlinkCommand: true,
        trustedDirs: ["/opt/homebrew"],
        args: ["kv", "get", "-field=OPENAI_API_KEY", "secret/openclaw"],
        passEnv: ["VAULT_ADDR", "VAULT_TOKEN"],
        jsonOnly: false,
      },
    },
  },
  models: {
    providers: {
      openai: {
        apiKey: { source: "exec", provider: "vault_openai", id: "value" },
      },
    },
  },
}

SOPS ​

json5
{
  secrets: {
    providers: {
      sops_openai: {
        source: "exec",
        command: "/opt/homebrew/bin/sops",
        allowSymlinkCommand: true,
        trustedDirs: ["/opt/homebrew"],
        args: ["-d", "--extract", '["providers"]["openai"]["apiKey"]', "/path/to/secrets.enc.json"],
        passEnv: ["SOPS_AGE_KEY_FILE"],
        jsonOnly: false,
      },
    },
  },
}

11.6.5 活跃表面过滤 ​

SecretRef 仅在有效活跃表面上验证：

活跃表面 ​

  • 已启用的渠道/账户
  • 工具/功能表面
  • 选中的 Web 搜索提供商密钥

非活跃表面 ​

  • 禁用的渠道/账户条目
  • 未使用的顶级渠道凭据
  • 未启用的工具功能
  • 未选中的 Web 搜索提供商密钥

诊断日志 ​

SECRETS_GATEWAY_AUTH_SURFACE
├── active  → SecretRef 是有效认证表面，必须解析
└── inactive → SecretRef 被忽略

11.6.6 密钥管理命令 ​

审计 ​

bash
# 审计密钥配置
openclaw secrets audit --check

审计发现包括：

  • 静态明文值 (openclaw.json, auth-profiles.json, .env)
  • 敏感 Provider Header 残留
  • 无法解析的 refs
  • 优先级阴影

配置 ​

bash
# 交互式配置
openclaw secrets configure

# 仅配置提供商
openclaw secrets configure --providers-only

# 跳过提供商设置
openclaw secrets configure --skip-provider-setup

应用 ​

bash
# 应用保存的计划
openclaw secrets apply --from /tmp/openclaw-secrets-plan.json

# 试运行
openclaw secrets apply --from /tmp/openclaw-secrets-plan.json --dry-run

11.6.7 运行时行为 ​

激活触发 ​

密钥激活在以下时机运行：

  • 启动时（预检 + 最终激活）
  • 配置重载热应用路径
  • 配置重载重启检查路径
  • 手动重载 via secrets.reload

降级与恢复 ​

当重载失败时：

  • 降级：运行时保留最后已知好的快照
  • 恢复：下次成功激活后发出信号

诊断事件：

  • SECRETS_RELOADER_DEGRADED
  • SECRETS_RELOADER_RECOVERED

11.6.8 安全策略 ​

单向安全策略 ​

OpenClaw 故意不写入包含历史明文密钥值的回滚备份。

安全模型 ​

1. 预检必须在写入模式前成功
2. 运行时激活在提交前验证
3. 应用使用原子文件替换

本节小结 ​

  1. SecretRef 契约：env/file/exec 三种来源
  2. 运行时快照：原子加载，快速失败
  3. 活跃表面：仅验证有效活跃的密钥引用
  4. 集成：1Password、Vault、SOPS 示例
  5. 命令：audit、configure、apply
