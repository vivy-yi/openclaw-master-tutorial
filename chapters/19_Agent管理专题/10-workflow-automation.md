# OpenClaw 工作流自动化原理

## 核心概念：三层架构

```
┌─────────────────────────────────────────────────────────┐
│ 现实世界的工作流 (用户业务需求)                           │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│ OpenClaw 自动化层                                        │
│  触发源 (Bindings/Cron) → Agent 处理 → 子 Agent 调度     │
└─────────────────────┬───────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────────────┐
│ OpenClaw 运行时 (Gateway + 插件)                         │
│  消息路由 / 任务执行 / 外部 API 调用                      │
└─────────────────────────────────────────────────────────┘
```

## 工作流转化流程

### 第一步：识别触发源

| 触发类型 | 配置位置 | 示例 |
|----------|----------|------|
| **消息触发** | `bindings` | 用户在 Telegram 群发送消息 |
| **定时触发** | `cron` | 每天 9:00 发送日报 |
| **事件触发** | `heartbeat` | 定期检查状态 |

### 第二步：定义 Agent 角色

```
~/.openclaw/workspaces/{agent-name}/
├── SOUL.md       ← 角色定义 (我是谁，我能做什么)
├── AGENTS.md     ← 行为指令 (如何处理任务)
├── TOOLS.md      ← 工具配置 (使用哪些技能)
└── USER.md       ← 用户信息 (服务对象是谁)
```

### 第三步：配置任务执行

| 执行方式 | 配置位置 | 说明 |
|----------|----------|------|
| **直接执行** | Agent 自身 | Agent 直接处理 |
| **子 Agent** | `subagents.allowAgents` | 调度其他 Agent |
| **外部脚本** | Cron jobs | 调用 shell/python |

### 第四步：定义输出/投递

| 投递方式 | 配置位置 |
|----------|----------|
| **返回群聊** | 消息路由回原群 |
| **发送邮件** | 需 skill 调用外部 API |
| **推送通知** | Telegram/Discord Webhook |

## 工作流定义示例

假设你要实现：**每天早上 9 点生成金融简报并发送到金融群**

### Step 1: 创建 Agent (jinrong)

```yaml
# ~/.openclaw/workspaces/jinrong/SOUL.md
# 角色: 金融助手
# 职责: 分析金融市场，生成简报
```

### Step 2: 配置 Cron 触发

```yaml
# openclaw.json plugins.cron
{
  "name": "金融-每日简报",
  "schedule": { "kind": "cron", "expr": "0 9 * * *" },
  "payload": {
    "kind": "agentTurn",
    "message": "生成今日金融简报"
  },
  "delivery": {
    "mode": "announce",
    "channel": "telegram",
    "to": "-1003633299525"
  }
}
```

### Step 3: Agent 执行逻辑

```yaml
# jinrong/SOUL.md
## 能力
1. 调用 stock-market-pro skill 获取行情
2. 调用 finance-news skill 获取新闻
3. 整合生成简报
```

## 实现自动化的核心文件

| 文件 | 作用 | 必填? |
|------|------|-------|
| `openclaw.json` | 全局配置 (agents, bindings, cron) | ✅ |
| `SOUL.md` | Agent 角色与能力定义 | ✅ |
| `AGENTS.md` | Agent 行为指令 | ✅ |
| `subagents` | 声明可调度的子 Agents | 可选 |
| `skills/` | 外部能力扩展 | 可选 |
| `cron` 配置 | 定时任务触发 | 可选 |
| `bindings` | 消息触发路由 | 可选 |

## 完整工作流示例

```
用户需求: "每天早上提醒我今天的待办事项"

┌──────────────────────────────────────────────────────┐
│ 1. 触发: Cron (每天 8:00)                             │
└─────────────────────┬────────────────────────────────┘
                      ↓
┌──────────────────────────────────────────────────────┐
│ 2. Agent: mo-richang (墨事)                           │
│    - 读取 todos.md 文件                              │
│    - 整理当日任务                                     │
└─────────────────────┬────────────────────────────────┘
                      ↓
┌──────────────────────────────────────────────────────┐
│ 3. 投递: 发送到生活助手群 (-5153278498)               │
└──────────────────────────────────────────────────────┘
```

## 总结：转化关键点

1. **明确触发源** - 消息/定时/事件
2. **定义 Agent 角色** - SOUL.md 说明能做什么
3. **编排执行逻辑** - AGENTS.md 定义怎么做
4. **配置投递方式** - 消息返回/外部通知
