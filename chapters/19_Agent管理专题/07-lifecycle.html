📊 三、Agent 学习/进化/更新 Loop ​

1. 学习机制（Memory） ​

┌─────────────────────────────────────────────┐
│              短期记忆 (Session)              │
│         当前对话上下文 (自动管理)             │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              每日记录 (Daily)                │
│    memory/YYYY-MM-DD.md (每日新增)          │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              长期记忆 (Long-term)           │
│           MEMORY.md (人工/自动提炼)         │
└─────────────────────────────────────────────┘

自动记忆 flush：

  • 当 Session 接近上下文压缩时
  • OpenClaw 会触发一个静默的 Agent 轮次
  • 提醒模型在压缩前写入持久记忆

2. 进化机制（Evolution） ​

进化方式说明
手动更新直接修改 SOUL.md / AGENTS.md
自动学习通过 memory_flush 自动写入偏好
用户反馈用户说 "记住这个" → 写入 MEMORY.md
上下文压缩summarization 保留关键信息

触发记忆写入：

markdown
用户: "记住我喜欢麻辣口味"
→ Agent 调用 memory_get/write
→ 写入 MEMORY.md 或 memory/YYYY-MM-DD.md

3. 更新 Loop 流程 ​

┌─────────────────────────────────────────────┐
│              触发条件                        │
│  1. 用户明确要求 ("记住...")                 │
│  2. 上下文即将压缩 (pre-compaction)         │
│  3. Session 结束 (/reset)                  │
│  4. 手动编辑配置文件                        │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              执行更新                        │
│  - 写入 memory/YYYY-MM-DD.md               │
│  - 更新 MEMORY.md (重要/持久信息)           │
│  - 更新 SOUL.md (角色变化)                  │
│  - 更新 AGENTS.md (指令变化)                │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              验证生效                        │
│  - 下次 Session 加载新上下文               │
│  - 通过 memory_search 验证记忆可检索        │
└─────────────────────────────────────────────┘

📊 五、Hook 拦截点 ​

可以在以下位置拦截和扩展：

Hook 事件时机
agent:bootstrapBootstrap 文件加载前
before_model_resolve模型解析前
before_prompt_buildPrompt 构建前
before_tool_call工具调用前
after_tool_call工具调用后
message:received收到消息时
message:sent消息发送后
session:compact:before上下文压缩前
session:compact:after上下文压缩后

这样就构成了一个完整的 感知 → 记忆 → 思考 → 行动 → 学习 → 进化 的 Agent 循环！
