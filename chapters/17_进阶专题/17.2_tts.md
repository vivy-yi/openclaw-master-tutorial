# 13.2 TTS 语音合成

OpenClaw 可以使用 ElevenLabs、OpenAI 或 Edge TTS 将回复转换为音频。TTS 可以在 OpenClaw 能发送音频的任何地方工作；Telegram 会收到圆形语音气泡。

---

## 13.2.1 支持的服务商

| 服务商 | 说明 | 特点 |
|--------|------|------|
| **ElevenLabs** | 主要或备用提供商 | 高质量、多语言、语音克隆 |
| **OpenAI** | 主要或备用提供商 | 也用于摘要 |
| **Edge TTS** | 无 API 密钥时的默认选项 | 免费、不需要密钥 |

### Edge TTS 说明

Edge TTS 使用 Microsoft Edge 的在线神经 TTS 服务，通过 `node-edge-tts` 库实现。它是一个托管服务（不是本地），使用 Microsoft 的端点，**不需要 API 密钥**。

> 注意：Edge TTS 是公共 Web 服务，没有发布的 SLA 或配额，应将其视为尽力而为的服务。如果需要保证的限制和支持，请使用 OpenAI 或 ElevenLabs。

---

## 13.2.2 配置密钥

### 可选密钥

如果需要 OpenAI 或 ElevenLabs：



![TTS架构](./.assets/diagrams/29_tts_architecture.png)

```bash
ELEVENLABS_API_KEY  # 或 XI_API_KEY
OPENAI_API_KEY
```

**Edge TTS 不需要 API 密钥**。如果未找到 API 密钥，OpenClaw 默认为 Edge TTS（除非通过 `messages.tts.edge.enabled=false` 禁用）。

如果配置了多个提供商，优先使用选择的提供商，其他作为自动备选。

---

## 13.2.3 基础配置

### 最小配置（启用 + 提供商）

```json5
{
  messages: {
    tts: {
      auto: "always",
      provider: "elevenlabs",
    },
  },
}
```

### OpenAI 优先，ElevenLabs 备用

```json5
{
  messages: {
    tts: {
      auto: "always",
      provider: "openai",
      summaryModel: "openai/gpt-4o-mini",
      openai: {
        apiKey: "openai_api_key",
        model: "gpt-4o-mini-tts",
        voice: "alloy",
      },
      elevenlabs: {
        apiKey: "elevenlabs_api_key",
        voiceId: "voice_id",
        modelId: "eleven_multilingual_v2",
      },
    },
  },
}
```

### Edge TTS（无需 API 密钥）

```json5
{
  messages: {
    tts: {
      auto: "always",
      provider: "edge",
      edge: {
        enabled: true,
        voice: "en-US-MichelleNeural",
        lang: "en-US",
        outputFormat: "audio-24khz-48kbitrate-mono-mp3",
        rate: "+10%",
        pitch: "-5%",
      },
    },
  },
}
```

## 13.2.4 自动模式详解

### auto 字段选项

| 值 | 说明 |
|----|------|
| `off` | 关闭自动 TTS |
| `always` | 始终发送音频 |
| `inbound` | 仅在收到语音消息后发送音频 |
| `tagged` | 仅当回复包含 `[[tts]]` 标签时发送音频 |

### 只在收到语音消息后回复音频

```json5
{
  messages: {
    tts: {
      auto: "inbound",
    },
  },
}
```

## 13.2.5 模型驱动的覆盖（默认开启）

默认情况下，模型可以为单次回复发出 TTS 指令。当 `messages.tts.auto` 为 `tagged` 时，需要这些指令来触发音频。

### 可用指令

```
[[tts:voiceId=pMsXgVXv3BLzUgSXRplE model=eleven_v3 speed=1.1]]
[[tts:text]](laughs) Read the song once more.[[/tts:text]]
```

支持的指令键：
- `provider` (`openai` | `elevenlabs` | `edge`)
- `voice` (OpenAI 语音) 或 `voiceId` (ElevenLabs)
- `model` (模型 ID)
- `stability`, `similarityBoost`, `style`, `speed`, `useSpeakerBoost`
- `languageCode` (ISO 639-1)

### 禁用所有模型覆盖

```json5
{
  messages: {
    tts: {
      modelOverrides: {
        enabled: false,
      },
    },
  },
}
```

## 13.2.6 输出格式

| 渠道 | 格式 |
|------|------|
| **Telegram** | Opus 语音便签 (48kHz / 64kbps) |
| **其他渠道** | MP3 (44.1kHz / 128kbps) |
| **Edge TTS** | 使用 `edge.outputFormat` |

---

## 13.2.7 Slash 命令使用

```
/tts off          # 关闭 TTS
/tts always       # 始终开启
/tts inbound      # 收到语音时开启
/tts tagged       # 标签模式
/tts status       # 查看状态
/tts provider     # 设置提供商
/tts limit 2000   # 设置长度限制
/tts summary off  # 关闭摘要
/tts audio Hello  # 生成一次性音频
```

---

## 13.2.8 自动 TTS 行为流程

```
回复 → TTS 已启用?
  否 → 发送文本
  是 → 包含媒体 / MEDIA: / 太短?
          是 → 发送文本
          否 → 长度 > 限制?
                   否 → TTS → 附加音频
                   是 → 摘要已启用?
                            否 → 发送文本
                            是 → 摘要 → TTS → 附加音频
```

## 本节小结

1. **三大提供商**：ElevenLabs、OpenAI、Edge TTS
2. **自动模式**：off/always/inbound/tagged 四种模式
3. **模型覆盖**：支持单次回复的语音自定义
4. **渠道适配**：Telegram 语音气泡、其他渠道 MP3
