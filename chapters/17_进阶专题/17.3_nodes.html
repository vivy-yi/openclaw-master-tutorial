13.3 移动端节点：iOS、Android 与语音交互 ​

OpenClaw 节点（Nodes）是运行在移动设备上的客户端，提供语音交互、语音唤醒等功能。

13.3.1 节点概述 ​

OpenClaw 支持三种主要节点平台：

平台语音唤醒语音通话语音转文字
macOS✅✅✅
iOS✅✅✅
Android手动✅✅

13.3.2 Voice Wake（语音唤醒） ​

OpenClaw 将唤醒词视为单一全局列表，由 Gateway 所有。

核心特性 ​

  • 无每节点自定义唤醒词
  • 任何节点/应用 UI 都可以编辑；更改由 Gateway 保留并广播给所有人
  • macOS 和 iOS 保留本地的 Voice Wake 启用/禁用开关
  • Android 目前保持 Voice Wake 关闭，使用语音标签中的手动麦克风流程

存储位置 ​

唤醒词存储在 Gateway 机器上：

~/.openclaw/settings/voicewake.json

格式 ​

json
{
  "triggers": ["openclaw", "claude", "computer"],
  "updatedAtMs": 1730000000000
}

协议 ​

方法：

  • voicewake.get → { triggers: string[] }
  • voicewake.set → { triggers: string[] }

事件：

  • voicewake.changed → { triggers: string[] }

谁接收：

  • 所有 WebSocket 客户端（macOS 应用、WebChat 等）
  • 所有连接的节点（iOS/Android），节点连接时作为初始"当前状态"推送

各平台行为 ​

平台行为
macOS使用全局列表控制 VoiceWakeRuntime 触发
iOS使用全局列表进行 VoiceWakeManager 触发检测
Android目前禁用，使用语音标签中的手动麦克风

13.3.3 Talk Mode（语音通话模式） ​

Talk mode 是一个持续的语音对话循环：


![移动节点]/assets/diagrams/20_nodes_mobile.png)


1. 聆听语音
2. 发送转录文本给模型（主会话，chat.send）
3. 等待响应
4. 通过 ElevenLabs 播放（流式播放）

macOS 行为 ​

  • 始终开启的覆盖层（启用 Talk 模式时）
  • Listening → Thinking → Speaking 阶段转换
  • 短暂停（静音窗口）时，当前转录被发送
  • 回复写入 WebChat（与打字相同）
  • 说话时中断（默认开启）：如果用户开始说话，停止播放并记录中断时间戳

回复中的语音指令 ​

助手可以在回复前加上 单个 JSON 行 来控制语音：


![移动节点]/assets/diagrams/20_nodes_mobile.png)

json
{ "voice": "<voice-id>", "once": true }

规则：

  • 仅第一行非空行
  • 未知键被忽略
  • once: true 仅适用于当前回复
  • 没有 once，语音成为 Talk 模式的新默认

支持的键：

  • voice / voiceId
  • model / modelId
  • speed, stability, similarity, style, speakerBoost

配置 ​

json5
{
  talk: {
    voiceId: "elevenlabs_voice_id",
    modelId: "eleven_v3",
    outputFormat: "mp3_44100_128",
    apiKey: "elevenlabs_api_key",
    silenceTimeoutMs: 1500,
    interruptOnSpeech: true,
  },
}

默认值：

  • interruptOnSpeech: true
  • silenceTimeoutMs: 平台默认（macOS 700ms，iOS 900ms）
  • voiceId: 回退到 ELEVENLABS_VOICE_ID
  • modelId: 默认为 eleven_v3

13.3.4 Audio / Voice Notes（语音消息） ​

OpenClaw 可以接收语音消息、自动转录并注入对话上下文。

工作流程 ​

1. 定位第一个音频附件，下载（如需要）
2. 强制执行 maxBytes 限制
3. 按顺序运行第一个符合条件的模型
4. 失败则尝试下一个
5. 成功时用 [Audio] 块替换 Body 并设置 {{Transcript}}

自动检测（默认） ​

如果未配置模型，OpenClaw 自动检测顺序：

  1. 本地 CLI：sherpa-onnx-offline、whisper-cli、whisper
  2. Gemini CLI
  3. 提供商：OpenAI → Groq → Deepgram → Google

支持的提供商 ​

  • OpenAI (gpt-4o-mini-transcribe)
  • Deepgram (nova-3)
  • Mistral (voxtral-mini-latest)

配置示例 ​


![移动节点]/assets/diagrams/20_nodes_mobile.png)

json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        maxBytes: 20971520,
        models: [
          { provider: "openai", model: "gpt-4o-mini-transcribe" },
        ],
      },
    },
  },
}

群聊中的语音消息 ​

当为群聊设置 requireMention: true 时，OpenClaw 在检查提及之前会转录音频。

本节小结 ​

  1. Voice Wake：全局唤醒词列表，Gateway 统一管理
  2. Talk Mode：持续语音对话循环，ElevenLabs TTS
  3. 语音消息：自动转录，注入对话上下文
  4. 平台差异：macOS 功能最全，Android 使用手动模式
