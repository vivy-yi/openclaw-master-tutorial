## 5.7.2 安装 ClawHub CLI

### 安装方式

```bash
# 使用 npm（推荐）
npm install -g clawhub

# 使用 pnpm
pnpm add -g clawhub

# 使用 yarn
yarn global add clawhub
```

### 验证安装

```bash
# 检查版本
clawhub --version

# 查看帮助
clawhub --help
```

### 登录 ClawHub

```bash
# 登录
clawhub login

# 查看当前用户
clawhub whoami

# 退出登录
clawhub logout
```

## 5.7.4 安装 Skills

### 基本安装

```bash
# 安装最新版本
clawhub install my-skill

# 指定版本
clawhub install my-skill@1.2.0

# 安装到指定目录
clawhub install my-skill --dir ./my-skills

# 指定工作目录
clawhub install my-skill --workdir ~/projects/myproject
```

### 安装示例

```bash
# 安装代码审查 Skill
clawhub install code-review

# 安装 GitHub 集成 Skill
clawhub install github-integration

# 安装数据库相关 Skills
clawhub install postgres-backup
clawhub install postgres-migration
```

### 批量安装

```bash
# 从文件批量安装
clawhub install -f skills.txt

# skills.txt 内容示例：
# code-review@1.0.0
# github-integration@2.1.0
# postgres-backup
```

## 5.7.5 更新 Skills

### 更新单个 Skill

```bash
# 更新到最新版本
clawhub update my-skill

# 更新到指定版本
clawhub update my-skill@1.2.0

# 强制更新
clawhub update my-skill --force
```

### 批量更新

```bash
# 更新所有 Skills
clawhub update --all

# 全部强制更新
clawhub update --all --force

# 跳过输入
clawhub update --all --no-input
```

### 更新原理

```
1. 计算本地文件 hash
2. 在 registry 查找匹配版本
3. 对比差异
4. 下载并替换
5. 更新本地索引
```

## 5.7.7 发布 Skills

### 准备 Skill 包

```
my-skill/
├── SKILL.md          # 必需：Skill 定义
├── README.md         # 可选：详细文档
├── tools.json        # 可选：工具配置
└── config.yaml       # 可选：Skill 配置
```

### SKILL.md 格式

```markdown
---
name: my-skill
description: "Skill 描述"
---

# My Skill

描述...

## 使用方法

## 示例
```

### 发布命令

```bash
# 发布 Skill
clawhub publish ./my-skill \
  --slug my-skill \
  --name "My Skill" \
  --version 1.0.0 \
  --changelog "Initial release"

# 发布新版本
clawhub publish ./my-skill \
  --slug my-skill \
  --version 1.1.0 \
  --changelog "Bug fixes"
```

### 发布参数

| 参数 | 说明 | 必需 |
|-----|------|------|
| `--slug` | 唯一标识符 | 是 |
| `--name` | 显示名称 | 是 |
| `--version` | 版本号 | 是 |
| `--changelog` | 变更日志 | 否 |
| `--private` | 私有 | 否 |

## 5.7.8 配置 ClawHub

### 环境变量

```bash
# 私有 Registry
export CLAWHUB_REGISTRY=https://registry.clawhub.com

# 工作目录
export CLAWHUB_WORKDIR=./skills
```

### 配置文件

```yaml
# .clawhub.yml
registry: https://clawhub.com
workdir: ./skills
dir: ./skills
```

### 多 Registry

```bash
# 使用私有 Registry
clawhub install my-skill --registry https://private.clawhub.com

# 默认 Registry
export CLAWHUB_REGISTRY=https://private.clawhub.com
```

## 5.7.9 常用场景

### 场景一：开发项目配置

```bash
# 1. 创建项目 Skills 目录
mkdir -p ./skills

# 2. 安装常用 Skills
clawhub install code-review --dir ./skills
clawhub install github-integration --dir ./skills
clawhub install unit-test-gen --dir ./skills

# 3. 更新
clawhub update --all --dir ./skills
```

### 场景二：团队共享

```bash
# 1. 导出团队 Skills
clawhub list > team-skills.txt

# 2. 队友安装
clawhub install -f team-skills.txt
```

### 场景三：自动化更新

```bash
# 添加到 CI/CD
# .github/workflows/update-skills.yml

- name: Update Skills
  run: |
    npm install -g clawhub
    clawhub update --all --no-input
```

## 5.7.10 常见问题

### 问题一：安装失败

```bash
# 检查网络
clawhub ping

# 查看详细错误
clawhub install my-skill --debug

# 使用国内镜像
export CLAWHUB_REGISTRY=https://mirror.clawhub.cn
```

### 问题二：权限不足

```bash
# 检查目录权限
ls -la ~/.clawhub/

# 修复权限
sudo chown -R $USER ~/.clawhub/
```

### 问题三：版本冲突

```bash
# 查看冲突
clawhub doctor

# 强制更新
clawhub update my-skill --force

# 清理缓存
clawhub cache clean
```
