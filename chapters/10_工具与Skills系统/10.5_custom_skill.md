# 5.5 自定义技能

## 本节目标
- 掌握自定义技能的创建方法

## 5.5.1 创建流程

### 自定义技能概述

```
自定义技能开发流程：

1. 规划技能
   → 确定功能
   → 定义输入输出
   └── 设计工作流

2. 创建文件
   → SKILL.md
   ├── prompt.md
   ├── tools.json
   └── config.yaml

3. 本地测试
   → 加载技能
   → 运行测试
   └── 修复问题

4. 发布分享
   → 打包发布
   └── 贡献社区
```

## 5.5.2 技能文件结构

### 目录结构

```
my-custom-skill/
├── SKILL.md           # 技能定义（必需）
├── prompt.md          # 提示词模板
├── tools.json         # 工具配置
├── examples.md        # 使用示例
├── config.yaml        # 配置文件
└── icon.svg          # 图标
```

### SKILL.md 结构

## 5.5.3 实战：创建代码审查技能

### Step 1: 创建目录

```bash
mkdir -p ~/.openclaw/skills/code-review
cd ~/.openclaw/skills/code-review
```

### Step 2: 创建 SKILL.md

```markdown
# Code Review

## Description
专业的代码审查技能，提供代码规范检查、安全扫描、性能优化建议。

## Capabilities
- 代码规范检查（ESLint、Prettier）
- 安全漏洞扫描
- 性能问题识别
- 代码可维护性评估
- 修复建议生成

## Usage
### 基本使用
@code-review /path/to/file

### 审查特定方面
@code-review --security /path/to/file
@code-review --performance /path/to/file

## Tools Used
- read: 读取代码文件
- grep: 搜索代码模式
- exec: 运行 lint 工具
```

### Step 3: 创建 prompt.md

```markdown
# Code Review Prompt

## Role
你是一个专业的代码审查专家，具有 10 年开发经验。

## Review Standards

### 代码规范
- 命名规范
- 代码格式
- 注释质量

### 安全检查
- SQL 注入
- XSS 漏洞
- 敏感信息泄露

### 性能
- 循环优化
- 缓存使用
- 数据库查询

### 可维护性
- 代码复杂度
- 重复代码
- 依赖管理

## Output Format
返回 JSON 格式：
{
  "summary": "总体评价",
  "issues": [
    {
      "severity": "critical|warning|info",
      "line": 行号,
      "message": "问题描述",
      "suggestion": "修复建议"
    }
  ]
}
```

### Step 4: 创建 tools.json

```json
{
  "tools": [
    {
      "name": "read",
      "enabled": true,
      "description": "读取代码文件"
    },
    {
      "name": "grep",
      "enabled": true,
      "description": "搜索代码模式"
    },
    {
      "name": "exec",
      "enabled": true,
      "description": "运行 lint 工具",
      "allowed_commands": ["eslint", "prettier", "npm"]
    }
  ]
}
```

### Step 5: 创建 config.yaml

```yaml
name: code-review
version: 1.0.0
author: your-name
description: Professional code review skill

config:
  lint_on: true
  security_scan: true
  max_issues: 50
  severity_threshold: warning
```

## 5.5.5 技能参数

### 定义参数

```yaml
parameters:
  - name: target
    type: string
    required: true
    description: 审查目标文件或目录

  - name: severity
    type: string
    required: false
    default: warning
    description: 最低严重级别
    enum: [error, warning, info]

  - name: rules
    type: array
    required: false
    description: 自定义规则列表
```

### 使用参数

```
用户：@code-review app.js --severity error
→ 解析参数：severity = error
→ 过滤输出：只显示 error 级别
```

## 5.5.6 高级特性

### 条件分支

```yaml
workflow:
  - if: security_scan
    steps:
      - tool: grep
        pattern: "SQL|script|eval"
      - tool: exec
        command: security-checker

  - if: lint_check
    steps:
      - tool: exec
        command: eslint
```

### 循环处理

```yaml
workflow:
  - for: file in files
    steps:
      - tool: read
        path: "{{file}}"
      - tool: analyze
```

### 错误处理

```yaml
on_error:
  - log: "Error in {{step}}"
  - notify: user
  - fallback: "审查失败，请检查文件路径"
```

## 5.5.8 常见问题

### Q1: 技能加载失败

```
错误：Skill load failed

解决：
1. 检查文件格式
2. 验证 SKILL.md 存在
3. 检查 YAML/JSON 语法
4. 查看错误日志
```

### Q2: 工具调用失败

```
错误：Tool not available

解决：
1. 在 tools.json 中启用
2. 检查权限配置
3. 确认工具存在
```

### Q3: 输出格式错误

```
问题：输出不符合预期

解决：
1. 在 prompt 中明确格式
2. 添加输出示例
3. 添加验证逻辑
```
