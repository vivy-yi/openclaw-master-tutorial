5.6.2 浏览器配置 ​

本节讲解 OpenClaw 浏览器的完整配置，包括多 Profile 管理、远程 CDP、Browserless、Browserbase 等。

5.6.2.1 基础配置 ​

配置文件位置 ​

浏览器设置位于 ~/.openclaw/openclaw.json：

json5
{
  browser: {
    enabled: true,           // 默认: true
    defaultProfile: "chrome", // 默认: "chrome"
    color: "#FF4500",        // 浏览器 UI 主题色
    headless: false,         // 是否无头模式
    noSandbox: false,        // 是否禁用沙箱
    attachOnly: false,       // true = 仅附着，不启动本地浏览器

    // SSRF 防护策略
    ssrfPolicy: {
      dangerouslyAllowPrivateNetwork: true, // 默认: true (信任网络模式)
      hostnameAllowlist: ["*.example.com", "example.com"],
      allowedHostnames: ["localhost"]
    },

    // 远程 CDP 超时配置
    remoteCdpTimeoutMs: 1500,
    remoteCdpHandshakeTimeoutMs: 3000,

    // 可执行文件路径
    executablePath: "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser",

    // 多 Profile 配置
    profiles: {
      openclaw: { cdpPort: 18800, color: "#FF4500" },
      work: { cdpPort: 18801, color: "#0066CC" },
      remote: { cdpUrl: "http://10.0.0.42:9222", color: "#00AA00" }
    }
  }
}

配置说明 ​

配置项类型默认值说明
enabledbooleantrue是否启用浏览器
defaultProfilestring"chrome"默认使用的 Profile
headlessbooleanfalse是否无头运行
noSandboxbooleanfalse是否禁用沙箱（Linux root 用户需要）
attachOnlybooleanfalse仅附着到已运行的浏览器
colorstring"#FF4500"浏览器 UI 主题色
ssrfPolicy.dangerouslyAllowPrivateNetworkbooleantrue是否允许访问私网

5.6.2.2 多 Profile 管理 ​

Profile 类型 ​

OpenClaw 支持三种类型的 Profile：

┌─────────────────────────────────────────────────────────────┐
│                     Profile 类型                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. openclaw-managed                                        │
│     └── 专用 Chromium浏览器实例，有独立用户数据目录            │
│                                                             │
│  2. remote                                                  │
│     └── 显式 CDP URL (在别处运行的 Chromium 浏览器)           │
│                                                             │
│  3. extension relay                                         │
│     └── 通过本地中继 + Chrome 扩展控制现有 Chrome Tab          │
│                                                             │
└─────────────────────────────────────────────────────────────┘

Profile 命令 ​

bash
# 列出所有 Profile
openclaw browser profiles

# 创建 Profile
openclaw browser create-profile --name work --color "#FF5A36"
openclaw browser create-profile --name remote-browser --cdp-url "http://10.0.0.42:9222"

# 删除 Profile
openclaw browser delete-profile --name work

Profile 分配规则 ​

  • openclaw Profile：自动创建（如果不存在）
  • chrome Profile：内置的 Chrome 扩展中继（默认指向 http://127.0.0.1:18792）
  • 本地 CDP 端口：默认从 18800–18899 分配

5.6.2.3 本地控制 vs 远程控制 ​

本地控制（默认） ​

Gateway 启动本地浏览器控制服务，可启动本地浏览器：

Gateway → 本地控制服务 → 浏览器

远程控制 ​

在有浏览器的机器上运行 Node Host，Gateway 代理浏览器操作到该节点：

Gateway → Node Host → 浏览器

远程 CDP ​

配置 browser.profiles.<name>.cdpUrl 连接到远程浏览器：

json5
{
  browser: {
    profiles: {
      remote: {
        cdpUrl: "http://10.0.0.42:9222",
        color: "#00AA00"
      }
    }
  }
}

CDP URL 可以包含认证信息：

json5
// 查询 token
{
  cdpUrl: "https://provider.example?token=<token>"
}

// HTTP Basic 认证
{
  cdpUrl: "https://user:pass@provider.example"
}

5.6.2.4 Browserless 集成 ​

Browserless 是一个托管的 Chromium 服务，通过 HTTPS 暴露 CDP 端点。

配置示例 ​

json5
{
  browser: {
    enabled: true,
    defaultProfile: "browserless",
    remoteCdpTimeoutMs: 2000,
    remoteCdpHandshakeTimeoutMs: 4000,
    profiles: {
      browserless: {
        cdpUrl: "https://production-sfo.browserless.io?token=<BROWSERLESS_API_KEY>",
        color: "#00AA00"
      }
    }
  }
}

使用方式 ​

bash
openclaw browser --browser-profile browserless start
openclaw browser --browserless open https://example.com

注意：替换 <BROWSERLESS_API_KEY> 为你的真实 Token

5.6.2.5 Browserbase 集成 ​

Browserbase 是一个云端浏览器平台，提供内置的 CAPTCHA 解决、隐身模式和住宅代理。

配置示例 ​

json5
{
  browser: {
    enabled: true,
    defaultProfile: "browserbase",
    remoteCdpTimeoutMs: 3000,
    remoteCdpHandshakeTimeoutMs: 5000,
    profiles: {
      browserbase: {
        cdpUrl: "wss://connect.browserbase.com?apiKey=<BROWSERBASE_API_KEY>",
        color: "#F97316"
      }
    }
  }
}

Browserbase 特点 ​

  • 免费套餐：1 个并发会话，每月 1 浏览器小时
  • 直接 WebSocket 连接，无需手动创建会话
  • 内置 CAPTCHA 解决和隐身模式

使用方式 ​

bash
openclaw browser --browser-profile browserbase start
openclaw browser --browserbase open https://example.com

5.6.2.6 Node Browser 代理（零配置默认） ​

如果你在有浏览器的机器上运行 Node Host，OpenClaw 可以自动将浏览器工具调用路由到该节点，无需额外配置。

工作原理 ​

┌─────────────┐          ┌─────────────┐
│   Gateway   │ ───────→ │  Node Host  │
│             │   代理    │ (浏览器所在) │
└─────────────┘          └─────────────┘

禁用自动代理 ​

json5
// 在 Node 上禁用
{
  nodeHost: {
    browserProxy: {
      enabled: false
    }
  }
}

// 或在 Gateway 上禁用
{
  gateway: {
    nodes: {
      browser: {
        mode: "off"
      }
    }
  }
}

5.6.2.7 SSRF 防护策略 ​

严格模式配置 ​

json5
{
  browser: {
    ssrfPolicy: {
      dangerouslyAllowPrivateNetwork: false,
      hostnameAllowlist: ["*.example.com", "example.com"],
      allowedHostnames: ["localhost"]
    }
  }
}

配置说明 ​

配置项说明
dangerouslyAllowPrivateNetwork是否允许访问私网（默认 true，信任网络模式）
hostnameAllowlist允许访问的域名模式
allowedHostnames允许访问的主机名（精确匹配）

本节小结 ​

  1. 基础配置：enabled、defaultProfile、headless、color
  2. 多 Profile：支持 openclaw-managed、remote、extension relay 三种类型
  3. 远程 CDP：支持 Browserless、Browserbase 等云服务
  4. Node 代理：零配置自动路由到 Node Host
  5. SSRF 防护：可配置严格模式的域名白名单
