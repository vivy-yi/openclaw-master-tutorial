# 5.6.4 快照与截图

本节讲解 OpenClaw 浏览器的快照和截图功能，包括 AI 快照、ARIA 快照、截图、PDF 导出等。

---

## 5.6.4.1 快照类型概述

OpenClaw 支持两种快照风格：

```
┌─────────────────────────────────────────────────────────────┐
│                      快照类型                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. AI 快照（默认）                                          │
│     ├── 使用数字 ref（如 12）                                 │
│     ├── 由 Playwright 的 aria-ref 支持                        │
│     └── 适合 Agent 操作                                      │
│                                                             │
│  2. Role 快照                                               │
│     ├── 使用角色 ref（如 e12）                                │
│     ├── 通过 getByRole() 解析                                 │
│     └── 适合复杂页面结构                                      │
│                                                             │
│  3. ARIA 快照                                               │
│     ├── 无障碍树格式                                         │
│     ├── 仅用于检查，不能用于操作                               │
│     └── 开发调试用                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

![浏览器快照](./.assets/diagrams/39_web_snapshot.png)

## 5.6.4.2 AI 快照（默认）

AI 快照是默认快照模式，返回带数字 ref 的文本快照：

```bash
# 获取 AI 快照（默认）
openclaw browser snapshot
# 或者
openclaw browser snapshot --format ai
```

### 输出示例

```
[12] 🔍 Search
     └─ input[type="text"]

[13] 🔗 Login
     └─ button[type="submit"]

[14] 🔗 Menu
     └─ button[aria-haspopup="menu"]

[15] 📝 Welcome message
     └─ div[aria-label="Welcome"]
```

### 使用快照进行操作

```bash
# 使用数字 ref 进行操作
openclaw browser click 12
openclaw browser type 13 "hello"
openclaw browser hover 14
```

---

## 5.6.4.3 Role 快照

Role 快照使用基于角色的引用，更适合复杂页面结构：

```bash
# 获取 Role 快照（交互式）
openclaw browser snapshot --interactive

# 紧凑模式
openclaw browser snapshot --compact

# 指定深度
openclaw browser snapshot --depth 6

# 指定选择器
openclaw browser snapshot --selector "#main"
```

### 输出示例

```
[e12] 🔍 Search
     role: textbox, name: "Search"

[e13] 🔗 Search button
     role: button, name: "Search"

[e14] 🔗 Menu
     role: button, name: "Menu", expanded: false
```

### 使用 Role Ref 进行操作

```bash
# 使用 role ref 进行操作
openclaw browser click e12
openclaw browser type e13 "hello"
openclaw browser highlight e14
```

### 带有标签的快照

```bash
# 添加视图截图并叠加 ref 标签
openclaw browser snapshot --interactive --labels

# 输出包含 MEDIA:/path/to/screenshot.png
```

## 5.6.4.4 ARIA 快照

ARIA 快照返回无障碍树格式，仅用于检查，不能用于操作：

```bash
# 获取 ARIA 快照
openclaw browser snapshot --format aria
```

### 输出示例

```
searchbox "Search"
  └─ (none)

button "Login"
  └─ (none)

menu "Menu"
  └─ menuitem "Settings"
  └─ menuitem "Profile"
```

## 5.6.4.5 快照配置

### 默认快照模式

在配置中设置默认快照模式：

```json5
{
  browser: {
    snapshotDefaults: {
      mode: "efficient"
    }
  }
}
```

### 高效快照模式

```bash
# 高效快照（紧凑 + 交互 + 深度限制）
openclaw browser snapshot --efficient
# 或者
openclaw browser snapshot --mode efficient
```

## 5.6.4.6 截图操作

### 基本截图

```bash
# 截图（当前视口）
openclaw browser screenshot
```

### 全页截图

```bash
# 截图（整个页面）
openclaw browser screenshot --full-page
```

### 元素截图

```bash
# 截图特定元素
openclaw browser screenshot --ref 12
openclaw browser screenshot --ref e12
```

### 截图输出

```
MEDIA:/path/to/screenshot.png
```

## 5.6.4.7 PDF 导出

```bash
# 导出当前页面为 PDF
openclaw browser pdf
```

### PDF 选项

```bash
# 带参数的 PDF
openclaw browser pdf --format A4
openclaw browser pdf --landscape
```

## 5.6.4.8 Ref 稳定性

```
★ Insight ─────────────────────────────────────
Ref 稳定性注意事项：
1. Ref 在页面导航后会变化
2. 如果操作失败，重新获取快照获取新 ref
3. Role 快照带 --frame 时，ref 作用域限于该 iframe
4. 数字 ref 和 role ref 可以混用
─────────────────────────────────────────────────
```

## 5.6.4.9 控制台与网络

### 查看控制台日志

```bash
# 查看所有控制台日志
openclaw browser console

# 只看错误
openclaw browser console --level error

# 清除日志
openclaw browser console --clear
```

### 查看页面错误

```bash
# 查看页面错误
openclaw browser errors

# 清除错误
openclaw browser errors --clear
```

### 查看网络请求

```bash
# 查看最近请求
openclaw browser requests

# 过滤请求
openclaw browser requests --filter api

# 清除请求日志
openclaw browser requests --clear
```

### 查看响应体

```bash
# 查看 API 响应
openclaw browser responsebody "**/api" --max-chars 5000
```

## 5.6.4.10 调试工作流

当操作失败时（如 "not visible"、"strict mode violation"、"covered"）：

```bash
# 1. 获取交互式快照
openclaw browser snapshot --interactive

# 2. 使用 ref 进行操作
openclaw browser click <ref>
openclaw browser type <ref> "text"

# 3. 如果仍然失败，高亮元素查看 Playwright 目标
openclaw browser highlight <ref>

# 4. 如果页面行为异常，检查错误和请求
openclaw browser errors --clear
openclaw browser requests --filter api --clear

# 5. 深度调试：录制 trace
openclaw browser trace start
# 复现问题
openclaw browser trace stop
# 输出: TRACE:/path/to/trace.zip
```

## 5.6.4.11 完整示例

```bash
# 1. 打开页面并截图
openclaw browser open https://example.com
openclaw browser screenshot

# 2. 获取 AI 快照
openclaw browser snapshot

# 3. 假设输出：
# [12] 🔍 Search input
# [13] 🔗 Search button

# 4. 在搜索框输入并搜索
openclaw browser type 12 "OpenClaw"
openclaw browser click 13

# 5. 等待结果，获取紧凑快照
openclaw browser wait --text "Results"
openclaw browser snapshot --compact

# 6. 截图保存
openclaw browser screenshot --full-page

# 7. 导出 PDF
openclaw browser pdf
```

---

## 本节小结

1. **AI 快照**：默认模式，数字 ref，适合 Agent 操作
2. **Role 快照**：`--interactive`，角色 ref，适合复杂页面
3. **ARIA 快照**：无障碍树，仅用于检查
4. **截图**：`screenshot`、`--full-page`、`--ref`
5. **调试**：`highlight`、`errors`、`requests`、`trace`
