**实际情况**:
OpenClaw 使用 **Compaction** 机制，不是 `history.strategy`。真实配置在 `agents.defaults.compaction` 下：
- `model`: 压缩模型
- `keepRecentTokens`: 保留最近 token 数
- `prompt`: 压缩提示词

**没有** `history.strategy`, `max_messages`, `compression.threshold` 等配置

**错误类型**: 虚构配置结构和机制名称

## 🟡 错误 5: 混淆 Compaction 与消息保留策略

**文件**: `6.2_message_history.md`
**位置**: 6.2.2 历史管理策略
**错误内容**:
```
1. 全部保留
2. 保留最近 N 条
3. 摘要压缩
4. 重要性筛选
```

**实际情况**:
OpenClaw 的消息处理机制是：
1. **Compaction**: 当会话接近 context limit 时，自动将旧消息压缩为摘要
2. **Memory flush**: 压缩前自动提醒 agent 将重要内容写入 memory 文件
3. **Session pruning**: 可配置会话修剪策略

**没有** "全部保留"、"保留最近 N 条"、"重要性筛选" 这些策略选项

**错误类型**: 混淆概念，添加不存在的策略

1. Daily notes (`memory/YYYY-MM-DD.md`)
2. Long-term memory (`MEMORY.md`)
3. Dream Diary (`DREAMS.md`) - 可选

**Layer 3-6** 都是墨家的自定义扩展，不是 OpenClaw 官方设计

**错误类型**: 将第三方定制框架描述为 OpenClaw 标准架构

## ✅ 正确的 OpenClaw 记忆架构 (对照)

根据 `openclaw/docs/concepts/memory.md` 官方文档：

### 官方记忆文件
```
workspace/
├── MEMORY.md              # 长期记忆 - 在每个 DM 会话开始时加载
├── memory/
│   └── YYYY-MM-DD.md      # 每日日志 - 今天和昨天自动加载
└── DREAMS.md              # 梦境日记 - 可选，需要启用 dreaming
```

### 官方记忆机制
1. **memory_search** - 语义搜索记忆
2. **memory_get** - 读取特定记忆文件
3. **Compaction** - 对话过长时自动压缩
4. **Memory flush** - 压缩前自动提醒保存重要内容
5. **Dreaming** - 可选的背景记忆整理

### 官方配置路径
```json
{
  "agents": {
    "defaults": {
      "memory": {
        "engine": "builtin"
      },
      "compaction": {
        "model": "ollama/qwen3:8b"
      }
    }
  }
}
```
