# 11.2 权限管理

## 本节目标
- 掌握权限配置方法
- 理解 Profile 机制
- 完成权限配置

---

## 11.2.1 权限模型

### 权限类型

```
权限类型：

1. 工具权限
   ├── read - 读取
   ├── write - 写入
   ├── exec - 执行命令
   └── browser - 浏览器

2. 路径权限
   ├── allowed_paths - 允许路径
   └── denied_paths - 禁止路径

3. API 权限
   ├── 端点权限
   └── 方法权限
```

---

## 11.2.2 Profile 配置

### 创建 Profile

```yaml
profiles:
  # 开发者 Profile
  developer:
    description: 开发者权限
    tools:
      read:
        enabled: true
      write:
        enabled: true
      exec:
        enabled: true
        allowed_commands:
          - npm
          - git
          - docker
        denied_commands:
          - rm -rf /
          - sudo

    paths:
      allowed:
        - ~/workspace/**
        - ~/projects/**
      denied:
        - ~/.ssh/**
        - /etc/**

  # 只读 Profile
  readonly:
    description: 只读权限
    tools:
      read:
        enabled: true
      write:
        enabled: false
      exec:
        enabled: false
```

### 用户配置

```yaml
users:

    name: 张三

  - user_id: user_002
    name: 李四
    profile: readonly
```

## 11.2.4 权限检查

### 检查流程

```
权限检查流程：

1. 获取用户 Profile
2. 检查工具权限
3. 检查路径权限
4. 检查命令权限
5. 决定是否允许
```

---

## 本节小结

1. **Profile**：权限集合
2. **工具**：read/write/exec
3. **路径**：allowed/denied
4. **分配**：用户关联 Profile
