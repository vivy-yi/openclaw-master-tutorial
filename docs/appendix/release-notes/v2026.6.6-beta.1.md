## ⚠️ 安全加固提示（重要必读）

本版本包含 **全面安全加固**，涉及 transcripts、sandbox binds、host 环境继承、MCP stdio、Codex HTTP 访问等核心模块。**如果你的配置涉及这些模块，强烈建议在测试环境充分验证后再升级。**

## 一、安全加固（🔴 P1 — 重大变更）

| 维度 | 内容 |
|------|------|
| **涉及范围** | transcripts、sandbox binds、host 环境继承、MCP stdio、Codex HTTP 访问 |
| **影响** | 以上模块的访问控制全面收紧 |
| **相关 PR** | #91529, #91618, #91615, #91619, #91741, #91745~#91752, #91763, #89938 |
| **升级建议** | 🔴 生产环境升级前务必在测试环境验证 |

**安全加固内容汇总**：

| 模块 | 变更类型 | 说明 |
|------|----------|------|
| Transcripts | 访问控制收紧 | 防止未授权访问 transcripts |
| Sandbox Binds | 权限收紧 | 减少 sandbox 环境逃逸风险 |
| Host 环境继承 | 范围收窄 | 减少 host 环境信息泄露 |
| MCP Stdio | I/O 边界强化 | 防止 MCP stdio 通信越权 |
| Codex HTTP 访问 | 默认关闭/受限 | 减少 Codex HTTP 暴露面 |

> 💡 **升级前检查清单**：
> 1. 检查 `openclaw.json` 中是否配置了以上相关模块
> 2. 在测试环境验证相关功能是否正常
> 3. 如果使用了自定义 sandbox binds，确认白名单配置是否需要更新

## 二、Telegram 渠道改进（🟡 P1）

| 维度 | 内容 |
|------|------|
| **变更数量** | 9 个 PR |
| **相关 PR** | #91189, #88682, #89588, #90212, #91876, #91874, #91904, #91478, #91915 |

### 2.1 账户级话题路由

支持在账户级别配置话题（Topic）路由规则，消息可以自动路由到对应的话题线程中。

### 2.2 流式文本支持工具调用

Telegram 消息中的流式文本现在支持工具调用（Tool Calls），提供更流畅的交互体验。

### 2.3 `/compact` 命令改进

`/compact` 命令现在工作在整个通用入口（General Entry），而非仅限于特定场景。

### 2.4 回调处理改进

改进回调（Callback）处理逻辑，减少消息丢失和处理延迟。

## 三、iMessage 渠道改进（🟡 P1）

| 维度 | 内容 |
|------|------|
| **变更数量** | 6 个 PR |
| **相关 PR** | #91335, #91449, #88969, #88530, #91783, #91785 |

### 3.1 始终在线入站重启

iMessage 渠道现在支持"始终在线入站重启"机制，提升消息接收可靠性。

### 3.2 持久化回显标记

消息回显（Echo）标记现在支持持久化存储，重启后不会丢失。

### 3.3 块流改进

改进消息块流处理逻辑，减少大消息传输时的截断问题。

### 3.4 空闲审批发现

新增空闲审批发现机制，当连接空闲时自动触发审批流程验证。

### 3.5 出站传输加固

出站消息传输增加更多错误处理和重试机制。

### 3.6 入站启动诊断

新增入站连接启动诊断功能，便于排查连接问题。

## 四、浏览器与 MCP 改进（🟡 P2）

| 维度 | 内容 |
|------|------|
| **相关 PR** | #91422, #89851, #91736, #91747, #91451, #80143 |

### 4.1 现有会话 CDP 支持

现有浏览器会话现在支持 CDP（Chrome DevTools Protocol）连接，便于调试和高级控制。

### 4.2 WebSocket 验证

MCP WebSocket 连接增加验证机制，提升安全性。

### 4.3 默认 Profile CDP URL 处理

改进默认 profile 的 CDP URL 处理逻辑，减少配置错误。

### 4.4 安全浏览器输出边界

强化浏览器输出的内容边界检查，防止恶意内容注入。
