## 一、核心功能更新

### 1.1 Parallel 搜索集成（⭐⭐⭐⭐⭐ 高优先级）

| 维度 | 内容 |
|------|------|
| **性质** | 🆕 捆绑的 `web_search` provider |
| **配置** | `PARALLEL_API_KEY` 环境变量 |
| **特性** | guarded endpoint、cache-safe session ids、onboarding picker 支持 |
| **PR** | #85158（贡献者 @NormallyGaussian） |
| **地位** | Q4 2025 Parallel API 终于官方集成 |

**配置示例**：
```bash
# 环境变量配置
PARALLEL_API_KEY=your_parallel_api_key

# 在 openclaw 配置中使用
providers:
  web_search:
    parallel:
      apiKey: ${PARALLEL_API_KEY}
```

**功能亮点**：
- 与现有的 Brave/Perplexity/Tavily 等搜索 provider 并列
- 支持 session id 缓存，避免重复 API 调用
- 提供 onboarding picker，新用户友好引导

### 1.2 QQBot Reasoning 内容过滤

| 维度 | 内容 |
|------|------|
| **功能** | QQBot 发送前自动删除 `<thinking>` 内容块 |
| **PR** | #89913, #90132（贡献者 @openperf） |
| **效果** | 防止内部模型推理内容泄露到频道回复 |

**原理**：
```
用户输入 → Agent推理(含<thinking>) → QQBot过滤<thinking> → 干净回复发送至QQ频道
```

**适用场景**：
- 使用 QQ频道且启用了 reasoning 模型（如 o4-mini、Claude）
- 需要保持公开回复简洁专业的场景

## 六、升级指南

### 6.1 从 v2026.6.1 升级

```bash
# 1. 停止 Gateway
openclaw gateway stop

# 2.升级 npm 包
npm install -g openclaw@latest

# 3. 执行 doctor 前置检查（会触发 Auth/Cron 数据迁移）
openclaw gateway doctor

# 4. 重启 Gateway
openclaw gateway start
```

### 6.2 Auth 数据迁移注意事项

>⚠️ 首次升级时 doctor preflight 会自动将 auth profiles 从文件迁移到 SQLite。
> 迁移过程中**请勿中断** Gateway 进程。

### 6.3 Parallel 搜索配置（如使用）

```bash
# 添加 Parallel API Key
export PARALLEL_API_KEY=your_key_here

# 在配置中指定
providers:
  web_search:
    default: parallel # 设置为默认搜索 provider
```
