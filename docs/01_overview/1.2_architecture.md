## 1.2.2 核心组件详解

### Gateway (网关)

Gateway 是 OpenClaw 的入口，是连接用户与 Agent 的桥梁。

```
┌─────────────────────────────────────────────────────────────┐
│                        Gateway 职责                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 消息路由                                                │
│     └── 将用户消息分发到对应的 Agent                          │
│                                                              │
│  2. 会话管理                                                │
│     └── 维护多用户多会话状态                                  │
│                                                              │
│  3. 渠道适配                                                │
│     └── 统一不同渠道的协议（飞书/Telegram/Discord）          │
│                                                              │
│  4. 权限控制                                                │
│     └── 基于 Profile 的细粒度权限                             │
│                                                              │
│  5. 安全拦截                                                │
│     └── 阻止危险操作                                         │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### Gateway 配置示例

```json
{
  "gateway": {
    "port": 18789,
    "host": "0.0.0.0",
    "channels": ["feishu", "telegram", "discord"],
    "rateLimit": {
      "enabled": true,
      "maxRequests": 100,
      "windowMs": 60000
    },
    "security": {
      "execApproval": false,
      "allowedCommands": ["ls", "cat", "grep"]
    }
  }
}
```

### Agent (智能体)

Agent 是 OpenClaw 的核心执行单元，是"大脑"。

```
┌─────────────────────────────────────────────────────────────┐
│                        Agent 职责                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. 意图理解                                                │
│     └── 解析用户指令，理解用户想要什么                        │
│                                                              │
│  2. 任务规划                                                │
│     └── 分解复杂任务为具体执行步骤                             │
│                                                              │
│  3. 工具选择                                                │
│     └── 决定使用哪些工具（read/write/exec/browser）         │
│                                                              │
│  4. 状态管理                                                │
│     └── 维护任务执行状态，处理中断恢复                         │
│                                                              │
│  5. 上下文维护                                              │
│     └── 管理对话上下文，记住之前对话内容                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

#### Agent 工作流程

```
用户指令
    │
    ▼
┌─────────────────────────────────────────────┐
│  意图理解 (Intent Understanding)            │
│  - 分析用户想做什么                         │
│  - 提取关键信息                             │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  任务规划 (Task Planning)                   │
│  - 分解为具体步骤                           │
│  - 确定执行顺序                             │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  工具选择 (Tool Selection)                 │
│  - 确定需要调用的工具                       │
│  - 准备参数                                 │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  执行与反馈 (Execution & Feedback)          │
│  - 调用工具执行                             │
│  - 收集结果                                 │
│  - 判断是否需要继续                         │
└─────────────────────────────────────────────┘
    │
    ▼
    返回结果给用户
```

#### Agent 配置示例

```json
{
  "agent": {
    "name": "assistant",
    "model": {
      "provider": "deepseek",
      "model": "deepseek-chat"
    },
    "tools": ["read", "write", "exec", "browser", "web"],
    "maxSteps": 100,
    "timeout": 300000,
    "maxRetries": 3
  }
}
```

### Model Layer (模型层)

模型层负责与各种 LLM 提供商交互。

```
┌─────────────────────────────────────────────────────────────┐
│                      模型层功能                               │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  • LLM 调用        与各种模型提供商交互                       │
│  • 模型路由        支持多模型配置                             │
│  • Failover        主模型失败时自动切换                       │
│  • 流式输出        实时返回生成内容                           │
│  • Token 统计      监控使用量和成本                           │
│  • 成本优化        自动选择性价比最高的模型                   │
│                                                              │

```

## 1.2.3 消息处理流程

当用户在飞书发送一条消息时，整个处理流程如下：

```
用户(飞书)
    │
    ▼
┌─────────────────────────────────────────────┐
│  1. Gateway 接收消息                        │
│     • 验证消息格式                           │
│     • 解析用户身份                           │
│     • 创建/查找会话                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  2. 消息路由                                │
│     • 根据渠道和用户匹配 Agent              │
│     • 检查权限 Profile                      │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  3. Agent 处理                              │
│     ┌────────────────────────────────────┐ │
│     │ a) 意图理解                          │ │
│     │    LLM 分析用户指令                  │ │
│     ├────────────────────────────────────┤ │
│     │ b) 任务规划                          │ │
│     │    分解为具体执行步骤                 │ │
│     ├────────────────────────────────────┤ │
│     │ c) 工具选择                          │ │
│     │    确定需要调用的工具                 │ │
│     └────────────────────────────────────┘ │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  4. 工具执行                                │
│     • 调用内置工具/Skills/Plugins          │
│     • 收集执行结果                          │
│     • 处理工具返回                          │
└─────────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────────┐
│  5. 响应生成                                │
│     • 将工具结果整合为自然语言               │
│     • 流式返回给用户                        │
└─────────────────────────────────────────────┘
    │
    ▼
返回给用户(飞书)
```

## 1.2.4 配置文件结构

OpenClaw 的核心配置文件是 `openclaw.json`，位于 `~/.openclaw/` 目录：

```json
{
  // 模型配置
  "model": {
    "provider": "deepseek",
    "model": "deepseek-chat",
    "apiKey": "${DEEPSEEK_API_KEY}"
  },

  // 网关配置
  "gateway": {
    "port": 18789,
    "channels": {
      "feishu": { "enabled": true },
      "telegram": { "enabled": true }
    }
  },

  // Agent 配置
  "agent": {
    "name": "assistant",
    "tools": ["read", "write", "exec", "browser"]
  },

  // Skills 配置
  "skills": {
    "enabled": true,
    "autoInstall": true
  },

  // 安全配置
  "security": {
    "profile": "default",
    "execApproval": true
  },

  // 记忆配置
  "memory": {
    "enabled": true,
    "type": "filesystem"
  }
}
```

## 本节小结

1. **分层架构**：用户层 → Gateway层 → Agent层 → Model层 → Tools层 → System层
2. **Gateway 职责**：消息路由、会话管理、渠道适配、权限控制、安全拦截
3. **Agent 职责**：意图理解、任务规划、工具选择、状态管理
4. **Tools 层**：内置工具 + Skills (1800+) + Plugins
5. **System 层**：文件系统、命令行、浏览器、日历、API、数据库、进程、画布、语音
6. **消息流程**：接收 → 路由 → 处理 → 执行 → 响应
7. **配置结构**：`~/.openclaw/openclaw.json`
8. **扩展性**：支持 Skills、渠道、工具、模型扩展
