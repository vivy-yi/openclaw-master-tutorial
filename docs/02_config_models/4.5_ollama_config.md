## 4.5.2 Ollama 安装

### macOS 安装

```bash
# 使用 Homebrew
brew install ollama

# 或者直接下载
# 访问 ollama.com/download
```

### Linux 安装

```bash
# 安装
curl -fsSL https://ollama.com/install.sh | sh

# 验证
ollama --version
```

### Windows 安装

```
1. 访问 ollama.com/download
2. 下载 Windows 版本
3. 双击安装
4. 命令行验证：ollama --version
```

## 4.5.3 模型下载

### 常用命令

```bash
# 查看可用模型
ollama list

# 下载模型
ollama pull llama3.1
ollama pull qwen2.5
ollama pull mistral

# 删除模型
ollama rm llama3.1

# 查看模型信息
ollama show llama3.1
```

### 推荐配置

```
入门配置：
- 16GB 内存
- Mistral (7B) 或 Phi-3 (14B)

推荐配置：
- 32GB+ 内存
- Llama 3.1 (70B) 或 Qwen 2.5 (72B)

发烧配置：
- 64GB+ 内存
- 多卡并行
- 大容量模型
```

## 4.5.5 Ollama 服务管理

### 启动服务

```bash
# 启动服务（默认端口 11434）
ollama serve

# 守护进程方式
# macOS
launchctl start ollama

# Linux
sudo systemctl start ollama
```

### 常用操作

```bash
# 查看运行状态
ollama ps

# 停止模型
ollama stop llama3.1

# 重新加载
# 修改模型后需要重启服务
pkill ollama
ollama serve
```

## 4.5.6 性能优化

### GPU 配置

```bash
# 查看 GPU 状态
nvidia-smi

# 设置 GPU 数量
export OLLAMA_GPU_LAYERS=99
```

### 内存优化

```
优化建议：
1. 关闭不必要的模型
2. 使用量化模型
3. 调整上下文长度
4. 限制并发请求
```

### 模型量化

```
量化级别（从高到低）：
- IQ2_XXS: 最小，最快，质量最低
- IQ2_S: 小，快速
- Q4_K_M: 平衡（推荐）
- Q5_K_S: 较大，更准确
- Q8_0: 最大，最准确

下载量化版本：
ollama pull llama3.1:q4_k_m
```

## 4.5.7 常见问题

### Q1: 显存不足

```
错误：Error: model requires more GPU memory

解决：
1. 使用更小的模型
2. 使用量化版本
3. 减少上下文长度
4. 增加 GPU
```

### Q2: 模型下载慢

```
解决：
1. 使用镜像站
2. 手动下载模型文件
3. 检查网络连接
```

### Q3: 响应速度慢

```
解决：
1. 使用 GPU 加速
2. 使用量化模型
3. 减少并发
4. 升级硬件
```

### Q4: 端口被占用

```
错误：listen tcp 0.0.0.0:11434: bind: address already in use

解决：
1. 检查端口占用：lsof -i :11434
2. 停止其他 Ollama 进程
3. 修改默认端口
```
