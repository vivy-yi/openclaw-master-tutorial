## 4.2.2 API 获取

### 注册账号

1. 访问 [platform.deepseek.com](https://platform.deepseek.com)
2. 注册账号（支持邮箱/手机）
3. 完成实名认证

### 创建 API Key

```
步骤：
1. 登录控制台
2. 点击左侧「API Keys」
3. 点击「创建 API Key」
4. 命名并复制（只显示一次！）
5. 充值账户（支持支付宝/微信）
```

## 4.2.3 OpenClaw 配置

### 方法一：配置文件

在 `openclaw.json` 中添加：

```json
{
  "models": {
    "deepseek": {
      "provider": "deepseek",
      "model": "deepseek-chat",
      "api_key": "sk-xxxxxxxxxxxxxxxx",
      "temperature": 0.7,
      "max_tokens": 4096
    },
    "deepseek-coder": {
      "provider": "deepseek",
      "model": "deepseek-coder",
      "api_key": "sk-xxxxxxxxxxxxxxxx",
      "temperature": 0.3,
      "max_tokens": 4096
    }
  }
}
```

### 方法二：环境变量

```bash
# 环境变量方式
export DEEPSEEK_API_KEY="sk-xxxxxxxxxxxxxxxx"
```

### 方法三：使用 OpenRouter

```json
{
  "models": {
    "deepseek-via-openrouter": {
      "provider": "openrouter",
      "model": "deepseek/deepseek-chat",
      "api_key": "sk-or-xxxxxxxxxxxxxxxx"
    }
  }
}
```

## 4.2.4 多模型切换

### 默认配置

```json
{
  "agent": {
    "model": "deepseek-chat",
    "fallback_models": ["deepseek-coder", "gpt-4o-mini"]
  }
}
```

### 按场景切换

```json
{
  "agent": {
    "default_model": "deepseek-chat",
    "model_scenarios": {
      "coding": "deepseek-coder",
      "writing": "deepseek-chat",
      "analysis": "deepseek-v3"
    }
  }
}
```

## 4.2.5 成本优化

### 使用技巧

```
💰 成本优化技巧：

1. 选用合适模型
   ├── 简单对话 → deepseek-chat
   ├── 代码任务 → deepseek-coder
   └── 复杂推理 → deepseek-v3

2. 设置合理参数
   ├── max_tokens: 按需设置，不过大
   ├── temperature: 0.3-0.7 即可
   └── 避免过长上下文

3. 使用缓存
   ├── 启用上下文缓存
   └── 重复内容只计算一次
```

### 成本计算示例

```
任务：处理 10,000 条用户咨询

方案一：GPT-4o
- 输入：$2.50/1M × 5M tokens = $12.50
- 输出：$10.00/1M × 2M tokens = $20.00
- 总计：$32.50

方案二：DeepSeek Chat
- 输入：$0.27/1M × 5M tokens = $1.35
- 输出：$1.10/1M × 2M tokens = $2.20
- 总计：$3.55

节省：约 90%
```

## 4.2.6 常见问题

### Q1: API Key 无效

```
错误：invalid api key

解决：
1. 检查 Key 是否正确复制
2. 确认 Key 已激活
3. 检查账户余额是否充足
```

### Q2: 余额不足

```
错误：insufficient quota

解决：
1. 登录 DeepSeek 控制台
2. 点击「账户充值」
3. 选择充值金额（最低 ¥10）
```

### Q3: 请求超时

```
错误：request timeout

解决：
1. 检查网络连接
2. 尝试使用代理
3. 增加超时设置：
   "timeout": 120
```

### Q4: 模型不存在

```
错误：model not found

解决：
1. 确认模型名称正确
2. 检查是否支持该模型
3. 常用模型：deepseek-chat, deepseek-coder
```
