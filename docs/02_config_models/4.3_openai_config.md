## 4.3.2 API 获取

### 注册 OpenAI 账号

```
步骤：
1. 访问 platform.openai.com
2. 使用邮箱注册
3. 完成手机验证
4. 添加付款方式（必须）
```

### 创建 API Key

```
流程：
1. 登录 OpenAI 控制台
2. 点击「API keys」
3. 点击「Create new secret key」
4. 设置名称和权限
5. 复制并保存（只显示一次！）
```

### 充值方式

```
支持的支付方式：
├── 信用卡/借记卡 (Visa, Mastercard, AMEX)
├── 支付宝 (中国区)
└── 微信支付 (中国区)

注意：
- 首次充值最低 $5
- 按用量计费，无月费
- 可设置消费上限
```

## 4.3.4 模型参数详解

### temperature

```
temperature 控制输出的随机0.性：

0 - 确定性输出
    ├── 相同输入总是相同输出
    └── 适合代码、数学等精确任务

0.7 - 平衡（默认）
    ├── 有创意但不离谱
    └── 适合一般对话

1.0+ - 高随机性
    ├── 更有创意但可能跑偏
    └── 适合创意写作
```

### max_tokens

```
max_tokens 控制最大输出长度：

- 设置过低：回答被截断
- 设置过高：浪费 token
- 建议：根据任务调整

常见设置：
- 短回答：512-1024
- 中等回答：2048-4096
- 长回答：8192-16384
```

### top_p (nucleus sampling)

```
top_p 控制词汇选择范围：

- top_p = 1.0：使用全部词汇
- top_p = 0.9：使用前 90% 词汇
- top_p = 0.1：只使用最高概率词汇

建议：
- 只设置 temperature 或 top_p 一个
- 不要同时设置
```

## 4.3.6 常见问题

### Q1: 账户被限制

```
错误：Your account has been limited

原因：
- 可疑活动
- 欠费
- 违反使用政策

解决：
1. 检查账户状态
2. 联系 support@openai.com
3. 提供相关证明
```

### Q2: 速率限制

```
错误：Rate limit exceeded

解决：
1. 等待后重试
2. 使用 o1-mini 等小模型
3. 增加缓存
4. 升级付费套餐
```

### Q3: 余额不足

```
错误：insufficient_quota

解决：
1. 登录控制台
2. 点击「Billing」→「Payment methods」
3. 添加支付方式或充值
```

### Q4: 国内无法访问

```
解决：
1. 使用代理服务
2. 使用 OpenRouter 转发
3. 使用国内模型替代
```
