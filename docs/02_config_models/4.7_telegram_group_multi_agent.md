## 4.7.2 基础配置步骤

### 步骤 1：创建 Telegram 机器人

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         创建 Telegram 机器人                                     │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  1. 打开 Telegram，搜索 @BotFather                                            │
│  2. 发送 /newbot 命令                                                         │
│  3. 按提示设置机器人名称和用户名（必须以 bot 结尾）                            │
│  4. 复制 bot token                                                            │
│                                                                                 │
│  可选设置：                                                                    │
│  • /setjoingroups — 控制是否允许添加到群组                                   │
│  • /setprivacy — 控制隐私模式（影响群组消息可见性）                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 步骤 2：配置 Token

编辑 `openclaw.json`：

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "1234567890:ABCdefGHIjklMNOpqrsTUVwxyz",
      dmPolicy: "pairing",
    },
  },
}
```

### 步骤 3：获取群组 ID

#### 方法一：转发消息给机器人

```
将群组中的任意消息转发给 @userinfobot 或 @getidsbot
机器人会回复该消息的聊天 ID（负数，如 -1001234567890）
```

#### 方法二：查看日志

```bash
# 启动 Gateway 后，发送消息到群组
# 然后查看日志
tail -f ~/.openclaw/logs/gateway.log | grep "telegram"
# 查找 chat.id 字段
```

#### 方法三：使用 Bot API

```bash
# 获取更新
curl "https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getUpdates"
# 在响应中查找 chat.id
```

## 4.7.3 完整配置示例

### 单群组配置

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_BOT_TOKEN",
      dmPolicy: "pairing",
      // 群组配置
      groups: {
        "-1001234567890": {
          allow: true,
          requireMention: false
        },
        "-1009876543210": {
          allow: true,
          requireMention: true  // 需要 @提及才回复
        }
      }
    }
  },
  bindings: [
    {
      agentId: "main",
      match: {
        channel: "telegram",
        peer: {
          kind: "group",
          id: "-1001234567890"
        }
      }
    }
  ]
}
```

### 多群组多 Agent 配置

```json5
{
  agents: {
    list: [
      {
        id: "main",
        name: "Main Assistant",
        workspace: "~/.openclaw/workspace"
      },
      {
        id: "coder",
        name: "Coding Expert",
        workspace: "~/.openclaw/workspace-coder"
      },
      {
        id: "docs",
        name: "Documentation Bot",
        workspace: "~/.openclaw/workspace-docs"
      }
    ]
  },
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_BOT_TOKEN",
      groups: {
        "-1001234567890": {
          allow: true,
          requireMention: false
        }
      }
    }
  },
  bindings: [
    {
      agentId: "main",
      match: {
        channel: "telegram",
        peer: {
          kind: "group",
          id: "-1001234567890"
        }
      }
    }
  ]
}
```

### 使用论坛话题实现多 Agent 隔离

由于广播组功能尚未支持 Telegram，可以使用论坛话题实现类似的多 Agent 隔离效果：

```json5
{
  channels: {
    telegram: {
      enabled: true,
      botToken: "YOUR_BOT_TOKEN",
      groups: {
        "-1001234567890": {
          allow: true,
          requireMention: false,
          topics: {
            // 代码审查话题
            "101": {
              requireMention: true,
              systemPrompt: "你是一个代码审查专家，专注于代码质量和安全"
            },
            // 文档话题
            "102": {
              requireMention: true,
              systemPrompt: "你是一个文档助手，专注于技术文档撰写"
            }
          }
        }
      }
    }
  }
}
```

## 4.7.4 群组策略详解

### groups vs groupPolicy 区别

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                      groups vs groupPolicy 区别                                │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  channels.telegram.groups:                                                     │
│  └── 控制哪些群组可以互动（群组允许列表）                                      │
│      • 无 groups 配置 = 允许所有群组                                           │
│      • 有 groups 配置 = 只允许列出的群组或 "*"                                 │
│                                                                                 │
│  channels.telegram.groupPolicy:                                                │
│  └── 控制群组中哪些发送者可以触发 Agent                                        │
│      • "open" = 允许所有发送者                                                 │
│      • "allowlist" = 只有 groupAllowFrom 中的可以                              │
│      • "disabled" = 禁用所有群组消息                                           │
│                                                                                 │
│  📌 大多数用户需要：groupPolicy: "allowlist" + groupAllowFrom + 在 groups 中  │
│     列出特定群组                                                               │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 常见配置组合

| 场景 | groups | groupPolicy | requireMention |
|------|--------|-------------|----------------|
| 公开群组，所有人可触发 | `"*": {}` | `"open"` | false |
| 特定群组，管理员触发 | `-100xxx: {}` | `"allowlist"` | false |
| 私密群组，需要 @提及 | `-100xxx: {}` | `"allowlist"` | true |
| 只接收命令 | `"*": {}` | `"allowlist"` | true |

## 4.7.5 隐私模式与权限

### 隐私模式说明

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         Telegram 隐私模式                                       │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  Telegram 机器人默认启用隐私模式：                                              │
│  • 机器人只能收到：@提及、命令、部分消息                                       │
│  • 不会收到群组中其他人的普通消息                                               │
│                                                                                 │
│  禁用隐私模式（两种方式）：                                                     │
│  1. @BotFather → /setprivacy → Disable                                        │
│  2. 将机器人添加为群组管理员                                                   │
│                                                                                 │
│  ⚠️ 更改隐私模式后，需要将机器人移除群组并重新添加                             │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 管理员权限

```json5
// 推荐：将机器人设为管理员以获得完全消息访问权限
// 在 Telegram 群组设置中配置
{
  channels: {
    telegram: {
      groups: {
        "-1001234567890": {
          allow: true,
          requireMention: false  // 禁用隐私模式后可设为 false
        }
      }
    }
  }
}
```

## 4.7.7 多 Agent 协作方案

### 当前可用方案

由于 Telegram 广播组功能尚未支持，以下是当前可行的多 Agent 协作方案：

#### 方案 1：Binding 路由（最常用）

```json5
{
  bindings: [
    {
      agentId: "main",
      match: {
        channel: "telegram",
        peer: {
          kind: "group",
          id: "-1001234567890"
        }
      }
    }
  ]
}
```

#### 方案 2：论坛话题隔离

```json5
{
  channels: {
    telegram: {
      groups: {
        "-1001234567890": {
          allow: true,
          requireMention: true,
          topics: {
            "1": {
              // 通用话题
              systemPrompt: "你是通用助手"
            },
            "2": {
              // 代码话题
              systemPrompt: "你是代码审查专家"
            }
          }
        }
      }
    }
  }
}
```

#### 方案 3：关键词触发（通过 Agent 内部逻辑）

在 Agent 的 AGENTS.md 中配置关键词路由规则。

## 4.7.9 常见问题

### Q1: 机器人没有收到群组消息

```bash
# 检查步骤：
1. 确认 Bot 已添加到群组
2. 确认 Bot 有管理员权限或隐私模式已禁用
3. 检查 groups 配置是否包含该群组
4. 查看日志：
tail -f ~/.openclaw/logs/gateway.log | grep telegram
```

### Q2: 机器人不响应非 @提及消息

```bash
# 原因：requireMention 默认为 true
# 解决：
# 方式1：配置 requireMention: false
# 方式2：在群组中发送 /activation always
# 方式3：禁用 Telegram 隐私模式
```

### Q3: 群组 ID 获取失败

```bash
# 确保：
1. 先发送消息到群组
2. 然后调用 getUpdates
3. 群组 ID 格式正确（超级群组以 -100 开头）
```

### Q4: 如何实现多 Agent 响应？

```
当前 Telegram 不支持广播组功能，可选方案：

方案 1：使用 WhatsApp 广播组
- 真正的多 Agent 并行响应
- 详见 broadcast-groups.md

方案 2：使用 Telegram 论坛话题
- 每个话题独立会话
- 通过不同话题路由到不同 Agent

方案 3：等待支持
- 关注官方更新
- Telegram 广播组开发中
```

## 本节小结

1. **groups vs groupPolicy**：groups 控制群组允许列表，groupPolicy 控制发送者
2. **requireMention**：默认 true，需要 @提及才回复
3. **群组 ID 格式**：Telegram 群组以 `-100` 开头（超级群组）
4. **隐私模式**：禁用后机器人可收到所有群组消息
5. **多 Agent 方案**：当前使用 Binding 路由，广播组开发中
6. **论坛话题**：Telegram 特有功能，可实现话题隔离
