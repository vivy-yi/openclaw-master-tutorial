## 5.2 db_query - 查询数据

### 功能说明

执行 SELECT 查询并返回结果。

### 参数

```json5
{
  "sql": "SELECT * FROM users WHERE id = ?",  // SQL 查询
  "params": [1],                               // 参数
  "timeout": 30000,                          // 超时
  "limit": 100                                // 限制返回行数
}
```

### 使用示例

#### 简单查询

```bash
# 查询所有用户
db_query("SELECT * FROM users")

# 带条件的查询
db_query("SELECT * FROM users WHERE status = ?", {
  params: ["active"]
})

# 分页查询
db_query("SELECT * FROM users LIMIT ? OFFSET ?", {
  params: [10, 0]
})
```

#### 聚合查询

```bash
# 统计数量
db_query("SELECT COUNT(*) as total FROM users WHERE status = ?", {
  params: ["active"]
})

# 分组统计
db_query("""
  SELECT status, COUNT(*) as count
  FROM users
  GROUP BY status
""")
```

#### JOIN 查询

```bash
# 多表关联
db_query("""
  SELECT u.id, u.name, o.order_id, o.total
  FROM users u
  LEFT JOIN orders o ON u.id = o.user_id
  WHERE u.created_at > ?
""", {
  params: ["2024-01-01"]
})
```

## 5.3 db_execute - 执行命令

### 功能说明

执行 INSERT、UPDATE、DELETE 等写操作。

### 参数

```json5
{
  "sql": "INSERT INTO users (name, email) VALUES (?, ?)",  // SQL 命令
  "params": ["张三", "zhangsan@example.com"]               // 参数
}
```

### 使用示例

#### INSERT 插入

```bash
# 插入单条
db_execute("""
  INSERT INTO users (name, email, status)
  VALUES (?, ?, ?)
""", {
  params: ["张三", "zhangsan@example.com", "active"]
})

# 插入并获取 ID
result = db_execute("""
  INSERT INTO users (name, email)
  VALUES (?, ?)
""", {
  params: ["李四", "lisi@example.com"],
  returning: "id"
})
print("Inserted ID: " + result.lastInsertId)
```

#### UPDATE 更新

```bash
# 更新单条
db_execute("""
  UPDATE users
  SET name = ?, status = ?
  WHERE id = ?
""", {
  params: ["新名字", "inactive", 1]
})

# 批量更新
db_execute("""
  UPDATE users
  SET status = 'inactive'
  WHERE last_login < ?
""", {
  params: ["2024-01-01"]
})
```

#### DELETE 删除

```bash
# 删除单条
db_execute("DELETE FROM users WHERE id = ?", {
  params: [1]
})

# 条件删除
db_execute("DELETE FROM users WHERE status = ? AND created_at < ?", {
  params: ["deleted", "2023-01-01"]
})
```

## 5.4 db_transaction - 事务操作

### 功能说明

执行事务操作，确保原子性。

### 参数

```json5
{
  "operations": [
    { "sql": "INSERT INTO accounts...", "params": [...] },
    { "sql": "UPDATE accounts...", "params": [...] }
  ],
  "isolation": "READ_COMMITTED"  // 隔离级别
}
```

### 使用示例

```bash
# 简单事务
db_transaction([
  {
    sql: "INSERT INTO orders (user_id, total) VALUES (?, ?)",
    params: [1, 100]
  },
  {
    sql: "UPDATE users SET balance = balance - ? WHERE id = ?",
    params: [100, 1]
  }
])

# 失败回滚
try {
  db_transaction([
    { sql: "INSERT INTO accounts...", params: [...] },
    { sql: "UPDATE users...", params: [...] }
  ])
} catch (error) {
  print("Transaction failed: " + error.message)
}
```

## 5.5 连接配置

### 全局配置

```json5
{
  tools: {
    allow: ["group:database"]
  }
}
```

### 数据库连接

```json5
{
  agents: {
    defaults: {
      database: {
        // 数据库类型
        type: "postgresql",
        // 连接配置
        connection: {
          host: "localhost",
          port: 5432,
          database: "myapp",
          user: "admin",
          // 密码通过环境变量
          password: "${DB_PASSWORD}"
        },
        // 连接池配置
        pool: {
          min: 2,
          max: 10
        }
      }
    }
  }
}
```

### 支持的数据库

| 数据库 | 类型标识 |
|--------|---------|
| PostgreSQL | postgresql |
| MySQL | mysql |
| SQLite | sqlite |
| MongoDB | mongodb |

## 5.6 查询优化

### 参数化查询

```bash
# ✅ 正确：使用参数防止 SQL 注入
db_query("SELECT * FROM users WHERE id = ?", {
  params: [userId]
})

# ❌ 错误：字符串拼接（SQL 注入风险）
db_query("SELECT * FROM users WHERE id = " + userId)
```

### 限制结果

```bash
# 始终限制结果数量
db_query("SELECT * FROM large_table LIMIT ?", {
  params: [1000]
})
```

### 索引提示

```bash
# 使用索引
db_query("""
  SELECT * FROM orders USE INDEX (idx_user_id)
  WHERE user_id = ?
""", {
  params: [1]
})
```

## 5.7 安全建议

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                       数据库操作安全建议                                        │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ✅ 推荐做法：                                                                  │
│  1. 使用参数化查询，防止 SQL 注入                                              │
│  2. 最小权限原则，Agent 使用只读账号                                           │
│  3. 敏感字段加密存储                                                          │
│  4. 记录所有数据库操作日志                                                    │
│                                                                                 │
│  ❌ 避免做法：                                                                  │
│  1. 字符串拼接 SQL                                                            │
│  2. 使用管理员账户进行日常操作                                                │
│  3. 在代码中硬编码数据库密码                                                   │
│  4. 允许 DROP TABLE 等危险操作                                               │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 权限配置

```json5
{
  agents: {
    list: [
      {
        id: "reader",
        tools: {
          allow: ["db_query"],
          deny: ["db_execute", "db_transaction"]
        }
      }
    ]
  }
}
```

### 禁止危险操作

```json5
{
  tools: {
    deny: [
      "DROP TABLE",
      "DROP DATABASE",
      "TRUNCATE",
      "GRANT",
      "REVOKE"
    ]
  }
}
```

## 5.8 错误处理

```bash
try {
  result = db_query("SELECT * FROM users WHERE id = ?", {
    params: [1]
  })
  print("Found: " + result.length + " users")
} catch (error) {
  print("Query error: " + error.message)
}

try {
  db_execute("UPDATE users SET name = ? WHERE id = ?", {
    params: ["新名字", 1]
  })
  print("Updated successfully")
} catch (error) {
  print("Execute error: " + error.message)
}
```
