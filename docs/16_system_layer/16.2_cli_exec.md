## 2.2 exec - 执行命令

### 功能说明

执行单条系统命令并返回输出。

### 参数

```json5
{
  "command": "npm install",        // 必需，要执行的命令
  "timeout": 60000,               // 可选，超时时间（毫秒）
  "description": "安装依赖"        // 可选，描述
}
```

### 使用示例

```bash
# 执行简单命令
exec("ls -la")

# 执行带参数的命令
exec("git status")

# 指定超时时间
exec("npm run build", { timeout: 120000 })

# 执行并获取结果
result = exec("cat /etc/hostname")
print(result.stdout)
```

### 输出结构

```json5
{
  "stdout": "命令标准输出",
  "stderr": "命令错误输出",
  "exitCode": 0,
  "timedOut": false
}
```

## 2.3 bash - 执行脚本

### 功能说明

执行多行 Shell 脚本，支持复杂操作。

### 参数

```json5
{
  "command": "#!/bin/bash\n...",   // 必需，脚本内容
  "timeout": 60000,                 // 可选，超时时间
  "description": "部署脚本"          // 可选，描述
}
```

### 使用示例

```bash
# 执行多行脚本
bash("""
#!/bin/bash
echo "Starting deployment..."
npm run build
docker build -t myapp .
docker push myapp
""")

# 使用变量
bash("""
#!/bin/bash
NAME="myapp"
VERSION="1.0.0"
echo "Building $NAME:$VERSION"
""")

# 条件执行
bash("""
#!/bin/bash
if [ -d "dist" ]; then
  echo "Build exists"
else
  echo "No build found"
  exit 1
fi
""")
```

## 2.4 工具配置

### 全局配置

```json5
{
  tools: {
    allow: ["exec", "bash"],
    deny: ["rm -rf /"]  // 禁止危险命令
  }
}
```

### 按 Agent 配置

```json5
{
  agents: {
    list: [
      {
        id: "deployer",
        tools: {
          allow: ["group:runtime"],
          deny: ["rm -rf"]  // 禁止删除操作
        }
      },
      {
        id: "reader",
        tools: {
          allow: ["exec", "bash"],
          deny: ["*"]  // 只读模式
        }
      }
    ]
  }
}
```

### 工具组简写

```json5
{
  tools: {
    // 等价于 ["exec", "bash"]
    allow: ["group:runtime"]
  }
}
```

## 2.5 安全建议

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                       命令行执行安全建议                                        │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ✅ 推荐做法：                                                                  │
│  1. 沙箱限制命令执行                                                          │
│  2. 禁止危险命令（rm -rf, dd, mkfs）                                         │
│  3. 设置命令超时防止挂起                                                    │
│  4. 记录所有执行的命令                                                      │
│                                                                                 │
│  ❌ 避免做法：                                                                  │
│  1. 允许无限制的 shell 访问                                                 │
│  2. 执行用户提供的未验证命令                                                │
│  3. 允许 sudo 或 root 权限命令                                              │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 危险命令黑名单

```json5
{
  tools: {
    deny: [
      "rm -rf",
      "dd",
      "mkfs",
      "fdisk",
      "> /dev/sda",
      "chmod 777",
      "chown -R"
    ]
  }
}
```

## 2.6 性能优化

### 超时配置

```json5
{
  agents: {
    defaults: {
      // 默认命令超时
      defaultExecTimeout: 60000,
      // 危险命令超时（更短）
      dangerousExecTimeout: 10000
    }
  }
}
```

### 建议超时

| 命令类型 | 建议超时 |
|---------|---------|
| 文件操作 | 30秒 |
| 构建命令 | 2分钟 |
| 部署命令 | 5分钟 |
| 长时间任务 | 使用后台执行 |

## 2.7 最佳实践

### 场景示例

```bash
# ✅ 正确：使用绝对路径
exec("/usr/bin/git status")

# ✅ 正确：设置超时
exec("npm install", { timeout: 300000 })

# ❌ 错误：危险的递归删除
exec("rm -rf node_modules")

# ❌ 错误：允许任意用户输入
exec("rm -rf " + userInput)  // 不要这样做！
```

### 错误处理

```bash
result = exec("npm run build")
if (result.exitCode !== 0) {
  print("Build failed: " + result.stderr)
  throw new Error("Build failed")
}
print(result.stdout)
```

## 本节小结

1. **exec** - 执行单条命令，返回 stdout/stderr/exitCode
2. **bash** - 执行多行 Shell 脚本
3. **工具组** - group:runtime 包含 exec 和 bash
4. **安全** - 配置黑名单，设置超时
5. **最佳实践** - 使用绝对路径，处理错误
