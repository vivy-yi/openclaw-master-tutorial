## 1.2 read - 读取文件

### 功能说明

读取文件或目录内容。

### 参数

```json5
{
  "path": "/path/to/file.txt",      // 必需，文件或目录路径
  "offset": 0,                        // 可选，起始行号
  "limit": 100                        // 可选，读取行数
}
```

### 使用示例

```bash
# 读取文件
read("/path/to/file.txt")

# 读取前 100 行
read("/path/to/file.txt", { limit: 100 })

# 读取第 50-100 行
read("/path/to/file.txt", { offset: 50, limit: 50 })

# 列出目录
read("/path/to/directory")

# 查看当前目录
read(".")
```

---

## 1.3 write - 写入文件

### 功能说明

写入或创建文件。

### 参数

```json5
{
  "path": "/path/to/file.txt",      // 必需，文件路径
  "content": "文件内容",              // 必需，写入内容
  "append": false                     // 可选，是否追加模式
}
```

### 使用示例

```bash
# 创建新文件
write("/path/to/newfile.md", "# Hello World")

# 覆盖写入
write("/path/to/file.txt", "新内容")

# 追加内容
write("/path/to/log.txt", "新日志行", { append: true })
```

## 1.4 edit - 编辑文件

### 功能说明

编辑文件内容，支持精确替换。

### 参数

```json5
{
  "path": "/path/to/file.txt",      // 必需，文件路径
  "oldString": "要替换的内容",        // 必需，被替换的内容
  "newString": "替换后的内容"         // 必需，新内容
}
```

### 使用示例

```bash
# 替换单行
edit("/path/to/file.txt", "旧文本", "新文本")

# 替换多行
edit("/path/to/file.txt", "const a = 1;\nconst b = 2;", "const sum = a + b;")

# 使用精确匹配
edit("/path/to/config.json", '"debug": false', '"debug": true')
```

### 编辑模式

| 模式 | 说明 |
|------|------|
| 精确替换 | 精确匹配 oldString，替换为 newString |
| 保留缩进 | 编辑时自动保留原始缩进 |

## 1.5 glob - 文件搜索

### 功能说明

搜索匹配模式的文件。

### 参数

```json5
{
  "pattern": "**/*.js",              // 必需，搜索模式
  "path": "/path/to/search",          // 可选，搜索路径
  "options": {
    "ignore": ["node_modules"]        // 可选，排除的目录
  }
}
```

### 模式语法

```
**       → 匹配任意目录
*        → 匹配任意字符
?        → 匹配单个字符
[abc]    → 匹配括号内任意字符
!pattern → 排除匹配的文件
```

### 使用示例

```bash
# 搜索所有 JavaScript 文件
glob("**/*.js")

# 搜索 src 目录下的 TypeScript 文件
glob("src/**/*.ts")

# 搜索特定文件
glob("**/*config*.json")

# 排除 node_modules
glob("**/*.js", { ignore: ["node_modules", "dist"] })
```

## 1.6 apply_patch - 应用补丁

### 功能说明

应用统一的 diff 补丁。

### 参数

```json5
{
  "path": "/path/to/file.txt",      // 必需，文件路径
  "patch": "patch 内容"              // 必需，diff 补丁
}
```

### 使用示例

```bash
# 应用补丁
apply_patch("/path/to/file.txt", """---
 file.txt | 2 +-
 1 file changed, 1 insertion(+), 1 deletion(-)

diff --git a/file.txt b/file.txt
index 1234567..89abcdef 100644
--- a/file.txt
+++ b/file.txt
@@ -1,3 +1,3 @@
-old line
+new line
""")
```

## 1.7 工具配置

### 全局配置

```json5
{
  tools: {
    allow: ["read", "write", "edit", "glob", "apply_patch"],
    deny: []
  }
}
```

### 按 Agent 配置

```json5
{
  agents: {
    list: [
      {
        id: "coder",
        tools: {
          allow: ["group:fs"],
          deny: ["write", "edit"]  // 只读
        }
      }
    ]
  }
}
```

### 工具组简写

```json5
{
  tools: {
    // 等价于 ["read", "write", "edit", "apply_patch"]
    allow: ["group:fs"]
  }
}
```

│  3. 允许 Agent 访问系统关键文件                                           │

{
  agents: {

## 本节小结

1. **read** - 读取文件或目录，支持分页
2. **write** - 写入或创建文件，支持追加
3. **edit** - 精确替换文件内容
4. **glob** - 按模式搜索文件
5. **apply_patch** - 应用 diff 补丁
6. **工具组** - group:fs 包含所有文件系统工具
