## 二、GitHub Issues 深度分析

### 2.1 🔴 P1: #83778 - Cron Jobs + Tools Delivery Failure

**基本信息**:
| 项目 | 值 |
|------|---|
| Issue # | #83778 |
| 标题 | Cron jobs with tools enabled cause delivery failure and silent fallback to DM |
| 标签 | P1, clawsweeper:needs-info, impact:message-loss, issue-rating: gold shrimp |
| 作者 | @alissonaguiar |
| 日期 | 2026-05-18 |

**问题描述**:
当 Cron job 配置了 `tools` 参数时，消息投递失败并静默回退到 DM 模式，导致：
- 预期的群组/频道通知无法送达
- 用户不知道消息被降级到 DM
- 无明显错误日志

**影响范围**:
- 所有使用 Cron + Tools 组合的用户
- 影响 message-loss 场景

**Workaround**:
```markdown
### #83778 Workaround

1. **确认工具必要性**: 检查 cron job 是否真正需要 tools
2. **降级方案**: 
   - 移除 tools 配置，改为在主 session 中处理
   - 或使用 `openclaw agents run` 手动触发带工具任务
3. **监控检查**: 定期检查 cron 执行日志，确认 DM 回退是否发生
```

**官方状态**: `clawsweeper:needs-info` - 等待更多信息

| 标题 | Skill registry: changes to skills.load.extraDirs do not propagate to existing per-session skillsSnapshot |
| 标签 | P2, clawsweeper:queueable-fix, impact:session-state, issue-rating: diamond lobster |
| 作者 | @nathansmithopenclaw-alt |
| 日期 | 2026-05-18 |

**问题描述**:
修改 `skills.load.extraDirs` 配置后，变更不会传播到现有 session 的 skillsSnapshot，需要重启 session 才会生效。

**Workaround**:
```markdown
### #83782 Workaround

1. **保存配置后重启**: 修改 extraDirs 后执行 `openclaw agents restart`
2. **新 session 验证**: 开启新 session 验证配置是否生效
3. **持久化方案**: 将 extraDirs 写入 workspace 的 openclaw.yaml，避免每次手动配置
```

**官方状态**: `clawsweeper:queueable-fix` - 计划修复
