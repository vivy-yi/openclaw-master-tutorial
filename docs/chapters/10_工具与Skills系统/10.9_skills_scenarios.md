## 5.8.2 开发场景

### 场景一：代码审查

**推荐 Skills**：

| Skill | 功能 | 安装 |
|-------|------|------|
| `code-review` | 代码审查 | `clawhub install code-review` |
| `github` | GitHub PR 检查 | 需安装 `gh` CLI |

**使用流程**：
```
用户：帮我审查这个 PR
→ 使用 github skill 查看 PR 详情
→ 使用 code-review skill 审查代码
← 返回审查报告
```

**配置示例**：
```bash
# 安装
npm install -g clawhub
clawhub install code-review
clawhub install github

# 认证 GitHub
gh auth login
```

**使用示例**：
```
用户：审查 https://github.com/owner/repo/pull/55
→ gh pr view 55 --repo owner/repo
→ 分析代码变更
← 返回：
  ✅ 代码规范通过
  ⚠️ 建议：
  - 第 23 行：建议添加类型注解
  - 第 45 行：考虑使用 try-catch
```

## 5.8.3 笔记场景

### 场景一：Obsidian 笔记管理

**推荐 Skills**：

| Skill | 功能 | 安装 |
|-------|------|------|
| `obsidian` | Obsidian 操作 | 内置 |

**前置要求**：
- Obsidian Vault 路径
- （可选）Obsidian Shell Commands

**使用示例**：
```
用户：搜索我笔记中关于 OpenClaw 的内容
→ obsidian search "OpenClaw"
← 返回相关笔记列表

用户：在笔记中添加今天的会议纪要
→ obsidian create "会议/2024-01-15.md"
← 返回创建的笔记内容
```

**常用命令**：
```bash
# 读取笔记
obsidian read "我的笔记"

# 搜索
obsidian search "关键字"

# 创建
obsidian create "新笔记.md" --content "# 内容"

# 列出最近
obsidian recent --days 7
```

### 场景二：Notion 集成

**推荐 Skills**：

| Skill | 功能 | 安装 |
|-------|------|------|
| `notion` | Notion 操作 | 需配置 |

**配置**：
```bash
# 获取 Notion Token
# 1. 访问 https://www.notion.so/my-integrations
# 2. 创建新 Integration
# 3. 获取 Internal Integration Token

# 配置
openclaw config set plugins.entries.notion.config.apiKey "secret_xxx"
```

**使用示例**：
```
用户：查看我的 Notion 任务列表
→ notion query --database-id "xxx"
← 返回任务列表

用户：在 Notion 中创建新任务
→ notion create --database-id "xxx" --properties '{"Name": "新任务"}'
← 返回创建的页面
```

**推荐 Skills**：

| Skill | 功能 | 安装 |
|-------|------|------|
| `healthcheck` | 系统健康检查 | 内置 |

**使用示例**：
```
用户：检查系统状态
→ healthcheck --all
← 返回：
  CPU: 45%
  Memory: 2.1GB / 4GB
  Disk: 120GB / 500GB

```

**自定义检查**：
```bash
# 检查特定服务
healthcheck --service nginx,mysql,redis

# 检查端口
healthcheck --port 80,443,18789
```

## 5.8.7 组合工作流

### 工作流示例：代码审查 + 通知

```json
{
  "workflow": {
    "name": "PR 审查流程",
    "steps": [
      {
        "skill": "github",
        "action": "get-pr",
        "params": {"pr_url": "{{input}}"}
      },
      {
        "skill": "code-review",
        "action": "review",
        "params": {"changes": "{{pr.changes}}"}
      },
      {
        "skill": "slack",
        "action": "notify",
        "params": {"message": "{{review.result}}"}
      }
    ]
  }
}
```

### 工作流示例：笔记同步

```json
{
  "workflow": {
    "name": "会议记录同步",
    "steps": [
      {
        "skill": "apple-notes",
        "action": "create",
        "params": {"title": "{{meeting.title}}"}
      },
      {
        "skill": "notion",
        "action": "sync",
        "params": {"content": "{{note.content}}"}
      },
      {
        "skill": "slack",
        "action": "share",
        "params": {"channel": "team"}
      }
    ]
  }
```
