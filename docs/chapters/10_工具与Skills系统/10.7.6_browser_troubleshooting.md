# 5.6.6 浏览器故障排除

本节讲解 OpenClaw 浏览器自动化中的常见问题与解决方案。

---

## 5.6.6.1 常见错误与解决

### "Browser disabled" 错误

**错误信息**：
```
Error: Browser disabled
```

**解决方案**：

1. 在配置中启用浏览器：

```json5
{
  browser: {
    enabled: true
  }
}
```

2. 重启 Gateway

---

### "Playwright is not available" 错误

**错误信息**：
```
Playwright is not available in this gateway build
```

**解决方案**：

某些功能（navigate/act/AI snapshot/role snapshot、element screenshots、PDF）需要 Playwright。

1. 安装完整 Playwright 包（不是 `playwright-core`）：

```bash
npm install playwright
```

2. 重启 Gateway

3. 如果使用 Docker：

```bash
docker compose run --rm openclaw-cli \
  node /app/node_modules/playwright-core/cli.js install chromium
```

4. 设置持久化路径：

```bash
export PLAYWRIGHT_BROWSERS_PATH=/home/node/.cache/ms-playwright
```

---

### "Not visible" 错误

**错误信息**：
```
Error: Element is not visible
```

**解决方案**：

1. 获取新的快照，确认元素可见：

```bash
openclaw browser snapshot --interactive
```

2. 滚动到元素使其可见：

```bash
openclaw browser scrollintoview <ref>
```

3. 等待元素可见：

```bash
openclaw browser wait "#selector" --state visible
```

---

### "Strict mode violation" 错误

**错误信息**：
```
Error: Strict mode violation: multiple elements found
```

**解决方案**：

1. 使用更精确的选择器获取快照：

```bash
openclaw browser snapshot --selector "#unique-id"
```

2. 使用 nth 索引（如果有）：

```bash
openclaw browser click <ref> --nth 0
```

---

### "Element is covered" 错误

**错误信息**：
```
Error: Element is covered by another element
```

**解决方案**：

1. 获取快照查看覆盖层：

```bash
openclaw browser snapshot --interactive
```

2. 高亮元素查看覆盖情况：

```bash
openclaw browser highlight <ref>
```

3. 关闭覆盖层（如弹窗）：

```bash
openclaw browser press Escape
```

## 5.6.6.2 Linux 特定问题

### Snap Chromium 问题

如果使用 snap 安装的 Chromium，可能需要额外配置：

```json5
{
  browser: {
    noSandbox: true
  }
}
```

或者使用系统浏览器：

```json5
{
  browser: {
    executablePath: "/usr/bin/google-chrome"
  }
}
```

### 安装 Playwright 浏览器

```bash
# 在 Docker 中
docker compose run --rm openclaw-cli \
  node /app/node_modules/playwright-core/cli.js install chromium
```

## 5.6.6.3 WSL2 + Windows 问题

### 场景描述

Gateway 运行在 WSL2，Chrome 运行在 Windows 上。

### 解决方案

1. **安装 Chrome 扩展**（WSL2 中）：

```bash
openclaw browser extension install
```

2. **配置 relayBindHost**（Windows Chrome 端）：

```json5
{
  browser: {
    enabled: true,
    relayBindHost: "0.0.0.0",
    defaultProfile: "chrome"
  }
}
```

3. **启动 Chrome 并附着**

详细参考：[WSL2 + Windows + remote Chrome CDP troubleshooting](https://docs.openclaw.ai/tools/browser-wsl2-windows-remote-cdp-troubleshooting)

## 5.6.6.4 调试技巧

### Trace 录制

录制操作 trace 进行深度调试：

```bash
# 1. 开始录制
openclaw browser trace start

# 2. 执行操作（复现问题）
openclaw browser click 12

# 3. 停止录制
openclaw browser trace stop
# 输出: TRACE:/path/to/trace.zip
```

### 高亮元素

```bash
# 高亮元素查看目标
openclaw browser highlight 12
```

### 检查错误

```bash
# 查看页面错误
openclaw browser errors

# 清除错误
openclaw browser errors --clear
```

### 检查网络请求

```bash
# 查看请求
openclaw browser requests --filter api

# 清除
openclaw browser requests --clear

# 查看响应体
openclaw browser responsebody "**/api" --max-chars 5000
```

## 5.6.6.5 安全与隐私

### 浏览器数据安全

```
★ Insight ─────────────────────────────────────
浏览器安全注意事项：
1. openclaw Profile 可能包含登录会话，视为敏感
2. browser act / evaluate 执行任意 JS，需防注入
3. 禁用 evaluate: browser.evaluateEnabled=false
4. Gateway/Node 保持私有（loopback 或 tailnet）
5. 远程 CDP 端点强大，需隧道和保护
─────────────────────────────────────────────────
```

### 禁用 JavaScript 执行

如果不需要 `evaluate` 功能，可以禁用：

```json5
{
  browser: {
    evaluateEnabled: false
  }
}
```

## 5.6.6.6 状态与环境控制

### Cookie 操作

```bash
# 查看 Cookie
openclaw browser cookies

# 设置 Cookie
openclaw browser cookies set session abc123 --url "https://example.com"

# 清除 Cookie
openclaw browser cookies clear
```

### Storage 操作

```bash
# Local Storage
openclaw browser storage local get
openclaw browser storage local set theme dark
openclaw browser storage local clear

# Session Storage
openclaw browser storage session get
openclaw browser storage session clear
```

### 模拟环境

```bash
# 离线模式
openclaw browser set offline on

# 自定义 Headers
openclaw browser set headers --headers-json '{"X-Debug":"1"}'

# HTTP 认证
openclaw browser set credentials user pass
openclaw browser set credentials --clear

# 地理位置
openclaw browser set geo 37.7749 -122.4194 --origin "https://example.com"
openclaw browser set geo --clear

# 媒体类型
openclaw browser set media dark

# 时区
openclaw browser set timezone America/New_York

# 地区
openclaw browser set locale en-US

# 设备
openclaw browser set device "iPhone 14"
```

## 5.6.6.7 诊断命令

### 完整诊断流程

```bash
# 1. 检查浏览器状态
openclaw browser status

# 2. 列出 Tab
openclaw browser tabs

# 3. 查看控制台错误
openclaw browser console --level error

openclaw browser errors

# 5. 查看网络请求
openclaw browser requests

# 6. 获取快照
openclaw browser snapshot --interactive
```
