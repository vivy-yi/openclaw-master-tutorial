## 5.3.3 Skill 触发机制 (hook)

### 什么是 hook

hook 是 SKILL.md frontmatter 中的关键参数，用于**控制 Skill 的触发时机**。

```yaml
---
name: my-skill
description: "我的 Skill 描述"
hook: SessionStart   # ← 这就是 hook
---
```

### 支持的 hook 类型

| hook | 触发时机 | 用途 | 示例 |
|------|---------|------|------|
| `SessionStart` | 每次会话开始 | 加载核心规范、项目结构 | 运行时指南、工作区说明 |
| `MessageReceived` | 收到每条消息 | 响应特定触发词 | 搜索技能、文档解析 |
| `CronExecuted` | Cron 任务执行后 | 处理任务结果 | 报告生成、数据处理 |
| `AgentSpawned` | 子 Agent 创建时 | 子 Agent 初始化 | 子 Agent 引导 |
| 无 hook | 手动调用 | 需要时使用 | 临时任务、专业技能 |

### SessionStart hook 示例

```yaml
---
name: openclaw-runtime-guide
description: "OpenClaw 运行时指南：核心规范、设计原则、安全约束。触发词：规范, 原则, 限制, 安全"
hook: SessionStart
---

# OpenClaw 运行时指南

本指南在每次会话开始时自动注入，帮助 Agent 理解核心规范。
```

### hook 的实际行为

**重要澄清**：带有 `hook: SessionStart` 的 Skill **不会自动注入内容到 prompt**。

```yaml
---
name: openclaw-runtime-guide
description: "OpenClaw 运行时指南：核心规范、设计原则、安全约束。触发词：规范, 原则, 限制, 安全"
hook: SessionStart
---
```

设置 `hook: SessionStart` 的实际效果：

1. Skill 出现在 **`<available_skills>` XML 目录**中
2. Agent 需要用 `read` 工具**主动读取** SKILL.md 才能使用
3. 内容**不是**直接注入到 system prompt

```
System Prompt 结构：
├── ## Skills (mandatory)
│   Before replying: scan <available_skills> <description> entries.
│   
│   <available_skills>
│     <skill>
│       <name>openclaw-runtime-guide</name>
│       <description>OpenClaw runtime guide...</description>
│       <location>/path/to/SKILL.md</location>   ← Agent 需主动 read
│     </skill>
│   </available_skills>
│
├── ## Project Context   ← 这是真正的自动注入
│   ## SOUL.md / USER.md / AGENTS.md 等文件内容直接在这里
```

**对比**：

| 注入方式 | 内容来源 | 如何获取 |
|----------|---------|---------|
| **文件加载**（Project Context） | SOUL.md, USER.md 等 | 直接在 prompt 中，**无需操作** |
| **Skills 目录** | SKILL.md | Agent 用 `read` 工具主动读取 |

### Plugin Hook 系统（真正注入）

OpenClaw 的 Plugin Hook 系统是**真正的内容注入机制**，与 Skills 的 `hook` 参数不同。

#### 支持的 Hook 类型

| Hook 名称 | 触发时机 | 用途 | 返回值 |
|-----------|---------|------|--------|
| `before_prompt_build` | 构建 system prompt 前 | 修改/注入内容 | `{ systemPrompt, prependSystemContext, appendSystemContext }` |
| `before_agent_start` | Agent 启动前（遗留） | 同上 | 同上 |
| `session_start` | 会话开始时 | 内存同步、通知等 | 无（fire-and-forget） |
| `session_end` | 会话结束时 | 归档、日志 | 无 |

#### 注入机制

`before_prompt_build` hook 可以返回以下字段来修改 system prompt：

```typescript
{
  systemPrompt?: string;        // 替换整个 system prompt
  prependSystemContext?: string; // 在 prompt 开头插入内容
  appendSystemContext?: string;  // 在 prompt 末尾追加内容
}
```

#### 注入流程

```
会话开始
    │
    ▼
构建 System Prompt
    │
    ▼
┌─────────────────────────────────────┐
│  before_prompt_build hook 执行      │
│  - Plugin 返回 { prependSystemContext,  │
│               appendSystemContext }  │
└──────────────┬──────────────────────┘
               │
               ▼
┌─────────────────────────────────────┐
│  合并到 System Prompt               │
│  prependSystemContext → prompt 开头   │
│  appendSystemContext → prompt 末尾   │
└──────────────┬──────────────────────┘
               │
               ▼
         发送给 LLM
```

#### 示例：内存同步 Plugin

```javascript
// Plugin 监听 session_start hook，执行内存预热
registerInternalHook('session_start', async (event) => {
  // 会话开始时预热内存搜索索引
  await memoryManager.sync({ reason: 'session-start' });
});
```

#### Skills hook vs Plugin Hook

| 对比项 | Skills `hook` 参数 | Plugin Hook |
|--------|-------------------|-------------|
| **定义位置** | SKILL.md frontmatter | Plugin 代码 |
| **触发机制** | 技能目录注册 | 事件驱动 |
| **内容注入** | ❌ 只是目录 | ✅ 可修改 system prompt |
| **用途** | 告诉 Agent 何时读取 | 真正插入内容 |
| **执行者** | Agent（读取 Skill） | OpenClaw（构建 prompt 时） |

### MessageReceived hook 示例

```yaml
---
name: web-search
description: "网页搜索技能。触发词：搜索, 帮我搜, look up"
hook: MessageReceived
---

当用户消息匹配触发词时，自动激活此 Skill。
```

### 无 hook（手动调用）

```yaml
---
name: code-review
description: "代码审查技能。使用 @code-review 调用"
# 无 hook，手动使用
---
```

```yaml
---
name: web-search
description: "网页搜索技能。触发词：搜索, 帮我搜, look up"
hook: MessageReceived
---

当用户消息匹配触发词时，自动激活此 Skill。
```

### 无 hook（手动调用）

```yaml
---
name: code-review
description: "代码审查技能。使用 @code-review 调用"
# 无 hook，手动使用
---
```

### hook 选择指南

| 场景 | 推荐 hook |
|------|----------|
| 核心规范、安全约束、设计原则 | `SessionStart`（放在 Skill 目录） |
| 需要理解项目结构、自我改进机制 | `SessionStart` |
| 工具类技能（搜索、解析、生成） | `MessageReceived` |
| 定时任务的后处理 | `CronExecuted` |
| 一次性专业任务 | 无 hook（手动调用） |

### 注意事项

- `hook` 参数是可选的，不写则默认为手动调用
- **Skills `hook: SessionStart` 只是加入技能目录**，不是自动注入内容到 prompt
- **真正注入内容需要 Plugin Hook 系统**（`before_prompt_build`）
- 多个 Skill 可以有相同的 hook，都会出现在目录中
- SessionStart 时过多 Skill 会增加 token 消耗（因为目录变大）

## 5.3.5 Skills 配置

### 启用 Skills

```json
{
  "skills": {
    "enabled": true,
    "auto_load": true,
    "paths": [
      "~/.openclaw/skills",
      "./skills"
    ]
  }
}
```

### 指定 Skill

```json
{
  "agent": {
    "skill": "code-review"
  }
}
```

### 多 Skill 组合

```json
{
  "agent": {
    "skills": ["code-review", "unit-test"]
  }
}
```

## 5.3.6 Skills 工作流

### 执行流程

```
用户：审查这个代码

═══════════════════════════════════════════════

Step 1: Skill 匹配
  → 识别需要 code-review Skill
  → 加载 Skill 配置

Step 2: 提示词组装
  → 加载 code-review prompt
  → 注入代码内容
  → 添加审查标准

Step 3: 工具准备
  → 准备 read 工具
  → 准备 grep 工具
  → 配置工具参数

Step 4: 执行推理
  → LLM 分析代码
  → 逐项检查
  → 生成建议

Step 5: 结果处理
  → 格式化输出
  → 添加修复建议
  → 返回审查报告

═══════════════════════════════════════════════

返回：
📋 代码审查报告
├── 代码规范：✅ 通过
├── 安全性：⚠️ 发现问题
├── 性能：✅ 良好
└── 建议：3 条
```

### 上下文传递

```
上下文包含：
├── 当前代码内容
├── 相关文件
├── 历史对话
└── 用户偏好

传递方式：
1. 自动注入
2. 按需加载
3. 手动指定
```

## 5.3.8 常见问题

### Q1: Skill 不生效

```
错误：Skill not found

解决：
1. 检查 Skill 名称是否正确
2. 确认 Skill 路径配置正确
3. 检查 Skill 文件格式
```

### Q2: Skill 冲突

```
问题：多个 Skill 功能重叠

解决：
1. 指定单一 Skill
2. 调整 Skill 优先级
3. 禁用冲突的 Skill
```

### Q3: Skill 执行慢

```
问题：Skill 响应慢

解决：
1. 检查 Skill 复杂度
2. 优化工具调用
3. 减少上下文
```
