## 5.5.1 创建流程

### 自定义技能概述

> ⚠️ **重要说明**：OpenClaw Skill **只需 SKILL.md 一个文件**，不需要 prompt.md、tools.json、config.yaml 等额外文件。

```
官方 Skill 开发流程：

1. 规划技能
   → 确定功能
   → 定义输入输出

2. 创建文件
   → 只需 SKILL.md 一个文件

3. 本地测试
   → 加载技能
   → 运行测试

4. 发布分享
   → 打包发布
   → 贡献社区
```

## 5.5.2 技能文件结构

### 目录结构

> ⚠️ **官方结构**：Skill 只需一个 `SKILL.md` 文件，不需要其他配置文件。

```
my-custom-skill/
└── SKILL.md           # 技能定义（唯一必需文件）
```

> **注**：加载优先级为 `workspace > personal > managed > bundled`

![Skill结构]/assets/diagrams/21_skill_anatomy.png)



### SKILL.md 结构

```markdown
# Skill Name

## Description
技能的功能描述

## Capabilities
- 功能 1
- 功能 2
- 功能 3

## Usage
### 命令
`@skill-name [参数]`

### 示例
示例 1
示例 2

## Requirements
- 要求 1
- 要求 2
```

## 5.5.3 实战：创建代码审查技能

### Step 1: 创建目录

```bash
mkdir -p ~/.openclaw/skills/code-review
cd ~/.openclaw/skills/code-review
```

### Step 2: 创建 SKILL.md

```markdown
# Code Review

## Description
专业的代码审查技能，提供代码规范检查、安全扫描、性能优化建议。

## Capabilities
- 代码规范检查（ESLint、Prettier）
- 安全漏洞扫描
- 性能问题识别
- 代码可维护性评估
- 修复建议生成

## Usage
### 基本使用
@code-review /path/to/file

### 审查特定方面
@code-review --security /path/to/file
@code-review --performance /path/to/file

## Tools Used
- read: 读取代码文件
- grep: 搜索代码模式
- exec: 运行 lint 工具

### Step 3: 测试技能

```bash
# 重新加载技能
openclaw skills reload

# 测试技能
openclaw chat @code-review test.js
```

### 调试技巧

```
调试方法：
1. 开启调试模式
   → openclaw debug on

2. 查看日志
   → tail -f ~/.openclaw/logs/skills.log

3. 单步执行
   → openclaw skills trace
```

## 5.5.5 技能参数

### 定义参数

```yaml
parameters:
  - name: target
    type: string
    required: true
    description: 审查目标文件或目录

  - name: severity
    type: string
    required: false
    default: warning
    description: 最低严重级别
    enum: [error, warning, info]

  - name: rules
    type: array
    required: false
    description: 自定义规则列表
```

### 使用参数

```
用户：@code-review app.js --severity error
→ 解析参数：severity = error
→ 过滤输出：只显示 error 级别
```

## 5.5.6 高级特性

### 条件分支

```yaml
workflow:
  - if: security_scan
    steps:
      - tool: grep
        pattern: "SQL|script|eval"
      - tool: exec
        command: security-checker

  - if: lint_check
    steps:
      - tool: exec
        command: eslint
```

### 循环处理

```yaml
workflow:
  - for: file in files
    steps:
      - tool: read
        path: "{{file}}"
      - tool: analyze
```

### 错误处理

```yaml
on_error:
  - log: "Error in {{step}}"
  - notify: user
  - fallback: "审查失败，请检查文件路径"
```

## 5.5.7 发布技能

### 发布到市场

```bash
# 打包技能
openclaw skills pack my-skill

# 发布技能
openclaw skills publish ./my-skill.tar.gz

# 设置价格（可选）
openclaw skills publish ./my-skill.tar.gz --price 0
```

## 5.5.8 常见问题

### Q1: 技能加载失败

```
错误：Skill load failed

解决：
1. 检查文件格式
2. 验证 SKILL.md 存在
3. 检查 YAML/JSON 语法
4. 查看错误日志
```

### Q2: 工具调用失败

```
错误：Tool not available

解决：
1. 在 tools.json 中启用
2. 检查权限配置
3. 确认工具存在
```

### Q3: 输出格式错误

```
问题：输出不符合预期

解决：
1. 在 prompt 中明确格式
2. 添加输出示例
3. 添加验证逻辑
```
