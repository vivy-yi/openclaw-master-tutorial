# Claude Code / OpenClaw 内置 Connector 完整指南

> 本指南详细介绍 Claude Code 和 OpenClaw 中内置的 Connector（连接器），涵盖所有已安装的技能（Skills）及其配置方法。

---

## 一、概述

### 1.1 什么是 Connector

Connector（连接器）是 OpenClaw 内置的技能模块，用于与外部服务进行集成和交互。它们以 **Skill** 的形式存在，位于：

```
~/.npm-global/lib/node_modules/openclaw/skills/
# 或
~/.claude/skills/
```

### 1.2 Connector 分类

| 分类 | 数量 | 示例 |
|------|------|------|
| **协作工具** | 8+ | GitHub, Slack, Discord, Trello |
| **笔记应用** | 4+ | Notion, Obsidian, Apple Notes, Bear |
| **日历/任务** | 4+ | Things, Apple Reminders, Calendar |
| **IoT/硬件** | 3+ | OpenHue, Sonos, OpenRGB |
| **系统工具** | 5+ | Tmux, Camera, Screen |
| **消息平台** | 5+ | iMessage, BlueBubbles, Telegram |
| **多媒体** | 5+ | Spotify, Video Frames, Voice Call |
| **API 工具** | 3+ | GitHub CLI, OrderCLI, WACLI |

## 二、协作工具 Connector

### 2.1 GitHub Connector

| 属性 | 说明 |
|------|------|
| **Skill 名称** | `github` |
| **描述** | GitHub 操作 via `gh` CLI：Issues、PRs、CI 运行、代码审查、API 查询 |
| **配置要求** | `gh auth login` |

#### 触发场景

```
✅ 使用
- 检查 PR 状态或 CI
- 创建/评论 Issues
- 列出/过滤 PRs 或 Issues
- 查看运行日志

❌ 不使用
- 需要手动浏览器流程的复杂 Web UI 交互
- 跨多仓库的批量操作
- gh auth 未配置时
```

#### 使用示例

```
Agent: 检查一下 main 分支的最新 CI 状态
Agent: 创建一个新 Issue 描述这个问题
Agent: 查看最近 5 个已合并的 PR
```

## 十二、配置管理

### 12.1 环境变量配置

```bash
# Notion
export NOTION_API_KEY="secret_xxx"

# GitHub（通过 gh CLI）
gh auth login

# Trello
export TRELLO_API_KEY="xxx"
export TRELLO_TOKEN="xxx"

# OpenHue
export HUE_BRIDGE_IP="192.168.1.x"
export HUE_API_KEY="xxx"
```

### 12.2 OpenClaw 配置

在 `openclaw.json` 中配置渠道：

```json
{
  "channels": {
    "slack": {
      "token": "xoxb-xxx"
    },
    "discord": {
      "token": "xxx"
    }
  }
}
```

## 十三、故障排除

### 常见问题

| 问题 | 解决方案 |
|------|---------|
| Connector 未找到 | 检查 Skill 是否正确安装 |
| API 认证失败 | 验证 API Key 是否正确 |
| 权限不足 | 检查 OAuth 权限范围 |
| 超时错误 | 增加 timeout 设置 |

### 诊断命令

```bash
# 检查 OpenClaw 状态
openclaw status

# 检查 Skill 安装
openclaw skills list

# 测试连接
openclaw doctor
```
