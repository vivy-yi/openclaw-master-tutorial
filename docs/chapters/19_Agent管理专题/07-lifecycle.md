## OpenClaw Agent 完整生命周期

### 📊 一、用户需求响应流程

```
┌─────────────────────────────────────────────────────────────────────┐
│                        1. 消息入口                                    │
│         用户在 Telegram 群发送消息 → Gateway 接收                     │
└─────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        2. Binding 匹配                               │
│         Gateway 根据 binding 规则 → 分发到对应 Agent                  │
└─────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        3. Session 管理                               │
│         加载/创建 Session → 加载 Workspace 上下文                     │
│         - SOUL.md (角色定义)                                        │
│         - AGENTS.md (指令)                                          │
│         - MEMORY.md (长期记忆)                                      │
│         - memory/YYYY-MM-DD.md (每日记录)                          │
└─────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        4. Prompt 组装                                │
│         构建系统提示：基础提示 + Skills + 上下文 + 引导               │
└─────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        5. 模型推理                                    │
│         调用 LLM (MiniMax) → 流式输出                                │
│         - thinking (推理)                                          │
│         - assistant (回复)                                          │
│         - tool (工具调用)                                          │
└─────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        6. 工具执行                                    │
│         Agent 调用工具 → 执行 → 返回结果                             │
│         - memory_search / memory_get (记忆)                          │
│         - sessions_spawn (调用子 Agent)                             │
│         - 其他工具                                                 │
└─────────────────────────────────────────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────────┐
│                        7. 响应返回                                   │
│         流式返回给用户 → Session 持久化                            │
└─────────────────────────────────────────────────────────────────────┘
```

### 📊 二、Agent 状态监控

| 监控维度 | 说明 | 工具/方法 |
|---------|------|----------|
| **Session 状态** | 活跃会话数、Token 使用量 | `openclaw status` |
| **运行时状态** | PID、内存、运行时间 | Gateway Dashboard |
| **心跳监控** | 定时健康检查 | `HEARTBEAT.md` |
| **错误追踪** | 错误日志、异常捕获 | `openclaw logs` |
| **Cron 任务** | 定时任务执行状态 | `openclaw cron list` |

**Session 存储位置**：
```
~/.openclaw/agents/<agent_id>/sessions/
├── <session_id>.jsonl  ← 会话历史
└── ...
```

### 📊 三、Agent 学习/进化/更新 Loop

#### 1. 学习机制（Memory）

```
┌─────────────────────────────────────────────┐
│              短期记忆 (Session)              │
│         当前对话上下文 (自动管理)             │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              每日记录 (Daily)                │
│    memory/YYYY-MM-DD.md (每日新增)          │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              长期记忆 (Long-term)           │
│           MEMORY.md (人工/自动提炼)         │
└─────────────────────────────────────────────┘
```

**自动记忆 flush**：
- 当 Session 接近上下文压缩时
- OpenClaw 会触发一个**静默的 Agent 轮次**
- 提醒模型在压缩前**写入持久记忆**

#### 2. 进化机制（Evolution）

| 进化方式 | 说明 |
|---------|------|
| **手动更新** | 直接修改 SOUL.md / AGENTS.md |
| **自动学习** | 通过 memory_flush 自动写入偏好 |
| **用户反馈** | 用户说 "记住这个" → 写入 MEMORY.md |
| **上下文压缩** | summarization 保留关键信息 |

**触发记忆写入**：
```markdown
用户: "记住我喜欢麻辣口味"
→ Agent 调用 memory_get/write
→ 写入 MEMORY.md 或 memory/YYYY-MM-DD.md
```

#### 3. 更新 Loop 流程

```
┌─────────────────────────────────────────────┐
│              触发条件                        │
│  1. 用户明确要求 ("记住...")                 │
│  2. 上下文即将压缩 (pre-compaction)         │
│  3. Session 结束 (/reset)                  │
│  4. 手动编辑配置文件                        │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              执行更新                        │
│  - 写入 memory/YYYY-MM-DD.md               │
│  - 更新 MEMORY.md (重要/持久信息)           │
│  - 更新 SOUL.md (角色变化)                  │
│  - 更新 AGENTS.md (指令变化)                │
└─────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────┐
│              验证生效                        │
│  - 下次 Session 加载新上下文               │
│  - 通过 memory_search 验证记忆可检索        │
└─────────────────────────────────────────────┘
```

### 📊 四、关键配置文件

| 文件 | 位置 | 作用 |
|------|------|------|
| **SOUL.md** | workspace/ | Agent 角色定义 |
| **AGENTS.md** | workspace/ | 指令和行为规范 |
| **MEMORY.md** | workspace/ | 长期记忆 |
| **memory/YYYY-MM-DD.md** | workspace/ | 每日记录 |
| **HEARTBEAT.md** | workspace/ | 心跳检查任务 |

### 📊 五、Hook 拦截点

可以在以下位置拦截和扩展：

| Hook 事件 | 时机 |
|----------|------|
| `agent:bootstrap` | Bootstrap 文件加载前 |
| `before_model_resolve` | 模型解析前 |
| `before_prompt_build` | Prompt 构建前 |
| `before_tool_call` | 工具调用前 |
| `after_tool_call` | 工具调用后 |
| `message:received` | 收到消息时 |
| `message:sent` | 消息发送后 |
| `session:compact:before` | 上下文压缩前 |
| `session:compact:after` | 上下文压缩后 |
