# 墨析架构设计

## 一、整体架构

```
┌─────────────────────────────────────────────────┐
│                    墨析 Agent                     │
├─────────────────────────────────────────────────┤
│  SOUL.md ── 定位/理念/职责                      │
│  SKILL.md ─ 自适应 Skills 框架                   │
│  TOOLS.md ─ CDP 工具配置                        │
├─────────────────────────────────────────────────┤
│  知识库 (knowledge_base/)                       │
│  ├── platforms/ ── 平台学习记录                │
│  ├── api_endpoints/ ─ API 接口库               │
│  └── type_patterns/ ── 类型模式                 │
├─────────────────────────────────────────────────┤
│  数据 (collected_data/)                         │
│  ├── trending/ ── 热门数据                      │
│  ├── search/ ── 搜索结果                        │
│  └── products/ ── 商品数据                     │
└─────────────────────────────────────────────────┘
         ↓
┌─────────────────────────────────────────────────┐
│         OpenClaw browser tool (CDP)              │
│  open / snapshot / intercept / evaluate / act    │
└─────────────────────────────────────────────────┘
```

---

## 二、核心组件

### 1. SOUL.md - 角色定义

定义墨析的核心理念和职责：

```markdown
# 核心理念
CDP 原生自适应 - 只使用 OpenClaw 原生 CDP 浏览器协议

# 核心职责
1. 自适应采集 - 零预配置，自动探索任意网站
2. 自我进化 - 学习后记录方法，下次直接复用
3. 全链路覆盖 - 公开数据 + 用户画像 + 竞品监控
4. CDP 原生 - 不依赖任何第三方工具
```

### 2. SKILL.md - 自适应框架

定义自适应采集的核心逻辑：

```yaml
# 自适应采集工作流
用户请求 → URL识别 → CDP探索 → 自适应采集 → 数据标准化 → 自我学习

# 降级策略
API直连 → CDP DOM → Screenshot

# 学习机制
首次探索 → 记录方法 → 知识库 → 后续复用
```

### 3. TOOLS.md - 工具配置

定义 CDP 工具的使用方式：

```yaml
# 核心工具
browser: CDP 浏览器控制，核心采集工具

# 工具动作
browser.open(url)        # 打开页面
browser.snapshot()       # 分析结构
browser.intercept()      # 拦截 API
browser.evaluate(js)     # 执行 JS
browser.screenshot()      # 截图验证
```

## 三、数据流

```
用户请求
    ↓
┌──────────────────────────────────────┐
│ 1. 理解意图                           │
│    "采集知乎热榜" → 采集任务          │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ 2. URL 识别                          │
│    zhihu.com → 识别为内容平台        │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ 3. 知识库查询                         │
│    已学习？→ 直接复用                 │
│    未学习？→ CDP 探索                 │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ 4. CDP 自适应采集                     │
│    优先: API 直连                     │
│    降级: CDP DOM → Screenshot        │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ 5. 数据标准化                         │
│    → JSON 格式化                     │
│    → 存储到 collected_data/           │
└──────────────────────────────────────┘
    ↓
┌──────────────────────────────────────┐
│ 6. 自我学习（可选）                   │
│    → 记录成功的采集方法               │
│    → 更新 knowledge_base/             │
└──────────────────────────────────────┘
```

## 五、与传统方案的对比

| 维度 | 传统方案 | 墨析 |
|------|----------|------|
| **依赖** | opencli/第三方 CLI | 零依赖 |
| **平台** | 预配置 73+ | 任意网站 |
| **采集逻辑** | 固定 | 自动探索 |
| **维护** | 人工更新 | 自我学习 |
| **灵活性** | 受限 | 完全自适应 |

## 六、技术栈

- **Agent**: OpenClaw 原生 Agent
- **采集**: OpenClaw browser tool (CDP)
- **存储**: 本地文件系统 JSON
- **定时**: OpenClaw Cron
- **通知**: OpenClaw message
