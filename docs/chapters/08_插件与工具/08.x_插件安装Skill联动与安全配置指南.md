# 插件安装、Skill 联动与安全配置指南

> 本文档补充 OpenClaw 插件系统完全指南，覆盖：插件与 Skill 的内置关系、install 命令的工作流程、plugins.allow 白名单配置、以及常见安装问题排查。
> 配套阅读：[OpenClaw插件系统完全指南.md](./OpenClaw插件系统完全指南.md)

---

## 1. 插件与 Skill 的内置关系

### 1.1 核心概念：插件 = Runtime Code + 内置 Skill

OpenClaw 的插件采用**高内聚**设计：一个插件不仅可以包含运行时代码，还可以**自带 SKILL.md 和参考文档**。这意味着插件的"使用说明"和"运行代码"是打包在一起的。

```
lossless-claw/                          # 插件根目录
├── dist/                               # 插件运行时代码
│   └── index.js
├── skills/                             # ← 内置 Skill 目录
│   └── lossless-claw/
│       ├── SKILL.md                    # ← Skill 定义（指令、触发条件）
│       └── references/                # ← Skill 参考文档
├── openclaw.plugin.json                # 插件清单
└── package.json
```

### 1.2 openclaw.plugin.json 中的 Skill 声明

插件通过 `openclaw.plugin.json` 的 `skills` 字段声明自己带了哪些 Skill：

```json
{
  "id": "lossless-claw",
  "name": "Lossless Claw",
  "kind": "context-engine",
  "version": "0.9.x",
  "skills": [
    "skills/lossless-claw"              // ← 声明内置 Skill
  ],
  "slots": {
    "contextEngine": "lossless-claw"    // ← 声明占用的 slot
  },
  "runtime": {
    "node": true,
    "piVersion": "^0.70.0"
  }
}
```

当插件安装时，OpenClaw 会：
1. 加载插件的 runtime code
2. 读取 `skills` 字段中声明的所有 Skill
3. 将这些 Skill 注册到系统中，使其可用

### 1.3 Skill 的加载时机

执行 `openclaw plugins install --link /path/to/lossless-claw` 时：

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1 | Record | 将插件记录到配置 `plugins.entries` |
| 2 | Enable | 设置 `plugins.entries.<id>.enabled: true` |
| 3 | Slot Selection | 检查 `slots` 字段，自动应用 slot 占用 |
| 4 | Skill Registration | 读取 `skills` 字段，注册所有内置 Skill |

### 1.4 为什么这样设计？

| 优点 | 说明 |
|------|------|
| **版本同步** | Skill 说明和 runtime code 必须一起更新，不会出现文档过时的问题 |
| **自包含** | 用户安装一个插件，直接获得所有需要的东西 |
| **按需加载** | Skill 只在匹配任务时触发，不影响正常性能 |
| **易于分发** | 一个 npm 包 / git 仓库包含所有内容 |

### 1.5 查看插件内置的 Skill

安装插件后，可以查看其内置的 Skill：

```bash
# 方法一：查看 plugins list 输出
openclaw plugins list

# 方法二：查看 skills 目录
ls ~/.openclaw/extensions/<plugin-id>/skills/

# 方法三：查看 skill 内容
cat ~/.openclaw/extensions/<plugin-id>/skills/<skill-name>/SKILL.md
```

## 2. openclaw plugins install 工作流程详解

### 2.1 命令格式

```bash
# 从本地目录安装（开发/调试用）
openclaw plugins install --link /path/to/lossless-claw

# 从本地 OpenClaw checkout 安装
pnpm openclaw plugins install --link /path/to/lossless-claw

# 从 npm 安装（稳定版）
openclaw plugins install @martian-engineering/lossless-claw

# 从 npm 安装特定版本
openclaw plugins install @martian-engineering/lossless-claw@0.9.2
```

### 2.2 install 命令的完整工作流程

当执行 `openclaw plugins install --link /path/to/lossless-claw` 时，内部执行以下步骤：

```
Step 1: 解析插件包
    ├── 读取 openclaw.plugin.json
    ├── 验证 id/name/kind/version
    └── 检查 requiredPiVersion

Step 2: 记录插件到配置
    ├── 在 plugins.entries 下创建条目
    ├── 设置 enabled: true
    └── 写入 package.json 路径（如果是 --link）

Step 3: 应用 Slot 选择
    ├── 读取 slots 字段（如果有）
    ├── 如果 slot 已被占用，跳过或报错
    └── 如果 slot 可用，设置为该插件

Step 4: 注册内置 Skills
    ├── 读取 skills 字段（如果有）
    ├── 扫描 skills/ 目录
    └── 注册每个 SKILL.md 到系统

Step 5: 验证安装
    ├── 运行 healthcheck
    └── 确认插件状态为 loaded
```

### 2.3 Slot 选择机制

Slot 是插件的"槽位"，用于控制哪些插件可以同时激活。OpenClaw 有以下主要 slot：

| Slot 名称 | 用途 | 示例插件 |
|-----------|------|----------|
| `contextEngine` | 对话上下文压缩/管理 | lossless-claw, openviking |
| `memory` | 长期记忆存储 | memory-lancedb-pro, memory-wiki |

**重要规则**：每个 slot 同时只能有一个插件占用。

```javascript
// 如果 openviking 已占用 contextEngine slot
plugins.slots.contextEngine = "openviking"  // ← 已占用

// 安装 lossless-claw 时
openclaw plugins install lossless-claw
// 结果：lossless-claw 的 contextEngine slot 不会生效
// 警告：slot 冲突，contextEngine 仍为 openviking
```

### 2.4 查看当前 Slot 分配

```bash
openclaw config get plugins.slots
```

输出示例：
```json
{
  "contextEngine": "lossless-claw",
  "memory": "memory-lancedb-pro"
}
```

### 2.5 手动切换 Slot

如果需要手动切换 slot：

```bash
# 切换 contextEngine slot 到 lossless-claw
openclaw config set plugins.slots.contextEngine lossless-claw

# 切换到 openviking
openclaw config set plugins.slots.contextEngine openviking

# 重启 Gateway 生效
openclaw gateway restart
```

## 3. plugins.allow 白名单配置

### 3.1 什么时候需要 plugins.allow？

当你启用了**安全模式**（`plugins.mode: "allowlist"`）时，必须显式声明允许哪些插件运行：

```json
{
  "plugins": {
    "mode": "allowlist",              // ← 启用白名单模式
    "allow": [
      "lossless-claw",                // ← 只允许这些插件
      "memory-lancedb-pro",
      "acpx"
    ]
  }
}
```

### 3.2 plugins.allow vs plugins.entries

| 配置项 | 作用 | 示例 |
|--------|------|------|
| `plugins.entries` | 已安装的插件列表 | `lossless-claw: { enabled: true }` |
| `plugins.allow` | 白名单（安全模式） | `["lossless-claw", "acpx"]` |
| `plugins.slots` | Slot 分配 | `{ contextEngine: "lossless-claw" }` |

### 3.3 常见错误：误将命令 token 加入 allow

**❌ 错误做法**：
```json
{
  "plugins": {
    "allow": [
      "lossless",                     // ← 错！这是命令 token，不是插件 ID
      "/lcm",                         // ← 错！这是命令别名
      "lossless-claw"                 // ← 对
    ]
  }
}
```

**✅ 正确做法**：
```json
{
  "plugins": {
    "allow": [
      "lossless-claw",                // ← 插件 ID
      "memory-lancedb-pro",
      "acpx"
    ]
  }
}
```

**原因**：`plugins.allow` 只接受**插件 ID**（即 `openclaw.plugin.json` 中的 `id` 字段），不接受命令 token 或别名。

### 3.4 插件 ID vs 命令 token

| 类型 | 示例 | 用于 |
|------|------|------|
| 插件 ID | `lossless-claw`, `memory-lancedb-pro` | `plugins.allow`, `plugins.entries` |
| 命令 token | `lossless`, `/lcm` | 聊天中触发插件命令 |

### 3.5 查看已安装插件的 ID

```bash
# 方法一：查看插件 list
openclaw plugins list

# 方法二：查看插件的 openclaw.plugin.json
cat ~/.openclaw/extensions/<plugin-id>/openclaw.plugin.json | grep '"id"'
```

## 4. 常见安装问题与故障排查

### 4.1 问题：Unable to resolve plugin runtime module

**错误信息**：
```
[ERROR] Unable to resolve plugin runtime module for lossless-claw
```

**原因**：插件的 `package.json` 中 `pi-*` 依赖版本与当前 OpenClaw 不兼容。

**解决方案**：
```bash
# 方案一：使用 --link 安装本地版本，然后手动更新依赖
cd /path/to/lossless-claw
pnpm install
pnpm build

# 方案二：等待插件更新到兼容版本
# 跟踪：https://github.com/Martian-Engineering/lossless-claw/issues/497
```

### 4.2 问题：Slot 冲突（contextEngine slot 被占用）

**错误信息**：
```
[WARNING] Plugin lossless-claw requests contextEngine slot, but it is already occupied by openviking
```

**原因**：两个插件都想占用 `contextEngine` slot，但同时只能有一个。

**解决方案**：
```bash
# 查看当前 slot 分配
openclaw config get plugins.slots

# 手动切换到 lossless-claw（如果确实需要）
openclaw config set plugins.slots.contextEngine lossless-claw
openclaw gateway restart

# 或者保持 openviking，禁用 lossless-claw 的 contextEngine 功能
```

### 4.3 问题：插件已安装但 Skill 不可用

**原因**：插件的 `skills` 字段没有正确声明，或者 Skill 文件不存在。

**排查步骤**：
```bash
# 1. 确认插件已安装且 enabled
openclaw plugins list | grep <plugin-id>

# 2. 检查插件目录结构
ls ~/.openclaw/extensions/<plugin-id>/skills/

# 3. 确认 SKILL.md 存在
cat ~/.openclaw/extensions/<plugin-id>/skills/<skill-name>/SKILL.md

# 4. 检查 OpenClaw 日志
openclaw logs | grep -i skill
```

### 4.4 问题：plugins.allow 白名单模式导致插件被阻止

**错误信息**：
```
[ERROR] Plugin <plugin-id> is not in plugins.allow list and plugins.mode is "allowlist"
```

**解决方案**：
```bash
# 将插件添加到白名单
openclaw config set plugins.allow '["lossless-claw", "memory-lancedb-pro", "acpx"]'

# 或者关闭白名单模式
openclaw config set plugins.mode "default"
```

### 4.5 问题：版本兼容性

**背景**：lossless-claw v0.9.2 已知与 OpenClaw ≥某些版本有兼容性问题（pi-* 依赖硬编码为 0.66.1）。

**检查当前版本**：
```bash
# 查看 OpenClaw 版本
openclaw --version

# 查看插件版本
openclaw plugins list | grep lossless-claw
```

**建议**：
1. 使用最新版本的 lossless-claw（检查 GitHub releases）
2. 或使用 `--link` 安装本地版本，手动调整 `package.json` 中的依赖版本

### 4.6 完整故障排查流程

```bash
# 1. 确认插件状态
openclaw plugins list

# 2. 查看详细日志
openclaw logs --level debug 2>&1 | grep -i "lossless\|plugin\|slot"

# 3. 检查配置
openclaw config get plugins

# 4. 检查 Slot 分配
openclaw config get plugins.slots

# 5. 尝试重新安装
openclaw plugins uninstall <plugin-id>
openclaw plugins install <plugin-id>

# 6. 重启 Gateway
openclaw gateway restart
```

## 5. 最佳实践

### 5.1 安装插件前的检查清单

- [ ] 确认 OpenClaw 版本兼容
- [ ] 确认 slot 不冲突（如果是 contextEngine 或 memory slot）
- [ ] 如果使用安全模式，准备好 `plugins.allow` 配置
- [ ] 查看 GitHub issues 了解已知问题

### 5.2 插件配置示例

```json
{
  "plugins": {
    "mode": "allowlist",
    "allow": [
      "lossless-claw",
      "memory-lancedb-pro",
      "acpx"
    ],
    "entries": {
      "lossless-claw": {
        "enabled": true,
        "config": {
          "summarizeModel": "claude-3-haiku"
        }
      }
    },
    "slots": {
      "contextEngine": "lossless-claw",
      "memory": "memory-lancedb-pro"
    }
  }
}
```

### 5.3 验证安装成功

```bash
# 检查插件状态
openclaw plugins list

# 检查 Skill 可用性
openclaw skills check | grep <plugin-id>

# 测试插件命令（如果有）
/lcm status
```
