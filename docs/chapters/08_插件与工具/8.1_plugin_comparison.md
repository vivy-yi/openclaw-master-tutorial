## 3. Lark CLI vs Google CLI

### 3.1 基本信息

| | **Lark CLI** (飞书) | **Google CLI** (gcloud) |
|---|---|---|
| **官网** | github.com/larksuite/cli | cloud.google.com/sdk/gcloud |
| **包管理** | npm `@larksuite/cli` | Google Cloud SDK |
| **认证方式** | OAuth / Access Token | gcloud auth |
| **交互模式** | REPL 交互 + 命令 | 命令 + 配置文件 |
| **跨平台** | ✅ macOS/Linux/Windows | ✅ 完整支持 |

### 3.2 核心命令对比

#### Lark CLI

```bash
# 文档操作
lark doc create          # 创建文档
lark doc read <token>    # 读取文档
lark doc write <token>   # 写入文档

# 消息
lark message send        # 发送消息
lark message receive     # 接收消息

# 日历
lark calendar list        # 列出日历
lark calendar create      # 创建日程

# 云空间
lark drive list          # 列出文件
lark drive upload         # 上传文件

# 通讯录
lark contacts search      # 搜索联系人

# 审批
lark approval list        # 列出审批
```

#### Google CLI (gcloud)

```bash
# 计算引擎
gcloud compute instances list
gcloud compute ssh <instance>

# 云存储
gcloud storage ls
gcloud storage cp <source> <dest>

# BigQuery
gcloud bigquery query <sql>
gcloud bigquery datasets list

# Kubernetes
gcloud container clusters list
kubectl get pods

# AI Platform
gcloud ai models list
gcloud ai endpoints list

# IAM 权限
gcloud iam service-accounts list
gcloud projects add-iam-policy-binding

# 日志
gcloud logging read <filter>
```

## 4. 最佳实践

### 4.1 飞书用户推荐配置

```yaml
# OpenClaw 配置
plugins:
  load:
    - feishu

channels:
  feishu:
    enabled: true
    tools:
      doc: true
      wiki: true
      drive: true
      bitable: true
      perm: false  # 按需启用

# 补充 Lark CLI（手动操作）
# npm install -g @larksuite/cli
```

### 4.2 工具组合策略

| 需求 | 推荐组合 |
|------|---------|
| Agent 自动化飞书文档 | OpenClaw Feishu 插件 |
| 主动发送飞书消息 | Lark CLI |
| 手动管理飞书 | Lark CLI |
| 飞书 + GCP 双管 | Lark CLI + gcloud CLI |
| 全功能飞书 + 自动化 | **OpenClaw Feishu 插件 + Lark CLI** |
