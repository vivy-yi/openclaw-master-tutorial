## 3.9.1 Session 核心概念

### 什么是 Session？

Session（会话）是 OpenClaw 管理对话历史和状态的核心机制。每个 Session 包含：

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                           Session 组成                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  Session = {                                                                  │
│    sessionKey: "agent:main:telegram:direct:123456789",                       │
│    sessionId: "uuid-v4",                                                      │
│    updatedAt: 1700000000000,                                                  │
│    model: "claude-sonnet-4-5",                                                │
│    contextTokens: 5000,                                                       │
│    totalTokens: 15000,                                                        │
│    ...                                                                        │
│  }                                                                            │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### Session Key 格式

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         Session Key 格式                                       │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  直接消息 (DM):                                                                │
│  • main:       agent:<agentId>:main                                           │
│  • per-peer:  agent:<agentId>:direct:<peerId>                                │
│  • per-channel-peer: agent:<agentId>:<channel>:direct:<peerId>              │
│  • per-account-channel-peer: agent:<agentId>:<channel>:<accountId>:direct:<peerId>│
│                                                                                 │
│  群组消息:                                                                    │
│  • agent:<agentId>:<channel>:group:<id>                                      │
│  • Telegram 话题: agent:<agentId>:telegram:group:<id>:topic:<threadId>       │
│                                                                                 │
│  其他来源:                                                                    │
│  • Cron: cron:<job.id>                                                       │
│  • Webhook: hook:<uuid>                                                      │
│  • Node: node-<nodeId>                                                       │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

## 3.9.2 DM 作用域配置

### dmScope 选项

`dmScope` 控制直接消息如何分组：

```json5
{
  session: {
    dmScope: "main"  // 选项: main | per-peer | per-channel-peer | per-account-channel-peer
  }
}
```

| dmScope | 说明 | 适用场景 |
|---------|------|---------|
| `main` | 所有 DM 共享主会话 | 单用户连续对话 |
| `per-peer` | 按发送者 ID 隔离 | 多用户简单隔离 |
| `per-channel-peer` | 按渠道+发送者隔离 | 多用户多渠道 |
| `per-account-channel-peer` | 按账户+渠道+发送者隔离 | 多账户多用户 |

### 安全 DM 模式（重要）

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         安全 DM 模式警告                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ⚠️ 如果你的 Agent 可以接收多人的 DM，强烈建议启用安全 DM 模式！               │
│                                                                                 │
│  问题场景：                                                                   │
│  • Alice 向 Agent 发送私人消息（医疗预约）                                    │
│  • Bob 向 Agent 发送消息"我们在聊什么？"                                     │
│  • 由于默认 DM 共享同一会话，Agent 可能用 Alice 的上下文回答 Bob           │
│                                                                                 │
│  解决方案：设置 dmScope 为 per-channel-peer                                   │
│                                                                                 │
│  {                                                                             │
│    session: {                                                                 │
│      dmScope: "per-channel-peer"                                              │
│    }                                                                          │
│  }                                                                             │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### identityLinks 配置

如果同一个人通过多个渠道联系你，可以使用 identityLinks 合并会话：

```json5
{
  session: {
    dmScope: "per-channel-peer",
    identityLinks: {
      "alice": ["telegram:123456789", "discord:987654321012345678"]
    }
  }
}
```

## 3.9.3 Session 存储位置

### 文件结构

```
~/.openclaw/
├── agents/
│   ├── main/
│   │   ├── agent/
│   │   │   └── auth-profiles.json
│   │   └── sessions/
│   │       ├── sessions.json          # 会话索引
│   │       ├── <SessionId>.jsonl      # 完整对话历史
│   │       └── ...
│   └── work/
│       └── sessions/
│           └── ...
└── workspace/
```

### 存储文件说明

| 文件 | 说明 |
|------|------|
| `sessions.json` | 会话索引（sessionKey -> metadata 映射）|
| `<SessionId>.jsonl` | 完整对话历史（JSONL 格式）|
| Telegram 话题会话 | `<SessionId>-topic-<threadId>.jsonl` |

## 3.9.4 Session 维护配置

### 默认配置

```json5
{
  session: {
    maintenance: {
      mode: "warn",           // warn | enforce
      pruneAfter: "30d",      // 删除多久的旧条目
      maxEntries: 500,        // 最大会话数
      rotateBytes: "10mb"     // sessions.json 轮转大小
    }
  }
}
```

### 维护模式说明

| 模式 | 说明 |
|------|------|
| `warn` | 报告将被清理的内容，但不实际删除 |
| `enforce` | 自动执行清理 |

### 清理执行顺序

1. 删除超过 `pruneAfter` 的旧条目
2. 限制条目数量到 `maxEntries`
3. 归档已删除条目的 transcript 文件
4. 清理旧的归档文件
5. 轮转超过 `rotateBytes` 的 sessions.json
6. 如果设置了 `maxDiskBytes`，强制磁盘预算

### 生产环境配置示例

```json5
{
  session: {
    maintenance: {
      mode: "enforce",
      pruneAfter: "45d",
      maxEntries: 800,
      rotateBytes: "20mb",
      resetArchiveRetention: "14d",
      maxDiskBytes: "1gb",
      highWaterBytes: "800mb"
    }
  }
}
```

### CLI 命令

```bash
# 预览清理结果（不执行）
openclaw sessions cleanup --dry-run

# 执行清理
openclaw sessions cleanup --enforce

# 查看会话列表
openclaw sessions --json

# 只看活跃会话（最近 N 分钟）
openclaw sessions --json --active 60
```

## 3.9.5 Session 重置策略

### 重置模式

```json5
{
  session: {
    reset: {
      mode: "daily",       // daily | idle | never
      atHour: 4,          // 每日重置时间（本地时区）
      idleMinutes: 120    // 空闲超时（分钟）
    }
  }
}
```

| 模式 | 说明 |
|------|------|
| `daily` | 每天指定时间重置 |
| `idle` | 空闲指定时间后重置 |
| `never` | 不自动重置 |

### 按类型重置

```json5
{
  session: {
    resetByType: {
      direct: { mode: "idle", idleMinutes: 240 },
      group: { mode: "idle", idleMinutes: 120 },
      thread: { mode: "daily", atHour: 4 }
    }
  }
}
```

### 按渠道重置

```json5
{
  session: {
    resetByChannel: {
      discord: { mode: "idle", idleMinutes: 10080 }  // Discord 保留 7 天
    }
  }
}
```

### 手动重置触发

| 触发方式 | 说明 |
|---------|------|
| `/new` | 开始新会话 |
| `/reset` | 重置当前会话 |
| `/new claude-opus-4-5` | 指定模型开始新会话 |

## 3.9.6 Session 工具

Session 工具允许 Agent 列出会话、获取历史和跨会话发送消息。

### 工具列表

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          Session 工具                                          │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  sessions_list     →  列出会话                                                │
│  sessions_history →  获取会话历史                                              │
│  sessions_send    →  向另一个会话发送消息                                      │
│  sessions_spawn   →  生成子 Agent 运行                                         │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### sessions_list - 列出会话

```json5
{
  kinds: ["main", "group", "cron"],  // 过滤器
  limit: 50,                          // 最大数量
  activeMinutes: 60,                  // 最近 N 分钟活跃
  messageLimit: 3                     // 包含最后 N 条消息
}
```

**返回格式：**

```json
[
  {
    "key": "agent:main:telegram:direct:123456789",
    "kind": "direct",
    "channel": "telegram",
    "displayName": "Alice",
    "updatedAt": 1701234567890,
    "sessionId": "abc123",
    "model": "claude-sonnet-4-5",
    "contextTokens": 5000,
    "totalTokens": 15000,
    "messages": [...]
  }
]
```

### sessions_history - 获取会话历史

```json5
{
  sessionKey: "agent:main:telegram:direct:123456789",
  // 或使用 sessionId:
  // sessionId: "abc123"
  limit: 20,
  includeTools: false  // 是否包含工具消息
}
```

### sessions_send - 发送消息（同步/异步）

```json5
{
  sessionKey: "agent:work:telegram:group:123456789",
  message: "请审查这段代码",
  timeoutSeconds: 120  // 0 = 即发即忘
}
```

**返回格式：**

```json
{
  "runId": "run_abc123",
  "status": "ok",
  "reply": "审查结果..."
}
```

**乒乓对话：**
- 最多 `session.agentToAgent.maxPingPongTurns` 轮（默认 5）
- 回复 `REPLY_SKIP` 停止对话

### sessions_spawn - 生成子 Agent

```json5
{
  task: "分析这个 Python 项目的架构",
  label: "code-analysis",
  agentId: "analyzer",
  model: "claude-opus-4-5",
  runTimeoutSeconds: 300,
  cleanup: "delete"
}
```

**返回格式：**

```json
{
  "status": "accepted",
  "runId": "run_abc123",
  "childSessionKey": "agent:analyzer:subagent:uuid"
}
```

## 3.9.7 启用跨 Agent 通信

默认情况下，Agent 间通信是**禁用**的。需要在配置中显式启用：

```json5
{
  tools: {
    agentToAgent: {
      enabled: true,
      allow: ["main", "work", "analyzer"]
    }
  }
}
```

### 配置项说明

| 配置项 | 类型 | 说明 |
|-------|------|------|
| `enabled` | boolean | 是否启用跨 Agent 通信 |
| `allow` | string[] | 允许通信的 Agent ID 列表 |
| `maxPingPongTurns` | number | 乒乓对话最大轮次 (0-5) |

## 3.9.8 Send Policy 配置

可以阻止特定会话类型的消息发送：

```json5
{
  session: {
    sendPolicy: {
      rules: [
        { action: "deny", match: { channel: "discord", chatType: "group" } },
        { action: "deny", match: { keyPrefix: "cron:" } }
      ],
      default: "allow"
    }
  }
}
```

### 运行时覆盖

| 命令 | 说明 |
|------|------|
| `/send on` | 允许此会话发送 |
| `/send off` | 禁止此会话发送 |
| `/send inherit` | 清除覆盖，使用配置规则 |

## 3.9.9 Session 隔离与可见性

### 沙箱隔离

每个 Agent 可以配置独立的沙箱：

```json5
{
  agents: {
    defaults: {
      sandbox: {
        sessionToolsVisibility: "spawned"  // 只看到自己生成的会话
        // 或 "all" - 可见所有会话
      }
    }
  }
}
```

### 可见性级别

| 级别 | 说明 |
|------|------|
| `self` | 只有当前会话 |
| `tree` | 当前会话 + 生成的子 Agent 会话 |
| `agent` | 当前 Agent 的所有会话 |
| `all` | 所有会话（需配合 agentToAgent）|

## 3.9.10 完整配置示例

```json5
{
  session: {
    // DM 作用域
    dmScope: "per-channel-peer",

    // 身份链接
    identityLinks: {
      "alice": ["telegram:123456789", "discord:987654321"]
    },

    // 重置策略
    reset: {
      mode: "daily",
      atHour: 4,
      idleMinutes: 120
    },
    resetByType: {
      direct: { mode: "idle", idleMinutes: 240 },
      group: { mode: "idle", idleMinutes: 120 }
    },
    resetTriggers: ["/new", "/reset"],

    // 维护策略
    maintenance: {
      mode: "enforce",
      pruneAfter: "30d",
      maxEntries: 500,
      rotateBytes: "10mb"
    },

    // 发送策略
    sendPolicy: {
      rules: [
        { action: "deny", match: { keyPrefix: "cron:" } }
      ],
      default: "allow"
    },

    // 主键
    mainKey: "main"
  },

  // 跨 Agent 通信
  tools: {
    agentToAgent: {
      enabled: true,
      allow: ["main", "work", "analyzer"]
    }
  }
}
```

## 本节小结

1. **Session 核心**：每个 Session 有独立的 sessionKey、sessionId、对话历史
2. **DM 作用域**：`dmScope` 控制 DM 会话隔离方式，建议多用户场景使用 `per-channel-peer`
3. **存储位置**：`~/.openclaw/agents/<agentId>/sessions/`
4. **维护策略**：通过 `maintenance` 配置自动清理
5. **重置策略**：支持每日重置、空闲重置、手动重置
6. **Session 工具**：sessions_list、sessions_history、sessions_send、sessions_spawn
7. **跨 Agent 通信**：需要显式启用 `tools.agentToAgent`
