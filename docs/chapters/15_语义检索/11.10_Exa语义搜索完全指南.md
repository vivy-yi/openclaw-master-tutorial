## 一、Exa 简介

### 1.1 什么是神经网络语义搜索

Exa 是一款基于神经网络的语义搜索引擎。与传统关键词匹配不同，Exa 能够：

- **理解查询意图**：输入 "苹果公司最近的AI布局"，Exa 理解你在找 Apple 的 AI 战略，而非字面上包含"苹果"和"AI"的水果内容
- **捕捉语义相似性**：找 "machine learning for healthcare" 时，同时覆盖 "deep learning in medical diagnosis"、"AI applications in hospitals" 等相关表述
- **返回相关性排序**：基于内容语义而非外链数量排序

### 1.2 传统关键词搜索 vs Exa 语义搜索

| 对比维度 | 传统搜索（Google/Bing） | Exa 语义搜索 |
|---------|----------------------|------------|
| 匹配方式 | 关键词共现、PageRank | 神经网络向量语义 |
| 查询理解 | 字面匹配 | 意图理解 |
| 专业领域 | 通用为主 | 可聚焦金融、医疗、技术 |
| 公司/人/代码搜索 | ❌ 不支持 | ✅ 原生支持 |
| 结构化过滤 | 有限 | 强大的域名/时间/类型过滤 |
| RAG 友好度 | 需后处理 | 原生 chunking + highlight |
| 复杂检索准确率 | — | **81%**（官方 benchmark） |

## 三、MCP 工具详解

> 以下内容基于 `exa-mcp-server` v3.2.1 源码分析

### 3.1 web_search_exa — 基础搜索

**源码位置**：`src/tools/search.ts`

**用途**：最常用的基础语义搜索工具。

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `query` | string | ✅ | 自然语言搜索查询 |
| `numResults` | number | ❌ | 返回结果数量（默认10） |
| `category` | string | ❌ | 限定类型，见下方语法 |

**category 语法**：使用 `category:<type>` 格式

| category 值 | 搜索范围 |
|------------|---------|
| `category:company` | 公司相关信息 |
| `category:people` | 人物公开资料 |
| `category:research paper` | 学术论文 |
| `category:news` | 新闻文章 |
| `category:pdf` | PDF 文档 |

**MCP 调用示例**：

```json
{
  "tool": "web_search_exa",
  "parameters": {
    "query": "OpenClaw AI agent framework",
    "numResults": 10
  }
}
```

**Python SDK 等效用法**：

```python
from exa_py import Exa

exa = Exa()
result = exa.search("OpenClaw AI agent framework", num_results=10)
```

**Node.js SDK 等效用法**：

```javascript
import Exa from 'exa-js';

const exa = new Exa(process.env.EXA_API_KEY);
const result = await exa.search("OpenClaw AI agent framework", {numResults: 10});
```

**category 搜索示例**：

```json
{
  "tool": "web_search_exa",
  "parameters": {
    "query": "Apple AI strategy latest news",
    "numResults": 10,
    "category": "company"
  }
}
```

### 3.2 web_search_advanced_exa — 高级搜索

**源码位置**：`src/tools/advanced.ts`

**用途**：提供更丰富的搜索选项，适合专业搜索场景。

**参数：**

| 参数 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `query` | string | 必填 | 自然语言搜索查询 |
| `numResults` | number | 10 | 返回结果数量 |
| `category` | string | null | 限定类型（见下表） |
| `includeDomains` | string[] | null | 只搜索指定域名 |
| `excludeDomains` | string[] | null | 排除指定域名 |
| `startPublishedDate` | string | null | 起始日期（YYYY-MM-DD） |
| `endPublishedDate` | string | null | 结束日期（YYYY-MM-DD） |
| `textMaxCharacters` | number | null | 正文最大字符数 |
| `contextMaxCharacters` | number | null | 上下文最大字符数 |
| `enableSummary` | boolean | false | 是否返回摘要 |
| `enableHighlights` | boolean | false | 是否返回高亮片段 |
| `subpages` | number | null | 抓取子页面数量 |
| `subpageTarget` | string | null | 子页面目标选择器 |

**category 可选值**（来自源码枚举）：

| category 值 | 搜索范围 |
|------------|---------|
| `company` | 公司相关信息 |
| `research paper` | 学术论文 |
| `news` | 新闻文章 |
| `pdf` | PDF 文档 |
| `github` | GitHub 代码 |
| `personal site` | 个人网站 |
| `people` | 人物资料 |
| `financial report` | 金融报告 |

**⚠️ 与 web_search_exa 的区别**：

| 特性 | web_search_exa | web_search_advanced_exa |
|------|----------------|------------------------|
| category 语法 | `category:company` | 直接传 `company` |
| 域名过滤 | ❌ 不支持 | ✅ include/excludeDomains |
| 日期范围 | ❌ 不支持 | ✅ start/endPublishedDate |
| 正文获取 | ❌ 不支持 | ✅ textMaxCharacters |
| 高亮片段 | ❌ 不支持 | ✅ enableHighlights |
| 子页面抓取 | ❌ 不支持 | ✅ subpages |

**MCP 调用示例**：

```json
{
  "tool": "web_search_advanced_exa",
  "parameters": {
    "query": "OpenClaw agent memory implementation",
    "category": "github",
    "includeDomains": ["github.com"],
    "numResults": 20,
    "startPublishedDate": "2025-01-01",
    "enableHighlights": true
  }
}
```

**带正文和高亮的搜索**：

```json

    "query": "what is retrieval augmented generation",
```

## 四、Python/Node.js SDK 用法

> 以下内容基于 `exa-js`（v2.8.0）和 `exa-py` SDK

### 4.1 Python SDK 基础搜索

```python
from exa_py import Exa

# 初始化 Exa（读取环境变量 EXA_API_KEY）
exa = Exa()

# 最简单的语义搜索
result = exa.search("OpenClaw AI agent framework")

# 打印所有结果
for item in result.results:
    print(f"📄 {item.title}")
    print(f"🔗 {item.url}")
    print(f"📊 得分: {item.score:.3f}")
    print(f"📅 发布日期: {item.published_date}")
    print("---")
```

**返回字段说明：**

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 网页标题 |
| `url` | string | 网页链接 |
| `score` | float | 相关度得分（0-1，越高越相关） |
| `published_date` | string | 发布日期（可能为空） |
| `text` | string | 正文内容（可选） |
| `highlights` | list | 高亮匹配段落（RAG 好帮手） |

### 4.2 Node.js SDK 基础搜索

```javascript
import Exa from 'exa-js';

// 初始化 Exa
const exa = new Exa(process.env.EXA_API_KEY);

// 基础搜索
const result = await exa.search("OpenClaw AI agent framework");

// 打印结果
result.results.forEach(item => {
    console.log(`📄 ${item.title}`);
    console.log(`🔗 ${item.url}`);
    console.log(`📊 得分: ${item.score.toFixed(3)}`);
    console.log("---");
});
```

然后配置：

```json
{
  "mcpServers": {
    "exa": {
      "command": "npx",
      "args": ["-y", "exa-mcp-server"]
    }
  }
}
```

### 5.3 在 Agent 指令中使用

一旦配置好，即可在 Agent 指令中直接使用 Exa MCP 工具：

```
你是一个研究助手。当用户提出研究问题时，使用 Exa 语义搜索工具：

示例搜索策略：
```

## 七、最佳实践

### 7.1 MCP 工具选择指南

| 场景 | 推荐工具 | 原因 |
|------|---------|------|
| 日常快速搜索 | `web_search_exa` | 简单直接 |
| 需要过滤/日期/正文 | `web_search_advanced_exa` | 功能最全 |
| 找代码示例 | `get_code_context_exa` | 专为此场景优化 |
| 获取页面完整内容 | `web_fetch_exa` | 精确抓取指定 URL |

### 7.2 RAG 场景应用

**推荐流程：**

```
1. web_search_advanced_exa（enableHighlights=true, textMaxCharacters=3000）
   ↓
2. web_fetch_exa（maxCharacters=5000）
   ↓
3. 将 highlights + text 组装为 context
   ↓
4. 送给 LLM 生成
```

**MCP 工具 RAG 示例**：

```json
// 第一步：搜索 + 高亮
{
  "tool": "web_search_advanced_exa",
  "parameters": {
    "query": "retrieval augmented generation RAG best practices",
    "numResults": 10,
    "enableHighlights": true,
    "textMaxCharacters": 3000
  }
}

// 第二步：抓取重要页面
{
  "tool": "web_fetch_exa",
  "parameters": {
    "urls": ["https://docs.example.com/rag-guide"],
    "maxCharacters": 5000
  }
}
```

### 7.3 查询优化

**好查询 vs 差查询：**

| ❌ 差查询 | ✅ 好查询 | 原因 |
|---------|---------|------|
| "AI" | "large language model fine-tuning best practices" | 过于宽泛 |
| "苹果" | "Apple product launches AI strategy 2026" | 语义歧义 |
| "机器学习" | "gradient descent optimization algorithms comparison" | 词不达意 |

**MCP category 限定**：

```json
{
  "tool": "web_search_advanced_exa",
  "parameters": {
    "query": "your research topic",
    "category": "research paper",
    "numResults": 20
  }
}
```

### 7.4 成本优化

1. **免费额度策略：**
   - 用 `numResults=5` 而非默认的 10
   - 优先使用 `enableHighlights` 而非 `textMaxCharacters`
   - 对批量查询加入缓存层

2. **批量查询节流**：
   ```python
   import time

   for i, query in enumerate(queries):
       result = exa.search(query, num_results=5)
       # 每秒最多 10 次请求
       if i % 10 == 9:
           time.sleep(1)
   ```

## 九、常见问题

### Q1: Exa 的免费额度用完了怎么办？

**答：** 免费额度为 1000次/月。超量后：
- 升级 Pro 计划（~$40/月）
- 切换到 Open WebSearch（免费）
- 使用 Jina Reader 作为补充
- 使用 Tavily 免费额度（1000次/月）

### Q2: Exa MCP 工具和 SDK 怎么选？

**答：** 
- **AI Agent / OpenClaw 场景** → 使用 MCP 工具（`web_search_exa` 等）
- **直接编程** → 使用 Python/Node.js SDK

两者底层都调用相同的 Exa API。

### Q3: `category` 参数在不同工具中语法一样吗？

**答：** 不一样！

| 工具 | category 语法 | 示例 |
|------|-------------|------|
| `web_search_exa` | `category:<value>` | `category:company` |
| `web_search_advanced_exa` | 直接传值 | `category: "company"` |

### Q4: Exa 能否替代 Google 搜索？

**答：** 不能完全替代，但可以在以下场景替代：
- 需要深度语义理解的专业查询
- RAG/知识库构建场景
- 公司/人/代码的专业搜索
- 通过 AI Agent 自动搜索（使用 MCP 工具）

### Q5: 为什么有些搜索结果质量不高？

**可能原因：**
1. 查询过于宽泛 → 改用更具体的描述
2. 需要加入 `startPublishedDate` 过滤旧内容
3. 目标内容在小众网站 → 加入 `includeDomains` 限定
4. 查询与目标内容存在语言差异 → 尝试英文查询

### Q6: Exa 支持中文搜索吗？

**答：** 支持。Exa 的神经网络模型支持多语言，包括中文。但对英文内容的索引更全面，**建议在专业场景下优先使用英文查询**，再对结果进行中文摘要。

### Q7: API Key 泄露了怎么办？

**答：**
1. 立刻去 Dashboard 删除该 Key：https://dashboard.exa.ai/api-keys
2. 创建新 Key
3. 更新所有使用该 Key 的地方
4. 检查 Dashboard 是否有异常调用记录

### Q8: `enableHighlights` 和 `textMaxCharacters` 有什么区别？

**答：**
| 参数 | 返回内容 | 消耗 | 适用场景 |
|------|---------|------|---------|
| `enableHighlights` | 查询词在原文中的匹配片段 | 低 | RAG 首选，精准 |
| `textMaxCharacters` | 页面正文（截断） | 高 | 需要全文上下文 |
