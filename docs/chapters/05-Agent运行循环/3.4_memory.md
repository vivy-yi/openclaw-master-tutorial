## 3.4.1 长期记忆概述

### 什么是长期记忆

长期记忆是 OpenClaw 的核心记忆系统，存储在 `workspace/memory/MEMORY.md` 文件中。它包含了 Agent 需要持久记住的核心知识、规则和偏好。

```

│                            长期记忆架构                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                      MEMORY.md                                        │   │
│   │   ┌─────────────────────────────────────────────────────────────┐  │   │
│   │   │  # 长期记忆                                                  │  │   │
│   │   │                                                             │  │   │
│   │   │  ## 核心知识                                                 │  │   │
│   │   │  - [知识条目 1]                                             │  │   │
│   │   │  - [知识条目 2]                                             │  │   │
│   │   │                                                             │  │   │
│   │   │  ## 重要规则                                                 │  │   │
│   │   │  - [规则 1]                                                 │  │   │

│                                                                                 │

```

### 长期记忆 vs 工作区文件

| 特性 | 长期记忆 (MEMORY.md) | 工作区文件 |
|-----|---------------------|-----------|
| 加载时机 | 每次会话自动加载 | 按需读取 |
| 存储内容 | 核心知识、规则、偏好 | 项目文件、数据 |
| 更新频率 | 低（长期积累） | 高（频繁变动） |
| 文件位置 | `workspace/memory/MEMORY.md` | `workspace/` 根目录 |

## 3.4.2 记忆文件结构

### MEMORY.md 标准结构

```markdown
# 长期记忆

## 核心知识
- 公司的核心技术栈是 React + Node.js
- 产品主要面向企业用户
- 重要客户包括：阿里巴巴、腾讯、字节跳动

## 重要规则
- 回复必须使用中文
- 代码注释使用英文
- 重要决策需要先确认
- 超过 100 行的代码需要分块展示

## 用户偏好
- 喜欢 TypeScript > JavaScript
- 偏好函数式编程风格
- 喜欢详细的代码注释
- 使用 2 空格缩进

## 交互习惯
- 早上 9 点到晚上 6 点在线
- 周一至周五工作时间
- 喜欢直接高效的沟通
- 需要详细的执行步骤

## 项目背景
- 当前项目：企业后台管理系统
- 技术栈：React + Ant Design + Node.js
- 团队规模：5 人
- 开发周期：3 个月

## 常见任务
- 代码审查
- Bug 修复
- 新功能开发
- 技术文档编写
```

### 分类建议

```
MEMORY.md 内容分类：

1. 核心知识 (Core Knowledge)
   - 公司/产品信息
   - 技术栈
   - 重要联系人

2. 重要规则 (Rules)
   - 行为准则
   - 代码规范
   - 安全要求

3. 用户偏好 (Preferences)
   - 编程语言偏好
   - 沟通风格
   - 输出格式

4. 交互习惯 (Habits)
   - 工作时间
   - 常用命令
   - 典型任务

5. 项目背景 (Project Context)
   - 当前项目
   - 团队信息
   - 里程碑
```

## 3.4.3 记忆管理命令

### CLI 命令

```bash
# 添加长期记忆
openclaw memory add "用户喜欢使用 TypeScript"

# 查看所有记忆
openclaw memory list

# 搜索记忆
openclaw memory search "偏好"

# 查看记忆文件
openclaw memory cat

# 编辑记忆
openclaw memory edit --line 10 --content "新内容"

# 重新加载记忆
openclaw memory reload
```

### 直接编辑

```bash
# 使用文本编辑器打开
vim ~/.openclaw/workspace/memory/MEMORY.md

# 或使用 openclaw 的 edit 命令
openclaw edit workspace/memory/MEMORY.md
```

## 3.4.5 最佳实践

### 记忆内容建议

```
✅ 推荐写入长期记忆的内容：
- 用户/公司核心信息（只需写入一次）
- 重要规则和约束
- 偏好和习惯
- 技术栈和项目背景
- 常见问题和解决方案

❌ 不推荐写入的内容：
- 频繁变动的数据（用每日记忆）
- 临时性信息（用工作区文件）
- 大段代码或文档（用项目文件）
- 敏感信息（用 credentials 配置）
```

### 更新时机

```markdown
# 建议更新 MEMORY.md 的时机：

1. 新开始一个项目
   - 添加项目背景
   - 添加技术栈
   - 添加团队成员

2. 用户明确偏好
   - "我更喜欢 TypeScript"
   - "回复要简洁"

3. 重要规则变化
   - "以后代码要用 4 空格缩进"
   - "重要决定前要确认"

4. 学习到新知识
   - "用户公司使用 Vue2"
   - "用户主要在下午工作"
```

### 维护建议

```bash
# 定期清理
- 每月检查一次记忆内容
- 删除过时信息
- 合并重复内容

# 保持简洁
- 每个条目简洁明了
- 使用列表格式
- 避免冗余

# 结构清晰
- 使用标题分层
- 分类明确
- 便于搜索
```

## 3.4.6 常见问题

### Q1: 记忆没有加载

```bash
# 检查文件是否存在
ls -la ~/.openclaw/workspace/memory/

# 检查文件格式
cat ~/.openclaw/workspace/memory/MEMORY.md

# 手动触发加载
openclaw memory reload

# 查看加载日志
openclaw logs | grep memory
```

### Q2: 记忆内容冲突

```markdown
# 问题：MEMORY.md 和 USER.md 内容重复

# 解决：明确分工
- USER.md: 用户基本信息（姓名、联系方式）
- MEMORY.md: 交互知识（偏好、习惯、规则）
```

### Q3: 记忆过长

```markdown
# 问题：MEMORY.md 内容太多

# 解决：
1. 拆分到多个文件
2. 使用引用
3. 保留核心内容到 MEMORY.md
4. 其他放到 workspace/ 下

# 示例结构：
workspace/
├── memory/
│   ├── MEMORY.md        # 核心记忆（精简）
│   ├── projects.md      # 项目详情
│   ├── team.md         # 团队信息
│   └── rules.md        # 详细规则
```

## 本节小结

1. **长期记忆**：存储在 `MEMORY.md`，每次会话自动加载
2. **内容分类**：核心知识、重要规则、用户偏好、交互习惯
3. **加载时机**：会话启动时，在 USER.md 之后加载
4. **管理方式**：CLI 命令或直接编辑
5. **最佳实践**：精简、分类清晰、定期维护
