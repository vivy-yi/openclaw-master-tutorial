## 三、v2026.6.6-beta.1 (2026-06-10)

**发布于**: 2026-06-10 18:13 UTC
**类型**: Beta（预发布）
**状态**: 🟡 **Latest Beta（最新预发布版）**

> 🔴 **重要安全警告**: 本次版本包含**14项安全加固 PR**，涵盖 transcripts 访问控制、sandbox binds 权限收紧、host 环境继承隔离、MCP stdio 安全、Codex HTTP 边界等多个维度。**建议生产用户尽快升级至正式版**（v2026.6.6 正式版发布后立即升级）。

### 核心亮点

| 模块 | 变更 | 相关 PRs | 重要性 |
|------|------|----------|--------|
| **🔐 安全加固** | transcripts 访问控制收紧、sandbox binds 权限收紧、host 环境继承隔离、MCP stdio 安全、Codex HTTP 边界收紧 | #91529, #91618, #91615, #91619, #91741, #91745~#91752, #91763, #89938 | ⭐⭐⭐⭐⭐ |
| **Telegram 改进** | 账户级话题路由、流式文本支持工具调用、`/compact` 工作在通用入口、回调处理优化 | #91189, #88682, #89588, #90212, #91876, #91874, #91904, #91478, #91915 | ⭐⭐⭐⭐ |
| **iMessage 改进** | 始终在线入站重启、持久化回显标记、块流、空闲审批发现、出站传输加固 | #91335, #91449, #88969, #88530, #91783, #91785 | ⭐⭐⭐⭐ |
| **浏览器/MCP** | 现有会话 CDP 支持、WebSocket 验证、默认 profile cdpUrl 处理、安全浏览器输出边界 | #91422, #89851, #91736, #91747, #91451, #80143 | ⭐⭐⭐⭐ |
| **性能优化** | 缓存模型元数据、移除启动目录等待、懒加载斜杠命令、首事件追踪 | #91531, #91538, #91568, #91583, #91598 | ⭐⭐⭐⭐ |
| **Provider 扩展** | OpenRouter OAuth onboarding、Claude Fable 5 自适应思考、Codex 会话正确压缩所有权、本地模型跳过 guardian 审查 | #91830, #91882, #91590, #88630, #88768, #91696 | ⭐⭐⭐⭐ |

### 安全加固详细说明

**收紧范围（按类型）**:
1. **transcripts** — 敏感内容访问控制收紧
2. **sandbox binds** — 沙箱绑定权限收紧
3. **host 环境继承** — 宿主机环境变量隔离
4. **MCP stdio** — MCP 标准输入输出安全
5. **Codex HTTP 访问** — Codex HTTP 请求边界

### 其他重要变更

```
🆕 CLI 进度: 发出 Claude CLI 注释进度事件，桥接工具间注释到 channel 进度
🆕 可观测性: 允许受信任诊断 channel 捕获工具输入/输出内容
🆕 Plugins/ClawHub: 干运行可跳过发布审批、报告托管插件版本漂移
🆕 Memory/providers: 将本地 llama.cpp runtime 移入 provider 插件、批量文件嵌入、持久化 agent 模型目录缓存
🆕 性能: 预热 TUI 运行时插件、去重插件自动启用、修剪密集文本增量快照
```


### 重要修复

```
✅ Agent/会话恢复: session rebind 后丢弃过时审批跟进、恢复过时的主回复和可见回复
✅ 用户可见内容边界: 抑制 Codex/Harmony 协议伪影、净化浏览器和 LanceDB 内存媒体指令
✅ Channel 投递: WhatsApp 捕获回复附加到后续控制器、Feishu 速率限制重试
✅ Gateway/config/auth: 共享审批运行时 socket token、替换 config.patch 中的数组
✅ Provider/Codex: 澄清配额错误、恢复 Codex 合成使用行、保留 Gemma 4 reasoning_content
```

### 重大变更：Operator Install Policy

这是 v2026.6.2-beta.1 最重要的安全变更：

- **替代** `dangerous-code` scanner
- **新机制**: 白名单 + 插件签名验证
- **影响**: 插件安装需要显式授权
- **相关文档**: `16_安全配置/16.11_operator_install_policy.md`
