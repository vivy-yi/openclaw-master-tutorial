# AI/Agent 领域术语表

> 统一解释 AI/Agent 领域常见概念，括号内标注英文原文与常用缩写

---

## A

### Agent（智能体 / AI Agent）
**定义**：能够感知环境、进行决策并执行动作的 AI 系统，具备自主性、反应性和社交能力。
**常见分类**：单 Agent、多 Agent、Agent 工作流（Agentic Workflow）
**对比**：传统 AI（输入→输出）vs Agent（输入→推理→工具调用→行动→观察→迭代）

### Agent Loop（Agent 执行循环）
**定义**：Agent 完成任务的核心循环：感知（Perceive）→ 推理（Reason）→ 行动（Act）→ 观察（Observe）
**OpenClaw 中的角色**：OpenClaw Gateway 的核心执行模型，区别于简单的一问一答模式

### API Key（API 密钥）
**定义**：调用大模型服务商 API 的身份凭证，通常为一段加密字符串
**常见格式**：`sk-xxxxx`（OpenAI）、`AIza...`（Google）

---

## B

### Boring AI（无聊 AI）
**定义**：指那些没有幻觉、能稳定输出、可靠完成任务的 AI，强调实用性和可靠性，而非"惊艳"
**对比**：Exciting AI（能激发想象）vs Boring AI（能稳定干活）

### Backend / Backend Model（后端模型）
**定义**：实际处理请求的底层大语言模型，如 GPT-4、Claude、DeepSeek 等
**OpenClaw 中的角色**：OpenClaw 的 `defaultModel` 或 `model` 配置项指定

### Bridge / Bridge Agent（桥接 Agent）
**定义**：在不同系统或平台之间进行协议转换、数据传递的中间层 Agent
**常见场景**：飞书 ↔ OpenClaw、Telegram ↔ OpenClaw

---

## C

### Channel（渠道）
**定义**：用户与 Agent 通信的方式或平台，如 Telegram、飞书、微信、Discord、Slack
**OpenClaw 中的角色**：`channel` 配置文件指定，如 `channels.telegram`

### Chat History（对话历史）
**定义**：用户与 Agent 之间的消息记录，用于提供上下文（Context）
**OpenClaw 中的实现**：通过 session 管理，支持长期记忆和上下文窗口

### Claude MCP（Model Context Protocol Client）
**定义**：Anthropic 提供的 MCP 客户端实现，用于连接外部工具和数据源
**与 OpenClaw 的关系**：OpenClaw 支持 MCP 协议，可接入 Claude 的工具生态

### Compaction（压缩 / 上下文压缩）
**定义**：当对话历史过长时，将旧的上下文压缩或摘要，以节省 Token 成本
**OpenClaw 中的相关配置**：`compact` 相关参数

### Context（上下文）
**定义**：LLM 在生成响应时参考的全部信息，包括 System Prompt、对话历史、工具描述等
**关键约束**：Context 有最大长度限制（由模型决定，如 128k tokens）

### Context Window（上下文窗口）
**定义**：模型一次能处理的最大 Token 数量，包括输入和输出
**常见值**：4K、8K、32K、128K、200K、1M tokens

### Cron（定时任务）
**定义**：基于 cron 表达式调度的周期性任务
**OpenClaw 中的应用**：定时触发 Agent 工作、发送消息、数据采集等

---

## D

### Direct Cut-off（直接截止）
**定义**：模型在生成时遇到 Context 窗口限制而被强制截断的情况
**问题**：可能截断关键内容，导致回答不完整

### Dispatcher（调度器）
**定义**：在多渠道或多租户场景下，决定请求由哪个 Agent 或处理单元接收的组件

---

## E

### Embedding（向量嵌入）
**定义**：将文本、图像等数据转换为固定维度数值向量的技术，用于语义检索和相似度计算
**常见用途**：RAG（检索增强生成）、知识库搜索

### Execution（执行 / 执行层）
**定义**：Agent 实际执行任务的过程，包括工具调用、API 请求、文件操作等

---

## F

### Fallback Model（降级模型）
**定义**：当主模型不可用时，自动切换使用的备选模型
**OpenClaw 中的配置**：`fallbacks` 数组

### Frontend Token（前端 Token）
**定义**：用户可见的输入 Token 数量（不含系统 Prompt 和历史）
**OpenClaw 显示**：`Context: 0/200k (0%)` 中的第一个数字

---

## G

### Gateway（网关）
**定义**：OpenClaw 的核心进程，负责管理 Agent、Session、Channel 和工具调度
**启动命令**：`openclaw gateway start/restart`
**配置文件**：`~/.openclaw/config.yaml`

### General Purpose AI（通用 AI）
**定义**：设计用于处理多种任务的 AI 系统，而非单一用途
**对比**：垂直 AI（Vertical AI）— 针对特定行业或场景定制

---

## H

### Hallucination（幻觉）
**定义**：LLM 生成的内容看似合理但实际错误或不存在的内容
**常见原因**：训练数据偏差、上下文不足、推理错误
**缓解方法**：RAG、引用验证、CoT（Chain of Thought）

### Heartbeat（心跳）
**定义**：定期向系统发送的存活信号，用于维持长连接或触发周期性任务
**OpenClaw 中的应用**：cron 任务调度、session 保活

---

## I

### Inference（推理）
**定义**：使用训练好的模型进行预测或生成的过程（相对于训练 Training）
**计费方式**：通常按输入/输出的 Token 数量计费

---

## J

### Jina Reader（Jina 阅读器）
**定义**：Jina AI 提供的网页内容提取服务，可将任意网页转为 Markdown
**用法**：`curl -s "https://r.jina.ai/<URL>"`
**常见场景**：采集微信公众号、知乎等平台的技术文章

---

## K

### Knowledge Base（知识库）
**定义**：存储结构化或非结构化知识，供 Agent 检索和使用的系统
**常见技术**：Vector DB（向量数据库）、RAG（检索增强生成）

---

## L

### LLM（Large Language Model，大语言模型）
**定义**：基于大规模文本数据训练，能够理解和生成自然语言的深度学习模型
**代表模型**：GPT-4、Claude、Gemini、DeepSeek、通义千问、文心一言

### Loop（循环 / 执行循环）
**定义**：Agent 反复执行"思考→行动→观察"的过程，直到任务完成或达到最大步数
**OpenClaw 中的控制参数**：`maxTurns`、`timeout`

---

## M

### MCP（Model Context Protocol，模型上下文协议）
**定义**：Anthropic 提出的开放协议，用于连接 AI 模型与外部工具、数据源
**官方 SDK**：`@modelcontextprotocol/sdk`
**OpenClaw 中的支持**：内置 MCP 工具调用能力

### Message（消息）
**定义**：Session 中的最小通信单元，包含 role（user/assistant/system）、content 等字段

### Middleware（中间件）
**定义**：介于 Channel 和 Agent 之间的处理层，负责消息预处理、后处理、路由等

### Model（模型）
**定义**：用于推理的 LLM，可通过 API 或本地部署调用
**OpenClaw 配置项**：`model`（主模型）、`fallbacks`（降级模型）

### Multi-Agent（多 Agent 系统）
**定义**：多个 Agent 协作完成复杂任务，每个 Agent 承担特定角色或子任务
**常见模式**：Supervisor-Agent、Hierarchical、Collaborative、Competitive

---

## N

### N-shot Learning（N 次学习）
**定义**：通过在 Prompt 中提供 N 个示例（Examples）来引导模型输出特定格式或行为
**常见类型**：Zero-shot（0个示例）、One-shot（1个）、Few-shot（几个）

---

## O

### OpenClaw（OpenClaw AI 助手框架）
**定义**：一个开源的 AI 助手运行时框架，支持多渠道接入、多 Agent 管理、定时任务、工具调用等
**官方文档**：docs.openclaw.ai
**开源地址**：github.com/openclaw/openclaw

### OpenAI API（OpenAI 接口）
**定义**：OpenAI 提供的 LLM 调用接口，兼容多种 SDK 和工具
**计费**：按 Token 计费，实时汇率结算

---

## P

### Payload（负载 / 消息体）
**定义**：通过 API 发送的完整请求数据，包括 model、messages、tools 等字段

### Plugin（插件）
**定义**：扩展 OpenClaw 功能的模块，可添加新工具、新渠道或新能力
**OpenClaw 中的插件目录**：`~/.openclaw/plugins/`

### Prompt（提示词）
**定义**：给 LLM 的输入文本，用于引导模型生成特定输出
**分类**：System Prompt、User Prompt、Assistant Prompt

### Provider（服务商 / 提供商）
**定义**：提供 LLM API 或其他 AI 服务的企业或平台
**常见 Provider**：OpenAI、Anthropic、Google、DeepSeek、Ollama

---

## R

### RAG（Retrieval-Augmented Generation，检索增强生成）
**定义**：结合信息检索与生成模型的技术，从知识库中检索相关文档作为上下文
**优势**：减少幻觉、提供最新知识、降低训练成本

### Reasoning（推理 / 思考）
**定义**：LLM 在生成回答前的内部思考过程（部分模型如 o1/o3 会显式展示）

### Response（响应）
**定义**：LLM 对用户请求的完整回复，包含文本、工具调用或混合内容

### Role（角色）
**定义**：Prompt 中的角色定义，如 system（系统）、user（用户）、assistant（助手）
**OpenClaw 中的标准角色**：`user`、`assistant`、`system`（未来支持 `tool`）

---

## S

### SDK（Software Development Kit，软件开发包）
**定义**：封装好的代码库，提供简化的 API 调用能力

### Session（会话 / 对话会话）
**定义**：一段持续的对话上下文，通常与一个用户或一个话题相关联
**OpenClaw 中的 Session**：`sessions_*` 系列工具管理会话状态
**Session Key**：唯一标识一个会话的字符串

### Skill（技能）
**定义**：为 Agent 预先配置的专项能力，如网页搜索、代码执行、文件处理等
**OpenClaw 中的 Skills**：位于 `~/.openclaw/skills/` 目录
**官方 Skills 仓库**：github.com/openclaw/openclaw/tree/main/skills

### Subagent（子 Agent）
**定义**：在父 Agent 上下文中派生的独立 Agent，用于并行处理子任务
**OpenClaw 中的实现**：`sessions_spawn` 工具

### System Prompt（系统提示词）
**定义**：在对话开始前预设的全局指令，定义 Agent 的角色、行为规范和边界
**OpenClaw 中的配置**：配置文件或代码中指定

---

## T

### Template（模板）
**定义**：预定义的 Prompt 结构或消息格式，用于快速创建特定类型的任务

### Thread（线程）
**定义**：特定主题下的消息序列，常用于支持多话题并行讨论
**OpenClaw 中的 Thread**：`threadId` 参数

### Timeout（超时）
**定义**：等待响应的最大时间，超过则终止请求或切换降级模型
**OpenClaw 中的配置**：`timeoutSeconds` 参数

### Token（令牌 / Token）
**定义**：LLM 处理文本的最小单元，英文约 0.75 个词，中文约 1-2 个字符
**计费依据**：输入 Token + 输出 Token

### Tool（工具）
**定义**：Agent 可以调用的外部能力，如搜索、计算、文件读写、API 请求等
**OpenClaw 中的工具**：通过 SKILL.md 定义能力边界

### Tool Call（工具调用）
**定义**：LLM 决定调用外部工具完成特定任务的行为（如调用计算器、搜索API）

### Tool Schema（工具描述）
**定义**：描述工具名称、参数、返回值结构的 JSON Schema，供 LLM 理解如何调用

### Tools Permission（工具权限）
**定义**：控制 Agent 能否调用特定工具的安全机制
**OpenClaw 中的配置**：`toolsAllow`、`toolsDeny` 参数

---

## U

### Usage（用量）
**定义**：实际消耗的资源量，如 Token 数量、API 调用次数
**OpenClaw 中的显示**：`session_status` 中的 `Usage` 字段

### User（用户）
**定义**：发起请求的最终使用者，对应 `role: "user"` 消息角色

---

## V

### Vector（向量）
**定义**：Embedding 生成的固定维度数值数组，用于表示语义信息
**存储**：向量数据库（Vector DB），如 Pinecone、Milvus、Chroma

### Vision（视觉能力）
**定义**：模型处理和理解图像/视频的能力
**OpenClaw 中的支持**：`image` 工具分析图片

---

## W

### Wake Event（唤醒事件）
**定义**：触发沉寂中的 Agent 重新活跃的信号，如定时任务到期、收到新消息

### Webhook（网络钩子）
**定义**：服务器主动向指定 URL 发送数据的机制，常用于事件通知

### Workspace（工作空间）
**定义**：Agent 的工作目录，用于存储文件、配置和运行时数据
**OpenClaw 中的工作空间**：`~/.openclaw/workspaces/<workspace-name>/`

---

## 附录：OpenClaw 特有概念

### SKILL.md
**定义**：定义技能/工具能力边界的元数据文件，描述工具用途、参数、使用方法
**位置**：`~/.openclaw/skills/<skill-name>/SKILL.md`

### Session Key
**定义**：唯一标识一个会话的字符串，用于 `sessions_*` 系列工具操作
**格式示例**：`agent:openclaw-assistant:webchat:abc123`

### Gateway Config
**定义**：OpenClaw Gateway 的运行时配置，通过 `~/.openclaw/config.yaml` 或 `config.patch` 管理

### Cron Job（定时任务）
**定义**：基于 cron 表达式调度的周期性或一次性任务
**OpenClaw 中的实现**：`cron` 工具

### Isolated Session（隔离会话）
**定义**：与其他会话隔离的独立执行环境，用于子 Agent 或临时任务
**OpenClaw 中的 Spawn**：`sessions_spawn context="isolated"`
