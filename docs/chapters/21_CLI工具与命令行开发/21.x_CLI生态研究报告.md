## 21.x.2 CLI 生态系统五大趋势

### 趋势一：MCP → CLI 化（Token 节省革命）

**背景问题**：MCP Server 的 Tool Schema 每次请求传递消耗大量 Token（10-15K/token），96-99% 的 Token 用于工具定义。

**解决方案**：`mcp2cli` 将 MCP Server 运行时反射为本地 CLI，Token 消耗降至 300-500/token。

| 项目 | Stars | 核心价值 |
|------|-------|---------|
| **mcp2cli** | ⭐ 1.7k | MCP → CLI，节省 96-99% Token |
| **mcp (官方)** | - | Model Context Protocol 标准 |
| **mcp-server-playwright** | - | Playwright MCP Server（OpenClaw 内置） |

## 21.x.5 本地 CLI 是 AI Agent 基础设施的判断依据

### 为什么本地 CLI 是重点方向？

| 论据 | 分析 |
|------|------|
| **Token 成本** | 云端 API 每次传 Tool Schema 浪费 96-99%，本地 CLI 无此开销 |
| **隐私/合规** | 数据不出本地，满足金融/医疗/法律等敏感场景 |
| **可靠性** | 不依赖外部 API，网络波动不影响任务执行 |
| **确定性输出** | CLI 返回结构化 JSON，解析成本低于浏览器 DOM |
| **生态成熟度** | GitHub 上 CLI 项目星标增长迅速，工具链完善 |
| **跨平台** | Unix 设计哲学，一个 CLI 各平台通用 |

### AI Agent 使用 CLI 的三层价值

```
第一层（执行层）：CLI = Agent 的手脚
  Claude Code → 写代码 → git push
  browser-use → 抓取网页 → 提取数据

第二层（工具发现层）：CLI = 工具标准化
  MCP Server → mcp2cli → Token 节省 96%
  API → CLI → 统一调用接口

第三层（能力扩展层）：CLI = Agent 技能库
  opencli → B站数据
  yt-dlp → YouTube 字幕
  notion → 笔记写入
```

## 21.x.6 CLI 工具评估框架

> 如何判断一个 CLI 是否值得集成到墨家 Agent 系统

### 六维评估模型

| 维度 | 权重 | 评估问题 |
|------|------|---------|
| **Agent 兼容性** | 30% | 输出是否为结构化 JSON/纯文本？是否需要 TTY 交互？ |
| **安装便捷性** | 20% | 一条命令安装还是有复杂依赖？跨平台吗？ |
| **API 覆盖度** | 20% | 是否覆盖目标平台的核心能力？ |
| **维护活跃度** | 15% | 最近 6 个月有更新吗？ Issues 响应如何？ |
| **认证复杂度** | 10% | OAuth/API Key/无认证？ |
| **可替换性** | 5% | 是否有更流行的替代品？ |

### 推荐集成优先级

```
P0（必须集成，立即可用）：
  ✅ xurl      → Twitter/X 全功能
  ✅ yt-dlp    → 视频/音频下载
  ✅ camoufox  → 反检测抓取
  ✅ markitdown → 任意URL→Markdown
  ✅ playwright → 浏览器自动化
  ✅ gh        → GitHub 操作
  ✅ notion    → 笔记同步

P1（推荐集成，高价值）：
  ⭐ opencli   → 50+平台内容采集
  ⭐ firecrawl → AI友好网站爬虫
  ⭐ litellm   → 统一LLM网关
  ⭐ ollama    → 本地模型推理
  ⭐ llm       → 终端LLM调用
  ⭐ twarc     → Twitter归档
  ⭐ lark-cli  → 飞书全功能

P2（按需集成，实验性）：
  🔎 CLI-Anything → 桌面软件CLI化
  🔎 mcp2cli      → Token节省
  🔎 searxng     → 隐私搜索
  🔎 joplin-cli  → 跨平台笔记
```
