# 21.x2 Agent Reach 平台生态全解——每个平台的可替换 CLI 工具图谱

> Agent Reach 是 AI Agent 全平台内容采集的脚手架工具（⭐ 17.8k GitHub Stars）。核心理念是**每个平台可插拔**，当前选型只是默认值，不满意随时替换。本节整理其 16 个平台各自的当前选型与替代方案。
> **研究日期：2026-04-20 | 数据来源：GitHub API + 官方 README**

## 21.x2.1 Agent Reach 是什么

### 定位：脚手架，不是框架

```
Agent Reach = AI Agent 互联网能力的安装脚本 + 每个平台的可替换工具清单
```

安装完成后，Agent 直接调用上游工具，不经过 Agent Reach 包装层。**每个渠道完全独立。**

### 安装方式

```bash
# 一键安装（复制给 Agent 执行）
帮我安装 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md

# 安全模式（不自动装系统包，只告诉你需要什么）
帮我安装 Agent Reach（安全模式）：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/install.md
# 安装时使用 --safe 参数

# 已安装？一键更新
帮我更新 Agent Reach：https://raw.githubusercontent.com/Panniantong/agent-reach/main/docs/update.md
```

### OpenClaw 前置要求

```bash
# 需要开启 exec 权限
openclaw config set tools.profile "coding"
openclaw gateway restart
```

## 21.x2.3 各平台工具详解

### 1. 🌐 读网页

| 工具 | Stars | 类型 | 认证 | 说明 |
|------|-------|------|------|------|
| **Jina Reader**（当前） | ⭐9.8k | `curl` | 无 | `curl -s "https://r.jina.ai/URL"` 即可，最简单 |
| **Firecrawl**（替代） | ⭐15k+ | Python/CLI | API Key可选 | AI友好结构化提取，支持JavaScript渲染，比Jina更强 |
| **Crawl4AI**（替代） | ⭐8k+ | Python | 无 | AI原生爬虫，专为LLM设计，支持异步和流式输出 |
| **markitdown** | - | CLI ✅ | 无 | 任意URL→Markdown（当前系统已装） |
| 官方API | - | REST | API Key | 各平台自有方案 |

**选型建议**：
- 简单读取 → Jina Reader（零配置）
- AI 提取质量 → Firecrawl（结构化Markdown）
- 本地完全免费 → Crawl4AI（开源）

### 2. 🐦 Twitter/X

| 工具 | Stars | 认证 | 能力 |
|------|-------|------|------|
| **xurl**（替代 ✅已装） | - | OAuth2 | 搜索/读推文/时间线/发推/媒体上传，全功能 |
| **twitter-cli**（当前） | ⭐2.1k | Cookie | 搜索/读推文/时间线/长文 |
| 官方API | - | API Key | 完整但付费 |
| **twarc** | ⭐5k | API Key | 推特归档，学术用途 |

**当前系统已安装 xurl**，它是官方X API v2的完整CLI包装，支持 OAuth2，比 twitter-cli 的 Cookie 方式更稳定。

**xurl 快速上手：**
```bash
xurl auth oauth2              # OAuth 认证（需注册 App）
xurl search "query" -n 10    # 搜索推文
xurl read TWEET_ID           # 读取单条推文
xurl timeline -n 20           # 时间线
xurl post "Hello!"           # 发推
```

### 3. 📺 YouTube / 📺 B站

| 工具 | Stars | 平台 | 字幕 | 搜索 | 认证 |
|------|-------|------|------|------|------|
| **yt-dlp**（当前） | ⭐154k | YouTube+B站+1800站 | ✅多语言 | ✅ | 无 |
| **bili-cli**（替代） | ⭐590 | B站专属 | ✅ | ✅热门/排行/动态 | 无 |

**yt-dlp vs bili-cli 能力对比：**

| 能力 | yt-dlp | bili-cli |
|------|--------|---------|
| 视频下载 | ✅ | ✅ |
| 字幕提取 | ✅（YouTube+B站） | ❌ |
| 搜索视频 | ✅（`ytsearch5:`） | ✅（专属搜索） |
| 获取热门/排行 | ✅ | ✅ |
| 获取用户动态 | 限YouTube | ✅ |
| 安装复杂度 | 需ffmpeg | 简单 |

**选型建议**：
- 需要字幕/转录 → yt-dlp（当前系统已装 ✅）
- 纯B站浏览/搜索 → bili-cli 更轻量

**yt-dlp 常用命令：**
```bash
# 视频元数据（不下载）
yt-dlp --dump-json "URL"

# 下载字幕
yt-dlp --write-sub --write-auto-sub --sub-lang "zh-Hans,zh,en" \
  --skip-download -o "/tmp/%(id)s" "URL"

# 搜索
yt-dlp --dump-json "ytsearch5:搜索词"

# B站字幕
yt-dlp --write-sub --write-auto-sub --sub-lang "zh-Hans,zh,en" \
  --convert-subs vtt --skip-download "https://www.bilibili.com/video/BVxxx"
```

## 21.x2.4 工具替换指南

### 为什么需要替换？

| 原因 | 说明 |
|------|------|
| **封号风险** | Twitter/小红书等用Cookie有被封风险 |
| **能力不足** | 某些平台需要更高级能力（如脚本提取） |
| **成本问题** | 官方API收费，替代方案免费 |
| **稳定性** | 某些IP被平台封禁，需要代理或换工具 |
| **认证简化** | 从Cookie迁移到OAuth/API Key更安全 |

### 替换操作步骤

Agent Reach 的可插拔架构使替换非常简单：

```
channels/        ← 只需替换对应 .py 文件
├── twitter.py   → 改用 xurl
├── bilibili.py  → 改用 bili-cli
└── ...
```

每个 `channel.py` 文件只负责两件事：
1. `check()` — 检测上游工具是否可用
2. 调用上游CLI（实际逻辑在Agent直接调用）

### 常用替换场景

| 从 | 替换为 | 原因 |
|----|--------|------|
| twitter-cli | **xurl** ✅ | OAuth比Cookie更稳定 |
| Jina Reader | **Firecrawl** | AI提取质量更高 |
| yt-dlp | **bili-cli** | B站专属搜索更丰富 |
| rdt-cli | 官方API | Cookie封号风险 |
| Exa | **Tavily** | 搜索结果更准确 |
