## MCP 协议简介

**MCP（Model Context Protocol）** 是 AI Agent 与外部工具交互的标准协议。浏览器相关 MCP Server 将浏览器能力以标准化工具形式暴露给 AI Agent。

```
┌─────────────────────────────────────────────────────────────────┐
│                    MCP 浏览器工具生态                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  AI Agent（Claude Desktop / Cursor / OpenClaw...）              │
│         │                                                        │
│         ▼                                                        │
│  MCP Client（标准协议）                                           │
│         │                                                        │
│         ▼                                                        │
│  MCP Server（playwright-mcp / bb-browser / mcp-chrome）          │
│         │                                                        │
│         ▼                                                        │
│  🌐 Browser（Playwright / Chrome 扩展 / CDP）                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 安装配置

```json
// Claude Desktop / Cursor MCP 配置
{
  "mcpServers": {
    "playwright": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"]
    }
  }
}
```

### 1.4 核心特点

| 优点 | 缺点 |
|------|------|
| ✅ 官方维护（Microsoft）| ❌ 无反检测能力 |
| ✅ 无障碍树（非像素），LLM 友好 | ❌ 需要浏览器已启动 |
| ✅ 确定性操作 | ❌ 无 AI 自主决策 |
| ✅ 适合探索性自动化 | ❌ Token 消耗中等 |

### 1.5 MCP vs CLI 取舍（官方说明）

Microsoft README 明确建议：

| 场景 | 推荐 |
|------|------|
| **探索性自动化** | MCP ✅ |
| **高吞吐量 Agent** | CLI + Skills ⭐ |
| **需要持久状态** | MCP ✅ |
| **上下文有限** | CLI + Skills ⭐ |
| **自愈测试** | MCP ✅ |

## 2. BB-Browser（⭐ 3,041）

### 2.1 项目信息

| 属性 | 值 |
|------|-----|
| **GitHub** | `epiral/bb-browser` |
| **Stars** | ⭐ 3,041 |
| **语言** | TypeScript |
| **安装** | `npm install -g bb-browser` |

### 2.2 核心特点

| 特点 | 说明 |
|------|------|
| **复用 Chrome 登录态** | 使用你已经登录的 Chrome session |
| **CLI + MCP 两种接口** | 灵活性高 |
| **零 API Key** | 不需要额外凭证 |
| **支持平台** | GitHub / Twitter 等 |

### 2.3 使用

```bash
# CLI 方式
bb-browser login github        # 配置 GitHub 登录态
bb-browser github search "query"   # 搜索
bb-browser twitter timeline --user elonmusk

# MCP 方式
bb-browser mcp start
```

## 3. MCP-Chrome（⭐ 10,994）

### 3.1 项目信息

| 属性 | 值 |
|------|-----|
| **GitHub** | `hangwin/mcp-chrome` |
| **Stars** | ⭐ 10,994 |
| **语言** | TypeScript |
| **安装** | Chrome 扩展 |

### 3.2 工作原理

```
┌─────────────────────────────────────────────────────────────────┐
│                   MCP-Chrome 架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  AI Assistant                                                    │
│         │                                                        │
│         ▼                                                        │
│  MCP Client                                                      │
│         │                                                        │
│         ▼                                                        │
│  Chrome Extension（MCP Server）                                   │
│         │                                                        │
│         ▼                                                        │
│  🌐 Chrome Browser（CDP）                                         │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 安装步骤

1. 下载扩展 from GitHub Releases
2. 打开 `chrome://extensions`
3. 启用开发者模式
4. 加载未打包扩展
5. 在 Claude Desktop 中配置 MCP

### 3.4 核心特点

| 优点 | 缺点 |
|------|------|
| ✅ Chrome 扩展方式（无需额外软件）| ⚠️ Claude Desktop 专用生态 |
| ✅ CDP 协议控制 | ❌ Claude 之外需适配 |
| ✅ 登录态自动复用 | ⚠️ Chrome 需保持运行 |

```bash
agent-browser navigate --url "https://github.com"

agent-browser fill --selector "input[name=q]" --value "search term"
```
