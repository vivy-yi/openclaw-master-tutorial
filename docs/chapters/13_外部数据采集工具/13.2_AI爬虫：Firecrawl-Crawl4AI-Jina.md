## 1. Firecrawl（⭐ 111,621，AI 结构化提取）

### 1.1 项目信息

| 属性 | 值 |
|------|-----|
| **GitHub** | `firecrawl/firecrawl` |
| **Stars** | ⭐ 111,621 |
| **语言** | TypeScript |
| **官网** | firecrawl.dev |
| **License** | SSPL（开源但主服务付费）|

### 1.2 核心特点

```
Firecrawl = 搜索引擎 + 页面抓取 + 页面交互 + AI 结构化提取

┌─────────────────────────────────────────────────────────────────┐
│                      Firecrawl 端点                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Search   → 搜索网络，返回完整内容摘要                            │
│  Scrape   → URL → Markdown / HTML / Screenshot / JSON           │
│  Crawl    → 整站抓取所有 URL                                     │
│  Map      → 快速发现网站 URL（无需完整抓取）                      │
│  Interact → 抓取后与页面交互（点击/滚动/输入）                   │
│  Batch    → 异步批量抓取数千个 URL                               │
│  Agent    → 描述需求，AI 自动完成数据收集                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.3 与 Scrapy 的核心区别

| 对比项 | Firecrawl | Scrapy |
|--------|----------|--------|
| **输出格式** | Markdown / JSON / Screenshot | HTML / XML |
| **AI 友好度** | ✅ 原生为 AI 设计 | ❌ 需后处理 |
| **JS 渲染** | ✅ 内置 | ❌ 需 splash/selenium |
| **反检测** | ✅ 内置 | ❌ 需自行处理 |
| **部署方式** | SaaS 或自部署 | 仅自部署 |
| **规模** | 云端弹性扩展 | 受限于服务器 |
| **定价** | 免费限额 + 付费 | 免费 |

### 1.4 安装与使用

```bash
# 安装
pip install firecrawl-py

# 或通过 npm
npm install -g firecrawl

# 使用
import firecrawl

app = firecrawl.FirecrawlApp(api_key="fc-xxxx")

# Scrape（抓取单页）
result = app.scrape_url(
    "https://example.com",
    formats=["markdown", "html", "screenshot"]
)
print(result["markdown"])

# Crawl（整站抓取）
crawl_result = app.crawl_url(
    "https://example.com",
    limit=10,
    formats=["markdown"]
)
```

### 1.4.1 Firecrawl 可用方法（从源码提取）

| 方法 | 功能 |
|------|------|
| `scrape` | 抓取单页（返回 Markdown/HTML/Screenshot） |

| `map` | 快速发现网站 URL |
| `search` | 搜索网络 |
| `extract` | AI 结构化提取 |
| `batch_scrape` | 批量抓取 |
| `agent` | AI Agent 自主任务 |
| `interact` | 与页面交互（点击/滚动/输入） |
| `parse` | 解析内容 |
| `watcher` | 监控网站变化 |
| **状态查询** | |
| `get_crawl_status` | 抓取状态 |
| `get_batch_scrape_status` | 批量抓取状态 |
| `get_extract_status` | 提取状态 |
| `get_agent_status` | Agent 状态 |
| **浏览器控制** | |
| `browser` | 创建浏览器实例 |
| `browser_execute` | 执行浏览器命令 |
| `scrape_execute` | 抓取后执行 |
| `delete_browser` | 删除浏览器 |
| `list_browsers` | 列出浏览器 |

### 1.5 适用场景

✅ **强烈推荐：**
- RAG pipeline 数据摄入（直接输出 Markdown）
- 快速原型（不想处理渲染/反爬/代理）
- 规模化数据采集（有预算，省运维）

## 2. Crawl4AI（⭐ 64,451，开源免费）

### 2.1 项目信息

| 属性 | 值 |
|------|-----|
| **GitHub** | `unclecode/crawl4ai` |
| **Stars** | ⭐ 64,451 |
| **语言** | Python |
| **官网** | crawl4ai.com |
| **最新版本** | v0.8.6（2026 年初）|
| **License** | MIT |

### 2.2 核心特点

**定位**：Firecrawl 的**开源替代**，专为 LLM 消费设计。

| 特性 | 说明 |
|------|------|
| **异步浏览器池** | 最小化网络延迟，内置缓存 |
| **Markdown 优先输出** | 带标题/表格/代码/引用提示的结构化输出 |
| **Fit Markdown** | 启发式过滤移除噪音内容 |
| **LLM 结构化提取** | 支持任意 LLM 做 schema-based 提取 |
| **Deep Crawl** | BFS 等策略，深层页面发现 |
| **Crash Recovery** | `resume_state` + `on_state_change` 回调 |
| **多浏览器** | Chromium / Firefox / WebKit |
| **Shadow DOM** | 扁平化 Shadow DOM 内容提取 |
| **3 层反检测** | 自动检测 bot 压力并升级代理 |

### 2.3 安装

```bash
pip install crawl4ai
crawl4ai-setup  # 自动安装浏览器依赖
```

### 2.4 使用

```python
import asyncio
from crawl4ai import AsyncWebCrawler

async def main():
    async with AsyncWebCrawler(verbose=True) as crawler:
        result = await crawler.arun(
            url="https://example.com",
            markdown=True,        # 输出 Markdown
            fit_markdown=True,    # 过滤噪音
            bypass_cache=True
        )
        print(result.markdown)

asyncio.run(main())
```

### 2.5 与 Firecrawl 对比

| 对比项 | Crawl4AI | Firecrawl |
|--------|----------|-----------|
| **开源** | ✅ 完全开源 | ✅ 开源但主服务付费 |
| **云托管** | 即将推出（封闭 beta） | ✅ 已成熟 |
| **安装** | pip + 浏览器依赖 | 注册 API key 即用 |
| **数据控制** | 完全自控 | 数据经过云端 |
| **并发规模** | 无限（自托管） | 有限（按 tier）|
| **定价** | 免费（自托管） | 按量付费 |

### 2.6 适用场景

✅ **强烈推荐：**
- 数据隐私敏感（数据不经过第三方）
- 大规模抓取（无限并发）
- RAG / 向量数据库（需要高质量 Markdown）
- 偏好开源（活跃社区 50k+ stars）

## 3. Jina Reader（极轻量内容读取）

### 3.1 核心原理

**原理极为直接**：调用 Jina 的 Reader API，传入 URL，返回净化后的文本内容。

```bash
# 最简单的用法：curl 即可
curl -s "https://r.jina.ai/https://example.com"

# 支持语言指定
curl -s "https://r.jina.ai/https://example.com@zh"   # 中文优化
curl -s "https://r.jina.ai/https://example.com@en"   # 英文优化
```

### 3.2 特点

| 优点 | 缺点 |
|------|------|
| ✅ 零配置，curl 即可 | ❌ 无 JS 渲染 |
| ✅ 免费调用限额 | ❌ 无交互能力 |
| ✅ AI 优化输出（去除广告/导航）| ❌ 无反检测 |
| ✅ 支持 Markdown 格式 | ❌ 免费额度有限制 |
| ✅ 嵌入 RAG pipeline 极简 | ❌ 无法处理需要登录的内容 |

### 3.3 适用场景

| 场景 | 推荐程度 |
|------|----------|
| 公开文章的快速读取测试 | ✅✅✅ |
| RAG pipeline 数据摄入（简单页面）| ✅✅ |
| 轻量爬虫的数据清洗 | ✅✅ |
| JS 重度页面 | ❌（用 Firecrawl/Crawl4AI）|
| 需要登录的内容 | ❌（用 Camoufox）|
