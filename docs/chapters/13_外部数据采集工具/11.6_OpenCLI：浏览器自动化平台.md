## 2. 核心定位

> **"Turn websites, browser sessions, Electron apps, and local tools into deterministic interfaces for humans and AI agents."**

OpenCLI 是**三层统一工具**：

```
┌─────────────────────────────────────────────────────────────────┐
│                    OpenCLI 三层架构                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Layer 1: 网站 → CLI 适配器（90+ 内置）                          │
│           Bilibili / 知乎 / 小红书 / Reddit / HackerNews 等      │
│                                                                  │
│  Layer 2: 浏览器自动化（Chrome 扩展 + CDP）                       │
│           复用你的登录态，操作任意网站                            │
│                                                                  │
│  Layer 3: 本地 CLI Hub                                          │
│           gh / docker / obsidian 等注册到统一界面                │
│                                                                  │
│  Layer 4: Electron 桌面 App 控制（CDP）                          │
│           Cursor / ChatGPT / Notion / Discord 等                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

---

## 3. 核心特点

| 特点 | 说明 |
|------|------|
| **零 Token 消耗** | 命令式执行，无 LLM 调用 |
| **确定性输出** | 相同命令 → 相同输出，可管道、脚本化 |
| **登录态复用** | Chrome 扩展一次性配置，所有平台共享 |
| **AI Agent 原生** | 内置 explore / synthesize / cascade 工具发现 |
| **反检测内置** | navigator.webdriver 补丁等 |
| **90+ 平台适配器** | 写新的只需 `opencli browser init` |

## 4. 安装与配置

```bash
# Step 1: 安装 Browser Bridge Extension
# 从 GitHub Releases 下载 opencli-extension.zip
# chrome://extensions → 开发者模式 → 加载已解压的扩展程序

# Step 2: 安装 OpenCLI
npm install -g @jackwener/opencli

# Step 3: 验证
opencli doctor

# Step 4: 安装 OpenClaw Skill
npx skills add jackwener/opencli
```

### 4.1 OpenClaw 前置要求

```bash
openclaw config set tools.profile "coding"
openclaw gateway restart
```

## 5. 核心命令

### 5.1 内置适配器命令（90+ 平台）

```bash
# 公开 API（无需浏览器）
opencli hackernews top --limit 5
opencli reddit hot --limit 10
opencli bilibili hot --limit 5
opencli zhihu hot --limit 10

# 浏览器命令（需要登录）
opencli bilibili hot --limit 5     # B站热门
opencli twitter trending --limit 10  # Twitter 趋势
opencli xiaohongshu search "关键词"  # 小红书搜索
```

### 5.2 浏览器控制命令（opencli browser）

**核心浏览器命令（从源码提取）：**

| 命令 | 功能 |
|------|------|
| **浏览器启动** | |
| `opencli browser open <url>` | 打开页面 |
| `opencli browser tab new [url]` | 新建标签页 |
| `opencli browser tab list` | 列出标签页 |
| `opencli browser tab select <targetId>` | 切换标签 |
| `opencli browser tab close` | 关闭当前标签 |
| **页面操作** | |
| `opencli state` | 获取页面状态 |
| `opencli click <selector>` | 点击元素 |
| `opencli type <selector> <text>` | 输入文本 |
| `opencli select <selector> <option>` | 选择下拉选项 |
| `opencli keys <key>` | 发送按键 |
| `opencli wait <ms>` | 等待（毫秒） |
| **内容获取** | |
| `opencli get <selector>` | 获取元素内容 |
| `opencli find <selector>` | 查找元素 |
| `opencli extract` | 智能提取内容 |
| `opencli eval <js>` | 执行 JavaScript |
| **页面视图** | |
| `opencli screenshot` | 截图 |
| `opencli scroll <px>` | 滚动（像素） |
| `opencli back` | 返回上一页 |
| **网络** | |
| `opencli network --filter "<pattern>"` | 网络请求过滤 |
| **标签页管理** | |
| `opencli tab list/new/select/close` | 标签页操作 |

**AI Agent 工具：**

| 命令 | 功能 |
|------|------|
| `opencli explore <url> --site <name>` | 探索网站能力 |
| `opencli synthesize <name>` | 生成适配器 |
| `opencli cascade <url>` | 级联检测认证策略 |
| `opencli generate <url> --goal "<goal>"` | 一站式生成适配器 |

### 5.4 输出格式

```bash
# 表格（默认）
opencli bilibili hot

# JSON / YAML / Markdown / CSV
opencli bilibili hot -f json
opencli bilibili hot -f yaml

# 详细日志
opencli bilibili hot -v
```

## 7. 与 Agent Reach 的对比

| 维度 | Agent Reach | OpenCLI |
|------|------------|---------|
| **核心思路** | 上游工具脚手架 | 统一浏览器自动化 |
| **浏览器控制** | ❌ 不支持 | ✅ Chrome 扩展 + CDP |
| **网站适配器** | 依赖独立工具组合 | 90+ 内置 |
| **Cookie 配置** | 各平台各自配置 | Chrome 扩展一次性 |
| **MCP 支持** | ✅ 集成 Exa | ❌ |
| **工具链复杂度** | 高（多工具组合）| 低（统一抽象）|
| **零成本** | ✅ | ✅ |

**推荐组合**：Agent Reach（一键安装上游工具）+ OpenCLI（需要浏览器操作时）。

## 8. 与 browser-use 的对比

| 对比项 | OpenCLI | browser-use |
|--------|---------|-------------|
| **Token 消耗** | 零 | 高（LLM 驱动）|
| **输出确定性** | ✅ 相同命令相同输出 | ❌ 依赖 LLM |
| **速度** | 快 | 慢 |
| **适合场景** | 已知网站 / 定时任务 | 未知网站探索 |
| **AI 理解能力** | ❌ | ✅ |

## 9. 适用场景

✅ **强烈推荐 OpenCLI：**

| 场景 | 示例 |
|------|------|
| 定时数据采集 | `crontab -e` → 每天 9 点 `opencli bilibili hot` |
| AI Agent 网站操作 | explore → synthesize → 自动注册工具 |
| 已知网站快速操作 | B站/知乎/小红书热门内容读取 |
| 复用登录态 | Chrome 已登录，一次配置所有平台 |
| 确定性管道任务 | `opencli hackernews top -f json \| jq` |

❌ **不适合的场景：**
- 大规模爬虫（用 Crawl4AI）
- 未知网站探索（用 browser-use）
- 需要 AI 理解的复杂判断（用 browser-use）
