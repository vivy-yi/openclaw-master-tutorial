## 2. 核心理念

**Agent Reach = 脚手架工具，不是框架**

```
传统框架：框架绑定工具，替换成本高
Agent Reach：脚手架帮你装工具，装完由 AI 直接调用上游 CLI
```

安装后，每个平台的工具完全独立，不经过 Agent Reach 包装层。


## 5. 各平台工具详解

### 5.1 Twitter/X：xurl（推荐替代）

**当前系统已安装 xurl**，它是官方 X API v2 的完整 CLI 包装，支持 OAuth2：

```bash
xurl auth oauth2              # OAuth 认证
xurl search "query" -n 10     # 搜索推文
xurl read TWEET_ID            # 读取单条推文
xurl timeline -n 20           # 时间线
xurl post "Hello!"            # 发推
```

**xurl 可用命令（从源码提取）：**
| 命令 | 功能 |
|------|------|
| **认证** | |
| `xurl auth check` | 检查认证状态 |
| `xurl auth extract` | 提取浏览器 Cookie |
| `xurl auth set` | 设置认证 |
| `xurl auth clear` | 清除认证 |
| `xurl auth browsers` | 列出可用浏览器 |
| **用户** | |
| `xurl user <handle>` | 获取用户资料 |
| `xurl followers <handle>` | 粉丝列表 |
| `xurl following <handle>` | 关注列表 |
| **推文** | |
| `xurl tweets <handle> -n N` | 用户推文 |
| `xurl tweet <urlOrId>` | 单条推文 |
| `xurl thread <urlOrId>` | 完整线程 |
| `xurl replies <urlOrId>` | 回复列表 |
| **搜索** | |
| `xurl search "query" -n N` | 搜索推文 |
| `xurl search "query" --type latest` | 最新推文 |
| **时间线** | |
| `xurl home` | 首页时间线 |
| `xurl home --following` | 关注者时间线 |
| `xurl bookmarks -n N` | 收藏 |
| `xurl likes <handle> -n N` | 点赞列表 |
| **列表** | |
| `xurl lists <handle>` | 用户列表 |
| `xurl list <list-id>` | 列表详情 |
| `xurl list-members <list-id>` | 列表成员 |
| `xurl list-tweets <list-id>` | 列表推文 |
| **私信** | |
| `xurl dms` | 发私信 |
| `xurl inbox` | 收件箱 |
| `xurl conversation <id>` | 对话详情 |
| **通知** | |
| `xurl notifications` | 通知列表 |
| `xurl mentions` | 提及 |
| `xurl verified-notifications` | 认证通知 |

vs twitter-cli（Cookie 方式）：OAuth2 更稳定，无 Cookie 过期问题。

### 5.2 视频：yt-dlp（当前系统已装）

yt-dlp 支持 YouTube + B站 + 1800+ 平台：

```bash
# 视频元数据（不下载）
yt-dlp --dump-json "URL"

# 下载字幕
yt-dlp --write-sub --write-auto-sub --sub-lang "zh-Hans,zh,en" \
  --skip-download -o "/tmp/%(id)s" "URL"

# B站字幕
yt-dlp --write-sub --sub-lang "zh-Hans,zh,en" \
  --convert-subs vtt --skip-download "https://www.bilibili.com/video/BVxxx"
```

### 5.3 搜索：Exa MCP（当前系统已装）

```bash
mcporter call 'exa.web_search_exa(query: "搜索词", numResults: 5)'
```

免费，无需 API Key，通过 mcporter 管理。

### 5.4 公众号：Camoufox（当前系统已装）

Camoufox 反检测浏览器是微信公众号读取的首选：

```python
import camoufox

with camoufox.API(antibot=True) as browser:
    page = browser.new_page()
    page.goto("公众号文章URL")
    content = page.content()
```

## 6. 工具替换指南

### 为什么需要替换？

| 原因 | 说明 |
|------|------|
| **封号风险** | Twitter/小红书 Cookie 方式有被封风险 |
| **能力升级** | 某些场景需要更高级能力（如视频脚本提取）|
| **成本优化** | 官方 API 收费，替代方案免费 |
| **稳定性** | OAuth 比 Cookie 更稳定 |

### 替换操作

每个平台对应一个独立的 `.py` 文件在 `channels/` 目录下：

```
channels/
├── twitter.py   → 替换为 xurl
├── bilibili.py  → 替换为 bili-cli
└── ...
```

**替换原则**：只替换检测逻辑和调用方式，上游工具完全独立。

## 7. 与 OpenCLI 的关系

**互补，不是替代：**

| 维度 | Agent Reach | OpenCLI |
|------|------------|---------|
| **核心思路** | 安装脚手架 + 上游工具组合 | 统一浏览器自动化 + 适配器 |
| **浏览器控制** | ❌ 不支持 | ✅ Chrome 扩展 + CDP |
| **网站适配器** | 依赖 yt-dlp/mcporter 等独立工具 | 90+ 内置，写新的只需 `opencli browser init` |
| **MCP 支持** | ✅ 集成 Exa 搜索 | ❌ 无 MCP |
| **Cookie 配置** | 各平台各自配置 | Chrome 扩展一次性配置 |

**推荐组合**：Agent Reach（一键安装）+ OpenCLI（需要浏览器控制时）。
