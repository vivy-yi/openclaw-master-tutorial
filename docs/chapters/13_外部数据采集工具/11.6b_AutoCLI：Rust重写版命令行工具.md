## 2. AutoCLI vs OpenCLI 核心区别

AutoCLI 是 OpenCLI（TypeScript）的 **Rust 重写版**，性能提升显著：

### 🚀 性能对比

| 指标 | 🦀 AutoCLI (Rust) | 📦 OpenCLI (Node.js) | 提升 |
|------|:-----------------:|:-----------------:|:----:|
| 内存占用（Public 命令）| 15 MB | 99 MB | **6.6x** |
| 内存占用（Browser 命令）| 9 MB | 95 MB | **10.6x** |
| 二进制大小 | 4.7 MB | ~50 MB | **10x** |
| 运行时依赖 | **无** | Node.js 20+ | **零依赖** |
| `bilibili hot` 耗时 | **1.66s** | 20.1s | 🔥 **12x** |
| `zhihu hot` 耗时 | **1.77s** | 20.5s | 🔥 **11.6x** |

> 数据来源：[AutoCLI GitHub README](https://github.com/nashsu/AutoCLI)，macOS Apple Silicon 自动化测试

### 🤔 选哪个？

| 场景 | 推荐 |
|------|------|
| **AI Agent 集成**（低内存、多任务）| AutoCLI ✅ |
| **已有 OpenCLI 跑得没问题** | 继续用也行 |
| **需要最新平台适配器** | OpenCLI（更新快）|
| **嵌入式 / 边缘设备** | AutoCLI ✅ |

**两者命令完全兼容**，可以无缝切换。

## 3. 安装

### 3.1 一键安装（推荐 macOS / Linux）

```bash
curl -fsSL https://raw.githubusercontent.com/nashsu/autocli/main/scripts/install.sh | sh
```

安装到 `/usr/local/bin/autocli`。

### 3.2 手动下载

从 [GitHub Releases](https://github.com/nashsu/autocli/releases/latest) 下载：

| 平台 | 文件 |
|------|------|
| macOS Apple Silicon | `autocli-aarch64-apple-darwin.tar.gz` |
| macOS Intel | `autocli-x86_64-apple-darwin.tar.gz` |
| Linux x86_64 | `autocli-x86_64-unknown-linux-musl.tar.gz` |
| Windows | `autocli-x86_64-pc-windows-msvc.zip` |

解压后放到 PATH 中。

### 3.3 从源码编译

```bash
git clone https://github.com/nashsu/autocli.git
cd autocli
cargo build --release
cp target/release/autocli /usr/local/bin/
```

### 3.4 更新

```bash
# 重新运行安装脚本即可
curl -fsSL https://raw.githubusercontent.com/nashsu/autocli/main/scripts/install.sh | sh
```

## 4. Chrome 扩展配置（Browser 命令必需）

Browser 命令（需要登录态的站点）需要安装 Chrome 扩展：

### 安装步骤

1. 从 [GitHub Releases](https://github.com/nashsu/autocli/releases/latest) 下载 `autocli-chrome-extension.zip`
2. 解压到任意目录
3. 打开 Chrome → `chrome://extensions`
4. 开启右上角「开发者模式」
5. 点击「加载已解压的扩展程序」
6. 选择解压后的文件夹

扩展会自动连接 autocli daemon。Public 命令（hackernews、devto 等）无需扩展。

## 5. 快速开始

```bash
# 查看所有命令
autocli --help

# 查看某个站点命令
autocli hackernews --help

# 获取 HN 热门（Public，无需浏览器）
autocli hackernews top --limit 10

# JSON 格式输出
autocli hackernews top --limit 5 --format json

# 获取 Bilibili 热门（需要浏览器）
autocli bilibili hot --limit 20

# 搜索 Twitter（需要登录）
autocli twitter search "rust lang" --limit 10

# 运行诊断
autocli doctor

# 生成 Shell 补全
autocli completion bash >> ~/.bashrc
autocli completion zsh >> ~/.zshrc
```

## 7. AI 原生能力

AutoCLI 保留了 OpenCLI 的 AI 发现能力，并增加了云端共享：

### 7.1 生成适配器

```bash
# AI 驱动（推荐）：AutoCLI.ai 分析页面，自动生成适配器
autocli generate https://www.example.com --goal hot --ai

# 规则驱动：无需 AI 的启发式分析
autocli generate https://www.example.com --goal hot
```

### 7.2 探索网站 API

```bash
# 探测端点、框架、Store
autocli explore https://www.example.com --site mysite

# 交互式模糊测试（触发隐藏 API）
autocli explore https://www.example.com --auto --click "评论,字幕"

# 自动探测认证策略（PUBLIC → COOKIE → HEADER）
autocli cascade https://api.example.com/hot
```

**发现能力：**
- `.json` 后缀探测（Reddit 风格 REST 发现）
- `__INITIAL_STATE__` 提取（Bilibili、小红书等 SSR 站点）
- Pinia/Vuex Store 发现和 Action 映射
- `--goal search` 自动发现搜索端点

### 7.3 搜索社区适配器

```bash
# 通过 URL 搜索
autocli search https://www.example.com

# 直接输入域名
autocli search example.com
```

从 [autocli.ai](https://autocli.ai) 搜索社区共享的适配器，选择后自动下载到本地。

### 7.4 AI 认证

```bash
autocli auth
```

1. 打开浏览器到 `https://autocli.ai/get-token`
2. 输入 Token
3. 验证并保存到 `~/.autocli/config.json`

### 8.3 在 OpenClaw 中使用

OpenClaw 内置浏览器执行命令时，可直接调用 AutoCLI：

```bash

```

AutoCLI 适合 OpenClaw Cron 定时任务：
- **零 Token 消耗**（确定性命令）
- **极低内存**（~15MB，vs Node.js ~100MB）
- **极速启动**（4.7MB 二进制，无需运行时）

## 9. 与 OpenCLI 共存

AutoCLI 和 OpenCLI 可以同时安装，通过别名区分：

```bash
# OpenCLI (Node.js)
opencli hackernews top --limit 5

# AutoCLI (Rust)
autocli hackernews top --limit 5
```

命令语法完全兼容，可以逐步迁移 Cron 任务从 OpenCLI 到 AutoCLI。

## 10. 常见问题

### Q: AutoCLI 支持 Windows 吗？
A: ✅ 支持，下载 `.zip` 文件，解压后把 `autocli.exe` 放到 PATH 即可。

### Q: AutoCLI 会替代 OpenCLI 吗？
A: 不会完全替代。OpenCLI 继续活跃开发（更多平台适配器），AutoCLI 专注于性能和 AI Agent 场景。两者命令兼容，可以共存。

### Q: 已有 OpenCLI 配置，需要重新配置 Chrome 扩展吗？
A: ✅ 不需要，AutoCLI 和 OpenCLI 共用同一个 Chrome 扩展。

### Q: 报错 "Chrome extension not connected"
A: 确保扩展已安装并在 Chrome 中启用，daemon 正在运行（`autocli doctor` 可诊断）。
