## 2. 核心原理：为何 Camoufox 不可检测

Camoufox 是**基于 Firefox 的反检测浏览器**，在 **C++ 层面修改 Firefox 源码**实现指纹注入，使 JavaScript 层面完全不可感知。

### 2.1 三层隔离架构

```
┌─────────────────────────────────────────────────────────────────┐
│                    Camoufox 架构                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  C++ 层（Firefox 源码修改）← 这里注入指纹，JS 看不到              │
│         │                                                        │
│         ▼                                                        │
│  SpiderMonkey JS Engine（Firefox JS 引擎）                        │
│         │                                                        │
│         ▼                                                        │
│  Web Content（网站 JS）← 只能看到被 C++ 层处理后的指纹           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

对比：playwright-stealth / undetected-chromedriver 的 JS 注入方式
  网站 JS  ←  ←  ←  ←  ←  ←  ←  （容易被绕过检测）

```

### 2.2 关键检测绕过能力

| 检测类型 | Camoufox | playwright-stealth | undetected-chromedriver |
|----------|----------|-------------------|------------------------|
| `navigator.webdriver` | ✅ C++ 层清空 | JS patch（可被检测） | Selenium patch |
| `window.__playwright__` | ✅ 完全隔离沙箱 | 变量隐藏 | 变量隐藏 |
| Canvas 指纹 | ✅ C++ 层随机化 | JS 注入（可被检测） | JS patch |
| WebGL 指纹 | ✅ C++ 层随机化 | JS 注入 | JS patch |
| 字体指纹 | ✅ 子像素抗锯齿匹配 | ❌ | ❌ |
| Human Cursor | ✅ C++ HumanCursor 算法 | ❌ | ❌ |
| 自动化相关变量 | ✅ 底层清除 | JS 覆盖（可被检测） | Selenium 覆盖 |

### 2.3 BrowserForge 指纹生成

Camoufox 使用 `browserforge` 库生成**符合真实市场分布**的指纹，而非随机生成：

- Linux 用户 → 5%（符合实际统计）
- macOS 分辨率 → 符合 Apple 设备分布
- Browser 版本 → 符合市场占有率的版本分布

这样可以躲避基于机器学习的流量分析（ML-based fingerprinting）。

### 2.4 联合欺骗

Camoufox 支持多维度联合 IP 欺骗：

| 维度 | 实现方式 |
|------|---------|
| **Geolocation** | 根据代理 IP 自动计算并设置 |
| **Timezone** | 与 IP 地理位置匹配 |
| **Locale** | 与地区匹配 |
| **WebRTC IP** | 协议级伪造（不依赖代理）|

## 3. 安装与使用

```bash
pip install camoufox

# 或通过 Agent Reach 自动安装（已配置）
```

### 3.1 基本使用（与 Playwright API 兼容）

```python
import camoufox

with camoufox.API(antibot=True) as browser:
    page = browser.new_page()
    page.goto("https://example.com")
    content = page.content()
    print(content)
```

### 3.2 异步模式

```python
import asyncio
import camoufox

async def main():
    async with camoufox.AsyncAPI(antibot=True) as browser:
        page = await browser.new_page()
        await page.goto("https://example.com")
        content = await page.content()
        print(content)

asyncio.run(main())
```

### 3.2.1 OpenClaw 可用工具（从源码提取）


**Camoufox 使用 Playwright Python API，主要方法：**
| 类别 | 方法 | 功能 |
|------|------|------|

| | `camoufox.NewBrowser(playwright, **kwargs)` | 启动新浏览器 |
| | `camoufox.AsyncAPI(**options)` | 异步浏览器实例 |
| | `camoufox.AsyncNewBrowser(playwright, **kwargs)` | 异步启动浏览器 |
| **页面操作** | `page.goto(url)` | 导航到 URL |
| | `page.content()` | 获取 HTML 内容 |
| | `page.title()` | 获取页面标题 |
| | `page.url` | 获取当前 URL |
| | `page.close()` | 关闭页面 |
| **交互操作** | `page.click(selector)` | 点击元素 |
| | `page.type(selector, text)` | 输入文本 |
| | `page.fill(selector, value)` | 填充表单 |
| | `page.check(selector)` | 勾选复选框 |
| | `page.select_option(selector, values)` | 选择下拉 |
| **内容获取** | `page.inner_html(selector)` | 获取元素 HTML |
| | `page.inner_text(selector)` | 获取元素文本 |
| | `page.evaluate(js)` | 执行 JavaScript |
| **截图与PDF** | `page.screenshot(**options)` | 截图 |

## 4. 与其他反检测方案对比

| 对比项 | Camoufox | undetected-chromedriver | playwright-stealth |
|--------|----------|------------------------|---------------------|
| **底层技术** | C++ 修改 Firefox 源码 | Selenium + Chromium patch | Playwright JS 注入 |
| **Chromium 支持** | ❌ 仅 Firefox | ✅ | ✅ |
| **指纹质量** | 极高（JS 不可见） | 中等 | 较低（JS 可绕过）|
| **维护状态** | 活跃（CloverLabs） | 较慢 | 较慢 |
| **性能** | 轻量（~200MB） | 普通 | 普通 |
| **Playwright API** | ✅ 兼容 | ❌ Selenium 独有 | ✅ 兼容 |
| **成本** | 免费 | 免费 | 免费 |

## 5. 适用场景

✅ **强烈推荐使用 Camoufox 的场景：**

| 场景 | 原因 |
|------|------|
| Cloudflare 强反爬 | 需绕过 Turnstile 等挑战 |
| Datadome / PerimeterX | 高强度 JS 行为分析 |
| 需要多身份轮换 | 指纹质量高，市场分布真实 |
| 金融/电商数据采集 | 指纹完整性要求极高 |
| Playwright Python 用户 | 零成本迁移（API 兼容）|

❌ **不适合的场景：**

| 场景 | 原因 |
|------|------|
| Chromium 指纹需求 | 仅支持 Firefox |
| 简单公开页面 | 杀鸡用牛刀，用 Jina Reader 即可 |
| 需要 Chrome 扩展 | Camoufox 是无头浏览器 |

---

## 6. 局限性

| 限制 | 说明 |
|------|------|
| **不支持 Chromium** | 部分 WAF 测试 Spidermonkey 引擎行为（Camoufox 无能为力）|
| **无头模式为主** | 不适合需要视觉验证的场景 |
| **License** | Proprietary，免费使用但非完全开源 |

## 7. 与 OpenClaw 的集成

Camoufox 可作为 OpenClaw 浏览器的**反检测增强**：

```python
# 概念示例：在 OpenClaw Skill 中使用 Camoufox
import camoufox

async def fetch_with_antibot(url: str):
    async with camoufox.AsyncAPI(antibot=True) as browser:
        page = await browser.new_page()
        await page.goto(url)
        return await page.content()
```

**推荐使用方式：**
- 对抗强反爬网站时作为 OpenClaw 浏览器的底层引擎
- 与 OpenClaw Cron 配合做定期数据采集
- 通过 Skill 封装为 `camoufox-fetch` 工具
