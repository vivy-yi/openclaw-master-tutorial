### 协同工作流

典型的数据采集任务会**从下往上**逐层尝试：

```python
# 协同示例：从简单到复杂的采集流程
task = "采集某网站的股票数据"

# Layer 1: 尝试最简单的 Jina Reader
content = jina_reader(url)  # 轻量快速

# Layer 2: 如果失败，尝试 Firecrawl（JS渲染）
if "blocked" in content:
    content = firecrawl.scrape(url)  # 内置JS渲染

# Layer 3: 如果需要交互，尝试 Playwright MCP
if "need_login" in content:
    content = playwright_mcp.click_and_scrape(url, "#login-btn")

# Layer 4: 如果需要多步骤决策，使用 browser-use
if "complex_form" in content:
    content = browser_use.agent.run("填写表单并提交")

# Layer 5: 如果被反爬检测，切换到 Camoufox
if "cloudflare_blocked" in content:
    content = camoufox.scrape(url)  # C++层指纹隐藏
```

## 1. 工具分类详解

### 1.1 Layer 1: 基础内容读取

适用于：**简单静态页面，无需登录**

| 工具 | 类型 | Token消耗 | 反检测 | 适用场景 |
|------|------|----------|--------|---------|
| **官方 Fetch** | MCP | 零 | ❌ | 零配置简单URL |
| **Jina Reader** | API | 低 | ❌ | 极轻量URL→Markdown |
| **curl** | CLI | 零 | ❌ | 快速原型 |

**Jina Reader 最简用法：**

```bash
# 一个 curl 即可获取 Markdown 内容
curl -s https://r.jina.ai/https://example.com
```

**适用场景：**
- 博客文章读取
- 公开 API 响应
- 简单新闻页面

---

### 1.2 Layer 2: AI 友好爬虫

适用于：**需要 JS 渲染或 AI 结构化输出**

| 工具 | 类型 | Stars | Token消耗 | JS渲染 | AI结构化 | 成本 |
|------|------|-------|----------|--------|--------|---------|------|
| **Firecrawl** | SaaS + 自部署 | ⭐ 111,621 | 中~高 | ✅ | ✅ Markdown/JSON | 免费限额+付费 |
| **Crawl4AI** | 开源自托管 | ⭐ 64,451 | 低 | ✅ | ✅ | 免费 |
| **Jina Reader** | API | — | 低 | ❌ | ✅ Markdown | 免费限额 |

**一句话选型：**
- **快速原型** → Firecrawl（注册 API key 即可）
- **数据隐私 + 无限规模** → Crawl4AI（自托管，完全可控）
- **极轻量快速** → Jina Reader（curl 即可）

**Firecrawl 基本用法：**

```python
import firecrawl

app = firecrawl.FirecrawlApp(api_key="fc-xxxx")

# Scrape（抓取单页）
result = app.scrape_url(
    "https://example.com",
    formats=["markdown", "html", "screenshot"]
)
print(result["markdown"])

# Agent（描述需求，AI 自动完成）
result = app.agent("找出所有股价大于100的股票")
```

**Crawl4AI 自托管部署：**

```bash
pip install crawl4ai
crawl4ai-setup  # 一键部署

# API 调用
curl -X POST http://localhost:8000/crawl \
  -d '{"urls": ["https://example.com"], "markdown": true}'
```

**browser-use 基本用法：**

```python
from browser_use import Agent, Browser, ChatBrowserUse
import asyncio

async def main():
    browser = Browser()

        llm=ChatBrowserUse(),  # 或使用 OpenAI/Anthropic
        browser=browser,
    )
    
    await agent.run()

asyncio.run(main())
```

### 1.5 Layer 5: 反检测增强

适用于：**强反爬网站（WAF/Cloudflare/Datadome）**

| 工具 | Stars | 技术 | 指纹质量 | 适用场景 |
|------|-------|------|-----------|---------|
| **Camoufox** | ⭐ 7,577 | C++ 修改 Firefox | 极高（JS不可见）| 强反爬 |
| **playwright-stealth** | ⭐ 9k | JS 注入 | 中等 | 普通反爬 |
| **undetected-chromedriver** | ⭐ 9k | Selenium | 较低 | Chromium |

**Camoufox 核心优势：**

```
┌─────────────────────────────────────────────────────────────────┐
│                    Camoufox 架构                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  C++ 层（Firefox 源码修改）← 指纹在这里注入，JS看不到            │
│         │                                                        │
│         ▼                                                        │
│  SpiderMonkey JS Engine（Firefox JS 引擎）                     │
│         │                                                        │
│         ▼                                                        │
│  Web Content（网站 JS）← 只能看到被C++层处理后的指纹           │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

对比：playwright-stealth 的 JS 注入方式
  网站 JS  ←  ←  ←  ←  ←  ←  （容易被检测绕过）
```

**Camoufox 检测绕过能力：**

| 检测类型 | Camoufox | playwright-stealth | undetected-chromedriver |
|----------|----------|-------------------|------------------------|
| navigator.webdriver | ✅ C++层清空 | JS patch（可被检测） | Selenium patch |
| window.__playwright__ | ✅ 完全隔离 | 变量隐藏 | 变量隐藏 |
| Canvas 指纹 | ✅ C++层随机化 | JS注入（可被检测） | JS patch |
| WebGL 指纹 | ✅ C++层随机化 | JS注入 | JS patch |
| 字体指纹 | ✅ 子像素匹配 | ❌ | ❌ |
| Human Cursor | ✅ C++算法 | ❌ | ❌ |

**Camoufox 基本用法：**

```python
import camoufox

# 基本用法（与 Playwright API 兼容）
with camoufox.API(antibot=True) as browser:
    page = browser.new_page()
    page.goto("https://example.com")
    print(page.content())

# 异步模式
import asyncio
import camoufox

async def main():
    async with camoufox.AsyncAPI(antibot=True) as browser:
        page = await browser.new_page()
        await page.goto("https://example.com")
        print(await page.content())

asyncio.run(main())

# 代理配置
with camoufox.API(proxy="http://user:pass@host:port") as browser:
    ...

# 指纹配置
with camoufox.API(
    fingerprint="desktop",
    os="win",
    browser="firefox"
) as browser:
    ...
```

## 3. 协同实战：从简单到复杂的采集流程

### 3.1 场景一：采集公开博客文章

**需求**：采集某技术博客的文章列表

**协同流程**：

```
Jina Reader（Layer 1）
    ↓ 成功
完成
```

```python
# 最简单的方案：Jina Reader
import requests

url = "https://example.com/blog"
content = requests.get(f"https://r.jina.ai/{url}").text
print(content)
```

**决策理由**：
- 博客文章通常是静态 HTML
- Jina Reader 零成本，无需配置
- 失败率极低

### 3.2 场景二：采集需要 JS 渲染的页面

**需求**：采集某电商网站的商品列表（需 JS 渲染）

**协同流程**：

```
Jina Reader（Layer 1）
    ↓ 失败（无JS渲染）
Firecrawl（Layer 2）
    ↓ 成功
完成
```

```python
import firecrawl

app = firecrawl.FirecrawlApp()

# Firecrawl 内置 JS 渲染
result = app.scrape_url(
    "https://example.com/products",
    formats=["markdown"]
)
print(result["markdown"])
```

**决策理由**：
- Jina Reader 不支持 JS 渲染
- Firecrawl 内置 Playwright，自动渲染
- 免费额度足够小规模使用

### 3.3 场景三：需要登录后操作

**需求**：采集某会员网站的个人数据

**协同流程**：

```
OpenCLI 登录（Layer 3）
    ↓ 一次登录，复用登录态
OpenCLI 爬取（Layer 3）
    ↓ 成功
完成
```

```bash
# Step 1: 登录（一次即可，Chrome扩展保存）
opencli browser login example

# Step 2: 爬取（自动复用登录态）
opencli scrape "https://example.com/my-data" --format markdown
```

**决策理由**：
- OpenCLI 一次登录，所有平台复用
- 不需要每次都输入账号密码
- 支持 90+ 平台适配器

### 3.4 场景四：复杂多步骤表单

**需求**：填写搜索表单，查询结果，导出数据

**协同流程**：

```
browser-use Agent（Layer 4）
    ↓ LLM自主决策执行多步骤
完成
```

```python
from browser_use import Agent, Browser, ChatOpenAI
import asyncio

async def main():
    browser = Browser()
    
    agent = Agent(
        task="""
        1. 打开 https://example.com/search
        2. 在搜索框输入 "iPhone"
        3. 点击搜索按钮
        4. 等待结果加载
        5. 导出第一页的所有商品标题
        """,
        llm=ChatOpenAI(model="gpt-4o"),
        browser=browser,
    )
    
    result = await agent.run()
    print(result)

asyncio.run(main())
```

**决策理由**：
- 表单填写是多步骤复杂操作
- browser-use 的 LLM 可以自主决策每一步
- 适合"说得出但写不出代码"的场景

### 3.5 场景五：强反爬网站

**需求**：采集被 Cloudflare 保护的网站

**协同流程**：

```
Jina Reader（Layer 1）
    ↓ 失败（Cloudflare拦截）
Firecrawl（Layer 2）
    ↓ 失败（Cloudflare更强拦截）
Camoufox（Layer 5）
    ↓ 成功（C++层指纹隐藏）
完成
```

```python
import camoufox

# Camoufox 的 C++ 层指纹完全不可检测
with camoufox.API(antibot=True) as browser:
    page = browser.new_page()
    page.goto("https://cloudflare-protected.com")
    print(page.content())
```

**决策理由**：
- Cloudflare 检测 JavaScript 层面的指纹修改
- Camoufox 在 C++ 层修改 Firefox 源码
- 指纹完全不可感知

### 3.6 场景六：生产环境定期采集

**需求**：每天定时采集某网站数据

**协同流程**：

```
OpenClaw Cron + OpenClaw 内置浏览器（Layer 3）

定时任务完成
```

```yaml
      - browser.evaluate()
```

**决策理由**：
- OpenClaw 内置浏览器零 Token 消耗
- 确定性操作，结果可预测
- Cron 定时执行，无需人工介入

### 4.2 协同问题诊断

**问题**：多个浏览器窗口

**原因分析**：
1. 多个 isolated cron session 同时调用 `browser.open()`
2. 每个 session 在 Chrome 里开新标签页
3. 如果是"多开浏览器窗口"而非"多标签页"，需要进一步诊断

**解决方案**：

| 方案 | 操作 | 效果 |
|------|------|------|
| A. headless | `browser.headless: true` | 无窗口，后台运行 |
| B. 复用CDP | 配置 browser 工具复用 | 一个Chrome，多标签页 |
| C. 错开cron | 修改 schedule | 减少同时触发 |

**推荐**：方案 A（headless），最简单有效
