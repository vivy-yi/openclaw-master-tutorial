## 一、概述

v2026.6.5-beta.1 起，**Parallel** 正式成为 OpenClaw 捆绑的 `web_search` provider，无需额外安装插件，直接通过环境变量配置即可使用。

| 维度 | 内容 |
|------|------|
| **性质** | 捆绑的 `web_search` provider |
| **最低版本** | v2026.6.5-beta.1 |
| **配置方式** | `PARALLEL_API_KEY` 环境变量 |
| **官方文档** | https://docs.parallel.ai |
| **PR** | #85158 |

## 二、核心特性

### 2.1 Parallel API 特色功能

| 特性 | 说明 |
|------|------|
| **Guarded Endpoint** | API 调用有速率限制保护 |
| **Cache-safe Session IDs** | 支持会话级缓存管理 |
| **Onboarding Picker** | 新用户引导式 API Key 配置 |
| **多搜索引擎聚合** | 支持 Google/Bing/DuckDuckGo 等多源搜索 |
| **结构化结果** | 返回标准化 JSON，含标题/URL/摘要/来源 |

### 2.2 与其他 Provider 对比

| Provider | 捆绑 | 需要插件 | 并发支持 | 缓存 |
|----------|------|----------|----------|------|
| **Parallel** | ✅ 是 | ❌ 否 | ✅ | ✅ |
| Tavily | ❌ 否 | ✅ 是 | ✅ | ✅ |
| Google Serp | ❌ 否 | ✅ 是 | ❌ | ❌ |
| DuckDuckGo | ❌ 否 | ✅ 是 | ❌ | ❌ |

## 三、配置方法

### 3.1 获取 Parallel API Key

1. 访问 [Parallel AI Dashboard](https://app.parallel.ai)
2. 注册/登录账户
3. 进入 **Settings → API Keys**
4. 创建新的 API Key（注意：免费版有 QPS 限制）

### 3.2 环境变量配置

```bash
# 标准配置
PARALLEL_API_KEY=your_parallel_api_key_here

# 可选：指定默认搜索引擎
PARALLEL_DEFAULT_ENGINE=google

# 可选：设置搜索结果数量
PARALLEL_MAX_RESULTS=10
```

### 3.3 OpenClaw 配置文件中设置

```yaml
# openclaw.yaml 或 .openclaw.yaml
providers:
  web_search:
    provider: parallel
    config:
      api_key: ${PARALLEL_API_KEY}
      engine: google          # 可选：google/bing/duckduckgo
      max_results: 10 # 可选：默认10条
      safe_search: true      # 可选：成人内容过滤
```

### 3.4 Docker 环境变量

```bash
docker run -e PARALLEL_API_KEY=your_key_here \
  openclaw/openclaw:latest
```

## 四、使用方法

### 4.1 基础搜索

```
/search OpenClaw tutorial
```

### 4.2 带参数搜索

```
/search OpenClaw v2026.6.5 features --provider parallel --max5
```

### 4.3 在 Workflow 中使用

```yaml

  steps:
    - name: search-docs
      tool: web_search
      provider: parallel
      params:
        query: "OpenClaw configuration guide"
        engine: google
        max_results: 5
```

## 五、高级配置

### 5.1 多引擎配置

```yaml
providers:
  web_search:
    provider: parallel
    config:
      api_key: ${PARALLEL_API_KEY}
      engines:
        - name: google
          priority: 1
          enabled: true
        - name: bing
          priority: 2
          enabled: true
        - name: duckduckgo
          priority: 3
          enabled: false
```

### 5.2 缓存策略

```yaml
providers:
  web_search:
    provider: parallel
    config:
      cache:
        enabled: true
        ttl_seconds: 3600      # 缓存1小时
        max_entries: 100 # 最多缓存100条
```

### 5.3 代理配置（国内环境）

```bash
# 使用代理
export HTTP_PROXY=http://127.0.0.1:7890
export HTTPS_PROXY=http://127.0.0.1:7890
```

## 七、故障排除

### 7.1 常见错误

| 错误码 | 原因 | 解决方案 |
|--------|------|----------|
| `401 Unauthorized` | API Key 无效 | 检查 `PARALLEL_API_KEY` 配置 |
| `429 Rate Limited` | 超出 QPS 限制 | 降低请求频率或升级套餐 |
| `403 Forbidden` | 账户余额不足 | 充值或等待配额重置 |
| `500 Internal` | Parallel 服务端错误 | 等待服务恢复或联系支持 |

### 7.2 调试方法

```bash
# 查看 Parallel provider 状态
openclaw gateway status --provider parallel

# 启用详细日志
OPENCLAW_LOG_LEVEL=debug openclaw gateway start
```
