## 11.1 工作空间核心理念

### 11.1.1 什么是工作空间

**工作空间 (Workspace)** 是 OpenClaw 的核心组织单元，它将以下所有垂直领域内容和服务全部集中在一起：

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                              工作空间架构                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ┌─────────────────────────────────────────────────────────────────────────┐   │
│  │                         WORKSPACE (工作空间)                             │   │
│  │                                                                          │   │
│  │   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐   │   │
│  │   │   Agent     │  │   Tools     │  │   Skills    │  │   Memory     │   │   │
│  │   │  智能体     │  │   工具集    │  │   技能系统   │  │   记忆系统   │   │   │
│  │   └─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘   │   │
│  │                                                                          │   │
│  │   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐   │   │
│  │   │  Context    │  │  外部知识库  │  │   流程       │  │   规则       │   │   │
│  │   │  上下文     │  │  API/向量库  │  │   自动化     │  │   安全策略   │   │   │
│  │   └─────────────┘  └─────────────┘  └─────────────┘  └─────────────┘   │   │
│  │                                                                          │   │
│  └─────────────────────────────────────────────────────────────────────────┘   │
│                                                                                 │
│                        隔离性 ✅ + 共享基础模块 ✅                                │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 11.1.2 核心设计原则

#### 隔离性 (Isolation)

每个工作空间是**完全独立的上下文单元**：

```
工作空间 A                    工作空间 B
┌─────────────┐              ┌─────────────┐
│ Agent 配置   │              │ Agent 配置   │
│ Skills 配置  │              │ Skills 配置  │
│ 记忆系统     │              │ 记忆系统     │
│ 外部知识库   │              │ 外部知识库   │
│ 规则策略     │              │ 规则策略     │
└─────────────┘              └─────────────┘
        ↓                          ↓
   不互相干扰                  不互相干扰
```

**隔离确保**：
- 不同项目/任务之间不会相互污染
- 配置文件、记忆、变量完全独立
- 便于独立测试和部署

#### 共享基础模块 (Shared Base Modules)

尽管工作空间隔离，但**基础模块是共用的**：

```
                    ┌──────────────────────┐
                    │   共享基础模块       │
                    ├──────────────────────┤
                    │ • OpenClaw 内核      │
                    │ • 内置 Tools (read/write/exec)
                    │ • 内置 Skills (基础库)
                    │ • Gateway 服务       │
                    │ • 渠道连接器         │
                    │ • 认证系统           │
                    └──────────────────────┘
                           ↑
        ┌──────────────────┼──────────────────┐
        ↓                  ↓                  ↓
   工作空间 A           工作空间 B           工作空间 C
   (项目A)             (项目B)             (项目C)
```

## 11.2 工作空间类型

### 11.2.1 类型总览

| 类型 | 说明 | 适用场景 |
|------|------|---------|
| **个人助手** | 单用户通用助手 | 日常办公、个人效率 |
| **团队协作** | 多用户共享 | 团队项目、内部服务 |
| **垂直领域** | 专注特定行业 | 客服、医疗、金融 |
| **开发环境** | 代码开发辅助 | 软件开发、测试 |
| **服务化部署** | 7x24在线服务 | SaaS、企业级应用 |

### 11.2.2 个人助手型

**特点**：轻量、快速、个性化

```
┌────────────────────────────────────────┐
│           个人助手工作空间               │
├────────────────────────────────────────┤
│  Agent: 我的专属助手                    │
│  SOUL.md: 朋友般轻松的语调              │
│  USER.md: 我的偏好和习惯                │
│  Skills: 日程、笔记、提醒              │
│  记忆: 个人知识库、偏好设置             │
│  外部API: 日历、邮件、备忘录            │
└────────────────────────────────────────┘
```

**典型配置**：
```json
{
  "workspace": {
    "type": "personal",
    "agent": {
      "model": "claude-sonnet"
    },
    "skills": ["calendar", "notes", "reminder"],
    "memory": {
      "personal": true
    }
  }
}
```

### 11.2.3 团队协作型

**特点**：多用户、多角色、权限控制

```
┌────────────────────────────────────────┐
│           团队协作工作空间               │
├────────────────────────────────────────┤
│  Agent: 角色化（客服/助理/顾问）         │
│  记忆: 团队知识库、共享上下文            │
│  Skills: 团队专用技能包                 │
│  规则: 权限分级、操作审批               │
│  外部: 企业知识库、工单系统              │
└────────────────────────────────────────┘
```

### 11.2.4 垂直领域型

**特点**：深度定制、行业知识

```
┌────────────────────────────────────────┐
│           垂直领域工作空间               │
├────────────────────────────────────────┤
│  医疗助手:                            │
│    • 医学知识库                        │
│    • 病历处理技能                      │
│    • HIPAA 合规规则                    │
│                                        │
│  金融助手:                            │
│    • 金融市场数据 API                   │
│    • 风险评估模型                       │
│    • 合规审计规则                       │
└────────────────────────────────────────┘
```

### 11.2.5 开发环境型

**特点**：代码为主、开发工具链

```
┌────────────────────────────────────────┐
│           开发环境工作空间               │
├────────────────────────────────────────┤
```

## 11.3 工作空间组成详解

### 11.3.1 Agent (智能体)

**作用**：工作空间的"大脑"，负责理解、规划、执行

```markdown
# Agent 配置示例

## 角色定义
你是一个专业的技术写作助手。

## 能力范围
- 技术文档撰写
- 代码注释生成
- API 文档整理

## 行为准则
- 回复简洁明了
- 代码示例要完整可运行
```

### 11.3.2 Tools (工具集)

**作用**：Agent 执行任务的能力载体

| 工具类型 | 示例 | 说明 |
|---------|------|------|
| 内置工具 | read/write/exec | OpenClaw 核心能力 |
| 文件操作 | glob/cat/edit | 文件系统操作 |
| 系统操作 | browser/nodes | 系统集成能力 |
| 网络操作 | web_search/api | 外部网络访问 |

### 11.3.3 Skills (技能系统)

**作用**：扩展 Agent 的专业能力

```
~/.openclaw/workspace/
├── skills/                    # 工作空间专属 Skills
│   ├── 中文润色/              # 文本处理
│   ├── 代码审查/              # 开发辅助
│   └── 数据分析/              # 分析技能
├── shared-skills/             # 共享 Skills (从全局 skills/ 链接)
```

### 11.3.4 Memory (记忆系统)

**作用**：持久化知识和上下文

```
workspace/
├── memory/                    # 长期记忆
│   ├── MEMORY.md             # 核心记忆（规则、偏好、知识）
│   └── kb/                   # 知识库文件
├── daily/                    # 每日记忆
│   ├── 2026-05-12.md
│   └── 2026-05-11.md
└── sessions/                 # 会话历史
```

### 11.3.5 Context (上下文)

**作用**：会话级上下文管理

```markdown
# 上下文加载优先级

1. IDENTITY.md        → 启动时加载（Agent 身份）
2. SOUL.md            → 启动时加载（人设）
3. USER.md            → 会话开始时加载（用户信息）
4. AGENTS.md          → 每次对话时加载（操作规则）
5. MEMORY.md          → 需要时加载（长期记忆）
6. daily/             → 需要时加载（每日记忆）
```

### 11.3.6 外部知识库/API

**作用**：连接外部数据和服务

| 类型 | 示例 | 用途 |
|------|------|------|
| 向量数据库 | Pinecone/Milvus | 语义搜索 |
| 知识图谱 | Neo4j | 关系推理 |
| API | GitHub/Notion | 数据同步 |
| 数据库 | PostgreSQL | 结构化存储 |

### 11.3.7 流程 (Process)

**作用**：定义自动化工作流

```
workspace/
├── workflows/                 # 工作流定义
│   ├── daily-brief.yaml     # 晨间简报
│   ├── weekly-report.yaml   # 周报生成
│   └── code-review.yaml    # 代码审查
├── triggers/                # 触发器
│   ├── cron.yaml           # 定时触发
│   └── webhook.yaml        # Webhook 触发
```

### 11.3.8 规则 (Rules)

**作用**：安全策略和操作约束

```markdown
# workspace/rules/security.md

## 安全规则
- 禁止删除生产环境文件
- 禁止执行 rm -rf /
- 敏感操作需二次确认

## 审批规则
- 高危操作需要审批
- 超过阈值需要通知

## 审计规则
- 所有操作记录日志
- 敏感操作留痕
```

## 11.4 工作空间配置

### 11.4.1 目录结构

```
~/.openclaw/
├── openclaw.json                    # 全局配置
├── credentials/                     # 认证凭据（共享）
├── skills/                          # 全局共享 Skills（共享）
├── gateway/                         # Gateway 配置（共享）
│
├── workspace/                       # 默认工作空间 (main agent)
│   ├── AGENTS.md                   # ⭐ Agent 操作指令
│   ├── SOUL.md                     # ⭐ 人设定义
│   ├── USER.md                     # ⭐ 用户信息
│   ├── IDENTITY.md                 # ⭐ 身份定义
│   ├── HEARTBEAT.md               # 心跳检查清单
│   ├── BOOT.md                     # 启动检查清单
│   ├── TOOLS.md                    # 工具说明
│   │
│   ├── memory/                     # 长期记忆
│   │   ├── MEMORY.md              # 核心记忆
│   │   └── kb/                    # 知识库
│   ├── daily/                      # 每日记忆
│   │
│   ├── skills/                     # 工作空间 Skills（隔离）
│   │   ├── custom-skill/
│   │   └── ...
│   │
│   ├── workflows/                  # 工作流（隔离）
│   ├── rules/                      # 规则策略（隔离）
│   │
│   ├── canvas/                     # Canvas UI
│   ├── downloads/                  # 下载文件
│   ├── uploads/                    # 上传文件
│   └── temp/                       # 临时文件
│
└── agents/                          # 多 Agent 目录
    ├── main/                       # main agent（使用默认 workspace/）
    │   ├── agent/                  # agent 状态
    │   └── sessions/              # 会话历史
    └── <agentId>/                 # 其他 agent
        ├── agent/
        ├── sessions/
        └── workspace/             # ⭐ 可选：独立的 workspace 目录
```

### 11.4.2 多工作空间配置

```json
// openclaw.json
{
  "agents": {
    "main": {
      "workspace": "~/.openclaw/workspace",
      "model": "claude-sonnet-4"
    },
    "work": {
      "workspace": "~/.openclaw/workspaces/work",
      "model": "claude-sonnet-4"
    },
    "coding": {
      "workspace": "~/.openclaw/workspaces/coding",
      "model": "claude-haiku-4"
    }
  }
}
```

### 11.4.3 共享 vs 隔离配置

```yaml
# 工作空间隔离级别配置

workspace:
  isolation:
    # 完全隔离（推荐用于多租户）
    full:
      - memory/
      - skills/
      - workflows/
      - rules/
    
    # 共享基础模块（推荐用于团队）
    shared:
      - skills/           # 共享全局 Skills
      - credentials/     # 共享认证（可选）
    
    # 可配置共享
    configurable:
      - skills: true      # 是否共享 Skills
      - credentials: false
      - tools: true      # 是否共享 Tools 配置
```

## 11.5 工作空间实践

### 11.5.1 创建新工作空间

```bash
# 方式1: 使用 CLI
openclaw workspace create my-project

# 方式2: 手动创建
mkdir -p ~/.openclaw/workspaces/my-project
cd ~/.openclaw/workspaces/my-project

# 创建核心文件
touch AGENTS.md SOUL.md USER.md IDENTITY.md

# 配置到 openclaw.json
openclaw agents add my-project --workspace ~/.openclaw/workspaces/my-project
```

### 11.5.2 工作空间切换

```bash
# 查看所有工作空间
openclaw agents list

# 切换到指定工作空间
openclaw chat --agent my-project

# 或在对话中切换
/agent my-project
```

### 11.5.3 工作空间迁移

```bash
# 导出工作空间配置
openclaw workspace export my-project --output ./backup/

# 导入到新环境
openclaw workspace import ./backup/my-project/ --name new-project
```

### 11.6.3 性能优化建议

| 优化项 | 建议 |

## 11.7 常见问题

### Q1: 工作空间之间如何共享 Skills？

```bash
# 在工作空间内创建软链接
ln -s ~/.openclaw/skills/shared ~/.openclaw/workspaces/my-ws/skills/shared
```

### Q2: 如何隔离外部 API 凭据？

```yaml
# workspace/rules/api-access.md

## API 隔离规则
- 每个工作空间独立的 API Key
- 不共享认证凭据
- 定期轮换密钥
```

### Q3: 工作空间迁移要注意什么？

1. **MEMORY.md** - 核心记忆是否需要保留
2. **Skills** - 是否有工作空间专属 Skills
3. **规则配置** - 安全规则是否需要调整
4. **外部依赖** - API 凭据是否需要更新
