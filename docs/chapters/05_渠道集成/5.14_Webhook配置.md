## 8.5.1 Webhook 概述

### 什么是 Webhook

```
┌─────────────────────────────────────────────┐
│           Webhook 概念                         │
├─────────────────────────────────────────────┤
│                                             │
│  Webhook = 回调 = HTTP 回调                   │
│                                             │
│  传统轮询：                                  │
│  客户端 ← 定时请求 ← 服务器                   │
│                                             │
│  Webhook：                                   │
│  客户端 → 注册回调 → 事件 → 服务器回调        │
│                                             │
│  优势：                                       │
│  ├── 实时性好                               │
│  ├── 资源消耗低                              │
│  └── 简单可靠                               │
│                                             │
└─────────────────────────────────────────────┘
```

### 应用场景

```
Webhook 场景：

1. 消息接收
   ├── 第三方平台消息回调
   └── 自定义事件

2. 事件通知
   ├── 任务完成通知
   └── 状态变更通知

3. 集成触发
   ├── GitHub Webhook
   ├── GitLab Webhook
   └── 自定义系统
```

## 8.5.2 配置 Webhook

### 接收配置

```json
{
  "channels": {
    "webhook": {
      "enabled": true,
      "config": {
        "path": "/webhook/custom",
        "auth": {
          "type": "bearer",
          "token": "your-webhook-token"
        }
      }
    }
  }
}
```

### 认证方式

```
支持的认证方式：

1. Bearer Token
   Header: Authorization: Bearer <token>

2. Basic Auth
   Header: Authorization: Basic <base64>

3. Signature
   Header: X-Signature: sha256=<signature>

4. Custom Header
   Header: X-Custom-Auth: <value>
```

## 8.5.4 第三方集成

### GitHub Webhook

```json
{
  "channels": {
    "webhook": {
      "github": {
        "enabled": true,
        "path": "/webhook/github",
        "events": [
          "push",
          "pull_request",
          "issues"
        ],
        "secret": "your-github-webhook-secret"
      }
    }
  }
}
```

### 处理事件

```
GitHub 事件处理：

1. push - 代码推送
   → 触发代码审查
   → 执行 CI/CD

2. pull_request - PR 创建
   → 自动审查代码
   → 运行测试

3. issues - Issue 创建
   → 自动分类
   → 生成解决方案
```

## 8.5.5 发送 Webhook

### 配置发送

```json
{
  "webhooks": {
    "notification": {
      "enabled": true,
      "url": "https://your-server.com/webhook",
      "method": "POST",
      "headers": {
        "Authorization": "Bearer token",
        "Content-Type": "application/json"
      },
      "events": [
        "task_completed",
        "task_failed"
      ]
    }
  }
}
```

### 消息格式

```json
{
  "event": "task_completed",
  "timestamp": 1705123456789,
  "data": {
    "task_id": "task_123",
    "result": "success",
    "output": "..."
  }
}
```

## 8.5.6 安全配置

### 签名验证

```json
{
  "channels": {
    "webhook": {
      "security": {
        "signature": {
          "enabled": true,
          "algorithm": "hmac-sha256",
          "header": "X-Signature",
          "secret": "your-secret"
        },
        "ip_whitelist": [
          "203.0.113.0/24"
        ]
      }
    }
  }
}
```

### 验证示例

```python
import hmac
import hashlib

def verify_signature(payload, signature, secret):
    expected = hmac.new(
        secret.encode(),
        payload.encode(),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, signature)
```

## 8.5.7 常见问题

### Q1: Webhook 无法接收

```
问题：服务器收不到回调

解决：
1. 检查 URL 是否可访问
2. 验证防火墙配置
3. 检查 SSL 证书
4. 确认请求方法
```

### Q2: 签名验证失败

```
问题：签名不匹配

解决：
1. 检查 secret 是否正确
2. 验证算法选择
3. 检查时间戳
4. 确认编码格式
```

### Q3: 重复接收

```
问题：收到重复消息

解决：
1. 使用消息 ID 去重
2. 实现幂等处理
3. 设置超时重试
```
