## 一、方案概述

### 什么是零部署

```
┌─────────────────────────────────────────────────────────────────┐
│                    传统部署 vs 零部署                               │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  传统部署：                                                       │
│  飞书 ──▶ 公网服务器 ──▶ OpenClaw Gateway                        │
│           ↑ 需要购买服务器、配置域名、ngrok                        │
│                                                                  │
│  零部署：                                                         │
│  飞书 ──▶ 本地桥接 ──▶ OpenClaw Gateway (本地)                   │
│           ↑ 无需公网，飞书官方支持长连接                          │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 核心原理

飞书支持 **WebSocket 长连接**接收事件，无需公网回调地址。

### 方案对比

| 方案 | Stars | 特点 | 适用场景 |
|------|-------|------|---------|
| **feishu-openclaw 桥接** | ⭐321 | 独立进程、媒体支持好 | 生产环境 |
| **openclaw-feishu-plugin** | ⭐22 | 插件形式、多账户 | 简单场景 |

### 安装步骤

#### 1. 准备工作

```bash
# 克隆仓库
git clone https://github.com/AlexAnys/feishu-openclaw
cd feishu-openclaw

# 安装依赖
npm install
```

#### 2. 配置飞书应用

```
1. 访问 https://open.feishu.cn/app
2. 创建企业自建应用
3. 获取 App ID 和 App Secret
4. 配置权限：
   - im:message
   - im:message:send_as_bot
   - im:message.p2p_msg:readonly
   - im:message.group_msg:readonly
5. 开启「使用长连接接收事件」
6. 发布应用
```

#### 3. 配置桥接

```bash
# 复制配置
cp config.example.json config.json

# 编辑配置
vim config.json
```

```json
{
    "feishu": {
        "app_id": "cli_xxxxxxxx",
        "app_secret": "your-app-secret"
    },
    "openclaw": {
        "gateway_url": "http://localhost:18789"
    }
}
```

#### 4. 启动桥接

```bash
# 启动
FEISHU_APP_ID=cli_xxxxxxxx FEISHU_APP_SECRET=your-secret node bridge.mjs

# 或者直接启动（会读取 config.json）
node bridge.mjs
```

#### 5. 配置 OpenClaw

在 OpenClaw 配置中添加飞书渠道（如果需要）。

### 媒体支持

```bash
# 2026.02.02 更新后支持：
# ✅ 飞书图片 → AI 能看懂
# ✅ 飞书视频/文件 → 桥接接收并下载
# ✅ AI 生成图片 → 自动回传飞书
```

### 调试模式

```bash

tail -f bridge.log
```

## 三、openclaw-feishu-plugin 插件

### 特点

- ✅ **OpenClaw 官方插件形式**
- ✅ **多账户配置**
- ✅ **无需公网 IP**
- ✅ **简单易用**

### 安装

```bash
# 通过 openclaw 命令安装
openclaw plugins install https://github.com/baidan4855/openclaw-feishu-plugin
```

### 配置飞书应用

#### 1. 创建飞书应用

```
1. 访问 https://open.feishu.cn/app
2. 点击「创建企业自建应用」
3. 填写应用名称和描述
```

#### 2. 获取凭证

在「凭证与基础信息」页面获取：
- **App ID**
- **App Secret**

#### 3. 配置权限

在「权限管理」添加：

| 权限名称 | 权限标识 | 用途 |
|---------|---------|------|
| 获取与发送单聊、群组消息 | `im:message` | 收发消息 |
| 读取用户发给机器人的单聊消息 | `im:message.p2p_msg:readonly` | 接收私聊 |
| 获取群组中所有消息 | `im:message.group_msg:readonly` | 接收群聊 |
| 以应用的身份发消息 | `im:message:send_as_bot` | 发送消息 |
| 获取用户基本信息 | `contact:user.base:readonly` | 获取用户信息 |

#### 4. 配置事件订阅

```
在「事件与回调」页面：
1. 选择「使用长连接接收事件」（推荐）
2. 添加事件：im.message.receive_v1
```

#### 5. OpenClaw 配置

```yaml
# openclaw.yaml
channels:
  feishu:
    enabled: true
    app_id: cli_xxxxxxxx
    app_secret: your-app-secret
    # 长连接模式（无需公网）
    use_websocket: true
```

## 四、两种方案对比

### feishu-openclaw 桥接

| 项目 | 说明 |
|------|------|
| **进程** | 独立 Node.js 进程 |
| **媒体处理** | ✅ 完整支持 |
| **进程隔离** | ✅ Gateway 隔离 |

| **维护** | 官方插件，更新方便 |
| **适用** | 简单场景、快速上手 |

### 简介

飞书官方开源了 **Lark CLI**（MIT 协议），让你的 AI Agent 直接操作飞书：

- 🔍 搜文档、读妙记
- 📅 查日历
- ✉️ 发消息
- 📊 操作多维表格
- **2500+ API，11 个业务领域，19 个 AI Agent Skills**

### 安装

```bash
# 只需告诉 AI 一句话 + 点一个授权链接

```
