## 4.2 NapCat 安装

### 下载与安装

1. 下载 [NapCat QQ](https://github.com/NapNeko/NapCatQQ/releases)
2. 解压并运行 `NapCat.exe`

### 配置 WebSocket 服务器

1. 打开 NapCat WebUI: `http://localhost:6099/webui/network`
2. 点击 **新建** → **Websocket 服务器**
3. 配置：
   - **名称**: `OpenClaw-WS-Server`
   - **Host**: `127.0.0.1`
   - **Port**: `3001`
   - **Token**: (留空)
   - **消息格式**: `Array`
   - **心跳间隔**: `30000`
   - **启用**: ✅

### 配置 HTTP 服务器

1. 点击 **新建** → **HTTP 服务器**
2. 配置：
   - **名称**: `OpenClaw-HTTP-API`
   - **Host**: `127.0.0.1`
   - **Port**: `8080`
   - **Token**: `your-token` (自定义，需与 OpenClaw 配置一致)
   - **消息格式**: `Array`
   - **启用**: ✅

## 4.4 OpenClaw 配置

### 配置 openclaw.json

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "channels": {
    "onebot": {
      "enabled": true,
      "wsUrl": "ws://localhost:3001",
      "httpUrl": "http://localhost:8080",
      "accessToken": "your-token"
    }
  }
}
```

### QQ 机器人配置（BytePioneer-AI）

```bash
# 启用 QQ 渠道
openclaw config set channels.qqbot.enabled true
openclaw config set channels.qqbot.appId your-app-id
openclaw config set channels.qqbot.clientSecret your-app-secret

# 可选配置
openclaw config set channels.qqbot.markdownSupport true
openclaw config set channels.qqbot.c2cMarkdownDeliveryMode proactive-all
openclaw config set channels.qqbot.c2cMarkdownChunkStrategy markdown-block
```

### 完整配置示例

```json
{
  "channels": {
    "qqbot": {
      "enabled": true,
      "appId": "your-app-id",
      "clientSecret": "your-app-secret",
      "markdownSupport": true,
      "c2cMarkdownDeliveryMode": "proactive-all",
      "c2cMarkdownChunkStrategy": "markdown-block",
      "autoSendLocalPathMedia": false,
      "asr": {
        "enabled": true,
        "appId": "your-tencent-app-id",
        "secretId": "your-tencent-secret-id",
        "secretKey": "your-tencent-secret-key"
      }
    }
  }
}
```

### 配置选项

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `enabled` | boolean | true | 是否启用 |
| `appId` | string | 必填 | 应用 ID |
| `clientSecret` | string | 必填 | 应用密钥 |
| `markdownSupport` | boolean | false | 启用 Markdown 支持 |
| `c2cMarkdownDeliveryMode` | string | "passive" | 私聊 Markdown 发送方式 |
| `c2cMarkdownChunkStrategy` | string | "length" | Markdown 切分策略 |

### 私聊 Markdown 模式

| 模式 | 说明 |
|------|------|
| `passive` | 被动发送 |
| `proactive-table-only` | 有表格时主动发送 |
| `proactive-all` | 全部主动发送 |

### Markdown 切分策略

| 策略 | 说明 |
|------|------|
| `markdown-block` | 按标题/表格/引用等边界切分 |
| `length` | 按长度切分 |

### 功能说明

## 4.8 主动发送功能

### 已知目标

- 机器人见过的用户或群会被记录
- 可直接给指定对象发送文字或媒体
- 记录保存到 `~/.openclaw/qqbot/data/known-targets.json`

### 发送代码示例

```typescript
import { sendProactiveQQBotMessage } from "@openclaw-china/qqbot";

await sendProactiveQQBotMessage({
  cfg: {
    channels: {
      qqbot: {
        appId: "your-app-id",
        clientSecret: "your-app-secret",
      }
    }
  },
  to: "user:your-openid",
  text: "这是一条主动发送的消息"
});
```

### 目标格式

| 格式 | 示例 | 说明 |
|------|------|------|
| `user:<openid>` | `user:u123456` | 发送给用户 |
| `group:<groupid>` | `group:g123456` | 发送给群 |

## 4.10 Reasoning 内容过滤

> 🆕 v2026.6.5 新增功能
> 相关 PR: #89913, #90132
> 适用版本: v2026.6.5-beta.1+

### 功能说明

当 OpenClaw 使用支持 extended thinking 的模型（如 Claude Opus 4）时，模型响应会包含 `<thinking>` 或 `<reasoning>` 标签包裹的推理过程。在 QQ 频道中直接发送这些内容会：

- 🔴 暴露模型内部推理给终端用户
- 🔴 造成消息长度激增（推理内容可能远超实际回复）
- 🔴 影响用户体验（用户看到的是思考过程而非结论）

QQBot reasoning stripping 功能会自动识别并过滤掉这些推理标记，确保频道消息干净、专业。

### 工作原理

```
用户提问 → OpenClaw 处理 → 模型响应（含 <thinking>...</thinking>）
                                              ↓
                              QQBot Stripping 过滤器
                                              ↓
                              纯净回复 → 发送到 QQ 频道
```

**过滤前**:
```
<thinking>
让我分析这个问题：用户询问的是关于 OpenClaw 的安装问题。
首先需要确认操作系统版本，然后检查 Node.js 是否正确安装。
我应该提供分步骤的解决方案。
</thinking>

根据您的描述，我来帮您解决安装问题。请先检查以下几点...
```

**过滤后**:
```
根据您的描述，我来帮您解决安装问题。请先检查以下几点...
```

### 启用方式

此功能在 v2026.6.5+ 中**默认启用**，无需额外配置。

### 验证方法

1. 使用支持 thinking 的模型（如 `claude-opus-4-5$(20252010)`）
2. 发送一条需要推理的问题
3. 检查回复中是否仍有 `<thinking>` 标签

```bash
# 检查 QQBot 插件版本
openclaw plugins list | grep qqbot

# 查看 reasoning stripping 日志
openclaw logs --channel qqbot --grep "reasoning" --since "1h"
```

### 已知问题

| 问题 | 状态 | 说明 |
|------|------|------|
| 非标准 reasoning 标签 | ⚠️ 部分过滤 | 目前支持 `<thinking>` 和 `<reasoning>` |
| 嵌套 reasoning 内容 | ✅ 已支持 | 递归过滤嵌套标签 |

### 故障排除

**问题**: 仍然看到 `<thinking>` 内容在回复中

**解决方案**:
1. 确认 OpenClaw 版本为 v2026.6.5+
   ```bash
   openclaw --version  # 应显示 v2026.6.5 或更高
   ```
2. 检查模型配置是否正确
   ```yaml
   models:
     - name: claude-opus-4
       provider: anthropic
       reasoning:
         enabled: true  # 确保 reasoning 已启用
   ```
3. 查看日志确认 stripping 正常工作
   ```bash
   openclaw logs --channel qqbot --grep "strip" --since "30m"
   ```

## 4.11 故障排除

### 问题 1: WebSocket 连接失败

**现象**: `WebSocket closed: 1005` 或 `1006`

**解决方案**:
1. 检查 NapCat WebSocket 服务器是否启用
2. 确保端口 3001 未被占用
3. 检查防火墙设置

### 问题 2: 消息发送失败 (403 Forbidden)

**现象**: `token verify failed!`

**解决方案**:
1. 确认 NapCat HTTP 服务器的 Token 与 OpenClaw 配置一致
2. 检查 Token 是否正确传递

### 问题 3: 收到消息但无回复

**现象**: NapCat 日志显示接收但无发送

**解决方案**:
1. 检查 OpenClaw 日志中的 `deliver(final)` 记录
2. 查看是否有 HTTP API 调用错误
3. 确认 `accessToken` 配置正确
