## 9.7.1 Cron 任务概述

### 什么是 Cron

```
┌─────────────────────────────────────────────────────────────┐
│                    Cron 定时任务                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Cron = Gateway 内置调度器                                  │
│  ├── 定时执行任务                                            │
│  ├── 持久化存储                                              │
│  └── 支持多种交付方式                                        │
│                                                              │
│  存储位置：~/.openclaw/cron/                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Cron vs Heartbeat

| 特性 | Cron | Heartbeat |
|-----|------|-----------|
| 触发方式 | 定时 | 定时+消息 |
| 运行环境 | Gateway 内部 | Gateway 内部 |
| 持久化 | ✅ | ✅ |
| 消息触发 | ❌ | ✅ |

## 9.7.2 创建 Cron 任务

### CLI 基本命令

```bash
# 添加一次性提醒
openclaw cron add \
  --name "Reminder" \
  --at "2026-02-01T16:00:00Z" \
  --session main \
  --system-event "Reminder: check the docs" \
  --wake now \
  --delete-after-run
```

### 循环任务

```bash
# 每日早报
openclaw cron add \
  --name "Morning Brief" \
  --cron "0 7 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "Summarize overnight updates." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

## 9.7.3 绑定到特定 Agent

### 指定 Agent 执行

```bash
# 绑定到特定 Agent
openclaw cron add \
  --name "Ops Check" \
  --cron "0 6 * * *" \
  --session isolated \
  --message "Check ops queue status." \

  --name "Work Report" \
  --cron "0 9 * * 1-5" \
  --session isolated \
  --message "Generate weekly work report." \
  --agent work
```

```json
        }
      }
    ]
  }
```

## 9.7.4 任务类型

### 主会话任务 (System Event)

```bash
# 主会话任务 - 使用心跳上下文
openclaw cron add \
  --name "Daily Standup" \
  --cron "0 9 * * 1-5" \
  --session main \
  --system-event "Daily standup: summarize progress." \
  --wake now
```

特点：
- 使用主会话上下文
- 通过 heartbeat 运行
- 适合需要完整上下文的场景

### 隔离任务 (Isolated)

```bash
# 隔离任务 - 独立会话
openclaw cron add \
  --name "Background Scan" \
  --cron "0 */4 * * *" \
  --session isolated \
  --message "Scan for updates." \
  --announce
```

特点：
- 独立会话，无历史
- 不干扰主对话
- 适合后台任务

### 当前会话任务

```bash
# 绑定到当前会话
openclaw cron add \
  --name "Project Update" \
  --cron "0 10 * * *" \
  --session current \
  --message "Update project status."
```

## 9.7.5 定时表达式

### 常用 Cron 表达式

| 表达式 | 含义 |
|-------|------|
| `0 * * * *` | 每小时 |
| `0 0 * * *` | 每天午夜 |
| `0 7 * * *` | 每天早上 7 点 |
| `0 9 * * 1-5` | 工作日早上 9 点 |
| `0 */2 * * *` | 每 2 小时 |
| `0 0 * * 0` | 每周日 |

### 相对时间

```bash
# 20 分钟后
--at "20m"

# 2 小时后
--at "2h"

# 明天
--at "tomorrow 9am"
```

## 9.7.6 交付模式

### 公告模式 (Announce)

```bash
# 发送结果到渠道
openclaw cron add \
  --name "Daily Report" \
  --cron "0 18 * * *" \
  --session isolated \
  --message "Generate daily report." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

### Webhook 模式

> **🔔 新增功能 #94279**: Cron 任务现已支持 Webhook 交付模式，可在任务完成后自动将结果 POST 到指定 URL。

```bash
# 发送到 Webhook
openclaw cron add \
  --name "API Notify" \
  --cron "0 8 * * *" \
  --session isolated \
  --message "Check API status." \
  --webhook "https://your-server.com/webhook"
```

**Webhook 配置示例**:
```json
{
  "delivery": {
    "mode": "webhook",
    "to": "https://your-server.com/webhook",
    "bestEffort": false
  }
}
```

| 参数 | 说明 |
|------|------|
| `mode` | 必须为 `webhook` |
| `to` | Webhook 目标 URL |
| `bestEffort` | `true`=失败不重试，`false`=保证交付 |

**关联 PR**: [#94279](https://github.com/openclaw/openclaw/pull/94279)

### 无交付

```bash
# 仅内部执行
openclaw cron add \
  --name "Internal Task" \
  --cron "0 */2 * * *" \
  --session isolated \
  --message "Internal check." \
  --no-announce
```

> 📌 **执行流程图请参考 [9.2 定时任务](../09_automation/9.2_scheduled_tasks.md) 章节**

## 9.7.7 管理 Cron 任务

### deleteAfterRun 行为 (v2026.5.18+)

> ⚡ v2026.5.18 Bug 修复：PR #83933

**修复内容**：手动运行时跳过 `deleteAfterRun`

| 运行方式 | deleteAfterRun 行为 | 说明 |
|---------|---------------------|------|
| 自动触发 (Cron/Heartbeat) | ✅ 执行 | 按配置删除 |
| 手动运行 (`cron run`) | ❌ 跳过 | 保留任务供后续使用 |

**原因**：避免手动触发 cron 任务后误删除任务，影响后续自动执行。

**影响**：
- 手动测试 cron 任务时不会意外删除任务
- 生产环境任务按预期行为执行
- 提升手动调试体验

**关联 Issue**：[GitHub #83933](https://github.com/openclaw/openclaw/pull/83933)

---

### 常用命令

```bash
# 列出所有任务
openclaw cron list

# 查看任务详情
openclaw cron info <job-id>

# 手动运行任务（不会触发 deleteAfterRun）
openclaw cron run <job-id>

# ✨ 等待任务完成并获取 run-id (v2026.5.16+)
openclaw cron run <job-id> --wait

# 查看运行历史
openclaw cron runs --id <job-id>

# 编辑任务
openclaw cron edit <job-id> --message "New message"

# 删除任务
openclaw cron remove <job-id>

# 禁用/启用任务
openclaw cron disable <job-id>
openclaw cron enable <job-id>
```

## 9.7.7.1 cron run --wait 新用法 (v2026.5.16+)

> ⚡ v2026.5.16-beta.3 新增功能

### 传统用法 vs --wait 用法

| 用法 | 说明 | 适用场景 |
|------|------|----------|
| `cron run <job-id>` | 后台异步执行，立即返回 | 快速触发，不关心结果 |
| `cron run <job-id> --wait` | 等待执行完成，精确 run-id 过滤 | 需要获取执行结果、做后续处理 |

### 使用示例

```bash
# 传统方式：只触发，返回 job-id
$ openclaw cron run daily-report
✅ Triggered job: daily-report (run-id: None，后台执行)

# 新方式：等待完成，返回精确 run-id
$ openclaw cron run daily-report --wait
⏳ Waiting for job daily-report to complete...
✅ Job completed (run-id: crn_01JXYZ...)
```

### 典型使用场景

```bash
# 场景 1：Cron 任务链 - 等待完成后触发下一个任务
JOB1_ID=$(openclaw cron run data-sync --wait --json | jq -r '.runId')
openclaw cron run notify-team --context "job1=$JOB1_ID"

# 场景 2：获取执行结果用于后续处理
RESULT=$(openclaw cron run analysis-report --wait --json)
RUN_ID=$(echo $RESULT | jq -r '.runId')
STATUS=$(echo $RESULT | jq -r '.status')
if [ "$STATUS" == "success" ]; then
  echo "Report ready: $RUN_ID"
fi

# 场景 3：CI/CD 集成
openclaw cron run deploy-check --wait
exit_code=$?
if [ $exit_code -eq 0 ]; then
  echo "Deploy check passed"
fi
```

### 注意事项

1. **超时设置**：默认等待 30 分钟，可通过 `--timeout` 调整
2. **后台任务**：`--wait` 会保持连接，适合短时任务
3. **run-id 用途**：
   - 精确追踪单次执行
   - 日志关联
   - 执行结果查询

```bash
# 带超时的等待
openclaw cron run long-task --wait --timeout 10m

# 查看特定 run-id 的日志
openclaw logs --run-id crn_01JXYZ...
```

**关联 Issue**：[GitHub #82441](https://github.com/openclaw/openclaw/issues/82441)

## 9.7.10 故障排除

### Cron 工具策略不生效

> **相关修复**: PR #91499
> **影响版本**: v2026.6.0 ~ v2026.6.4
> **修复版本**: v2026.6.5+

#### 问题描述

定时任务执行时发现缺少预期的工具限制，配置的 `toolsAllow` 或 `toolsDeny` 未生效。

#### 原因分析

Cron scheduled turn 的 tool policy 未被正确保留，导致定时任务执行时使用了默认工具策略而非配置的自定义策略。

#### 解决方案

**升级到 v2026.6.5+**

```bash
brew upgrade openclaw
```

**或手动配置修复**：


升级后，在 cron 任务配置中显式指定 tool policy：

```yaml
jobs:
  - jobId: "my-task"
    name: "My Task"
    schedule:
      kind: "cron"
      expr: "0 6 * * *"
      tz: "Asia/Shanghai"
    sessionTarget: "isolated"
    payload:
      kind: "agentTurn"
      message: "Check system status."
      toolsAllow:
        - exec
        - read
    delivery:
      mode: "announce"
```

#### 验证方法

1. 升级后重启 Gateway
2. 运行 cron 任务并检查日志：
   ```bash
   openclaw cron runs --id <job-id>
   ```
3. 确认工具限制已生效
