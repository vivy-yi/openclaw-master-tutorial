# Skills 与 Plugins 对比

> 理解 Skills 和 Plugins 的区别，选择合适的扩展方式。

---

## 一、核心区别

| 维度 | Skills | Plugins |
|------|--------|---------|
| **定位** | 轻量级任务封装 | 运行时扩展 |
| **定义方式** | SKILL.md 文件 | TypeScript 模块 |
| **复杂度** | 低（模板化） | 高（Hook 系统） |
| **执行时机** | 对话中触发 | 运行时加载 |
| **工具注册** | 通过 skill 定义 | 直接注册 |
| **Hook 支持** | ❌ | ✅ |
| **生命周期** | 一次性 | 持久化 |

---

## 二、架构对比图

```
用户请求
    ↓
┌─────────────────────────────────────────┐
│  Skills 层                              │

└─────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────┐

│  - Hook 系统                           │
│  - 工具注册                           │
└─────────────────────────────────────────┘
    ↓
运行时
```

---

## 三、选择指南

| 场景 | 推荐 |
|------|------|
| 简单任务封装（笔记、天气、搜索） | ✅ Skills |
| 需要 Hook 事件监听 | ❌ 必须用 Plugins |
| 需要注册 Tool 到 Agent | 两者都可以 |
| 需要深度系统集成 | Plugins |
| 快速原型验证 | Skills |
| 生产级扩展 | Plugins |

---

## 四、什么时候用哪个？

### 用 Skills 的场景

```markdown
# 1. 封装简单任务
使用 skill: weather 查询天气

# 2. 快速封装工具
使用 skill: obsidian 同步笔记

# 3. 轻量级自动化
使用 skill: taskflow 创建工作流
```

### 用 Plugins 的场景

```javascript
// 1. 需要 Hook 事件
registerHook({
  event: 'onAgentTurn',
  handler: (ctx) => { ... }
});

// 2. 需要深度集成
registerTool({
  name: 'deep-search',
  handler: async (query) => { ... }
});

// 3. 需要持久化运行
export function activate() { ... }
```

## 五、组合使用

两者可以组合：

```
Plugins 提供底层能力
    ↓
Skills 封装高级任务
    ↓
用户通过对话触发
```
