## 3.2 浏览器控制基础

### 3.2.1 action: status - 检查浏览器状态

```json
{
  "action": "status"
}
```

返回：浏览器运行状态、CDP 连接信息、可用配置文件

### 3.2.2 action: start - 启动浏览器

```json
{
  "action": "start",
  "profile": "openclaw"
}
```

### 3.2.3 action: stop - 停止浏览器

```json
{
  "action": "stop"
}
```

### 3.2.4 action: doctor - 诊断问题

```json
{
  "action": "doctor"
}
```

检测：浏览器可执行文件、配置文件、CDP 连接、Chrome MCP 配置

### 3.2.5 action: profiles - 管理配置文件

```json
{
  "action": "profiles"
}
```

## 3.3 页面导航操作

### 3.3.1 action: open - 打开新标签页

```json
{
  "action": "open",
  "url": "https://docs.openclaw.ai",
  "label": "docs"  // 稳定句柄
}
```

**label** 参���用于后续稳定引用该标签页

### 3.3.2 action: navigate - 导航当前标签页

```json
{
  "action": "navigate",
  "targetId": "docs",  // 使用 label 或 tabId
  "url": "https://example.com"
}
```

### 3.3.3 action: close - 关闭标签页

```json
{
  "action": "close",
  "targetId": "docs"
}
```

### 3.3.4 action: snapshot - 获取 DOM

```json
{
  "action": "snapshot",
  "targetId": "docs",
  "refs": "aria",  // 持久引用
  "maxChars": 50000
}
```

**推荐使用 `refs="aria"` 获取 Playwright 持久引用（如 `ax12`）**

## 3.4 交互操作（act action）

### 3.4.1 act 支持的 12 种 kind

| kind | 说明 | 示例 |
|------|------|------|
| `click` | 点击元素 | `{"kind": "click", "ref": "ax12"}` |
| `clickCoords` | 坐标点击 | `{"kind": "clickCoords", "x": 100, "y": 200}` |
| `type` | 输入文本 | `{"kind": "type", "ref": "input1", "text": "hello"}` |
| `press` | 按键 | `{"kind": "press", "key": "Enter"}` |
| `hover` | 悬停 | `{"kind": "hover", "ref": "menu1"}` |
| `drag` | 拖拽 | `{"kind": "drag", "startRef": "src", "endRef": "target"}` |
| `select` | 下拉选择 | `{"kind": "select", "ref": "select1", "values": ["option1"]}` |
| `fill` | 填写表单 | `{"kind": "fill", "ref": "input1", "text": "value"}` |
| `resize` | 调整元素 | `{"kind": "resize", "width": 800, "height": 600}` |
| `wait` | 等待条件 | `{"kind": "wait", "selector": "#loaded", "state": "visible"}` |
| `evaluate` | 执行 JS | `{"kind": "evaluate", "fn": "return document.title"}` |
| `close` | 关闭元素 | `{"kind": "close", "ref": ".modal"}` |

### 3.4.2 act 示例

```json
{
  "action": "act",
  "targetId": "docs",
  "request": {
    "kind": "click",
    "ref": "ax12"
  }
}
```

```json
{
  "action": "act",
  "targetId": "docs",
  "request": {
    "kind": "fill",
    "ref": "search-input",
    "text": "OpenClaw tutorial"
  }
}
```

```json
{
  "action": "act",
  "targetId": "docs",
  "request": {
    "kind": "wait",
    "selector": "#results",
    "state": "visible",
    "timeoutMs": 30000
  }
}
```

## 3.6 工具配置

### 全局配置

```json5
{
  tools: {
    allow: ["group:web"]
  }
}
```

### 按 Agent 配置

```json5
{
  agents: {
    list: [
      {
        id: "scraper",
        tools: {
          allow: ["group:web"],
          deny: ["browser_evaluate"]  // 禁止执行 JS
        }
      }
    ]
  }
}
```

## 3.8 错误处理与恢复

### Stale Ref 恢复

当元素引用失效时：

1. 重新 snapshot 获取新引用
2. 使用新的 ref 重试
3. 如果进入阻塞状态，报告问题

```json
{
  "action": "snapshot",
  "targetId": "docs",
  "refs": "aria"
}
```

### 异常处理

```bash
try {
  browser action 失败
  → 记录错误
  → screenshot 保存现场
  → 报告具体问题（如需要登录、验证码等）
} catch (error) {
  print("Browser error: " + error.message)
}
```

## 3.9 安全建议

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                       浏览器自动化安全建议                                      │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  ✅ 推荐做法：                                                                  │
│  1. 限制可访问的域名（ssrfPolicy）                                         │
│  2. 禁用危险操作（evaluate 执行未验证脚本）                               │
│  3. 设置合理的超时（actionTimeoutMs）                                      │
│  4. 处理意外弹窗（dialog action）                                          │
│  5. 使用 label 管理稳定标签页                                               │
│                                                                                 │
│  ❌ 避免做法：                                                                  │
│  1. 在生产环境无限制访问互联网                                         │
│  2. 执行未验证的用户提供的脚本                                           │
│  3. 允许访问敏感域名（内部服务）                                        │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 域名限制配置

```json5
{
  browser: {
    ssrfPolicy: {
      enabled: true,
      allowPrivateNetwork: false,
      allowedHosts: ["docs.openclaw.ai"]
    }
  }
}
```
