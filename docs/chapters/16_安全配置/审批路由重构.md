## 一、功能概述

### 1.1 变更背景

OpenClaw 的审批系统原本存在多个分散的审批逻辑实现，本次的重构将**原生审批路由门（Native Approval Route Gates）**进行统一，共享审批逻辑。

### 1.2 重构目标

| 目标 | 说明 |
|------|------|
| **代码复用** | 统一审批逻辑，消除重复代码 |
| **安全性提升** | 集中化管理审批流程 |
| **一致性** | 所有审批场景使用相同逻辑 |
| **可维护性** | 便于后续审计和扩展 |

## 四、配置说明

### 4.1 启用审批门

```yaml
# openclaw.yaml
security:
  approval:
    enabled: true
    routes:
      - route: file_delete
        require_approval: true
      - route: external_send
        require_approval: true
```

### 4.2 审批日志

```bash
# 查看审批日志
tail -f ~/.openclaw/logs/openclaw.log | grep -i "approval"

# 查看安全日志
tail -f ~/.openclaw/logs/openclaw.log | grep -i "security"
```
