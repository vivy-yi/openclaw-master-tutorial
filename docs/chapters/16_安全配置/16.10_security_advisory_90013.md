## 漏洞描述

`scripts/install.sh` 中有 4 条代码路径通过 `download_file` 下载远程脚本，然后用 `bash "$tmp"` 执行，**完全没有任何验证**。

虽然 HTTPS/TLS1.2 保护了传输层，但以下情况会被静默执行：

- **截断的下载文件**
- **HTML 错误页面**（如 CDN 403 响应）
- **空响应内容**

**受影响路径**（共4条）：

| 路径 | 下载来源 | 用途 |
|------|---------|------|
| `run_remote_bash` | `raw.githubusercontent.com` | Homebrew 安装脚本 |
| NodeSource apt/deb | `deb.nodesource.com/setup_*.x` | Node.js apt 安装 |
| NodeSource dnf/rpm | `rpm.nodesource.com/setup_*.x` | Node.js dnf 安装 |
| NodeSource yum/rpm | `rpm.nodesource.com/setup_*.x` | Node.js yum 安装 |

**对比**：`bootstrap_gum_temp` 函数对下载内容做了 **SHA256 校验和验证**——说明项目中已有安全下载的示范，但安装脚本的 4 条路径完全遗漏了这一保护。

**失败行为**：

- `run_quiet_step` 的 `if` 包装器会**抑制 `set -e`**，导致下载失败被静默忽略
- 不执行任何结构性验证
- 下载失败时**不会中止**，而是继续执行空/错误内容

## 影响范围

| 安装方式 | 影响 | 严重度 |
|---------|------|--------|
| 使用官方 install.sh 脚本 | 🔴 受影响 | 可能执行截断/错误内容 |
| 使用 npm/pip 包管理器安装 | 🟢 不受影响 | 走标准包管理器 |
| 手动下载并检查后执行 | 🟢 缓解 | 手动验证可发现异常 |
| 网络不稳定环境 | 🔴 风险更高 | 截断响应概率更高 |

## 漏洞验证

### 安装前验证命令

```bash
# 手动下载并检查内容再执行
curl -fsSL "https://raw.githubusercontent.com/openclaw/openclaw/main/scripts/install.sh" -o /tmp/install_check.sh
head -5 /tmp/install_check.sh  # 确认是 shell 脚本
wc -l /tmp/install_check.sh    # 确认非空

# 确认输出以 #!/ 开头（如 #!/bin/bash）
```

### 安装后检查

如果刚安装完 OpenClaw 出现异常行为，检查是否有意外的 cron 任务或 SSH 密钥添加：

```bash
# 检查 cron 任务
crontab -l 2>/dev/null

# 检查 SSH authorized_keys
cat ~/.ssh/authorized_keys 2>/dev/null

# 检查异常网络连接
lsof -i 2>/dev/null | head -20
```

### 方案一：等待上游修复（推荐）


### 方案二：安装前手动验证

```bash
# 1. 下载安装脚本
curl -fsSL "https://raw.githubusercontent.com/openclaw/openclaw/main/scripts/install.sh" -o /tmp/openclaw_install.sh

# 2. 检查非空
wc -l /tmp/openclaw_install.sh
# 确保行数 > 0

# 3. 检查 shebang
head -1 /tmp/openclaw_install.sh
# 确保输出为 #!/bin/bash 或类似

# 4. 如果以上都正常，再执行
bash /tmp/openclaw_install.sh
```

### 方案三：使用 npm 安装（无此漏洞）

```bash
# 使用 npm 全局安装（不受此漏洞影响）
npm install -g openclaw

# 验证安装
openclaw --version
```
