# 🔴 安全通告：Hook ingress token 绕过密码模式 Gateway 认证

> ⚠️ 发布日期：2026-05-20
> 🔴 严重度：高
> 🦉 来源：墨客-深度调研报告 2026-05-20

---

## 概述

**漏洞 ID**: #84337 / #84338

**漏洞类型**: 认证绕过 (Authentication Bypass)

**影响版本**: v2026.5.18

**影响系统**: 使用密码模式 (password-mode) Gateway 的所有用户

**状态**: 🟠 OPEN | **修复 PR 已提交**: [#84338](https://github.com/openclaw/openclaw/pull/84338)

---

## 漏洞描述

在 v2026.5.18 中，Hook ingress token 可以**绕过密码模式 Gateway 认证**。攻击者如果获取了 Hook ingress token，无需知道 Gateway 密码即可访问 Gateway API。

```
正常认证流程（密码模式）：
用户 → Gateway (密码验证) → API 访问 ✅

漏洞认证流程（Hook token 绕过）：
Hook ingress → Gateway (跳过密码验证) → API 访问 ✅❌
```

---

## 影响范围

| 配置 | 影响 | 严重度 |
|------|------|--------|
| Gateway 启用密码模式 + 配置 Hook | 🔴 严重 | 认证完全失效 |
| Gateway 启用密码模式 (无 Hook) | 🟢 不受影响 | 无 |
| Gateway 使用其他认证模式 | 🟢 不受影响 | 无 |

---

## 漏洞验证

### 检查是否受影响

```bash
# 1. 检查 Gateway 认证模式
openclaw gateway config | grep -i "auth\|password\|mode"

# 2. 检查是否配置了 Hook ingress
openclaw config get hooks.ingress 2>/dev/null

# 3. 检查 Gateway 日志中是否有异常认证
openclaw gateway logs | grep -i "hook\|token\|auth" | head -50
```

### 如果您使用 Hook ingress

```bash
# 临时措施：禁用 Hook ingress（如果不需要）
openclaw config set hooks.ingress.enabled false
openclaw gateway restart

# 或：使用更严格的访问控制
# 限制 Hook ingress token 的访问范围
```

---

## 修复方案

### 方案一：等待官方修复（推荐）

关注 PR [#84338](https://github.com/openclaw/openclaw/pull/84338)，合入后升级。

```bash
# 等待官方发布修复版本
# 升级命令：
openclaw update
```

### 方案二：禁用 Hook ingress（临时缓解）

```bash
# 编辑配置文件 (~/.openclaw/config.yaml)
# 禁用 hooks.ingress

hooks:
  ingress:
    enabled: false  # 临时禁用

# 重启 Gateway
openclaw gateway restart
```

### 方案三：使用非密码认证模式

如果业务允许，可以切换到其他认证模式：

```bash
# 查看可用认证模式
openclaw gateway config --help

# 切换认证模式（如 API key 模式）
openclaw config set gateway.auth.mode "api-key"
openclaw gateway restart
```
