## 一、漏洞概述

### 1.1 问题描述

OpenClaw 的 Secret resolver（密钥解析系统）在运行时无法将 **file-source secrets**（从文件读取的密钥）正确注入到 **ElevenLabs TTS provider**。

这意味着：
- 即使通过文件配置了 ElevenLabs API key，TTS 功能也无法正常工作
- 依赖文件密钥注入的自动化部署场景存在安全隐患

### 1.2 技术根因

```
Secret Resolver 支持多种 secret 来源：
├── env:        环境变量
├── file:       从文件读取（file-source）← 问题所在
├──1password:1Password vault
└── ghsecret:   GitHub Secrets

当前问题：file-source secrets 在运行时未正确传递给 ElevenLabs provider
```

**受影响路径**：
- `providers.tts.elevenlabs.apiKey` 使用 `file:` 前缀时
- 容器化/自动化部署场景

### 1.3 影响范围

| 影响项 | 详情 |
|--------|------|
| **受影响组件** | ElevenLabs TTS provider |
| **触发条件** | 使用 `file:` 前缀配置 ElevenLabs API key |
| **影响程度** | TTS 功能完全不可用（即使 key 正确配置） |
| **安全影响** | 机密信息可能回退到明文环境变量，增加泄露风险 |
| **影响版本** | 所有使用 ElevenLabs TTS 的版本 |

## 二、临时缓解方案

### 2.1 方案一：使用环境变量（推荐）

```bash
# 在 shell 环境或 docker-compose 中设置环境变量
export ELEVENLABS_API_KEY=sk_your_api_key_here
```

```yaml
# openclaw.yaml
providers:
  tts:
    elevenlabs:
      apiKey: ${ELEVENLABS_API_KEY}  # 使用 env: 前缀或直接引用
```

> ✅ **优点**：最简单可靠，官方推荐方式  
> ⚠️ **缺点**：key 以明文形式存在于环境变量中

### 2.2 方案二：使用 1Password（安全推荐）

```bash
# 安装 openclaw-secret-1password插件
openclaw plugins install secret-1password

# 配置 1Password vault
openclaw config set secrets.provider=1password
```

```yaml
# openclaw.yaml
providers:
  tts:
    elevenlabs:
      apiKey: ${op://vault/openclaw/eleveLabs-api-key/credential}
```

> ✅ **优点**：密钥安全存储，支持审计
> ⚠️ **缺点**：需要1Password 订阅和插件配置

### 2.3 方案三：使用 GitHub Secrets（CI/CD 场景）

```yaml
# .github/workflows/openclaw.yml
env:
  ELEVENLABS_API_KEY: ${{ secrets.ELEVENLABS_API_KEY }}
```

> ✅ **优点**：与 CI/CD 流水线集成良好  
> ⚠️ **缺点**：仅适用于 GitHub Actions 环境

## 三、配置验证

### 3.1 验证 TTS 配置

```bash
# 运行 doctor 检查 TTS 配置
openclaw gateway doctor --check tts

# 测试 TTS 功能
openclaw tts test --provider elevenlabs
```

### 3.2 正确配置示例

```yaml
# openclaw.yaml（完整配置）
gateway:
  host: 0.0.0.0
  port: 18789

providers:
  tts:
    elevenlabs:
      # ✅正确方式 1：环境变量引用
      apiKey: ${ELEVENLABS_API_KEY}
      
      # ✅ 正确方式 2：1Password 引用
      # apiKey: ${op://vault/openclaw/elevenvlabs-api-key/credential}
      
      #❌ 受影响方式：file-source（当前漏洞）
      # apiKey: ${file:/run/secrets/elevenvlabs.key}
      
      voice: en-US-Neural
      model: eleven_multilingual_v2
      stability: 0.5
```

### 3.3 检查当前配置

```bash
# 查看当前 TTS provider 配置（脱敏输出）
openclaw config show providers.tts

# 验证 API key 是否正确加载
openclaw gateway doctor --check secrets
```
