# SecretRef 认证迁移故障排除

> **更新时间**: 2026-05-26
> **适用版本**: v2026.4.29+ / v2026.5.x
> **数据来源**: GitHub Issue #86756

---

## 一、问题概述

### 1.1 问题描述

| 项目 | 说明 |
|------|------|
| Issue | [#86756](https://github.com/openclaw/openclaw/issues/86756) |
| 严重程度 | 🔴 **P1 - 数据丢失风险** |
| 标签 | `impact:data-loss` `clawsweeper:needs-maintainer-review` |
| 影响范围 | auth-profiles.json 中的 OAuth 配置 |

### 1.2 故障现象

```
SecretRef migration silently drops OAuth profiles from auth-profiles.json
```

SecretRef 迁移过程中，**静默丢失**了 auth-profiles.json 中的 OAuth 配置，没有错误提示。

### 1.3 影响

| 影响类型 | 说明 |
|----------|------|
| 🔴 数据丢失 | OAuth profiles 在迁移后消失 |
| 🔴 认证失败 | 依赖 OAuth 的渠道/提供商无法工作 |
| ⚠️ 静默失败 | 无错误提示，难以发现 |

## 三、问题诊断

### 3.1 检查是否受影响

```bash
# 1. 检查 auth-profiles.json 是否存在
cat ~/.openclaw/auth-profiles.json

# 2. 检查 OAuth 配置是否存在
openclaw config get --key auth.profiles | grep -A5 oauth

# 3. 检查 SecretRef 迁移日志
openclaw logs | grep -i "secretref\|migration\|oauth"

# 4. 验证 OAuth 状态
openclaw auth verify --profile oauth
```

### 3.2 检查迁移状态

```bash
# 检查迁移是否已完成
openclaw migration status

# 检查 SecretRef 配置
openclaw secrets list --type secretref

# 检查传统配置
openclaw secrets list --type legacy
```

## 四、临时解决方案

### 4.1 备份当前配置

```bash
# 备份 auth-profiles.json
cp ~/.openclaw/auth-profiles.json ~/.openclaw/auth-profiles.json.backup.$(date +%Y%m%d)

# 备份所有认证配置
openclaw config export --all --output ~/openclaw-auth-backup.yaml
```

### 4.2 手动恢复 OAuth 配置

```bash
# 方案1：从备份恢复
cp ~/.openclaw/auth-profiles.json.backup.20260526 ~/.openclaw/auth-profiles.json
openclaw gateway restart

# 方案2：禁用 SecretRef 迁移，使用传统配置
# 在 openclaw.yaml 中添加
migration:
  secretRef: false

# 方案3：使用环境变量替代
export DISCORD_CLIENT_ID=xxx
export DISCORD_CLIENT_SECRET=xxx
openclaw gateway restart
```

### 4.3 防止迁移执行

```bash
# 阻止 SecretRef 迁移执行
openclaw migration block --type secretref

# 检查迁移阻止状态
openclaw migration status --detail
```

## 五、永久解决方案

### 5.1 等待官方修复

Issue #86756 状态为 `needs-maintainer-review`，建议：
- 关注该 Issue 获得修复通知
- 等待官方发布修复版本

### 5.2 使用 SecretRef API

```bash
# 创建 SecretRef 替代传统配置
openclaw secrets create \
  --name discord-oauth \
  --type secretref \
  --value "secret://discord/oauth-token"

# 验证 SecretRef
openclaw secrets verify --name discord-oauth
```

### 5.3 配置示例

```yaml
# openclaw.yaml - 使用 SecretRef
secrets:
  discord-oauth:
    type: secretref
    uri: "secret://discord/oauth-token"
  
  slack-oauth:
    type: secretref
    uri: "secret://slack/oauth-token"

channels:
  discord:
    auth:
      type: oauth
      secretRef: discord-oauth
      
  slack:
    auth:
      type: oauth
      secretRef: slack-oauth
```
