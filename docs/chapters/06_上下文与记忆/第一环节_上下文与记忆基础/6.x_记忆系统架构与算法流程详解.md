## 1. 记忆系统全景架构

### 1.1 两套分层体系

OpenClaw 记忆系统有**两套相互关联的分层体系**：

| 体系 | 关注点 | 层数 |
|------|--------|------|
| **数据/日志分层** | 数据如何存储、蒸馏、遗忘 | 六层 |
| **运行时 Slot 分层** | 运行时各插件如何协作 | 四层 |

两套体系通过 **memory_recall** 和 **active-memory** 钩子连接：

```
数据层（用户产生数据）
    │
    │ 写入
    ▼
┌─────────────────────────────────────────────────────────────┐
│              六层数据/日志架构                                │
│  Layer 1: 每日日志 → Layer 2: MEMORY.md → ...              │
└─────────────────────────────────────────────────────────────┘
                    │
                    │ memory_recall / active-memory 注入
                    ▼
┌─────────────────────────────────────────────────────────────┐
│              四层运行时 Slot 架构                             │
│  Layer0: contextEngine → Layer1: before_agent →            │
│  Layer2: memory slot → Layer3: wiki slot                    │
└─────────────────────────────────────────────────────────────┘
                    │
                    │ LLM 看到的结果
                    ▼
               对话上下文
```

## 5. lossless-claw 核心算法

### 5.1 Compaction 算法流程

```
┌─────────────────────────────────────────────────────────────┐
│                   Compaction 触发条件                          │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│   条件 1: Token 超限                                         │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  当前 token 数 > 模型上限 × 0.9                      │   │
│   └─────────────────────────────────────────────────────┘   │
│                      OR                                       │
│   条件 2: 时间阈值                                           │
│   ┌─────────────────────────────────────────────────────┐   │
│   │  距上次 compaction > 30 分钟                         │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
                         │
                         ▼
┌─────────────────────────────────────────────────────────────┐
│                   Compaction 执行流程                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Step 1: 选择待压缩区间                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  [D0][D0][D0][D0][D0][D0][D0][Fresh Tail]           │   │
│  │        ↑                                             │   │
│  │   选择最早的 D0 块作为压缩目标                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                    │
│                         ▼                                    │
│  Step 2: 生成摘要（D0 → D1）                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  输入：D0 块中的所有原始消息                           │   │
│  │  模型：summarizeModel（默认同主模型）                  │   │
│  │  输出：压缩后的摘要文本                                │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                    │
│                         ▼                                    │
│  Step 3: 更新 DAG 结构                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  ┌─────┐    ┌─────┐    ┌─────────┐                 │   │
│  │  │ D1  │───▶│ D1  │───▶│  Fresh  │                 │   │
│  │  │摘要1│    │摘要2│    │  Tail   │                 │   │
│  │  └─────┘    └─────┘    └─────────┘                 │   │
│  │                              ▲                        │   │
│  │                              │                        │   │
│  │         D0 块已被替换为 D1    │                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                         │                                    │
│                         ▼                                    │
│  Step 4: 验证 token 减少                                     │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  如果 token 仍超限，继续压缩下一个 D0 块               │   │
│  │  直到 token 在安全范围内                             │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 6. memory-lancedb-pro 核心算法

### 6.1 混合检索三阶段架构

```
┌─────────────────────────────────────────────────────────────┐
│                 混合检索三阶段架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户查询: "OpenClaw 配置问题"                               │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Stage 1: Dense Vector Search (向量检索)              │  │
│  │  ┌────────────────────────────────────────────────┐ │  │
│  │  │  query → embedding → 1536-dim vector           │ │  │
│  │  │       │                                        │ │  │
│  │  │       ▼                                        │ │  │
│  │  │  LanceDB ANN index                            │ │  │
│  │  │       │                                        │ │  │
│  │  │       ▼                                        │ │  │
│  │  │  Top-50 candidates (by similarity)             │ │  │
│  │  └────────────────────────────────────────────────┘ │  │
│  └──────────────────────┬───────────────────────────────┘  │
│                         │                                   │
│  ┌──────────────────────▼───────────────────────────────┐  │
│  │  Stage 2: Sparse Keyword Search (关键词检索)           │  │
│  │  ┌────────────────────────────────────────────────┐ │  │
│  │  │  query → BM25 / TF-IDF                        │ │  │
│  │  │       │                                        │ │  │
│  │  │       ▼                                        │ │  │
│  │  │  Inverted index                               │ │  │
│  │  │       │                                        │ │  │
│  │  │       ▼                                        │ │  │
│  │  │  Top-50 candidates (by BM25 score)           │ │  │
│  │  └────────────────────────────────────────────────┘ │  │
│  └──────────────────────┬───────────────────────────────┘  │
│                         │                                   │
│  ┌──────────────────────▼───────────────────────────────┐  │
│  │  Stage 3: RRF Fusion (融合)                           │  │
│  │  ┌────────────────────────────────────────────────┐ │  │
│  │  │  RRF = Σ 1/(k + rank_i)                        │ │  │
│  │  │  k = 60 (常量)                                 │ │  │
│  │  │                                                │ │  │
│  │  │  结果1: vector rank=3  → 1/(60+3) = 0.0159    │ │  │
│  │  │  结果2: BM25  rank=1  → 1/(60+1) = 0.0164    │ │  │
│  │  │                                                │ │  │
│  │  │  两者都命中 → 累加得分                          │ │  │
│  │  └────────────────────────────────────────────────┘ │  │
│  └──────────────────────┬───────────────────────────────┘  │
│                         │                                   │
│                         ▼                                   │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Stage 4: Cross-encoder Rerank (精排)                  │  │
│  │  ┌────────────────────────────────────────────────┐ │  │
│  │  │  [query, document] → cross-encoder           │ │  │
│  │  │       │                                        │ │  │
│  │  │       ▼                                        │ │  │
│  │  │  重排 top-20                                   │ │  │
│  │  │       │                                        │ │  │
│  │  │       ▼                                        │ │  │
│  │  │  最终 top-10 结果                              │ │  │
│  │  └────────────────────────────────────────────────┘ │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.2 Weibull 衰减遗忘算法

```
┌─────────────────────────────────────────────────────────────┐
│                 Weibull 衰减遗忘模型                         │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  衰减函数:                                                   │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  importance(t) = importance_0 ×                     │   │
│  │                   exp(-(t/λ)^k)                     │   │
│  │                                                     │   │
│  │  λ = 特征衰减率（可配置，默认 30 天）                 │   │
│  │  k = 形状参数（控制衰减速度）                         │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                              │
│  衰减曲线示意:                                               │
│  importance                                                  │
│    1.0 │●                                                   │
│        │  ●                                                 │
│    0.7 │    ●                                               │
│        │      ●                                             │
│    0.5 │        ●                                           │
│        │          ●    ●    ●                               │
│    0.3 │                  ●                                 │
│        │                      ●                             │
│    0.1 │                          ●────────────            │
│        └────────────────────────────────────────            │
│              0    30    60    90   120   150  days          │
│                     λ=30, k=2                              │
│                                                              │
│  🔑 特点:                                                    │
│  - 初期衰减慢（重要记忆容易被强化）                           │
│  - 后期衰减快（长期不用的记忆逐渐消退）                       │
│  - 触发阈值时自动归档或删除                                  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 6.3 Smart Extraction 智能提取流程

```
┌─────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────┤
│                                                              │
│  输入: 对话消息列表                                          │

│  │                                                      │   │

│  │  类型 3: 偏好 (Preference)                           │   │
│  │    "用户喜欢重口味麻辣烫"                             │   │
│  │                                                      │   │
│  │  类型 4: 模式 (Pattern)                              │   │
│  │    "用户经常在晚上点外卖"                             │   │
│  │                                                      │   │
│  │  类型 5: 实体 (Entity)                               │   │
│  │    "墨星是内容运营 Agent"                             │   │
│  │                                                      │   │
│  │  类型 6: 反思 (Reflection)                           │   │

│                      ▼                                    │
│  Step 2: 重要性评分                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  评分维度:                                          │   │
│  │  - 对用户的重要性 (impact)                          │   │
│  │  - 频率 (frequency)                                │   │
│  │  - 稳定性 (stability) - 是否会变化                  │   │
│  │  - 新颖性 (novelty)                                 │   │
│  │                                                      │   │
│  │  score = w1×impact + w2×freq + w3×stability         │   │
│  │         + w4×novelty                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                      │                                    │
│                      ▼                                    │
│  Step 3: 知识蒸馏                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  if score > 0.8:                                   │   │
│  │      → 立即写入 MEMORY.md                           │   │
│  │  elif score > 0.5:                                 │   │
│  │      → 写入 pending 队列                           │   │

│  │  else:                                              │   │
│  │      → 丢弃                                        │   │
│  └─────────────────────────────────────────────────────┘   │
│                      │                                    │
│                      ▼                                    │
│  输出: 已提取的记忆条目                                      │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 9. 关键配置参数

### 9.1 lossless-claw 配置

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `summarizeModel` | 同主模型 | 摘要生成模型 |

| `maxDepth` | 3 | 最大摘要层级 (D0-D1-D2-D3) |

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `decayLambda` | 30 (天) | Weibull 衰减率 |
