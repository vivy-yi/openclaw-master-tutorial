```
┌─────────────────────────────────────────────────────────────┐
│                   Mem0 三层记忆体系                          │
├─────────────────────────────────────────────────────────────┤

│  │              Agent Memory (最高层)                   │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  跨用户、跨会话的 Agent 级别知识              │   │   │

│  └─────────────────────────────────────────────────────┘   │
│                           ↑                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │              Session Memory (中间层)                │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  单个会话内的上下文记忆                       │   │   │
│  │  │  例: "用户正在询问持仓数据"                    │   │   │
│  │  │  例: "当前对话主题是股票分析"                  │   │   │

## 4. ADD 流程 (记忆写入)

### 4.1 ADD-Only 单通道提取

Mem0 的新算法采用 **ADD-Only** 策略，记忆只累积不覆盖：

```
┌─────────────────────────────────────────────────────────────┐
│                   ADD 流程                                  │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  输入: 对话消息列表                                          │
│                                                              │
│  Step 1: 消息预处理                                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  messages = [                                       │   │
│  │    {"role": "user", "content": "帮我点个外卖"},    │   │
│  │    {"role": "assistant", "content": "好的，想吃什么？"}│   │
│  │    {"role": "user", "content": "麻辣烫"}           │   │
│  │  ]                                                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                    │                                        │
│                    ▼                                        │
│  Step 2: 单次 LLM 调用提取                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  prompt = "从以下对话中提取关键信息...              │   │
│  │           返回结构化的记忆条目"                      │   │
│  │                                                      │   │
│  │  ┌─────────────────────────────────────────────┐   │   │
│  │  │  LLM 一次性输出所有记忆                      │   │   │
│  │  │  - 不需要 UPDATE                            │   │   │
│  │  │  - 不需要 DELETE                            │   │   │
│  │  │  - 只做 ADD                                  │   │   │
│  │  └─────────────────────────────────────────────┘   │   │
│  └─────────────────────────────────────────────────────┘   │
│                    │                                        │
│                    ▼                                        │
│  Step 3: 实体链接                                           │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  对于每条提取的记忆:                                 │   │
│  │  1. 提取实体列表                                     │   │
│  │  2. 查询已有实体库                                   │   │
│  │  3. 建立实体关联                                     │   │
│  │                                                      │   │
│  │  例: "麻辣烫" → 与已有实体 "重口味" 关联             │   │
│  └─────────────────────────────────────────────────────┘   │
│                    │                                        │
│                    ▼                                        │
│  Step 4: 向量化 + 存储                                       │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  1. 生成 embedding                                  │   │
│  │  2. 存储到 Vector DB                                │   │
│  │  3. 存储实体关系到 Graph (可选)                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                    │                                        │
│                    ▼                                        │
│  输出: 新增的记忆条目                                        │
│                                                              │
│  🔑 优势:                                                   │
│  - 极简流程: 1 次 LLM 调用                                   │
│  - 无覆盖风险: 记忆累积不删除                                │
│  - 实体增强: 跨记忆的实体网络                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

## 5. SEARCH 流程 (记忆检索)

### 5.1 多信号融合检索

```
┌─────────────────────────────────────────────────────────────┐
│                   SEARCH 流程                                │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  用户查询: "用户最近的饮食偏好是什么？"                        │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Step 1: 三路并行检索                               │  │
│  │                                                      │  │
│  │            查询                                       │  │
│  │             │                                         │  │
│  │     ┌──────┼──────┐                                │  │
│  │     ▼      ▼      ▼                                │  │
│  │  ┌────┐ ┌────┐ ┌────┐                             │  │
│  │  │语义│ │BM25 │ │实体│                             │  │
│  │  │检索│ │搜索│ │匹配│                             │  │
│  │  └──┬─┘ └──┬─┘ └──┬─┘                             │  │
│  │     │      │      │                               │  │
│  │     ▼      ▼      ▼                               │  │
│  │  top-k   top-k   top-k                            │  │
│  └─────┼──────┼──────┼───────────────────────────────┘  │
│        │      │      │                                   │
│        └──────┼──────┘                                   │
│               ▼                                           │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Step 2: RRF 融合                                    │  │
│  │                                                      │  │
│  │  RRF(k) = Σ 1/(k + rank_i)                          │  │
│  │                                                      │  │
│  │  例: k=60                                            │  │
│  │  - 语义 rank=2 → 1/(60+2) = 0.0161                  │  │
│  │  - BM25 rank=1 → 1/(60+1) = 0.0164                  │  │
│  │  - 实体 rank=3 → 1/(60+3) = 0.0159                  │  │
│  │                                                      │  │
│  │  该记忆总分 = 0.0161 + 0.0164 + 0.0159 = 0.0484    │  │
│  └──────────────────────┬──────────────────────────────┘  │
│                         │                                   │
│                         ▼                                   │
│  ┌──────────────────────────────────────────────────────┐  │
│  │  Step 3: 返回 Top-K 结果                            │  │
│  │                                                      │  │
│  │  [                                                    │  │
│  │    {memory: "用户喜欢重口味麻辣烫", score: 0.92},   │  │
│  │    {memory: "用户经常晚上点外卖", score: 0.78},     │  │
│  │    {memory: "用户住在上海松江区", score: 0.65}      │  │
│  │  ]                                                   │  │
│  └──────────────────────────────────────────────────────┘  │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 多层记忆检索

```
