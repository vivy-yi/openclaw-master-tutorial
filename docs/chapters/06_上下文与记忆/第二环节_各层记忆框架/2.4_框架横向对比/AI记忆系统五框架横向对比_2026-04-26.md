## 1. memory-lancedb-pro

### 核心能力
- **存储**：LanceDB (本地向量数据库)
- **检索**：Hybrid (Vector + BM25) + Cross-encoder Rerank
- **智能提取**：LLM-powered Smart Extraction (6类)
- **生命周期**：Weibull 衰减模型
- **隔离**：Multi-scope (agent/user/session)

### 架构
```
Layer 0: openviking (对话压缩)
Layer 1: active-memory (记忆注入)
Layer 2: memory-lancedb-pro (混合检索)
Layer 3: memory-wiki (Obsidian 本地知识库)
```

### 工具
- `memory_store` / `memory_recall` / `memory_forget` / `memory_update`
- `self_improvement_log` / `self_improvement_extract_skill`

### 优点
- 生产级稳定性
- 混合检索 + Rerank 精度高
- 自适应衰减减少噪音
- 完全本地，无需外部服务

### 缺点
- 依赖外部 API (embedding/rerank/LLM)
- 配置复杂 (4档方案)

## 2. lossless-claw

### 核心能力
- **存储**：SQLite (每个对话一个表)
- **压缩**：DAG-based summarization (叶子 → 浓缩)
- **无损**：所有消息持久化，只压缩不删除
- **可检索**：`lcm_grep` / `lcm_describe` / `lcm_expand`

### 工作流
1. 消息写入 SQLite
2. 累积到阈值 → 摘要 (LLM)
3. 摘要形成 DAG 节点
4. 上下文组装 = 摘要 DAG + 最近原始消息

### 工具
- `/lcm` - 状态查看
- `/lcm backup` - 备份
- `/lcm rotate` - 转录重写
- `/lcm doctor` - 诊断

### 优点
- 真正的无损 (原始消息永不过期)
- DAG 可扩展 (不只是一级摘要)
- Agent 可 drill-down 检索细节

### 缺点
- 只解决**上下文压缩**，不解决长期记忆
- SQLite 存储随时间增长
- 需要 LLM 做摘要 (成本)

## 3. openviking

### 核心能力
- **范式**：文件系统风格 URI (`viking://resources/...`)
- **分层**：L0 (抽象) → L1 (概览) → L2 (完整内容)
- **检索**：目录递归检索 (优于 flat vector)
- **MCP**：自带 MCP server

### 工具
- `ov_import` - 导入 resource/skill
- `ov_search` - 检索

### 架构
```
客户端 (OpenClaw plugin)
    ↓ HTTP
OpenViking Server (本地 2033 端口)
    ↓
向量存储 (LanceDB)
    ↓
LLM (Volcengine/Ark)
```

### 优点
- 分层加载按需拉取
- 目录递归检索精度高
- 自带 MCP，生态好

### 缺点
- 需要自建 server
- 依赖 Volcengine API
- 不是专门给 Agent 的记忆层

## 4. mem0

### 核心能力
- **定位**：Universal, Self-improving memory layer
- **部署**：开源 (自托管) / 托管平台
- **存储**：Qdrant (默认) / Postgres+pgvector
- **特性**：
  - Adaptive memory (自动学习用户偏好)
  - Multimodal support
  - Async memory
  - Reranker-enhanced search

### 默认配置
- LLM: OpenAI GPT-4.1-nano
- Embedding: text-embedding-3-small
- Vector store: Postgres + pgvector

### 优点
- 开源 + 托管灵活选择
- 自适应学习 (不只是存储)
- 生态成熟 (LangChain, CrewAI 集成)

### 缺点
- **未集成 OpenClaw**
- 偏向应用层，非 Agent 专用

---

## 5. supermemory

### 核心能力
- **定位**：Context infrastructure for AI agents
- **五层架构**：
  1. User Profiles (行为理解)
  2. Memory Graph (向量图，ontology-aware)
  3. Retrieval (hybrid vector + keyword, <300ms)
  4. Extractors (PDF/web/image/audio)
  5. Connectors (Notion/Slack/GDrive/S3/Gmail)

### 性能
- **延迟**：<300ms p95
- **规模**：100B+ tokens/月
- **基准**：LongMemEval 85.2%, LoCoMo #1, ConvoMem #1

### 集成
- Claude Code, **OpenClaw**, Cursor, OpenCode, Windsurf
- Chrome Extension
- TypeScript, Python, REST API

### 优点
- 五层一体 (不只是向量存储)
- 用户理解模型 (不只是 recall)
- Consumer plugin 完整

### 缺点
- **未集成 OpenClaw** (虽有官方 integration 页面)
- 偏向 Consumer/应用层

## 当前 OpenClaw 架构

```
┌─────────────────────────────────────────────┐
│              OpenClaw Gateway               │
├─────────────────────────────────────────────┤
│  Slot: memory      → memory-lancedb-pro     │
│  Slot: contextEngine → lossless-claw        │
│  Tools: ov_import/ov_search → openviking    │
└─────────────────────────────────────────────┘
```

**结论**：当前已集成的三个框架已覆盖：
- 长期记忆 (lancedb-pro)
- 无损压缩 (lossless-claw)
- RAG/知识库 (openviking)

mem0 和 supermemory 的优势在于**应用层生态**，但对于 OpenClaw 这类 Agent 框架，现有集成已足够。
