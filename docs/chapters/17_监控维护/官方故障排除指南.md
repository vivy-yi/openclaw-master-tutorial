## 二、常见问题与解决方案

### 2.1 Gateway 无法启动

#### 问题表现

```
$ openclaw gateway start
Error: EADDRINUSE: address already in use :::18789
```

或

```
Gateway cannot start. Check logs at ~/.openclaw/logs/
```

#### 相关问题

| 问题 | 参考文档 |
|------|----------|
| npm 插件安装失败导致 Gateway 无法重启 | [npm-plugin-failure-recovery.md](./npm-plugin-failure-recovery.md) |
| Zen/Go API Key 独立配置 | [opencode-zen-go-api-key.md](../02_快速部署/opencode-zen-go-api-key.md) |
#### 诊断流程

```
┌─────────────────────────────────────────────────────────────┐
│                   Gateway 启动诊断流程                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 检查 Node 版本                                           │
│     $ node --version                                        │
│     预期: Node 22+                                          │
│     如版本过低 → 升级 Node                                   │
│                                                             │
│  2. 检查端口占用                                             │
│     $ lsof -i :18789                                        │
│     如有进程占用 → 释放端口 或 更换端口                        │
│                                                             │
│  3. 检查配置文件                                             │
│     $ cat ~/.openclaw/openclaw.json                         │
│     验证 JSON 格式正确                                       │
│                                                             │
│  4. 检查日志                                                 │
│     $ tail -100 ~/.openclaw/logs/openclaw.log                │
│     查找具体错误信息                                          │
│                                                             │
│  5. 重启 Gateway                                             │
│     $ openclaw gateway restart                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 解决方案

**方案 1: 升级 Node.js**

```bash
# 使用 nvm 安装
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.zshrc
nvm install 22
nvm use 22

# 验证
node --version  # 应显示 v22.x.x
```

**方案 2: 释放端口**

```bash
# 查找占用端口的进程
lsof -i :18789

# 终止进程 (替换 PID)
kill -9 <PID>

# 或更换端口启动
openclaw gateway start --port 18889
```

**方案 3: 重置配置**

```bash
# 备份当前配置
cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak

# 使用默认配置
openclaw config reset

# 重启
openclaw gateway restart
```

### 2.2 频道连接失败

#### Telegram 连接问题

**问题现象**: Bot 无法接收/发送消息

**诊断步骤**:

```
1. 验证 Bot Token
   $ openclaw channels test telegram

2. 检查 Webhook 设置
   $ openclaw channels debug telegram

3. 查看频道日志
   $ openclaw logs --channel telegram
```

**常见错误**:

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `Unauthorized` | Bot Token 错误 | 检查 `openclaw.json` 中的 token |
| `Forbidden: bot was blocked by user` | 用户封禁了 Bot | 无法干预 |
| `Conflict: took too long to respond` | Webhook 响应超时 | 检查 Gateway 状态 |

#### Discord 连接问题

**问题现象**: Discord Bot 无响应

**诊断步骤**:

```bash
# 1. 验证 Bot Token
openclaw channels test discord

# 2. 检查 Intents 配置
# 确保在 Discord Developer Portal 开启了:
# - MESSAGE CONTENT INTENT
# - PRESENCES INTENT (如需要)

# 3. 检查 WebSocket 连接
openclaw gateway status
```

**常见错误**:

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| `DiscordAPIError: Unknown Message` | 消息已删除 | 正常，忽略 |
| `DiscordGatewayError: Invalid Intents` | Intents 配置错误 | 检查 Developer Portal |
| `WebSocket connection failed` | 网络问题 | 检查代理设置 |

#### 飞书连接问题

**问题现象**: 飞书机器人无响应

**诊断步骤**:

```bash
# 1. 验证 App ID 和 App Secret
openclaw channels test feishu

# 2. 检查 Webhook 地址
# 确保使用公网可访问的地址

# 3. 查看飞书日志
openclaw logs --channel feishu
```

### 2.3 Session 问题

#### 查看活跃会话

```bash
# 列出所有会话
openclaw sessions list

# 查看特定会话详情
openclaw sessions status <session-id>

# 查看会话上下文大小
openclaw sessions detail <session-id>
```

#### 常见问题

**会话丢失**:

```
原因: Context 超出限制被截断
解决: 
1. 使用 sessions_list 查看当前会话
2. 检查上下文大小
3. 考虑启用轻量化上下文模式
```

**上下文异常**:

```bash
# 重置会话上下文
openclaw sessions reset <session-id>

# 或创建新会话
openclaw sessions new
```

**Session ID 格式**:

| 类型 | 格式 | 说明 |
|------|------|------|
| 主会话 | `main` | 固定不变 |
| Cron 会话 | `cron:<jobId>` | 定时任务专用 |
| 独立会话 | `isolated:<uuid>` | 每次新建 |

### 2.4 工具调用失败

#### exec 工具问题

**权限被拒绝**:

```bash
# 检查 exec 配置
openclaw config get tools.exec

# 启用 exec
openclaw config set tools.exec.allowFrom "all"

# 或指定白名单
openclaw config set tools.exec.allowFrom "192.168.1.0/24"
```

**命令执行超时**:

```json
// 在 openclaw.json 中调整超时
{
  "tools": {
    "exec": {
      "timeoutMs": 60000
    }
  }
}
```

#### browser 工具问题

**浏览器无法启动**:

```bash
# 1. 检查 Chrome/Chromium 是否安装
which google-chrome || which chromium

# 2. 安装 Chrome (macOS)
brew install --cask google-chrome

# 3. 验证浏览器自动化
openclaw browser test
```

**页面加载超时**:

```bash
# 增加超时时间
openclaw config set tools.browser.timeout 30000
```

#### 工具权限检查

```bash
# 查看所有工具状态
openclaw tools status

# 测试特定工具
openclaw tools test <tool-name>

# 查看工具调用日志
openclaw logs --tool exec
```

## 三、日志分析

### 3.1 日志位置

```
~/.openclaw/logs/
├── openclaw.log        # 主日志
├── gateway.log         # Gateway 日志
├── error.log           # 错误日志
└── sessions/          # 会话日志
    ├── main.log
    └── cron-xxx.log
```

### 3.2 日志级别

```bash
# 设置日志级别
openclaw config set logs.level debug  # 开发环境
openclaw config set logs.level info   # 生产环境

# 查看实时日志
tail -f ~/.openclaw/logs/openclaw.log
```

### 3.3 日志过滤

```bash
# 只看错误
grep -i error ~/.openclaw/logs/openclaw.log

# 只看特定工具
grep -i "tool=exec" ~/.openclaw/logs/openclaw.log

# 只看特定会话
grep "session=main" ~/.openclaw/logs/openclaw.log
```

## 四、性能问题

### 4.1 Gateway 响应慢

```
诊断步骤:
1. 检查 Gateway 状态
   $ openclaw gateway status

2. 检查系统资源
   $ top -o cpu
   $ free -h

3. 检查并发会话数
   $ openclaw sessions list

4. 检查内存使用
   $ openclaw gateway stats
```

### 4.2 内存占用过高

```bash
# 1. 查看内存使用
openclaw gateway stats

# 2. 限制会话数量
openclaw config set gateway.maxSessions 10

# 3. 设置会话过期时间
openclaw config set sessions.retention 24h

# 4. 清理旧会话
openclaw sessions prune --older-than 7d
```

### 4.3 磁盘空间不足

```bash
# 1. 查看日志大小
du -sh ~/.openclaw/logs/

# 2. 清理旧日志
find ~/.openclaw/logs -name "*.log" -mtime +7 -delete

# 3. 清理会话数据
openclaw sessions prune --all

# 4. 清理临时文件
rm -rf ~/.openclaw/workspace/temp/*
```
