## 本节目标
- 掌握认证配置方法

## 10.3.1 认证方式

### 支持的认证

```
认证方式：

1. API Key
   └── 简单密钥认证

2. Bearer Token
   └── OAuth / JWT

3. Basic Auth
   └── 用户名密码

4. 自定义
   └── 自定义验证逻辑
```

---

## 10.3.2 API Key 配置

### 配置方法

```yaml
gateway:
  auth:
    type: api_key
    api_keys:
      - key: sk-test-xxxxx
        name: test_user
        role: user

      - key: sk-admin-xxxxx
        name: admin
        role: admin
```

### 使用方式

```
Header: X-API-Key: sk-test-xxxxx
```

---

## 10.3.3 JWT 配置

### 配置方法

```yaml
gateway:
  auth:
    type: jwt
    jwt:
      secret: your-secret-key
      issuer: your-app
      algorithms:
        - HS256
      claims:
        - sub
        - role
        - exp
```

### 使用方式

```
Header: Authorization: Bearer <jwt_token>
```

---

## 10.3.4 OAuth 配置

### 配置方法

```yaml
gateway:
  auth:
    type: oauth
    oauth:
      providers:
        - name: google
          client_id: xxx
          client_secret: xxx
          authorize_url: https://accounts.google.com/o/oauth2/v2/auth
          token_url: https://oauth2.googleapis.com/token

        - name: github
          client_id: xxx
          client_secret: xxx
```

---

## 10.3.5 权限控制

### 角色配置

```yaml
gateway:
  auth:
    roles:
      - name: admin
        permissions:
          - "*"

      - name: user
        permissions:
          - /api/v1/agent:post
          - /api/v1/sessions:get

      - name: readonly
        permissions:
          - /api/v1/sessions:get
```

## 本节小结

1. **API Key**：简单密钥认证
2. **JWT**：Token 认证
3. **OAuth**：第三方登录
4. **权限**：角色 + 权限
