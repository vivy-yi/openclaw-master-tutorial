## 一、问题概述

### 1.1 问题描述

在 OpenClaw v2026.6.8 中，**会话状态管理出现回归（Regression）**。当 agent 在生成内容过程中被中断（`aborted-before-content`），该会话会进入"死亡螺旋"状态：

1. 会话被标记为损坏
2. 无法接收新消息
3. 尝试恢复导致进一步错误
4. 会话彻底不可用

这与 v2026.6.7 及之前版本的行为不一致，是一个**回归 Bug**。

### 1.2 影响范围

| 受影响版本 | 场景 | 严重程度 |
|-----------|------|---------|
| v2026.6.8 | 中断后会话恢复 | 🔴 P1 |
| v2026.6.8 | 长对话轮次 | 🔴 P1 |
| v2026.6.7 及之前 | ❌ 不受影响 | ✅ 正常 |

### 1.3 根本原因

v2026.6.8 中会话状态持久化逻辑变更后，未正确处理 `aborted-before-content` 边界情况，导致损坏的状态被写入会话存储，后续恢复时无法解析。

## 二、错误日志

### 2.1 典型错误信息

```text
[agent: main] ERROR: Session state corrupted after abort
[agent: main] WARNING: Attempting recovery...
[agent: main] ERROR: Failed to restore session: malformed state blob
[agent: main] FATAL: Session entering unrecoverable state
```

### 2.2 问题触发场景

```text
用户: 发送复杂查询
  ↓
Agent: 开始生成超长响应
  ↓
用户/系统: 中断操作 (Ctrl+C / timeout)
  ↓
Agent: aborted-before-content 状态写入会话
  ↓
下一轮: 会话无法恢复，进入"死亡螺旋"
```

## 三、触发条件

### 3.1 检查清单

```text
□ 使用 OpenClaw v2026.6.8
□ Agent 生成长响应时发生中断
□ 中断类型: 用户取消 / 超时 / 资源限制
□ 会话在中断后无法继续
□ 错误日志包含 "malformed state" 或 "unrecoverable"
```

### 3.2 影响评估

```bash
# 检查当前版本
openclaw --version

# 查看会话状态
openclaw sessions list --json | jq '.[] | {id, status}'

# 查看详细错误
openclaw logs --filter "session" --level error --since 24h
```

## 四、workaround 临时方案

### 4.1 方案一：手动重置会话

```bash
# 列出损坏的会话
openclaw sessions list --filter corrupted

# 重置特定会话
openclaw sessions reset <session-id>

# 或强制开启新会话
openclaw agent --new-session
```

### 4.2 方案二：降级版本

```bash
# 降级到 v2026.6.7
npm uninstall -g openclaw
npm install -g openclaw@2026.6.7

# 验证
openclaw --version
```

### 4.3 方案三：禁用长响应自动保存

```json
// openclaw.json
{
  "agents": {
    "main": {
      "session": {
        "persistInterval": 0,
        "autoRestore": false
      }
    }
  }
}
```
