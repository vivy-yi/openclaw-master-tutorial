## 一、60秒诊断流程（新增）

当 OpenClaw 出现问题时，按以下顺序执行诊断：

```bash
# 步骤1: 检查网关状态
openclaw status

# 步骤2: 检查所有渠道详细状态
openclaw status --all

# 步骤3: 检查网关探针
openclaw gateway probe

# 步骤4: 检查 systemd 服务状态
openclaw gateway status

# 步骤5: 运行医生检查
openclaw doctor

# 步骤6: 检查渠道探针
openclaw channels status --probe

# 步骤7: 跟踪日志
openclaw logs --follow
```

> **说明**: `openclaw channels status --probe` 是 v2026.4.23+ 新增命令，可探测所有渠道连通性。

## 五、常见问题解决方案更新

### 5.1 Anthropic 429 错误（长上下文）

**错误信息**:
```
HTTP 429: rate_limit_error: Extra usage is required for long context requests
```

**解决方案**: 参考 [官方文档](https://docs.openclaw.ai/gateway/troubleshooting#anthropic-429-extra-usage-required-for-long-context)

### 5.2 suspicious ownership 修复

**Docker 安装**:
```bash
sudo chown -R 1000:1000 /path/to/openclaw-config /path/to/openclaw-workspace
openclaw doctor --fix
```

**Root 运行**:
```bash
sudo chown -R root:root /path/to/openclaw-config/npm
openclaw doctor --fix
```

### 5.3 插件安装失败修复

确保 `openclaw.extensions` 配置正确：

```json
{
  "name": "@openclaw/my-plugin",
  "version": "1.2.3",
  "openclaw": {
    "extensions": ["./dist/index.js"]
  }
}
```
