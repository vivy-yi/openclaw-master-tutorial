# 15.6 Discord WebSocket 排错专题

## 本节目标

深入掌握 Discord WebSocket 连接不稳定、崩溃等问题的系统化排查与解决。

---

## 15.6.1 常见场景分类

Discord Bot 基于 WebSocket 与 Discord 官方保持实时连接。以下是高频问题场景：

| 场景 | 严重程度 | 典型表现 |
|------|---------|---------|
| 频繁断线重连 | 🟡 中 | 日志出现大量 `WebSocket reconnecting...` |
| stale-socket 崩溃 | 🔴 高 | Gateway 崩溃、无自动恢复、进程退出 |
| Intent 权限不足 | 🟢 低 | Bot 收不到特定事件（如 Reaction） |
| Rate Limit 触发 | 🟡 中 | 429 错误、消息发送被拒 |
| Gateway URL 过期 | 🟡 中 | 断开后无法找到有效 Gateway |
| 身份验证失败 | 🔴 高 | 401/403 错误，Bot 完全无法连接 |

---

```
Discord Bot 无响应

    │
    └─ Bot 能连接但收不到事件？
         └─ 查 15.6.7「Intent 权限问题」
```

## 15.6.3 崩溃类问题

### 问题 1：stale-socket 导致 Gateway 崩溃（#56274）

**严重程度**：🔴 高

**现象**：
- Discord Bot 完全崩溃，Gateway 无法响应
- WebSocket 连接处于 `stale-socket` 状态
- 大量 `uncaught exception` 循环
- 进程退出，无自动重连

**排查步骤**：

**Step 1：确认进程状态**

```bash
# 检查 Gateway 进程是否存活
ps aux | grep openclaw | grep -v grep

# 查看进程退出原因（dmesg 或 journalctl）
dmesg | grep openclaw
```

**Step 2：提取崩溃日志**

```bash
# 筛选 Discord 相关错误
openclaw gateway logs | grep -i discord

# 进一步过滤崩溃相关行
openclaw gateway logs | grep -i "discord" | grep -iE "error|exception|crash|fatal|stale"
```

**典型崩溃日志**：

```
[Discord] WebSocket state: stale-socket
[Discord] Uncaught exception in WebSocket handler
[Gateway] Unhandled promise rejection: DiscordWebSocketError
[Process] Worker exited with code 1
```

**Step 3：检查 Gateway 状态**

```bash
openclaw gateway status
openclaw gateway info
```

**根因**：

Discord WebSocket 处理 `stale-socket` 状态时未做防御性检查，当连接变为 stale 后直接抛异常导致进程崩溃。详见 [Issue #56274](https://github.com/openclaw/openclaw/issues/56274)。

**Workaround 方案**：

**方案 A：监控 + 自动重启（推荐生产环境使用）**

```bash
#!/bin/bash
# 保存为 /usr/local/bin/openclaw-watchdog.sh
# 并加入 crontab: * * * * * /usr/local/bin/openclaw-watchdog.sh

LOG_FILE="/var/log/openclaw-watchdog.log"
STATUS=$(openclaw gateway status 2>/dev/null)

if ! echo "$STATUS" | grep -q "running"; then
    echo "$(date '+%Y-%m-%d %H:%M:%S'): Gateway down, restarting..." >> "$LOG_FILE"
    openclaw gateway restart >> "$LOG_FILE" 2>&1
    echo "$(date '+%Y-%m-%d %H:%M:%S'): Restart done" >> "$LOG_FILE"
fi
```

**方案 B：限制 Discord 重连频率（防止重连风暴）**

在 `openclaw.yml` 中配置 Discord 重连参数：

```yaml
channels:
  discord:
    # 最大重连间隔（秒），防止指数级重连风暴
    maxReconnectInterval: 60
    # 重连最大次数，超出后放弃
    maxReconnectAttempts: 10
    # 启用指数退避
    useExponentialBackoff: true
```

**方案 C：切换到 REST Long Polling（备选）**

如果 WebSocket 持续不稳定，可降级使用 REST Long Polling 模式（稳定性优先场景）：

```yaml
channels:
  discord:
    useWebSocket: false
    pollInterval: 30000  # 30秒轮询
```

**官方修复跟踪**：
- [Issue #56274](https://github.com/openclaw/openclaw/issues/56274) — 活跃 Bug
- [PR #56287](https://github.com/openclaw/openclaw/pull/56287) — Fix: Discord can not find function withFileLockadd

**状态**：活跃 Bug，建议生产环境部署监控脚本，持续关注 PR 合入。

### 问题 2：大量 uncaught exception 循环

**现象**：
- 日志中反复出现 `Unhandled promise rejection`
- 内存持续增长
- 最终 OOM 进程被系统杀掉

**排查**：

```bash
# 查看异常堆栈
openclaw gateway logs | grep -iE "uncaught|unhandled|TypeError|ReferenceError" | tail -50

# 检查内存使用
ps aux | grep openclaw | awk '{print $6/1024 " MB RSS"}'
```

**根因**：Discord SDK 内部异常未被捕获，逐层上抛。

**临时应对**：
1. 重启 Gateway 清理异常状态
2. 开启 `uncaughtException` 处理器（需修改代码）
3. 关注 [Discord SDK Issues](https://github.com/discordjs/discord.js/issues)

## 15.6.4 断连类问题

### 问题 1：WebSocket 频繁断线重连

**现象**：
- 日志中大量 `WebSocket reconnecting in Xs...`
- Bot 时通时不通
- 用户感知明显延迟

**排查步骤**：

**Step 1：查看连接统计**

```bash
# 查看 Discord 连接详情
openclaw channels status | grep -i discord

# 查看详细连接日志
openclaw logs --channel discord --limit 200 | grep -iE "connect|disconnect|reconnect"
```

**Step 2：检查网络延迟**

```bash
# 测试到 Discord Gateway 的网络质量
ping -c 10 gateway.discord.gg

# 测试 SSL 连接
openssl s_client -connect gateway.discord.gg:443 -brief
```

**Step 3：检查本地资源**

```bash
# 检查 fd 使用量（WebSocket 消耗 fd）
lsof -p $(pgrep -f openclaw) | wc -l

# 检查系统 fd 限制
ulimit -n
```

**常见原因与对策**：

| 原因 | 症状 | 对策 |
|------|------|------|
| 网络不稳定 | 特定时间段断连 | 部署在网络稳定的环境 |
| 防火墙限制 | 突然断连无重连 | 开放 `443`、`7000-7002` 端口 |
| 负载过高 | 高并发时断连 | 降低 `maxConcurrence` |
| Discord 主动断开 | 无规律断连 | 增加心跳频率 |

**配置优化**：

```yaml
channels:
  discord:
    # 心跳间隔（毫秒），Discord 默认 41250ms，过高可能丢包
    heartbeatInterval: 30000
    # 连接超时（毫秒）
    connectionTimeout: 30000
    # WebSocket 压缩（降低带宽占用）
    compress: true
```

---

### 问题 2：Gateway URL 过期后无法重连

**现象**：
- 日志出现 `Invalid Gateway URL`
- 断开后不尝试重连
- 需手动重启才能恢复

**排查**：

```bash
openclaw gateway logs | grep -iE "gateway|url|invalid"
```

**根因**：Discord Gateway URL 有缓存，过期后 SDK 未主动刷新。

**Workaround**：

```bash
# 强制刷新 Gateway URL
openclaw channels restart discord
```

或重启 Gateway：

```bash
openclaw gateway restart
```

## 15.6.5 认证类问题

### 问题 1：401 Unauthorized / 403 Forbidden

**现象**：
- Bot 完全无法连接
- 日志出现 `401` 或 `403` 错误

**排查**：

```bash
# 检查 Token 配置
openclaw config get channels.discord.token
# （确认 Token 存在且未截断）

# 测试 Token 有效性（在线工具）
# https://discord.com/developers/applications
```

**典型错误日志**：

```
[Discord] Authentication failed: 401 Unauthorized
[Discord] Bot not found or token invalid
```

**根因与解决**：

| 错误码 | 原因 | 解决 |
|--------|------|------|
| 401 | Token 错误或已重置 | 重新获取 Bot Token |
| 403 | Token 被撤消 | 重新授权 |
| 403 | 权限不足 | 检查 Intent 申请 |

**解决步骤**：
1. 访问 [Discord Developer Portal](https://discord.com/developers/applications)
2. 选择对应 Application → Bot → Reset Token
3. 更新 `openclaw.yml` 中的 `channels.discord.token`
4. 重启 Gateway

---

### 问题 2：Intent 权限不足

**现象**：
- Bot 能连接但不响应特定事件
- 收不到 Reaction、Voice、Typing 等事件
- 日志无明显报错

**排查**：

**Step 1：检查 Intent 配置**

Discord Bot 需要在 Developer Portal 和代码中同时声明 Intent：

```bash
# 查看当前 Intent 配置
openclaw config get channels.discord.intents
```

**Step 2：对比 Developer Portal**

登录 [Discord Developer Portal](https://discord.com/developers/applications)：
- 勾选 Bot → Privileged Intents：
  - ✅ `PRESENCE INTENT`（需要）
  - ✅ `SERVER MEMBERS INTENT`（需要）
  - ✅ `MESSAGE CONTENT INTENT`（需要）

**Step 3：确认代码配置**

```yaml
channels:
  discord:
    intents:
      - GUILDS
      - GUILD_MESSAGES
      - MESSAGE_CONTENT
      - GUILD_MESSAGE_REACTIONS
      # 根据需要添加
```

**常见缺失 Intent 与影响**：

| 缺失 Intent | 影响 |
|------------|------|
| `MESSAGE_CONTENT` | 无法读取消息内容（Bot 必须申请） |
| `GUILD_MESSAGE_REACTIONS` | 无法接收 Reaction 事件 |
| `GUILD_MEMBERS` | 无法获取成员列表/昵称 |
| `GUILD_VOICE_STATES` | 无法响应语音事件 |

## 15.6.6 限流类问题（Rate Limit）

### 问题 1：触发 Rate Limit（429 错误）

**现象**：
- 日志出现 `429 Too Many Requests`
- 消息发送失败或延迟
- 重连被拒绝

**排查**：

```bash
# 查看限流相关日志
openclaw gateway logs | grep -i "429\|rate.limit\|retry.after"
```

**Rate Limit 类型**：

| 类型 | 限制 | 说明 |
|------|------|------|
| Global | 1 req/s | 所有 API 请求共享 |
| Channel | 120/60s | 按 Channel 独立计算 |
| Guild | 120/60s | 按 Server 独立计算 |
| WebSocket | 120/60s | Gateway 连接数 |

**应对策略**：

```yaml
channels:
  discord:
    # 启用请求队列，避免超限
    useRequestQueue: true
    # 全局速率限制（请求/秒）
    ratelimitMaxRetries: 3
    # 超过限流后等待时间（毫秒）
    ratelimitRetryAfter: 5000
```

**代码层面优化**：
- 批量操作使用 `bulk` 端点（如 `bulkDeleteMessages`）
- 减少高频轮询（如不在循环中查 Member）
- 缓存常用数据避免重复请求

## 15.6.7 日志分析实战

### 完整排查命令清单

```bash
# 1. 确认进程存活
ps aux | grep openclaw | grep -v grep

# 2. 查看 Gateway 状态
openclaw gateway status

# 3. 查看 Discord 连接日志（最近 200 行）
openclaw gateway logs | grep -i discord | tail -200

# 4. 提取所有错误
openclaw gateway logs | grep -iE "error|exception|crash|warn" | grep -i discord

# 5. 查看 WebSocket 状态
openclaw gateway logs | grep -i websocket | tail -50

# 6. 检查系统资源
ulimit -n && df -h && free -h

# 7. 测试网络连通性
curl -s --max-time 5 https://gateway.discord.gg && echo "Gateway reachable"
```

### 日志关键词速查

| 关键词 | 含义 | 严重度 |
|--------|------|--------|
| `stale-socket` | 连接过期未刷新 | 🔴 |
| `reconnecting` | 正在重连 | 🟡 |
| `401 Unauthorized` | Token 无效 | 🔴 |
| `403 Forbidden` | 权限不足 | 🔴 |
| `429 Too Many Requests` | 触发限流 | 🟡 |
| `Invalid Gateway URL` | Gateway URL 过期 | 🟡 |
| `heartbeat timeout` | 心跳超时 | 🟡 |
| `connection refused` | 连接被拒绝 | 🔴 |
| `ENOTFOUND` | DNS 解析失败 | 🔴 |

## 15.6.8 配置建议总结

### 生产环境推荐配置

```yaml
channels:
  discord:
    # === 连接稳定性 ===
    useWebSocket: true          # WebSocket 优先
    heartbeatInterval: 30000    # 30秒心跳（Discord 默认 41.25s）
    connectionTimeout: 30000    # 30秒连接超时
    compress: true              # 启用压缩节省带宽
    
    # === 重连策略 ===
    maxReconnectAttempts: 10    # 最多重试10次
    maxReconnectInterval: 60    # 最大间隔60秒
    useExponentialBackoff: true # 指数退避
    
    # === 速率控制 ===
    useRequestQueue: true       # 请求排队
    ratelimitMaxRetries: 3      # 限流重试次数
    
    # === Intent（按需启用）===
    intents:
      - GUILDS
      - GUILD_MESSAGES
      - MESSAGE_CONTENT
      - GUILD_MESSAGE_REACTIONS
      - GUILD_MESSAGE_TYPING
```

### 开发环境调试配置

```yaml
channels:
  discord:
    # 开启详细日志
    debug: true
    # 禁用压缩方便抓包
    compress: false
    # 缩短心跳方便观察
    heartbeatInterval: 10000
```

## 15.6.9 快速自检清单

出现 Discord 连接问题时，按顺序检查：

- [ ] **进程**：Gateway 进程是否存活？
- [ ] **Token**：`channels.discord.token` 是否正确？
- [ ] **权限**：Bot 是否已在 Server 中？Intent 是否申请？
- [ ] **网络**：`gateway.discord.gg:443` 是否可达？
- [ ] **日志**：是否有 stale-socket / 401 / 429 等关键词？
- [ ] **资源**：`ulimit -n` 是否足够？内存是否充足？
- [ ] **配置**：`openclaw.yml` 中 Discord 配置是否正确？
