## 二、性能问题：重启后高延迟

### 2.1 排查步骤

**步骤一：测量延迟基准**

```bash
# 方法1：通过 Gateway 日志测量响应时间
openclaw gateway logs 2>&1 | grep -E "request|response|latency" | tail -50

# 方法2：通过 API 直接测量
time curl -X POST http://localhost:18789/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message": "你好"}'

# 正常响应时间：< 2 秒
# 高延迟：> 5 秒
# 严重延迟：> 10 秒
```

**步骤二：检查 Gateway 配置**

```bash
# 查看 Gateway 配置
openclaw gateway status

# 检查资源配置
cat ~/.openclaw/openclaw.json | jq '.gateway'

# 预期配置示例
{
  "gateway": {
    "port": 18789,
    "workers": 4,
    "timeout": 30000,
    "keepalive": 60
  }
}
```

**步骤三：检查资源瓶颈**

```bash
# CPU 占用
top -o cpu | grep openclaw

# 内存占用
top -o mem | grep openclaw

# Node.js 内存限制
# 如果内存不足，LLM 调用会变慢
```

### 2.2 常见原因与修复

**原因1：Worker 数量不足**

```json
// openclaw.json
{
  "gateway": {
    "workers": 4    // ← 增加到可用 CPU 核心数
  }
}
```

**原因2：超时设置过短**

```json
{
  "gateway": {
    "timeout": 60000   // ← 生产环境建议 60 秒
  }
}
```

**原因3：缺少 Keep-Alive**

```json
{
  "gateway": {
    "keepalive": 120   // ← 保持连接复用，减少建连开销
  }
}
```

**原因4：磁盘 I/O 瓶颈**

```
诊断：
  → Gateway 重启后需要重新加载 session 文件
  → 如果 workspace 在网络磁盘（NFS）上，I/O 会很慢

解决：
  → 将 workspace 迁移到本地 SSD
  → 修改配置：
```

```json
{
  "workspace": {
    "path": "/Users/d/.openclaw/workspace"   // ← 本地 SSD
  }
}
```

## 三、UI问题：侧边栏会话冻结

### 3.1 排查步骤

**步骤一：检查 WebSocket 连接**

```bash
# 在浏览器中打开 Webchat
# 打开开发者工具 (F12) → Network → WS

# 观察 WebSocket 连接状态
# 正常：Connection ID: xxx, Status: Open
# 异常：Connection: close, Status: 1006

# WebSocket 断开常见原因：
#   - Gateway 重启
#   - 网络中断
#   - Idle timeout 超时
```

**步骤二：检查前端日志**

```javascript
// 在浏览器控制台执行
console.log("WebSocket state:", websocket.readyState);
// 0 = CONNECTING
// 1 = OPEN
// 2 = CLOSING
// 3 = CLOSED
```

**步骤三：清除缓存重试**

```
方法：
  1. 浏览器开发者工具 → Application → Clear storage
  2. 刷新页面 (Cmd+Shift+R)
  3. 检查问题是否复现
```

### 3.2 常见原因与修复

**原因1：Session ID 映射丢失**

```bash
# 诊断：检查 localStorage 中的 session 映射
# 在浏览器控制台执行：
Object.keys(localStorage).filter(k => k.includes('session'))

# 如果 key 存在但值不对，清除后重试
localStorage.clear()
location.reload()
```

**原因2：Gateway 重启导致 WebSocket 断连**

```
预防配置：
  • 启用 WebSocket 自动重连
  • 调整 ping/pong 间隔
```

```json
{
  "webchat": {
    "websocket": {
      "reconnect": true,
      "reconnect_interval": 3000,
      "ping_interval": 30
    }
  }
}
```

## 四、UI问题：XML 标签显示异常

### 4.1 问题现象

```
发送包含 XML 的消息时：
  ❌ 错误显示：XML 标签被解析为 HTML
  <config>...</config> → <config>...</config>

  ✅ 正确显示：代码块包裹，标签转义
  ```xml
  <config>...</config>
  ```
```

### 4.2 修复方案

**前端修复**（需要 Webchat 更新到支持版本）：

```javascript
// 使用 DOMPurify 净化 HTML
import DOMPurify from 'dompurify';

const safeContent = DOMPurify.sanitize(userContent, {
  ALLOWED_TAGS: ['code', 'pre'],
  ALLOWED_ATTR: []
});
```

**用户侧workaround**：

```
临时解决方案：
  1. 将 XML 内容放在代码块中发送
  2. 使用 ```xml 包裹代码
  3. 避免直接发送裸 XML
```

## 五、预防与监控

### 5.1 性能监控配置

```json
{
  "gateway": {
    "metrics": {
      "enabled": true,
      "port": 9090,
      "export_interval": 60
    }
  }
}

# Prometheus 指标：
#   - openclaw_request_duration_seconds
#   - openclaw_active_sessions
#   - openclaw_gateway_memory_bytes
```

### 5.2 延迟告警

```
阈值建议：
  • > 3 秒：P2 告警（可能有问题）
  • > 8 秒：P1 告警（需要立即处理）
  • > 15 秒：P0 告警（服务不可用）

告警方式：
  • 日志监控：grep "latency.*[0-9]{2,}" gateway.log
  • Prometheus + AlertManager
```
