# 10.4 限流配置

## 本节目标
- 掌握限流配置方法
- 理解限流策略
- 完成限流配置

---

## 10.4.1 限流概述

### 限流作用

```
限流的价值：

1. 保护服务
   ├── 防止过载
   └── 保障可用性

2. 成本控制
   ├── 控制 API 调用
   └── 预算保护

3. 公平使用
   ├── 防止滥用
   └── 资源公平
```

---

## 10.4.2 限流配置

### 全局限流

```yaml
gateway:
  rate_limit:
    enabled: true
    global:
      requests: 1000
      window: 60  # 每分钟

    # 突发限流
    burst:
      enabled: true
      multiplier: 2
```

### 按用户限流

```yaml
gateway:
  rate_limit:
    by_user:
      requests: 100
      window: 60

    by_ip:
      requests: 50
      window: 60
```

## 10.4.3 限流策略

### 固定窗口

```
固定窗口：
- 简单实现
- 边界可能有突发

示例：每分钟 100 次
|------60秒------|
|████████░░░░░░░| 100 次后限流
```

### 滑动窗口

```
滑动窗口：
- 更平滑
- 实现复杂

示例：每分钟 100 次
|-----------60秒-----------|
| ████████████░░░░░░░░░░░ | 实时滑动
```

### 令牌桶

```
令牌桶：
- 允许突发
- 平均速率

示例：
- 桶容量：100
- 添加速率：10/秒
- 突发：可一次用完桶中令牌
```

## 10.4.4 限流响应

### 响应格式

```json
{
  "error": "rate_limit_exceeded",
  "message": "请求过于频繁，请稍后重试",
  "retry_after": 30
}
```

### HTTP 状态

```
限流响应：

- 429 Too Many Requests
- Header: Retry-After: 30
```

## 10.4.5 白名单

### 配置白名单

```yaml
gateway:
  rate_limit:
    whitelist:
      ips:
        - 192.168.1.1
        - 10.0.0.0/8

      users:
        - admin
        - internal
```
