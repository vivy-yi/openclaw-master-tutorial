# 多模态记忆系统完整指南

> 本指南覆盖多模态记忆的完整知识体系：架构设计、模型选型、工具链、OpenClaw 现状、以及未来扩展方向。

**整理时间**: 2026-04-25
**适用版本**: OpenClaw 2026.3+

---

## 1. 什么是多模态记忆

### 1.1 单模态 vs 多模态

```
单模态（现状）：
  文字 → embedding → 向量存储 → 检索 → 文字结果

多模态（目标）：
  文字  ──→ embedding ──→ 向量存储 ──→ 检索 ──→ 文字结果
  图片  ──→ embedding ──→ 向量存储 ──→ 检索 ──→ 图片结果
   │                                              │
   └──────── 跨模态检索 ←───────────────────────────┘
```

### 1.2 多模态记忆的价值

| 场景 | 单模态能做到 | 多模态能做到 |
|------|------------|-------------|
| UI设计 | 文字描述"红色按钮" | 直接匹配设计稿截图 |
| 截图记忆 | 只能描述截图内容 | 直接存储和检索截图本身 |
| 视频理解 | 只能文字转录 | 关键帧+文字联合检索 |
| 语音备忘 | 只能转写文字 | 保留音频特征+转写 |
| 图表分析 | 只能描述图表内容 | 直接匹配同类图表 |

## 2. 多模态记忆架构

### 2.1 分层架构

```
┌────────────────────────────────────────────────────┐
│              模态输入层                            │
│  文字 / 图片 / 音频 / 视频 / PDF / 网页          │
└──────────────────────┬─────────────────────────────┘
                       │
┌──────────────────────▼─────────────────────────────┐
│              向量化层（Embedding）                  │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ │
│  │ 文字向量    │ │ 图片向量    │ │ 音频向量    │ │
│  │ jina-v5    │ │ jina-v4     │ │ Whisper→文本│ │
│  │ CLIP        │ │ CLIP ViT    │ │ AudioCLIP   │ │
│  └──────────────┘ └──────────────┘ └──────────────┘ │
└──────────────────────┬─────────────────────────────┘
                       │
┌──────────────────────▼─────────────────────────────┐
│              向量存储层（Vector DB）                │
│  LanceDB / Pinecone / Weaviate / Qdrant / Milvus  │
│  支持多模态索引：dense + sparse + multi-vector    │
└──────────────────────┬─────────────────────────────┘
                       │
┌──────────────────────▼─────────────────────────────┐
│              检索层（Retrieval）                   │
│  向量相似度 / BM25 / 混合检索 / Rerank           │
│  支持跨模态：text→image / image→text             │
└──────────────────────┬─────────────────────────────┘
                       │
┌──────────────────────▼─────────────────────────────┐
│              LLM 层（理解 + 生成）                  │
│  MiniMax M2.7 / GPT-4V / Claude 3 / Gemini       │
│  多模态输入理解 → 结构化输出 → 记忆存储            │
└────────────────────────────────────────────────────┘
```

### 2.2 多模态检索类型

```
1. 文字 → 文字（现状）
   "用户偏好" → 匹配文字记录

2. 图片 → 图片
   截图 → 匹配相似设计图

3. 文字 → 图片（跨模态）
   "找类似的UI设计" → 匹配设计稿图片

4. 图片 → 文字（跨模态）
   设计稿截图 → 匹配文字设计规范说明

5. 音频 → 文字
   语音备忘 → 匹配文字笔记

6. 视频 → 关键帧 → 向量
   视频片段 → 匹配相关截图+文字
```

## 3. 多模态 Embedding 模型

### 3.1 模型对比表

#### 文字 Embedding

| 模型 | 厂商 | 维度 | MTEB 分数 | 费用 | 多模态 |
|------|------|------|----------|------|--------|
| **jina-embeddings-v5-text-small** | Jina | 1024 | 高 | 低 | ❌ 纯文本 |
| **jina-embeddings-v4** | Jina | 1024 | 更高 | 中 | ✅ 文本+图片 |
| **text-embedding-3-large** | OpenAI | 3072 | 最高 | 高 | ❌ 纯文本 |
| **gemini-embedding-2-preview** | Google | 3072 | 新高 | 中 | ✅ 文本+图片+音频 |
| **mxbai-embed-large** | Ollama | 1024 | 高 | 免费 | ❌ 纯文本 |
| **snowflake-arctic-embed2** | Ollama | 1024 | 中 | 免费 | ❌ 纯文本 |
| **nomic-embed-text:v1.5** | Ollama | 768 | 中 | 免费 | ❌ 纯文本 |

#### 图片 Embedding（多模态）

| 模型 | 厂商 | 维度 | 特点 | 费用 |
|------|------|------|------|------|
| **CLIP ViT-L/14** | OpenAI/LAION | 768 | 图文对齐最强 | 免费(Ollama) |
| **CLIP ViT-B/32** | OpenAI | 512 | 轻量版 | 免费(Ollama) |
| **SIGLIP** | Google | 1024 | 高精度 | 免费(Ollama) |
| **LLaVA embedding** | 开源 | 4096 | VLM-based | 免费(Ollama) |
| **E5-Mistral-7B** | Microsoft | 4096 | 多模态+推理 | 免费(Ollama) |
| **jina-embeddings-v4** | Jina | 1024 | Qwen2.5-VL内核 | API |

#### 音频 Embedding

| 模型 | 厂商 | 特点 | Pipeline |
|------|------|------|---------|
| **Whisper → jina** | OpenAI+Jina | ASR → 文本 → embedding | 串行 |
| **AudioCLIP** | 开源 | 音频=图片=文字 | 原生 |
| **HuBERT** | Meta | 自监督语音 | 串行 |
| **MiniLM-LL-V1** | Microsoft | 轻量音频 | API |

### 3.2 OpenClaw 当前支持的 Embedding

```
memory-lancedb-pro 配置项：
  embedding.provider = "openai-compatible"
  embedding.model   = jina-embeddings-v5-text-small ← 当前（纯文本）
  embedding.model   = jina-embeddings-v4 ← 可升级（图文）

rerank.provider = "jina"
  rerank.model   = jina-reranker-v3 ← 当前（纯文本）
  rerank.model   = jina-reranker-m0 ← 可升级（多模态）
```

### 3.3 多模态 Embedding 选择决策树

```
需要处理什么模态？
    │
    ├── 仅文字 → jina-embeddings-v5-text-small ✅
    │
    ├── 文字 + 图片（Obsidian截图）
    │      → jina-embeddings-v4 ✅（同API，升级即可）
    │
    ├── 文字 + 图片 + 音频
    │      → gemini-embedding-2-preview ✅
    │
    └── 全本地免费方案
           Ollama:
           ├─ 文字 → mxbai-embed-large
           ├─ 图片 → CLIP ViT-L/14
           └─ 音频 → Whisper → mxbai-embed-large
```

## 4. 多模态向量数据库

### 4.1 数据库对比

| 数据库 | 多模态 | 混合检索 | 免费版 | 说明 |
|--------|--------|---------|--------|------|
| **LanceDB** | ✅ | dense+sparse | ✅ | memory-lancedb-pro 底层 |
| **Pinecone** | ✅ | dense | ❌ | 云服务，图片+向量 |
| **Weaviate** | ✅ | dense+sparse+BM25 | ✅ | 内置 CLIP multi2vec |
| **Qdrant** | ✅ | dense+sparse | ✅ | 多模态+过滤 |
| **Milvus/Zilliz** | ✅ | hybrid | ✅ | 企业级多模态 |
| **Chroma** | ⚠️ | 仅dense | ✅ | 简单场景 |
| **Elasticsearch** | ⚠️ via plugin | KNN | ✅ | 需单独向量插件 |

### 4.2 选型建议

| 场景 | 推荐 | 原因 |
|------|------|------|
| OpenClaw 生态 | LanceDB | 原生集成 ✅ |
| 图文混合 | Weaviate | CLIP 内置 |
| 企业级大规模 | Milvus/Zilliz | 成熟生态 |
| 简单原型 | Qdrant | 部署简单 |
| 成本敏感 | LanceDB / Qdrant | 都有免费托管版 |

### 5.1 当前支持情况

| 能力 | 状态 | 说明 |
|------|------|------|
| **LLM 图片理解** | ✅ | MiniMax M2.7 原生支持 |

```

│  图片输入  │ ──→ │ jina-v4    │ ──→ │  (multi-   │
└─────────────┘     └─────────────┘     │   vector)   │ ──→ LLM理解
┌─────────────┐     ┌─────────────┐     │            │
│  音频输入  │ ──→ │ Whisper→  │ ──→ │            │
```

## 7. 多模态 Pipeline 设计

### 7.1 图片记忆 Pipeline

```
用户发图 / Obsidian 笔记含图
    │
    ▼
图片预处理
    ├─→ 缩放（>2MB 则压缩）
    ├─→ 格式转换（统一 PNG/JPEG）
    └─→ 提取图片描述（LLM/MiniMax）
    │
    ▼
多模态 embedding
    ├─→ jina-embeddings-v4（图文向量）
    └─→ LLM 提取 alt-text / caption
    │
    ▼
LanceDB 存储
    ├─→ 图片向量（vector）
    ├─→ 文字描述（metadata）
    ├─→ alt-text（metadata）
    └─→ 来源路径（metadata）
    │
    ▼
检索时
    ├─→ 文字查询 → 匹配描述/alt-text
    └─→ 图片查询 → 向量相似度匹配
```

### 7.2 音频记忆 Pipeline

```
语音备忘 / 播客录音 / 视频音频
    │
    ▼
Whisper 转写
    │  (MiniMax 已有音频理解，或用 Whisper CLI)
    ▼
文本输出 + 时间戳
    │
    ▼
jina embedding（文字向量）
    │
    ▼
LanceDB 存储
    ├─→ 文字向量
    ├─→ 原始文本（metadata）
    ├─→ 时长/时间戳（metadata）
    └─→ 来源（metadata）
    │
    ▼
检索时
    ├─→ 文字 → 匹配文本内容
    └─→ 可回播对应时间戳的音频片段
```

### 7.3 视频记忆 Pipeline

```
视频文件 / 播客视频
    │
    ▼
关键帧提取（FFmpeg）
    │  每 N 秒一帧，或场景切换时提取
    ▼
帧图片 → CLIP / jina-v4 embedding
    │
    ▼
Whisper 转写（音频部分）
    │
    ▼
LLM 综合摘要
    │  "视频讲了什么"
    ▼
LanceDB 存储
    ├─→ 帧向量列表
    ├─→ 文字摘要
    ├─→ 转写文本
    └─→ 时间轴
    │
    ▼
检索时
    ├─→ 文字 → 匹配摘要+转写
    └─→ 截图 → 匹配相似帧 → 回推时间点
```

### 7.4 PDF/文档 Pipeline

```
PDF / 扫描件
    │
    ▼
处理路径
    ├─→ PDF文字版 → pdftotext → jina embedding
    └─→ 扫描件 → tesseract OCR → jina embedding
    │
    ▼
LLM 提取结构化信息
    │  标题、关键段落、图表描述
    ▼
LanceDB 存储
    ├─→ 文字向量
    ├─→ 结构化 metadata
    └─→ 图表图片向量（如有）
    │
    ▼
检索时
    ├─→ 文字 → 匹配文档内容
    └─→ 图片查询 → 匹配文档内图表
```

## 8. 多 Agent 多模态协同

### 8.1 典型场景

```
用户上传设计稿截图
    │
    ▼
meta-team Agent（路由）
    │
    ├─→ openviking 存储图片记忆
    │      (jina-v4 embedding)
    │
    ├─→ mo-finance Agent
    │      分析设计成本/商业价值
    │
    ├─→ mo-law Agent
    │      分析设计版权/合规
    │
    └─→ mo-yunying Agent
           分析用户反馈/市场定位
```

### 8.2 多 Agent 共享图片记忆

```
每 Agent 维护自己的 memory-lancedb-pro
    │
    但图片向量存储在共享层
    │
    ├─→ shared/images/ (原始文件)
    ├─→ shared/vectors/ (图片向量)
    └─→ shared/metadata/ (图片描述)
    │
    各 Agent 引用共享图片向量
    而不是各自 embedding 同一张图
```

## 9. 工具配置示例

### 9.1 升级到多模态（jina-v4）

```json
{
  "plugins": {
    "entries": {
      "memory-lancedb-pro": {
        "config": {
          "embedding": {
            "provider": "openai-compatible",
            "apiKey": "jina_...",
            "model": "jina-embeddings-v4",  // 升级：图文
            "baseURL": "https://api.jina.ai/v1",
            "dimensions": 1024,
            "normalized": true
          },
          "retrieval": {
            "mode": "hybrid",
            "rerank": "cross-encoder",
            "rerankProvider": "jina",
            "rerankModel": "jina-reranker-m0",  // 多模态rerank
            "rerankEndpoint": "https://api.jina.ai/v1/rerank",
            "rerankApiKey": "jina_..."
          }
        }
      }
    }
  }
}
```

### 9.2 FFmpeg 关键帧提取

```bash
# 每30秒提取一帧
ffmpeg -i video.mp4 -vf "fps=1/30" frames/frame_%03d.jpg

# 场景切换时提取
ffmpeg -i video.mp4 -vf "select='gt(scene,0.3)',showinfo" frames/frame_%03d.jpg

# 获取视频字幕
yt-dlp --write-subs --sub-lang zh video_url
```

### 9.3 Whisper 转写

```bash
# 安装
brew install whisper

# 转写（支持中文）
whisper audio.m4a --model medium --language Chinese

# 输出时间戳字幕
whisper audio.m4a --model medium --language Chinese --output_format srt
```

### 9.4 CLIP 本地 embedding（Ollama）

```bash
# 安装 Ollama
brew install ollama

# 拉取 CLIP 模型
ollama pull clip

# 文字 embedding
curl http://localhost:11434/api/embeddings -d '{
  "model": "clip",
  "prompt": "a red button design"
}'

# 图片 embedding
curl http://localhost:11434/api/embeddings -d '{
  "model": "clip",
  "prompt": "image_path_or_url"
}'
```
