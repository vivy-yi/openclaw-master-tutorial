# 多源知识系统完整配置指南

> OpenClaw 接入多源知识的核心原则：**Skill 优先 → CLI 兜底 → MCP 最后手段**

**整理时间**: 2026-04-25
**最后更新**: 2026-04-25
**适用版本**: OpenClaw 2026.3+

---

## 0. 核心原则：工具接入优先级

```
OpenClaw 工具接入优先级
    │
    ├── 1️⃣ Skill（最高优先）
    │       openclaw skills install → 自动配置，即装即用
    │
    ├── 2️⃣ CLI（次优先）
    │       已有官方 CLI → exec 直接调用
    │
    └── 3️⃣ MCP（最后手段）
            无 Skill + 无 CLI → 才考虑 MCP
```

**为什么 Skill 优先？**
- OpenClaw Skill 是原生集成，`openclaw skills install` 一行搞定
- 自动配置，不需要手动写 JSON
- 和 OpenClaw 深度绑定，支持内置 prompt 和最佳实践

**为什么 MCP 是最后手段？**
- MCP 主要是给 Claude Desktop / Cursor 用的
- OpenClaw 有更原生的 Skill 系统
- MCP 需要手动配置 JSON，复杂度高

## 3. 第二优先级：CLI 工具

### 3.1 判断标准

有以下任意特点优先用 CLI：
- 平台提供官方 CLI（如 `gh`, `zotero`, `gh`)
- CLI 认证简单（`gh auth login`）
- CLI 输出可解析（JSON/YAML）
- OpenClaw 的 `exec` 工具可直接调用

### 3.2 GitHub CLI（gh）

**状态**：已安装 ✅

```bash
# 认证
gh auth login

# 搜索仓库
gh search repos "openclaw" --limit 5

# 列出 issues
gh issue list --repo owner/repo --state open

# 创建 issue
gh issue create --title "Bug" --body "描述"

# 查看 PR
gh pr list --repo owner/repo
```

**OpenClaw 中调用**：
```bash
gh search code "filename:.github/workflows" --owner your-org
```

### 3.3 Zotero CLI

**安装**：
```bash
brew install zotero
```

**功能**：
- 文献检索
- 附件管理
- 库同步

### 3.4 Airtable CLI

**方案**：使用 Airtable REST API + `curl`：

```bash
# 列出 base
curl -H "Authorization: Bearer $AIRTABLE_API_KEY" \
  "https://api.airtable.com/v0/meta/bases"

# 查询表
curl -H "Authorization: Bearer $AIRTABLE_API_KEY" \
  "https://api.airtable.com/v0/BASE_ID/TABLE_NAME"
```

### 3.5 数据库 CLI

| 数据库 | CLI 工具 | 安装 |
|--------|---------|------|
| PostgreSQL | `psql` | `brew install postgresql` |
| MySQL | `mysql` | `brew install mysql` |
| SQLite | `sqlite3` | 内置 |

**OpenClaw 中调用**：
```bash
psql "postgresql://user:pass@host/db" -c "SELECT * FROM table"
```

## 4. 第三优先级：MCP（最后手段）

### 4.1 为什么 MCP 是最后手段

1. **配置复杂** — 需要写 JSON，手动管理进程
2. **不是 OpenClaw 原生** — Skill 系统更深度集成
3. **Claude Desktop 导向** — 很多 MCP 是为 Claude Desktop 设计的

### 4.2 何时才用 MCP

满足以下**全部条件**才用：
- ❌ 没有 OpenClaw Skill
- ❌ 没有官方 CLI
- ❌ 必须用该平台的数据

### 4.3 MCP 安装方式

```bash
# 方式 1：npm 全局安装（官方 MCP）
npm install -g @modelcontextprotocol/server-github

# 方式 2：npx 直接运行（社区 MCP）
npx makenotion/notion-mcp-server

# 方式 3：pip 安装（Python MCP）
pip install mcp-server-zotero
```

### 4.4 MCP 配置到 OpenClaw

```json
{
  "mcpServers": {
    "notion": {
      "command": "npx",
      "args": ["makenotion/notion-mcp-server"],
      "env": {
        "NOTION_API_KEY": "env:NOTION_API_KEY"
      }
    }
  }
}
```

### 4.5 MCP 选择决策树

```
是否有 OpenClaw Skill？
    │
    ├── ✅ 有 → 直接 install Skill
    │
    └── ❌ 没有 → 是否有官方 CLI？
              │
              ├── ✅ 有 → 用 CLI
              │
              └── ❌ 没有 → 才考虑 MCP
```

## 6. 故障排除

### 6.1 Skill 安装失败

```bash
# 查看详细错误
openclaw skills install <skill-name> --verbose

# 检查网络
curl -I https://clawhub.ai

# 重试
openclaw skills install <skill-name>
```

### 6.2 CLI 命令找不到

```bash
# 检查是否安装
which gh  # GitHub CLI
which zotero  # Zotero

# 重新安装
brew install gh
```

### 6.3 MCP 连接失败

```bash
# 测试 MCP server 独立运行
npx makenotion/notion-mcp-server

# 检查 API Key
echo $NOTION_API_KEY

# 查看 OpenClaw 日志
openclaw gateway logs --tail 50
```
