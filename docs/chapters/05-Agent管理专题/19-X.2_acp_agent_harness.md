## 一、什么是 ACP Agent Harness

### 1.1 基本概念

**ACP（Agent Client Protocol）** 是一种让 OpenClaw 连接外部编码 Agent 的协议。

```
OpenClaw（主控）
    │
    ▼
ACP 协议
    │
    ├── 路由到 Claude Code
    ├── 路由到 Codex
    ├── 路由到 Cursor
    └── 路由到其他编码 Agent
```

**Harness = "马具"**，把外部 Agent 套进来，让它们为 OpenClaw 打工。

### 1.2 支持的 Agent

| Agent | 用途 | 命令 |
|-------|------|------|
| Claude Code | 专业编码 | `claude` |
| Codex | OpenAI 编码 | `codex` |
| Cursor | IDE 集成 | `cursor` |
| Gemini CLI | Google 编码 | `gemini` |
| Qwen Code | 阿里编码 | `qwen` |
| Pi | 对话型编码 | `pi` |
| OpenCode | 开源编码 | `opencode` |
| Kimi | 月之暗面编码 | `kimi` |
| Kiro | 字节编码 | `kiro` |
| ... | ... | ... |

完整列表见 [acp-router skill](https://github.com/openclaw/openclaw/tree/main/dist/extensions/acpx/skills/acp-router)。

## 二、acp-router 工作原理

### 2.1 路由决策流程

```
用户请求："用 Claude Code 帮我写一个排序算法"
        │
        ▼
   acp-router 拦截
        │
        ▼
   解析请求意图
        │
        ├── 包含 "Claude Code" / "Codex" / "Pi" 等关键词
        │
        ▼
   选择适配器
        │
        ├── claude → npx -y @zed-industries/claude-agent-acp@0.21.0
        ├── codex → npx @zed-industries/codex-acp@^0.9.5
        └── ...
        │
        ▼
   执行并返回结果
```

### 2.2 内置适配命令

```javascript
// acpx 内置适配命令
{
  "openclaw": "openclaw acp",
  "claude": "npx -y @zed-industries/claude-agent-acp@0.21.0",
  "codex": "npx @zed-industries/codex-acp@^0.9.5",
  "copilot": "copilot --acp --stdio",
  "cursor": "cursor-agent acp",
  "droid": "droid exec --output-format acp",
  "gemini": "gemini --acp",
  "iflow": "iflow --experimental-acp",
  "kilocode": "npx -y @kilocode/cli acp",
  "kimi": "kimi acp",
  "kiro": "kiro-cli acp",
  "opencode": "npx -y opencode-ai acp",
  "pi": "npx pi-acp@^0.0.22",
  "qwen": "qwen --acp"
}
```

## 三、两种调用路径

### 3.1 ACP Runtime 路径（OpenClaw 管理）

通过 `sessions_spawn` 调用，OpenClaw 管理线程生命周期。

```javascript
// 在 OpenClaw 中调用
sessions_spawn({
  runtime: "acp",
  agentId: "claude",
  mode: "session",    // 持久化线程
  thread: true,        // 创建线程
  task: "帮我写一个排序算法"
})
```

**特点**：
- OpenClaw 管理会话
- 支持线程持久化
- 可以后续继续对话

### 3.2 Direct acpx 路径（CLI 直连）

通过 `acpx` CLI 直接调用，不需要 OpenClaw。

```bash
# 直接执行
acpx codex exec -- "帮我写一个排序算法"

# 持久化会话
acpx claude session --persist

# 一次性命令
acpx pi -- "帮我写一个排序算法"
```

**特点**：
- 无需 OpenClaw
- 适合一次性任务
- 更轻量

## 四、Think + ACP 组合使用

### 4.1 何时组合

| 场景 | Think 模式 | ACP Harness |
|------|-----------|-------------|
| 简单对话 | ❌ 关闭 | ❌ 不需要 |
| 复杂推理任务 | ✅ 开启 | ❌ 不需要 |
| 需要专业编码 | ❌ 可选 | ✅ 需要 |
| 复杂推理 + 专业编码 | ✅ 开启 | ✅ 需要 |

### 4.2 组合示例

```
用户：帮我写一个排序算法，然后用 Think 模式解释它的复杂度

Step 1: 用 Claude Code 写代码（ACP Harness）
Step 2: 用 Think ON 模式解释复杂度（OpenClaw 自身）
```

## 五、配置与管理

### 5.1 环境变量

```bash
# ACP 配置
OPENCLAW_ACP_ENABLED=true
CLAUDE_API_KEY=sk-...
CLAUDE_API_ENDPOINT=https://api.anthropic.com

# 其他 Agent 配置
OPENAI_API_KEY=sk-...
GEMINI_API_KEY=...
```

### 5.2 线程管理

```javascript
// 创建持久化线程
sessions_spawn({
  runtime: "acp",
  agentId: "claude",
  mode: "session",
  thread: true,
  task: "开发一个电商网站"
})

// 后续继续对话
sessions_send(sessionKey, "添加用户登录功能")
```

## 六、与 Skill Harness 的区别

| 维度 | ACP Agent Harness | Skill Harness |
|------|-------------------|---------------|
| **目的** | 让外部 Agent 打工 | 测试优化自身 Skill |
| **控制权** | 外部 Agent 执行 | 自我优化 |
| **验证方式** | Agent 自我验证 | 渲染测试 + 归因 |
| **迭代方式** | Agent 修复 | eval→analyze→optimize 循环 |
