## 一、什么是 CLI Harness

### 1.1 核心理念

> **"Today's Software Serves Humans👨💻. Tomorrow's Users will be Agents🤖"**
> 
> 今天软件服务于人类，明天软件服务于 Agent。

**CLI Harness = 把任意软件封装成 Agent 可调用的标准 CLI**

```
任意软件（GUI / 复杂工具）
       │
       ▼
  CLI-Anything 自动分析
       │
       ▼
  生成标准 CLI Harness
       │
       ▼
  Agent 通过标准 CLI 调用任意软件
```

### 1.2 为什么需要 CLI Harness

| 痛点 | 解决 |
|------|------|
| Agent 无法调用复杂 GUI 软件 | 用标准 CLI 封装 |
| 每个软件 API 不同，难以统一调用 | 统一 Click CLI 接口 |
| Agent 不知道软件有哪些功能 | 自动生成 `--help` 文档 |
| Agent 难以理解软件输出 | 结构化 JSON 输出 |

## 三、7 阶段生成管道

### 3.1 完整流程

```bash
/cli-anything <software-path>
        │
        ▼
Phase 1: 🔍 Analyze
  └─ 扫描源代码，分析 GUI 功能映射到 API
        
Phase 2: 📐 Design  
  └─ 设计命令组、状态模型、输出格式

Phase 3: 🔨 Implement
  └─ 构建 Click CLI（REPL、JSON输出、undo/redo）

Phase 4: 📋 Plan Tests
  └─ 创建 TEST.md（单元 + E2E 测试计划）

Phase 5: 🧪 Write Tests
  └─ 实现测试套件

Phase 6: 📝 Document
  └─ 更新文档

Phase 6.5: 🎯 SKILL.md Generation
  └─ 生成 AI 可发现的 Skill 定义

Phase 7: 🔄 Refine
  └─ 根据测试结果优化
```

### 3.2 各阶段详解

#### Phase 1: Analyze（分析）

```bash
# 分析 GIMP 源代码
/cli-anything ./gimp

# 输出：
# - 分析 500+ PDB procedures
# - 映射 GUI 功能到 API
# - 识别数据格式（XCF、Pillow）
```

#### Phase 2: Design（设计）

```yaml
# 设计命令结构
commands:
  - group: image
    commands:
      - open
      - save
      - resize
      - crop
  - group: layer
    commands:
      - add
      - remove
      - merge
  - group: filter
    commands:
      - blur
      - sharpen
      - color
```

#### Phase 3: Implement（实现）

```python
# 生成 Click CLI
@click.group()
def cli():
    """GIMP CLI - Agent-native image editing"""
    pass

@cli.command()
@click.argument('path')
def open(path):
    """Open an image file"""
    # 实现...
```

#### Phase 4-5: Test（测试）

```markdown
# TEST.md
## Unit Tests
- test_open_image
- test_resize
- test_filter_blur

## E2E Tests
- test_full_workflow: 打开 → 调整 → 导出
```

#### Phase 6.5: SKILL.md（技能定义）

```yaml
# SKILL.md
name: gimp
description: Agent-native GIMP image editing CLI
commands:
  - open, save, resize, crop
  - layer operations
  - filters (blur, sharpen, color)
```

## 四、生成的 CLI 示例

### 4.1 GIMP CLI

```bash
# 标准命令
gimp-cli open /path/to/image.png
gimp-cli resize --width 800 --height 600
gimp-cli filter blur --radius 5
gimp-cli export /output.png --format png

# JSON 输出（Agent 友好）
gimp-cli status --format json
```

### 4.2 Blender CLI

```bash
blender-cli render --scene Scene --frame 1 --output /tmp/
blender-cli object add --type Cube --location 0 0 0
```

### 4.3 Zotero CLI

```bash
zotero-cli search "machine learning"
zotero-cli add -- DOI 10.1234/example
zotero-cli export --format bibtex
```

## 七、与前两种 Harness 的区别

| 维度 | ACP Agent Harness | Skill Harness | CLI Harness |
|------|-------------------|--------------|-------------|
| **目的** | 让外部 Agent 打工 | 测试优化 Skill | 封装软件为 Agent 工具 |
| **输入** | 编码任务 | Skill 代码 | 任意软件/GUI |
| **输出** | Agent 执行结果 | 优化的 Skill | 标准 CLI |
| **验证** | Agent 自我验证 | 渲染测试 | 单元 + E2E |
| **迭代** | Agent 修复 | eval→render→optimize | 测试驱动 |
| **规模** | 14+ 种 Agent | 针对特定 Skill | 7400+ 软件 |

## 八、快速开始

```bash

/plugin marketplace add HKUDS/CLI-Anything

/plugin install cli-anything
```

```bash
npx skills add HKUDS/CLI-Anything --skill gimp -g -y
```

```bash
pip install cli-anything-gimp
```
