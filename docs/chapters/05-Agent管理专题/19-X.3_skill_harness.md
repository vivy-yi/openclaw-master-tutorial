## 一、什么是 Skill Harness

**Skill Harness = 让 Skill 自己测试、自己进化的自动化框架**

```
传统 Skill 开发：
  写 Skill → 人工测试 → 发现问题 → 手动修复 → 重复

Harness 驱动开发：
  写 Skill → Harness 自动测试 → 发现问题 → LLM 自动修复 → 重复
```

## 三、Harness 架构详解

### 3.1 目录结构

```
harness/
├── controller.js          # 主入口，驱动整个优化循环
├── eval-agent.js          # 调用 eval CLI，生成测试结果文件
├── render-agent.js        # Headless 浏览器渲染测试，返回失败 case
├── analyze-agent.js       # 将错误 case 归因到对应 skill 文件
├── optimize-agent.js      # LLM 重写 skill 文档修复错误
├── index-agent.js         # 重建 BM25 检索索引
├── config.js              # Library 注册表（g2 / g6 元信息）
├── config-manager.js      # 多源配置合并（CLI > 配置文件 > env > 默认值）
├── error-classifier.js    # 错误分类与恢复策略
├── memory.js              # 跨迭代记忆（skill 优化历史）
├── retry-utils.js         # 重试工具
└── package.json
```

### 3.2 循环流程

```
┌─────────────────────────────────────────────────────┐
│  每次迭代                                            │
│                                                     │
│  1. eval-agent      生成代码并写入 result JSON        │
│  2. render-agent    渲染测试，识别 error / blank case │
│  3. analyze-agent   将 case 归因到 skill 文件         │
│  4. optimize-agent  LLM 重写出错的 skill 文档         │
│  5. index-agent     重建 BM25 检索索引               │
│                                                     │
│  连续 --passes 次零错误 → 退出                        │
└─────────────────────────────────────────────────────┘
```

## 四、关键组件详解

### 4.1 eval-agent

**作用**：调用 LLM 生成代码，并写入 result JSON。

```javascript
// 简化流程
async function evalAgent(task, model) {
  // 1. 检索相关 Skill
  const skills = retrieve(task, library, topk)
  
  // 2. 调用 LLM 生成代码
  const code = await llmGenerate(task, skills, model)
  
  // 3. 写入结果
  await writeResultJSON(task, code)
  
  return code
}
```

### 4.2 render-agent

**作用**：用 Headless 浏览器渲染测试，识别错误。

```javascript
// 简化流程
async function renderAgent(results) {
  // 1. 启动 Headless 浏览器
  const browser = await launchBrowser()
  
  // 2. 逐个渲染
  for (const result of results) {
    await browser.goto(result.url)
    await browser.waitForRender()
    
    // 3. 检查是否有错误
    const errors = await checkErrors()
    if (errors.length > 0) {
      result.status = 'fail'
      result.errors = errors
    }
  }
  
  return failedCases
}
```

### 4.3 analyze-agent

**作用**：将失败 case 归因到具体的 Skill 文件。

```javascript
// 简化流程
async function analyzeAgent(failedCases) {
  // 1. 读取失败 case 的错误信息
  // 2. 检索相关的 Skill 内容
  // 3. LLM 分析：哪个 Skill 的哪个部分导致错误
  // 4. 输出归因结果
  
  return {
    skillFile: 'antv-g2-chart.md',
    errorSection: '## 柱状图 - stackY 配置',
    reason: 'stackY 在某些数据格式下会失效'
  }
}
```

### 4.4 optimize-agent

**作用**：LLM 重写出错的 Skill 文档。

```javascript
// 简化流程
async function optimizeAgent(attributions) {
  for (const attr of attributions) {
    // 1. 读取出错的 Skill 文件
    const skill = readFile(attr.skillFile)
    
    // 2. LLM 基于错误 case 重写
    const fixed = await llmRewrite(skill, attr.errors)
    
    // 3. 写回 Skill 文件
    writeFile(attr.skillFile, fixed)
  }
}
```

## 六、快速开始

### 6.1 安装

```bash
cd harness && npm install
```

### 6.2 运行

```bash
# 最小运行（g2，采样 10 条）
node controller.js

# 指定 library 和采样量
node controller.js --library=g6 --sample=30

# 全量数据集
node controller.js --library=g2 --full
```

### 6.3 CLI 参数

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--library <id>` | `g2` | 目标库：`g2` \| `g6` |
| `--sample <n>` | `10` | 随机采样条数 |
| `--full` | `false` | 使用完整数据集 |
| `--retrieval <strategy>` | `tool-call` | 检索策略：`tool-call` \| `bm25` \| `context7` |
| `--passes <n>` | `3` | 连续多少次零错误才停止 |
| `--max-iterations <n>` | `20` | 最大迭代轮次上限 |
| `--optimize-model <id>` | 同 `AI_MODEL` | 优化阶段使用的模型 |
| `--score` | `false` | 开启 VL 视觉质量打分 |

## 七、与 ACP Agent Harness 的区别

| 维度 | ACP Agent Harness | Skill Harness |
|------|-------------------|---------------|
| **目的** | 让外部 Agent 打工 | 测试优化自身 Skill |
| **控制权** | 外部 Agent 执行 | 自我优化 |
| **验证方式** | Agent 自我验证 | 渲染测试 + 归因 |
| **迭代方式** | Agent 修复 | eval→analyze→optimize 循环 |
| **使用场景** | 需要外部编码能力 | 提升 Skill 质量 |
