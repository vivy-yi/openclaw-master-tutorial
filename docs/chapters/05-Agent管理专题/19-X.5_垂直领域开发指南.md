## 一、垂直领域开发的核心问题

### 问题：需要 Harness 吗？

| 开发阶段 | 需要 Harness 吗 | 说明 |
|---------|----------------|------|
| **学习/原型验证** | ❌ 不需要 | 直接用，看效果 |
| **个人使用** | ❌ 不需要 | 发现问题手动改 |
| **小团队内部** | ⚠️ 可选 | Skill + Superpowers |
| **公开发布** | ✅ 建议 | 必须有评测保证 |
| **商业化** | ✅ 必须 | 需要完整的质量体系 |

---

## 二、什么场景用什么 Harness

### 场景 1：开发一个"法律合同审查助手"

| 组件 | 需要什么 | 为什么 |
|------|---------|-------|
| **Skill** | ✅ 需要 | 合同审查流程、判断标准 |
| **Knowledge** | ✅ 需要 | 法条库、判例参考 |
| **MCP Server** | ✅ 可选 | 法律数据库 API |
| **Superpowers** | ⚠️ 建议 | 标准化开发流程 |
| **ACP Agent Harness** | ❌ 不需要 | 不需要外部编码 Agent |
| **Skill Harness** | ⚠️ 可选 | 如果要持续迭代优化 |
| **CLI Harness** | ❌ 不需要 | 没有复杂软件需要封装 |

**结论**：Skill + Superpowers 足够了。

---

### 场景 2：开发一个"BI 数据可视化助手"

| 组件 | 需要什么 | 为什么 |
|------|---------|-------|
| **Skill** | ✅ 需要 | 可视化规范、图表选择 |
| **Knowledge** | ✅ 需要 | BI 模板、配色方案 |
| **CLI Harness** | ⚠️ 建议 | 需要封装 BI 工具（Tableau、PowerBI）|
| **Superpowers** | ⚠️ 建议 | 标准化开发流程 |
| **Skill Harness** | ⚠️ 可选 | 如果要自动化测试生成质量 |
| **ACP Agent Harness** | ❌ 不需要 | 不需要外部编码 |

**结论**：优先用 CLI Harness 封装 BI 工具，加 Superpowers 规范流程。

---

### 场景 3：开发一个"游戏 AI 助手"

| 组件 | 需要什么 | 为什么 |
|------|---------|-------|
| **Skill** | ✅ 需要 | 游戏开发工作流 |
| **Superpowers** | ✅ 需要 | 标准化开发流程 |
| **ACP Agent Harness** | ✅ 需要 | 需要 Claude Code/Codex 写代码 |
| **Skill Harness** | ⚠️ 可选 | 如果要测试 Skill 生成质量 |
| **CLI Harness** | ✅ 需要 | 需要封装 Godot/Unity/Unreal 等游戏引擎 |

**结论**：四种 Harness 都需要。

## 三、四种 Harness 的适用场景

### 3.1 ACP Agent Harness - 何时需要

| 场景 | 用 ACP Agent Harness |
|------|---------------------|
| 需要写复杂代码 | ✅ Claude Code / Codex |
| 需要多轮代码迭代 | ✅ 持久化线程 |
| 需要专业 IDE 集成 | ✅ Cursor |
| 日常对话/简单任务 | ❌ 不需要 |
| 只做领域判断/分析 | ❌ 不需要 |

---

### 3.2 Skill Harness - 何时需要

| 场景 | 用 Skill Harness |
|------|-----------------|
| Skill 需要持续迭代 | ✅ eval→optimize 循环 |
| 需要自动化测试 | ✅ 渲染测试 |
| 要公开发布 Skill | ✅ 必须有评测报告 |
| 个人学习用 Skill | ❌ 不需要 |
| 一次性任务 | ❌ 不需要 |

---

### 3.3 CLI Harness - 何时需要

| 场景 | 用 CLI Harness |
|------|--------------|
| 需要调用复杂 GUI 软件 | ✅ GIMP、Blender、Unity |
| 需要封装专用工具 | ✅ CAD、GIS，专业软件 |
| 要让 Agent 调用外部系统 | ✅ 封装 API 为 CLI |
| 纯文本处理 | ❌ 不需要 |
| 简单 API 调用 | ❌ 不需要 |

---

### 3.4 Superpowers - 何时需要

| 场景 | 用 Superpowers |
|------|---------------|
| **任何开发任务** | ✅ 强烈建议 |
| 需要标准化工作流程 | ✅ brainstorm→plan→execute→verify |
| 多 Agent 协作开发 | ✅ subagent-driven-development |
| 需要质量门禁 | ✅ verification-before-completion |
| 简单对话/纯分析 | ❌ 不需要 |

**为什么 Superpowers 建议优先？**
- 165K Stars 验证的方法论
- 不需要额外安装工具（只是 Skill）
- 适用于所有编程语言和项目类型
- 让 Agent 从"能干活"变成"会干活"

## 四、设计指南

### 4.1 最小化原则

**不要过度工程**。先从简单开始。

```
阶段 1：只用 Skill + Superpowers
  └─ 写好 Skill 文档，用标准流程工作

阶段 2：需要测试
  └─ 加简单的测试 harness

阶段 3：需要自动化
  └─ 加 eval→optimize 循环（Skill Harness）

阶段 4：需要外部集成
  └─ 加 ACP Agent Harness

阶段 5：需要封装软件
  └─ 加 CLI Harness
```

### 4.2 判断标准

| 问题 | 回答 "是" 的数量 | 建议 |
|------|-----------------|------|
| Skill 需要频繁更新？ | ≥3 | 加 Skill Harness |
| 需要调用复杂软件？ | ≥2 | 加 CLI Harness |
| 需要写复杂代码？ | ≥1 | 加 ACP Agent Harness |
| 要标准化开发流程？ | ≥1 | **加 Superpowers** |
| 要公开发布？ | =1 | 加基础测试 |

---

## 五、实战建议

### 5.1 推荐的开发顺序

```
1. 先做 Skill 原型 + Superpowers
   └─ 写 SKILL.md，用 OpenClaw 测试
   └─ 用标准流程工作（brainstorm→plan→verify）

2. 再加简单测试
   └─ 写几个测试用例，验证效果

3. 如果要发布，加自动化
   └─ 接入 CI/CD，自动测试

4. 如果需要，加外部集成
   └─ 按需接入 ACP Agent Harness / CLI Harness
```

### 5.2 常见错误

| 错误 | 后果 | 正确做法 |
|------|------|---------|
| 一开始就上全套 Harness | 过度复杂，难以迭代 | 从简单开始 |
| 不做测试就发布 | Skill 质量无法保证 | 至少有几个测试用例 |
| 用 ACP Harness 做简单任务 | 浪费资源 | 简单任务直接用 Skill |
| 忽略 Superpowers | Agent 缺乏标准流程 | **优先考虑加入** |
| 忽略 CLI Harness | Agent 无法调用复杂软件 | 按需封装 |

---

## 六、一句话总结

| 场景 | 用的 Harness | 一句话 |
|------|-------------|--------|
| 垂直领域 Skill 开发 | Skill + Superpowers | 让 Skill 按标准流程进化 |
| 需要调用复杂软件 | CLI Harness | 把软件变成 Agent 工具 |
| 需要专业编码能力 | ACP Agent Harness | 让外部 Agent 打工 |
| **任何开发任务（推荐）** | **Superpowers** | **让 Agent 按标准流程工作** |
| 日常对话/简单任务 | 不需要 | 直接用 OpenClaw |
