# 19-X.1 四种 Harness 类型概述

## 🚀 章节目标

理解四种不同的 Harness 思想：ACP Agent Harness、Skill Harness、CLI Harness、Superpowers。

---

## 一、什么是 Harness

### 1.1 一句话定义

> **Harness = 让 AI 在受控环境中可靠执行的控制系统**

### 1.2 核心本质

**Harness（马具）的本义**：套在马身上的装备，让马能被人控制。

```
没有 Harness：
  AI 自由发挥 → 结果随机 → 不可靠

有 Harness：
  AI + 受控环境 + 验证机制 → 结果可控
```

### 1.3 组成公式

```
Harness = 约束 + 验证 + 迭代

约束：AI 只能在指定范围内行动
验证：有明确的标准判断对错
迭代：失败 → 反馈 → 修复 → 再验证
```

### 1.4 为什么需要 Harness

| 问题 | 没有 Harness | 有 Harness |
|------|------------|-----------|
| AI 行为 | 随意发挥 | 受约束 |
| 验证 | 人工检查 | 自动验证 |
| 迭代 | 随机尝试 | 有方向的反馈循环 |
| 结果 | 不可靠 | 可复现 |

### 1.5 七种 Harness 的共同本质

| 类型 | 控制什么 | 验证什么 |
|------|---------|---------|
| **ACP Agent** | 外部编码 Agent | Agent 的输出 |
| **Skill** | Skill 的质量 | 渲染测试通过/失败 |
| **CLI** | 软件调用方式 | 单元 + E2E 测试 |
| **Superpowers** | Agent 工作流程 | 质量门禁 |
| **Wiki** | 知识积累方式 | metadata 完整性 |
| **Autoresearch** | 研究实验方向 | val_bpb 降低 |
| **Coding Benchmark** | 代码生成质量 | 测试用例通过 |

**共同点**：都是 **控制系统 + 验证机制 + 反馈循环**

### 1.6 一图总结

```
┌─────────────────────────────────────────┐
│              Harness                      │
│                                         │
│   ┌─────────────────────────────────┐   │
│   │  控制系统    验证机制    反馈循环 │   │
│   └─────────────────────────────────┘   │
│                                         │
│   作用：让 AI 的行为从"随意"变成"可靠"   │
└─────────────────────────────────────────┘
```

### 1.7 最重要的一句话

> **用了什么类型的 Harness 不重要，重要的是：你控制了什么、验证了什么、如何迭代。**

## 二、四种 Harness 一览

| 类型 | 全称 | 作用 | Stars | 代表项目 |
|------|------|------|-------|---------|
| **ACP Agent Harness** | Agent Client Protocol Harness | 连接外部编码 Agent | — | OpenClaw acp-router |
| **Skill Harness** | Skill 评测 Harness | 测试 + 优化 Skill 质量 | 238 | antvis/chart-visualization-skills |
| **CLI Harness** | CLI 封装 Harness | 把软件变成 Agent 可用工具 | 32K | HKUDS/CLI-Anything |
| **Superpowers** | 工作流方法论 Harness | 控制 Agent 的行为模式 | **165K** | obra/superpowers |

---

## 三、核心对比

| 维度 | ACP Agent Harness | Skill Harness | CLI Harness | Superpowers |
|------|-------------------|--------------|-------------|-------------|
| **Stars** | — | 238 | 32K | **165K** |
| **目的** | 让外部编码 Agent 打工 | 测试优化 Skill | 封装软件为 Agent 工具 | **控制工作流** |
| **输入** | 编码任务 | Skill 代码 | 任意软件/GUI | 任意任务 |
| **输出** | Agent 执行结果 | 优化的 Skill | 标准 CLI | 标准流程执行 |
| **验证** | Agent 自我验证 | 渲染测试 + 错误分类 | 单元 + E2E | **质量门禁** |
| **迭代** | Agent 自己修复 | eval→render→analyze→optimize | 测试驱动 | **Plan→Do→Verify** |
| **规模** | 14+ 种外部 Agent | 针对特定 Skill | 7400+ 软件 | 方法论通用 |
| **典型场景** | OpenClaw 集成外部 Agent | Skill 开发团队 | 软件 Agent 化 | **所有开发任务** |

## 四、四种 Harness 的关系

```
AI Agent (OpenClaw / Claude Code / Pi)
       │
       ├── ACP Agent Harness
       │      └─ 连接外部编码 Agent
       │          ├─ Claude Code ⭐
       │          ├─ Codex
       │          ├─ Cursor
       │          └─ ...
       │
       ├── Skill Harness
       │      └─ 测试优化 Skill
       │          ├─ eval-agent → 生成代码
       │          ├─ render-agent → 渲染测试
       │          ├─ analyze-agent → 错误归因
       │          └─ optimize-agent → LLM 修复
       │
       ├── CLI Harness
       │      └─ 封装任意软件为 CLI
       │          ├─ 分析软件
       │          ├─ 设计 CLI
       │          ├─ 实现 Click CLI
       │          └─ 测试验证
       │
       └── Superpowers（工作流方法论）
              └─ 控制 Agent 的行为模式
                  ├─ brainstorming → 设计先行
                  ├─ writing-plans → 计划驱动
                  ├─ subagent-driven → 子 Agent 执行
                  └─ verification → 质量门禁
```

## 五、一句话总结

| Harness 类型 | 一句话 | 关键问题 |
|-------------|--------|---------|
| **ACP Agent Harness** | 让外部编码 Agent 为我打工 | "谁能帮我写代码？" |
| **Skill Harness** | 让 Skill 自己测试、自己进化 | "我的 Skill 质量够好吗？" |
| **CLI Harness** | 把任意软件变成 Agent 可用的 CLI | "怎么让 Agent 调用这个软件？" |
| **Superpowers** | 控制 Agent 的工作方法论 | "如何让 Agent 按标准流程工作？" |

---

## 六、选择指南

| 你的需求 | 用什么 Harness |
|---------|--------------|
| 需要外部编码能力 | ACP Agent Harness |
| 开发高质量 Skill | Skill Harness |
| 需要 Agent 调用复杂软件 | CLI Harness |
| **标准化开发流程** | **Superpowers** |
| 日常对话/简单任务 | 不需要 Harness |

---

## 七、Superpowers 的独特价值

Superpowers 是唯一一个 **165K Stars** 验证的方法论：

```
165K Stars = 165,000 开发者认可的工作方法论
```

**为什么 Stars 这么高？**
- 不需要安装额外的工具
- 只是给 AI 编程工具一套标准的工作方法
- 适用于所有编程语言和项目类型
- 让 AI 从"能干活"变成"会干活"
