## 7.4.1 工作流概述

### 工作流概念

```
工作流 = 任务的自动化编排

包含：
├── 触发条件
├── 执行步骤
├── 分支逻辑
├── 错误处理
└── 完成条件
```

### 核心要素

```
工作流核心要素：

1. 触发器 (Trigger)
   └── 启动条件

2. 任务 (Task)
   └── 具体工作

3. 流程控制 (Control)
   └── 顺序、分支、循环

4. 数据传递 (Data)
   └── 输入、输出、转换

5. 错误处理 (Error)
   └── 重试、跳过、终止
```

## 7.4.2 工作流定义

### YAML 定义

```yaml
name: code_review_workflow
version: 1.0.0
description: 自动化代码审查工作流

trigger:
  type: event
  event: pull_request

agents:
  - name: scanner
    role: security_scan
  - name: reviewer
    role: code_review
  - name: reporter
    role: report

flow:
  - task: scan
    agent: scanner
    input: code
    output: scan_results

  - task: review
    agent: reviewer
    input: code
    output: review_results

  - task: merge
    agent: reporter
    input: [scan_results, review_results]
    output: final_report

error_handling:
  on_failure: retry
  max_retries: 3
```

## 7.4.3 触发器

### 触发类型

```
触发器类型：

1. 事件触发
   ├── PR 创建/合并
   ├── Issue 创建
   ├── 定时任务
   └── Webhook

2. 手动触发
   ├── 用户命令
   ├── API 调用
   └── 按钮点击

3. 条件触发
   ├── 关键词匹配
   ├── 状态变化
   └── 阈值触发
```

### 配置示例

```yaml
trigger:
  type: webhook
  events:
    - pull_request.opened
    - pull_request.synchronize

  filters:
    - type: branch
      pattern: "feature/*"

  payload:
    action: review_request
```

## 7.4.4 任务节点

### 节点类型

```
任务节点类型：

1. Agent 节点
   └── 调用 Agent 执行

2. Tool 节点
   └── 调用工具

3. Condition 节点
   └── 条件判断

4. Transform 节点
   └── 数据转换

5. Delay 节点
   └── 等待/延迟
```

### 节点配置

```yaml
nodes:
  - id: analyze
    type: agent
    agent: analyzer
    input: "{{trigger.payload}}"
    output: analysis_result
    timeout: 300

  - id: check
    type: condition
    condition: "analysis_result.issues > 0"
    on_true: fix
    on_false: approve

  - id: transform
    type: transform
    template: "发现 {{issues}} 个问题"
    output: message
```

## 7.4.5 数据传递

### 变量传递

```
变量引用：
{{node_id.output}}

示例：
{{scan_results.security_issues}}
{{review_results.score}}
```

### 数据转换

```yaml
transforms:
  - name: format_report
    type: template
    template: |
      ## 审查报告

      安全性：{{security_score}}/10
      代码质量：{{quality_score}}/10

      {{#if issues}}
      发现问题：
      {{#each issues}}
      - {{this}}
      {{/each}}
      {{/if}}
```

## 7.4.6 错误处理

### 处理策略

```
错误处理策略：

1. 重试 (retry)
   ├── 自动重试 N 次
   └── 间隔递增

2. 跳过 (skip)
   ├── 跳过失败步骤
   └── 继续执行

3. 终止 (terminate)
   ├── 停止工作流
   └── 通知管理员

4. 回退 (rollback)
   ├── 恢复到之前状态
   └── 清理副作用
```

### 配置示例

```yaml
error_handling:
  strategy: retry_then_notify
  max_retries: 3
  retry_interval: 5
  exponential_backoff: true
  fallback:
    action: skip
    notify: true
  notifications:
    - type: email
      to: admin@example.com
    - type: slack
      channel: "#alerts"
```

## 7.4.7 监控日志

### 状态追踪

```yaml
monitoring:
  enabled: true

  track:
    - node_execution_time
    - success_rate
    - token_usage
    - cost

  logging:
    level: info
    outputs:
      - file: logs/workflow.log
      - console: true

  metrics:
    - name: workflow_duration
      type: histogram
    - name: task_success
      type: counter
```

### 查看状态

```bash
# 查看工作流状态
openclaw workflow status my-workflow

# 查看执行历史
openclaw workflow history my-workflow

# 查看日志
openclaw workflow logs my-workflow --run-id xxx
```

## 7.4.8 常见问题

### Q1: 工作流执行失败

```
问题：工作流中途失败

解决：
1. 查看错误日志
2. 检查输入数据
3. 验证节点配置
4. 检查权限
```

### Q2: 无限等待

```
问题：某个节点一直等待

解决：
1. 设置超时时间
2. 检查依赖数据
3. 验证触发条件
```

### Q3: 数据格式错误

```
问题：节点间数据不匹配

解决：
1. 检查输出格式
2. 使用 Transform 转换
3. 添加数据验证
```
