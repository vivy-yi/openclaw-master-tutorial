## 🚀 5分钟快速启动流程

以下步骤可在 5 分钟内完成 OpenClaw 的安装、配置与首次验证。

### Step 1: 安装 OpenClaw

根据你的操作系统选择合适的安装命令：

```bash
# macOS / Linux
curl -fsSL https://openclaw.ai/install.sh | bash

# Windows (PowerShell)
iwr -useb https://openclaw.ai/install.ps1 | iex

# npm 安装 (全平台通用)
npm install -g openclaw@latest
```

>💡 安装完成后可通过 `openclaw --version` 验证安装是否成功。

---

### Step 2: 运行引导配置

运行引导式安装程序，它会自动完成配置文件创建、服务注册等操作：

```bash
openclaw onboard --install-daemon
```

引导程序会自动完成以下操作：
- 创建配置文件 `~/.openclaw/openclaw.json`
- 注册 Gateway 服务 (后台守护进程)
- 配置默认 Pi Agent (RPC 模式)
- 生成初始工作区结构

---

### Step 3: 验证网关运行状态

检查 OpenClaw Gateway 是否正常运行：

```bash
openclaw gateway status
```

正常输出示例：

```
Runtime:     running
Gateway:     connected
Pi agent:    active
Control:     http://localhost:18789
```

>🔑 **控制面板默认端口**: `18789`

---

### Step 4: 打开控制面板 (可选)

启动 Web 控制面板，直观查看和管理 OpenClaw：

```bash
openclaw dashboard
```

在浏览器打开 `http://localhost:18789` 即可进入控制面板。

---

### Step 5: 发送第一条消息

通过任意已配置的渠道（如 Telegram、Discord）向 OpenClaw 发送消息，即可开始使用。

> 📖 如果尚未配置渠道，请参考 [2.2 安装与配置](./2.2_installation.md) 中的渠道配置章节。

## ❓ 常见问题

### Q1: 安装后 `openclaw: command not found`

确保 PATH 中包含 npm 全局模块路径：

```bash
# 添加到 ~/.zshrc 或 ~/.bashrc
export PATH="$(npm root -g):$PATH"
```

然后重新加载配置：
```bash
source ~/.zshrc   # macOS 默认 shell
```

### Q2: `openclaw onboard` 卡住不动

尝试以管理员权限运行，或先运行诊断：
```bash
openclaw doctor
```

### Q3: Gateway 显示 `disconnected`

检查 Node 版本是否满足要求（22.19+），然后重启：
```bash
openclaw gateway restart
```

### Q4: 控制面板无法访问

确认端口 18789 未被占用：
```bash
# macOS
lsof -i :18789

# Linux
ss -tlnp | grep 18789
```
