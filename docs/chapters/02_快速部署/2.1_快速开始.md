## 什么是 OpenClaw？

OpenClaw 是一个**自托管 AI 网关**，用于将聊天应用（Discord、Telegram、QQ 等）连接到 AI 编码代理（如 Claude Code、OpenAI等）。

**适用人群**：希望自己掌控数据、不依赖第三方托管服务的开发者和技术用户。

---

## 前置要求

| 项目 | 要求 |
|------|------|
| **Node.js** | Node 24（推荐）或 Node 22.16+ |
| **包管理器** | npm / pnpm / yarn |
| **API Key** | 至少一个：Anthropic / OpenAI / Google 等 |
| **操作系统** | macOS / Linux / Windows (WSL) |
| **磁盘空间** | 约 100 MB |

## 安装步骤

### 方式一：官方安装脚本（推荐 ⭐）

```bash
curl -fsSL https://openclaw.ai/install.sh | bash
```

> 官方安装脚本会自动检测你的操作系统，下载并安装对应版本的 OpenClaw。

### 方式二：npm 全局安装

```bash
npm install -g openclaw@latest
```

### 方式三：Homebrew（macOS）

```bash
brew install openclaw
```

---

## 初始化并安装服务

安装完成后，运行引导程序：

```bash
openclaw onboard --install-daemon
```

**引导程序自动完成以下操作**：
- 创建配置文件 `~/.openclaw/openclaw.json`
- 注册 Gateway 系统服务（daemon 模式）
- 配置默认 Pi agent（RPC 模式）
- 验证 API Key 配置

---

## 验证安装

### 检查 Gateway 运行状态

```bash
openclaw gateway status
```

**正常输出示例**：
```
Runtime: running
Gateway: connected
Pi agent: active
```

### 打开控制台界面

```bash
openclaw dashboard
```

> 默认浏览器会自动打开 `http://localhost:8080`，进入 OpenClaw Web 控制台。

### 发送第一条消息

通过控制台或任意已配置的渠道发送消息，验证 Agent 响应是否正常。

## 高级配置选项

### 自定义 Control UI 构建

如果需要使用自定义构建的 Control UI：

```bash
openclaw dashboard --custom-build /path/to/your/build
```

### 环境变量配置

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| `OPENCLAW_HOME` | OpenClaw 根目录 | `~/.openclaw` |
| `OPENCLAW_STATE_DIR` | 状态文件目录 | `~/.openclaw/state` |
| `OPENCLAW_CONFIG_PATH` | 配置文件路径 | `~/.openclaw/openclaw.json` |

**配置示例**：
```bash
export OPENCLAW_HOME=/opt/openclaw
export OPENCLAW_CONFIG_PATH=/opt/openclaw/config.json
```

### API Key 配置

编辑 `~/.openclaw/openclaw.json` 或通过控制台添加：

```json
{
  "providers": {
    "anthropic": {
      "apiKey": "sk-ant-your-key-here"
    }
  }
}
```

## 常见问题

### Q1: `openclaw: command not found`

确保 npm 全局 bin 目录在 PATH 中：

```bash
export PATH="$(npm root -g)/bin:$PATH"
```

或将此行添加到 `~/.zshrc` / `~/.bashrc`。

### Q2: Node.js 版本过低

```bash
# 使用 nvm 升级 Node.js
nvm install 24
nvm use 24
```

### Q3: Gateway 无法连接

```bash
# 重启 Gateway
openclaw gateway restart

# 查看详细日志
openclaw gateway logs
```

### Q4: API Key 无效

检查 `openclaw.json` 中的 API Key 是否正确，确认提供商账户有效且额度充足。
