## 7.1 Star-Office-UI 概述

### 什么是 Star-Office-UI

Star-Office-UI 是一个**像素风格的 AI 办公室看板**，可以把 AI 助手的工作状态实时可视化，让你直观看到"谁在做什么、昨天做了什么、现在是否在线"。

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                         Star-Office-UI 核心功能                                │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│  🎮 状态可视化        6 种状态自动映射到办公室不同区域                        │
│  📝 昨日小记         自动从 memory/*.md 读取工作记录                        │
│  👥 多 Agent 协作     邀请其他 Agent 加入办公室                               │
│  🌐 中英日三语       CN / EN / JP 一键切换                                  │
│  🎨 AI 生图装修      接入 Gemini API 换背景                                 │
│  📱 移动端适配        手机直接查看                                           │
│  🔒 安全加固          密码保护、Session 加固                                 │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 6 种状态映射

| 状态 | 办公室区域 | 触发场景 |
|------|-----------|---------|
| `idle` | 🛋 休息区（沙发） | 待命 / 任务完成 |
| `writing` | 💻 工作区（办公桌） | 写代码 / 写文档 |
| `researching` | 💻 工作区 | 搜索 / 调研 |
| `executing` | 💻 工作区 | 执行命令 / 跑任务 |
| `syncing` | 💻 工作区 | 同步数据 / 推送 |
| `error` | 🐛 Bug 区 | 报错 / 异常排查 |

## 7.2 快速部署

### 方式一：让 OpenClaw 自动部署（推荐）

如果正在使用 OpenClaw，直接发送以下消息给 Agent：

```
请按照这个 SKILL.md 帮我完成 Star Office UI 的部署：
https://github.com/ringhyacinth/Star-Office-UI/blob/master/SKILL.md
```

Agent 会自动完成：
- clone 仓库
- 安装依赖
- 启动后端
- 配置状态同步
- 返回访问地址

### 方式二：手动部署

```bash
# 1) 下载仓库
git clone https://github.com/ringhyacinth/Star-Office-UI.git
cd Star-Office-UI

# 2) 安装依赖（需要 Python 3.10+）
python3 -m pip install -r backend/requirements.txt

# 3) 准备状态文件
cp state.sample.json state.json

# 4) 启动后端
cd backend
python3 app.py
```

打开 **http://127.0.0.1:19000** 即可访问。

## 7.3 与 OpenClaw 深度集成

### 状态自动同步配置

在 Agent 的 `SOUL.md` 或规则文件中加入：

```markdown
## Star Office 状态同步规则

- 接到任务时：先执行 `python3 set_state.py <状态> "<描述>"` 再开始工作
- 完成任务后：执行 `python3 set_state.py idle "待命中"` 再回复
```

### 配置步骤

#### Step 1：获取推送脚本

将 `set_state.py` 和 `office-agent-push.py` 复制到 Agent 可访问的目录：

```bash
cp set_state.py ~/.openclaw/
cp office-agent-push.py ~/.openclaw/
```

#### Step 2：设置状态文件路径

编辑 `set_state.py`，确保状态文件路径正确：

```python
STATE_FILE = os.path.expanduser("~/.openclaw/state.json")
```

#### Step 3：在 Agent 规则中使用

在 Agent 的系统提示中添加：

```markdown
## 状态管理

当你开始工作时，使用以下命令更新状态：

- 写代码/文档时：`python3 set_state.py writing "正在写代码"`
- 执行命令时：`python3 set_state.py executing "正在执行命令"`
- 搜索调研时：`python3 set_state.py researching "正在搜索信息"`
- 同步数据时：`python3 set_state.py syncing "正在同步数据"`
- 遇到错误时：`python3 set_state.py error "发现问题"`

完成任务后：`python3 set_state.py idle "待命中"`
```

## 7.4 多 Agent 协作

### 邀请其他 Agent 加入

#### Step 1：准备 join key

首次启动后端时，会自动生成 `join-keys.json`，包含示例 key（例如 `ocj_example_team_01`）。

可以自行修改 key：

```json
{
  "keys": [
    {
      "key": "ocj_starteam01",
      "max_agents": 5,
      "description": "主团队"
    }
  ]
}
```

#### Step 2：访客 Agent 配置

访客 Agent 下载 `office-agent-push.py`，填写配置：

```python
JOIN_KEY = "ocj_starteam01"           # 分配的 key
AGENT_NAME = "小明的龙虾"            # 显示名称
OFFICE_URL = "https://office.example.com"  # 办公室地址
```

#### Step 3：启动推送

```bash
python3 office-agent-push.py
```

脚本会自动：
- 加入办公室
- 每 15 秒推送一次状态
- 根据状态自动走到对应区域

## 7.5 常用命令

### 状态切换

```bash
# 写代码状态
python3 set_state.py writing "正在整理文档"

# 执行命令状态
python3 set_state.py executing "正在部署服务"

# 搜索状态
python3 set_state.py researching "正在搜索资料"

# 同步状态
python3 set_state.py syncing "同步进度中"

# 错误状态
python3 set_state.py error "发现问题，排查中"

# 空闲状态
python3 set_state.py idle "待命中"
```

### API 调用

```bash
# 设置状态
curl -X POST http://127.0.0.1:19000/set_state \
  -H "Content-Type: application/json" \
  -d '{"state": "writing", "description": "正在写代码"}'

# 获取状态
curl http://127.0.0.1:19000/status

# 获取昨日小记
curl http://127.0.0.1:19000/yesterday-memo

# 获取所有 Agent
curl http://127.0.0.1:19000/agents
```

## 7.6 公网访问配置

### 使用 Cloudflare Tunnel

```bash
cloudflared tunnel --url http://127.0.0.1:19000
```

获取 `https://xxx.trycloudflare.com` 链接即可分享。

### 安全配置

首次部署后，创建 `.env` 文件：

```bash
cp .env.example .env
```

设置以下环境变量：

```bash
# Flask 密钥（必须设置强随机值）
FLASK_SECRET_KEY=your-random-secret-key

# 资产管理员密码
ASSET_DRAWER_PASS=your-strong-password
```
