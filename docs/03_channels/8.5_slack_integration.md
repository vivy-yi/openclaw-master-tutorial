# 8.5 Slack 集成

本节讲解如何将 Slack 与 OpenClaw 集成，实现企业级团队协作。

---

## 8.5.1 概述

Slack 是企业常用的团队协作工具。OpenClaw 支持两种连接模式：

- **Socket Mode（默认）**：通过 WebSocket 长连接
- **HTTP Events API**：通过 HTTP 请求

---

## 8.5.2 快速开始

### 步骤 1：创建 Slack App

1. 访问 [Slack API](https://api.slack.com/apps)
2. 创建新应用，选择"From scratch"
3. 选择工作区

### 步骤 2：配置 App 权限

启用以下功能：

```
┌─────────────────────────────────────────────────────────────┐
│                    需要启用的功能                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. Socket Mode                                             │
│     └── 启用并创建 App Token (xapp-...)                      │
│         需 connections:write 权限                            │
│                                                             │
│  2. Event Subscriptions                                     │
│     ├── app_mention                                          │
│     ├── message.channels                                    │
│     ├── message.groups                                      │
│     ├── message.im                                          │
│     ├── reaction_added                                      │
│     └── member_joined_channel                               │
│                                                             │
│  3. OAuth Scope                                             │
│     ├── chat:write                                           │
│     ├── channels:read                                       │
│     ├── groups:read                                         │
│     ├── im:read                                             │
│     └── reactions:write                                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 步骤 3：配置 OpenClaw

```json5
{
  channels: {
    slack: {
      enabled: true,
      mode: "socket",
      appToken: "xapp-...",
      botToken: "xoxb-...",
    },
  },
}
```

或使用环境变量：

```bash
SLACK_APP_TOKEN=xapp-...
SLACK_BOT_TOKEN=xoxb-...
```

## 8.5.3 Socket Mode 配置（默认）

```json5
{
  channels: {
    slack: {
      enabled: true,
      mode: "socket",
      appToken: "xapp-你的AppToken",
      botToken: "xoxb-你的BotToken",
    },
  },
}
```

### App Token 权限

创建 App Token 需要以下权限：
- `connections:write`

---

## 8.5.4 HTTP Events API 配置

```json5
{
  channels: {
    slack: {
      enabled: true,
      mode: "http",
      botToken: "xoxb-你的BotToken",
      signingSecret: "你的SigningSecret",
      webhookPath: "/slack/events",
    },
  },
}
```

### 配置步骤

1. 在 Slack App 设置中启用 **Event Subscriptions**
2. 设置 Request URL 为：`https://你的Gateway/slack/events`
3. 启用需要的事件订阅

## 8.5.5 访问控制

### DM 策略

```json5
{
  channels: {
    slack: {
      dmPolicy: "pairing",  // pairing | allowlist | open | disabled
      allowFrom: ["user1", "user2"],
      dm: {
        enabled: true,
        groupEnabled: false,
      },
    },
  },
}
```

| 策略 | 说明 |
|------|------|
| `pairing` | 需要配对批准 |
| `allowlist` | 仅允许白名单用户 |
| `open` | 开放（需设置 allowFrom: ["*"]） |
| `disabled` | 禁用 DM |

### 频道策略

```json5
{
  channels: {
    slack: {
      groupPolicy: "allowlist",  // open | allowlist | disabled
      channels: {
        "C1234567890": {
          requireMention: true,
          skills: ["coding"],
        },
      },
    },
  },
}
```

## 8.5.6 消息与交互

### 触发方式

```
┌─────────────────────────────────────────────────────────────┐
│                    消息触发方式                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. @mention: <@BotID>                                    │
│  2. 提及模式: agents配置的mentionPatterns                   │
│  3. 线程回复: 在Bot创建的线程中回复                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 支持的功能

- 文本消息
- 媒体消息（图片、文件）
- 消息反应
- 频道/群组/DM
- Slash 命令

## 8.5.7 多账户配置

```json5
{
  channels: {
    slack: {
      accounts: {
        default: {
          botToken: "xoxb-default-...",
        },
        work: {
          botToken: "xoxb-work-...",
          accountId: "work-workspace",
        },
      },
    },
  },
}
```

## 本节小结

1. **两种模式**：Socket Mode（默认）和 HTTP Events API
2. **配置要点**：App Token、Bot Token、事件订阅
3. **访问控制**：DM 策略和频道策略
4. **触发方式**：@mention、提及模式、线程回复
