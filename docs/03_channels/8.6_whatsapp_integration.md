# 8.6 WhatsApp 集成

本节讲解如何将 WhatsApp 与 OpenClaw 集成，使用 WhatsApp Web（Baileys）协议。

---

## 8.6.1 概述

WhatsApp 是全球最流行的即时通讯工具之一。OpenClaw 通过 WhatsApp Web（Baileys）实现连接。

```
★ Insight ─────────────────────────────────────
WhatsApp 集成特点：
1. Gateway 管理配对的会话
2. 需要 QR 码配对
3. 支持 DM 和群组
4. 建议使用独立号码
─────────────────────────────────────────────────
```

---

## 8.6.2 快速开始

### 步骤 1：配置访问策略

```json5
{
  channels: {
    whatsapp: {
      dmPolicy: "pairing",
      allowFrom: ["+15551234567"],
      groupPolicy: "allowlist",
      groupAllowFrom: ["+15551234567"],
    },
  },
}
```

### 步骤 2：QR 配对

```bash
# 登录 WhatsApp
openclaw channels login --channel whatsapp

# 指定账户
openclaw channels login --channel whatsapp --account work
```

### 步骤 3：启动 Gateway

```bash
openclaw gateway
```

### 步骤 4：批准配对请求

```bash
# 查看配对请求
openclaw pairing list whatsapp

# 批准配对
openclaw pairing approve whatsapp <CODE>
```

## 8.6.2 访问控制

### DM 策略

```json5
{
  channels: {
    whatsapp: {
      dmPolicy: "pairing",  // pairing | allowlist | open | disabled
      allowFrom: ["+15551234567", "+15559876543"],
    },
  },
}
```

| 策略 | 说明 |
|------|------|
| `pairing` | 需要配对批准 |
| `allowlist` | 仅允许白名单号码 |
| `open` | 开放（需设置 allowFrom: ["*"]） |
| `disabled` | 禁用 DM |

### 群组策略

```json5
{
  channels: {
    whatsapp: {
      groupPolicy: "allowlist",  // open | allowlist | disabled
      groupAllowFrom: ["+15551234567"],
    },
  },
}
```

## 8.6.4 多账户配置

```json5
{
  channels: {
    whatsapp: {
      accounts: {
        default: {
          dmPolicy: "pairing",
        },
        work: {
          accountId: "work-number",
          dmPolicy: "allowlist",
          allowFrom: ["+15551234567"],
        },
      },
    },
  },
}
```

## 8.6.5 故障排除

### 常见问题

**Q: QR 码无法扫描**

```
解决：
1. 检查网络连接
2. 确保 WhatsApp Web 已退出其他设备
3. 重新生成 QR 码
```

**Q: 消息发送失败**

```
解决：
1. 检查配对是否过期
2. 重新登录：openclaw channels login --channel whatsapp
3. 查看日志：openclaw logs
```

**Q: 配对请求超时**

```
说明：配对请求 1 小时后过期
解决：重新发起登录请求
```
