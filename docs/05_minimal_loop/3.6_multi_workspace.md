## 3.6.1 多工作空间概述

### 什么是多工作空间

多工作空间是 OpenClaw 的核心架构特性，每个 Agent 拥有完全独立的工作空间，实现环境隔离和专业化分工。

```
┌─────────────────────────────────────────────────────────────────────────────────┐
│                          多工作空间架构                                         │
├─────────────────────────────────────────────────────────────────────────────────┤
│                                                                                 │
│                    ~/.openclaw/                                              │
│    ┌─────────────────────────────────────────────────────────────────────┐   │
│    │                                                                   │   │
│    │   ┌─────────────┐  ┌─────────────┐  ┌─────────────┐             │   │
│    │   │   main      │  │   work     │  │   coding    │   ...     │   │
│    │   │  Agent      │  │  Agent      │  │  Agent      │             │   │
│    │   ├─────────────┤  ├─────────────┤  ├─────────────┤             │   │
│    │   │ workspace/  │  │ workspace/  │  │ workspace/  │             │   │
│    │   │ 独立文件系统 │  │ 独立文件系统 │  │ 独立文件系统 │             │   │
│    │   ├─────────────┤  ├─────────────┤  ├─────────────┤             │   │
│    │   │ agent/     │  │ agent/     │  │ agent/     │             │   │
│    │   │ 独立状态   │  │ 独立状态   │  │ 独立状态   │             │   │
│    │   ├─────────────┤  ├─────────────┤  ├─────────────┤             │   │
│    │   │ sessions/  │  │ sessions/  │  │ sessions/  │             │   │
│    │   │ 独立会话   │  │ 独立会话   │  │ 独立会话   │             │   │
│    │   └─────────────┘  └─────────────┘  └─────────────┘             │   │
│    │                                                                   │   │
│    └─────────────────────────────────────────────────────────────────────┘   │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘
```

### 每个工作空间包含

| 组件 | 说明 |
|-----|------|
| **Workspace** | 文件系统、AGENTS.md/SOUL.md/USER.md 等配置文件 |
| **Agent Directory** | 认证配置、模型注册表、Agent 专属状态 |
| **Session Store** | 对话历史和路由状态 |
| **Auth Profiles** | 独立的 API 密钥和 OAuth 令牌 |
| **Skills** | 可配置独立的技能列表 |

## 3.6.2 创建工作空间

### CLI 命令创建

```bash
# 创建新的工作空间 (Agent)
openclaw agents add work

# 创建指定模型的工作空间
openclaw agents add coding --model claude-sonnet-4-20250514

# 创建多个工作空间
openclaw agents add research
openclaw agents add writing
openclaw agents add analysis
```

### 手动配置

在 `openclaw.json` 中配置：

```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "default": true,
        "name": "Main Agent",
        "workspace": "~/.openclaw/workspace"
      },
      {
        "id": "work",
        "name": "Work Agent",
        "workspace": "~/.openclaw/workspace-work"
      },
      {
        "id": "coding",
        "name": "Coding Agent",
        "workspace": "~/.openclaw/workspace-coding",
        "model": "anthropic/claude-sonnet-4-20250514"
      }
    ]
  }
}
```

### 配置字段说明

| 字段 | 类型 | 说明 |
|-----|------|------|
| `id` | string | Agent 唯一标识 (小写字母、数字、连字符) |
| `name` | string | 显示名称 |
| `workspace` | string | 工作空间路径 |
| `model` | string | 使用的模型 |
| `default` | boolean | 是否为默认 Agent |

## 3.6.4 工作空间管理命令

### Agent 管理

```bash
# 列出所有 Agent
openclaw agents list

# 查看 Agent 详情
openclaw agents info <agent-id>

# 查看 Agent 状态
openclaw agents status

# 删除 Agent
openclaw agents remove <agent-id>

# 重置 Agent
openclaw agents reset <agent-id>
```

### 工作空间切换

```bash
# 切换到指定 Agent 的工作空间
openclaw chat --agent work

# 使用指定 Agent 执行命令
openclaw exec --agent coding "帮我写一个排序算法"

# 指定工作空间启动
openclaw gateway --workspace ~/.openclaw/workspace-coding
```

### 状态查看

```bash
# 查看当前工作空间
openclaw status | grep workspace

# 查看 Agent 会话
openclaw sessions list --agent work

# 查看工作空间文件
openclaw files list workspace/
```

## 3.6.6 多工作空间配置

### 配置多个工作空间

```json
{
  "agents": {
    "defaults": {
      "workspace": "~/.openclaw/workspace",
      "model": "deepseek/deepseek-chat"
    },
    "list": [
      {
        "id": "main",
        "default": true,
        "name": "日常助手",
        "workspace": "~/.openclaw/workspace"
      },
      {
        "id": "work",
        "name": "工作专家",
        "workspace": "~/.openclaw/workspace-work",
        "model": "anthropic/claude-sonnet-4-20250514"
      },
      {
        "id": "coding",
        "name": "编程助手",
        "workspace": "~/.openclaw/workspace-coding",
        "model": "anthropic/claude-opus-4-6"
      }
    ]
  },
  "bindings": [
    {
      "agentId": "main",
      "match": {
        "channel": "telegram"
      }
    },
    {
      "agentId": "work",
      "match": {
        "channel": "slack"
      }
    },
    {
      "agentId": "coding",
      "match": {
        "channel": "discord",
        "guildId": "123456789"
      }
    }
  ]
}
```

### 工作空间权限配置

```json
{
  "agents": {
    "list": [
      {
        "id": "restricted",
        "workspace": "~/.openclaw/workspace-restricted",
        "tools": {
          "allow": ["read", "grep", "glob"],
          "deny": ["write", "edit", "exec", "bash"]
        },
        "sandbox": {
          "mode": "readonly",
          "scope": "workspace"
        }
      }
    ]
  }
}
```

### 工具限制说明

```json
// 工具限制配置
{
  "tools": {
    "allow": [
      "read",      // 读取文件
      "glob",      // 搜索文件
      "grep",      // 搜索内容
      "sessions_list",    // 列出会话
      "sessions_history" // 查看会话历史
    ],
    "deny": [
      "write",     // 写入文件
      "edit",      // 编辑文件
      "exec",      // 执行命令
      "bash",      // 运行脚本
      "browser",   // 浏览器操作

    ]
  }
```

### 隔离原则

```bash
# 推荐配置原则

✅ 应该隔离：
- 不同客户的代码/文档
- 生产环境和开发环境
- 敏感项目和普通项目
- 不同团队的资产

❌ 不需要隔离：
- 日常对话
- 简单查询
- 临时任务
```

### 维护建议

```bash
# 定期维护任务

1. 清理会话

2. 清理临时文件
openclaw workspace clean

3. 备份重要配置
openclaw config export

4. 检查磁盘空间
df -h ~/.openclaw/
```

## 3.6.8 常见问题

### Q1: 工作空间创建失败

```bash
# 检查权限
ls -la ~/.openclaw/

# 手动创建
mkdir -p ~/.openclaw/workspace-new
touch ~/.openclaw/workspace-new/AGENTS.md

# 重新加载配置
openclaw gateway restart
```

### Q2: 消息没有路由到正确的工作空间

```bash
# 检查 bindings 配置
openclaw config get bindings

# 检查路由规则
openclaw agents list --bindings

# 测试路由
openclaw gateway call message.route --params '{"source":"telegram","peer":"123456"}'
```

### Q3: 工作空间之间共享数据

```bash
# 使用共享目录
# 1. 创建共享目录
mkdir ~/.openclaw/shared

# 2. 在各工作空间创建软链接
ln -s ~/.openclaw/shared ~/.openclaw/workspace-work/shared
ln -s ~/.openclaw/shared ~/.openclaw/workspace-coding/shared
```

## 本节小结

1. **多工作空间**：每个 Agent 拥有独立的文件系统、状态目录、会话存储
2. **创建方式**：agents add CLI 或手动配置 JSON
3. **核心文件**：AGENTS.md、SOUL.md、USER.md、IDENTITY.md
4. **路由机制**：通过 bindings 配置消息路由
5. **权限控制**：工具白名单/黑名单、沙箱隔离
