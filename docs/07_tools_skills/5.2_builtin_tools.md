## 5.2.1 文件操作工具

### read - 读取文件

```
功能：读取文件或目录内容

参数：
├── path: 文件/目录路径（必填）
├── limit: 读取行数（可选）
└── offset: 起始行号（可选）

示例：
用户：读取 /home/user/test.txt 的前 100 行
```

```json
{
  "tool": "read",
  "params": {
    "path": "/home/user/test.txt",
    "limit": 100
  }
}
```

### write - 写入文件

```
功能：创建或覆盖文件

参数：
├── path: 文件路径（必填）
├── content: 文件内容（必填）
└── mode: 写入模式（可选，write/append）

示例：
用户：创建一个新文件
```

```json
{
  "tool": "write",
  "params": {
    "path": "~/workspace/notes.txt",
    "content": "这是文件内容"
  }
}
```

### edit - 编辑文件

```
功能：修改文件内容

参数：
├── path: 文件路径（必填）
├── old_string: 原内容（必填）
├── new_string: 新内容（必填）
└── replace_all: 全部替换（可选）

示例：
用户：把文件中的 "foo" 改成 "bar"
```

```json
{
  "tool": "edit",
  "params": {
    "path": "~/workspace/test.txt",
    "old_string": "foo",
    "new_string": "bar"
  }
}
```

### glob - 文件搜索

```
功能：按模式搜索文件

参数：
├── pattern: 搜索模式（必填）
│   ├── *.txt - 所有 txt 文件
│   ├── **/*.js - 递归搜索
│   └── src/**/*.{ts,tsx} - 多类型
├── path: 搜索路径（可选）
└── limit: 数量限制（可选）

示例：
用户：搜索所有 Python 文件
```

```json
{
  "tool": "glob",
  "params": {
    "pattern": "**/*.py",
    "path": "~/workspace"
  }
}
```

## 5.2.2 命令执行工具

### exec - 执行命令

```
功能：执行系统命令

参数：
├── command: 命令（必填）
├── timeout: 超时时间（可选，默认 30s）
├── description: 描述（可选）
└── workdir: 工作目录（可选）

示例：
用户：运行 git status
```

```json
{
  "tool": "exec",
  "params": {
    "command": "git status",
    "timeout": 30000,
    "workdir": "~/projects/myapp"
  }
}
```

### bash - Shell 脚本

```
功能：执行 Shell 脚本

参数：
├── script: 脚本内容（必填）
├── timeout: 超时时间（可选）
└── env: 环境变量（可选）

示例：
用户：批量重命名文件
```

```json
{
  "tool": "bash",
  "params": {
    "script": "for f in *.txt; do mv \"$f\" \"${f%.txt}.md\"; done"
  }
}
```

## 5.2.3 网络请求工具

### web - 网页请求

```
功能：发送 HTTP 请求

参数：
├── url: 请求 URL（必填）
├── method: 方法（可选，默认 GET）
├── headers: 请求头（可选）
├── body: 请求体（可选）
└── timeout: 超时时间（可选）

示例：
用户：获取网页内容
```

```json
{
  "tool": "web",
  "params": {
    "url": "https://api.example.com/data",
    "method": "GET",
    "headers": {
      "Authorization": "Bearer token"
    }
  }
}
```

### browser - 浏览器自动化

```
功能：控制浏览器

命令：
├── goto: 访问页面
├── click: 点击元素
├── input: 输入文字
├── screenshot: 截图
├── extract: 提取内容
└── wait: 等待元素

示例：
用户：打开百度并搜索
```

```json
{
  "tool": "browser",
  "params": {
    "action": "goto",
    "url": "https://www.baidu.com"
  }
}
```

## 5.2.4 数据处理工具

### json - JSON 处理

```
功能：处理 JSON 数据

操作：
├── parse: 解析 JSON
├── stringify: 转为字符串
├── extract: 提取字段
├── query: JSONPath 查询
└── validate: 验证格式

示例：
用户：从 JSON 中提取 name 字段
```

```json
{
  "tool": "json",
  "params": {
    "action": "extract",
    "data": {"name": "张三", "age": 30},
    "path": "name"
  }
}
```

### grep - 内容搜索

```
功能：在文件中搜索内容

参数：
├── pattern: 搜索模式（必填）
├── path: 搜索路径（必填）
├── regex: 正则表达式（可选）
├── ignore_case: 忽略大小写（可选）
└── context: 上下文行数（可选）

示例：
用户：搜索所有包含 "TODO" 的文件
```

```json
{
  "tool": "grep",
  "params": {
    "pattern": "TODO",
    "path": ".",
    "context": 3
  }
}
```

## 5.2.5 系统工具

### ps - 进程列表

```
功能：查看进程信息

参数：
├── all: 显示全部（可选）
├── filter: 过滤条件（可选）
└── limit: 数量限制（可选）

示例：
用户：查看所有 Node 进程
```

```json
{
  "tool": "ps",
  "params": {
    "filter": "node"
  }
}
```

### env - 环境变量

```
功能：读取环境变量

参数：
├── name: 变量名（可选）
└── all: 全部变量（可选）

示例：
用户：查看 PATH 变量
```

```json
{
  "tool": "env",
  "params": {
    "name": "PATH"
  }
}
```

## 5.2.6 工具使用示例

### 场景一：读取并修改配置

```
用户：把 config.json 中的端口改成 8080
```

```
Step 1: 读取配置
→ read("config.json")
← {"port": 3000, "host": "localhost"}

Step 2: 修改配置
→ edit("config.json", "3000", "8080")
← 已修改

Step 3: 验证修改
→ read("config.json")
← {"port": 8080, "host": "localhost"}
```

### 场景二：执行构建命令

```
用户：运行 npm build
```

```
Step 1: 执行命令
→ exec("npm run build", timeout=120000)
← 构建成功，输出已保存
```

### 场景三：网页内容提取

```
用户：获取天气信息
```

```
Step 1: 访问天气网站
→ browser.goto("https://weather.example.com")

Step 2: 提取内容
→ browser.extract(".temperature")
← 25°C
```

## 5.2.7 常见问题

### Q1: 文件路径错误

```
错误：File not found

解决：
1. 检查路径是否正确
2. 使用绝对路径
3. 确认文件存在
```

### Q2: 权限不足

```
错误：Permission denied

解决：
1. 检查文件权限
2. 使用 chmod 修改权限
3. 以管理员身份运行
```

### Q3: 命令执行超时

```
错误：Command timeout

解决：
1. 增加 timeout 参数
2. 优化命令逻辑
3. 检查系统资源
```
