# 5.6.5 Chrome 扩展中继

本节讲解如何使用 Chrome 扩展中继模式，让 Agent 控制你已登录的现有 Chrome 标签页。

---

## 5.6.5.1 什么是 Chrome 扩展中继

Chrome 扩展中继允许 OpenClaw 控制**你现有的 Chrome 标签页**（而非独立的 OpenClaw 浏览器实例），通过本地 CDP 中继 + Chrome 扩展实现。

```
┌─────────────────────────────────────────────────────────────┐
│                 Chrome 扩展中继架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌──────────┐      CDP       ┌─────────────────────────┐   │
│  │ Gateway/ │ ──────────────→ │  CDP 中继服务器          │   │
│  │ Node Host │   (loopback)  │  (127.0.0.1:18792)      │   │
│  └──────────┘                └───────────┬─────────────┘   │
│                                          │                  │
│                                    CDP   │                  │
│                                          ↓                  │
│                                   ┌──────────────┐           │
│                                   │ Chrome 扩展  │           │
│                                   │ (OpenClaw   │           │
│                                   │  Browser    │           │
│                                   │  Relay)     │           │
│                                   └──────────────┘           │
│                                          │                  │
│                                          ↓                  │
│                                   ┌──────────────┐           │
│                                   │ Chrome 标签页 │           │
│                                   │ (你已登录)   │           │
│                                   └──────────────┘           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 使用场景

- **需要已登录状态**：如 Gmail、GitHub、Slack 等需要登录的网站
- **使用已有 Chrome 配置**：保留你的书签、扩展、密码管理器等

---

## 5.6.5.2 安装扩展

### 步骤 1：安装扩展

```bash
# 安装扩展到稳定路径
openclaw browser extension install

# 查看扩展路径
openclaw browser extension path
```

### 步骤 2：在 Chrome 中加载扩展

1. 打开 Chrome，访问 `chrome://extensions`
2. 启用 **"开发者模式"** (Developer mode)
3. 点击 **"加载已解压的扩展程序"** (Load unpacked)
4. 选择上一步输出的路径
5. 固定扩展图标到工具栏

### 步骤 3：使用扩展

1. 打开你要控制的标签页
2. 点击扩展图标（徽章显示 `OFF`）
3. 徽章变为 `ON`，表示已附着

> 注意：扩展**不会自动附着**，每次需要手动点击

---

## 5.6.5.3 使用 Chrome Profile

### 基本命令

```bash
# 查看 chrome Profile 的 Tab
openclaw browser --browser-profile chrome tabs

# 使用 chrome Profile 打开网页
openclaw browser --browser-profile chrome open https://gmail.com

# 使用 chrome Profile 获取快照
openclaw browser --browser-profile chrome snapshot

# 断开：再次点击扩展图标
```

### 沙箱会话注意

如果 Agent 会话是沙箱化的，`browser` 工具可能默认使用 `target="sandbox"`（沙箱浏览器）。Chrome 扩展中继需要主机浏览器控制，需要：

1. 运行非沙箱会话，或
2. 设置 `agents.defaults.sandbox.browser.allowHostControl: true` 并在调用时使用 `target="host"`

---

## 5.6.5.4 自定义 Profile

### 创建自定义 Chrome 中继 Profile

```bash
openclaw browser create-profile \
  --name my-chrome \
  --driver extension \
  --cdp-url http://127.0.0.1:18792 \
  --color "#00AA00"
```

### 配置参数

| 参数 | 说明 |
|------|------|
| `--name` | Profile 名称 |
| `--driver` | 驱动类型 (`extension`) |
| `--cdp-url` | CDP 中继地址 |
| `--color` | 主题色 |

---

## 5.6.5.5 跨网络配置

如果 Gateway 运行在不同网络命名空间（如 WSL2 上的 Gateway，Chrome 在 Windows 上），需要配置中继绑定地址。

### WSL2 + Windows 示例

```json5
{
  browser: {
    enabled: true,
    relayBindHost: "0.0.0.0",  // 允许跨命名空间访问
    defaultProfile: "chrome"
  }
}
```

> 注意：保持周围网络私有和认证

---

## 5.6.5.6 与 openclaw 模式对比

| 特性 | openclaw 模式 | chrome 模式 |
|------|---------------|-------------|
| 浏览器实例 | 独立的新 Chrome | 你现有的 Chrome |
| 登录状态 | 需要重新登录 | 保持现有登录 |
| 扩展支持 | 不支持 | 支持 |
| 配置隔离 | 完全隔离 | 共享配置 |
| 无头模式 | 支持 | 不支持 |
| 安装需求 | 无 | 需要安装扩展 |

---

## 5.6.5.7 完整示例

```bash
# 1. 确保扩展已安装
openclaw browser extension path
# 输出: /Users/xxx/.openclaw/browser-extensions/xxx

# 2. 设置默认 Profile 为 chrome
openclaw config set browser.defaultProfile "chrome"

# 3. 打开 Chrome 并附着目标标签页
# (手动点击扩展图标)

# 4. 使用 Agent 控制
# Agent 会使用 chrome Profile 进行操作

# 5. 交互式快照
openclaw browser snapshot --interactive
```

---

## 5.6.5.8 注意事项

```
★ Insight ─────────────────────────────────────
Chrome 扩展中继注意事项：
1. 扩展需要手动附着，不能自动附着
2. 只能控制已附着的标签页
3. 关闭标签页会自动解除附着
4. CDP 中继默认仅限 loopback
5. 沙箱会话可能需要额外配置
─────────────────────────────────────────────────
```

---

## 本节小结

1. **扩展中继**：控制现有 Chrome 标签页
2. **安装步骤**：extension install → Load unpacked → 点击附着
3. **使用场景**：需要保持登录状态
4. **沙箱注意**：需要配置 allowHostControl
5. **跨网络**：配置 relayBindHost
