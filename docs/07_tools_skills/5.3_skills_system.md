## 5.3.1 什么是 Skills

### Skills vs Tools

```
┌─────────────────────────────────────────────┐
│            Skills 概念                        │
├─────────────────────────────────────────────┤
│                                             │
│  Skills = 能力封装 = 提示词 + 工具 + 工作流    │
│                                             │
│  例如：代码审查 Skill                         │
│  ├── 提示词：审查标准、关注点                │
│  ├── 工具：read、grep、exec                │
│  └── 工作流：读取 → 分析 → 输出             │
│                                             │
│  1800+ Skills 生态 ⭐                        │
│                                             │
└─────────────────────────────────────────────┘
```

### Skills 的价值

```
为什么使用 Skills：

1. 复用专家知识
   ├── 代码审查标准
   ├── 设计模式
   └── 最佳实践

2. 降低使用门槛
   ├── 无需编写复杂提示词
   ├── 一键启用专家能力
   └── 标准化输出

3. 组合扩展
   ├── Skills 可以组合
   ├── 支持嵌套调用
   └── 可定制工作流
```

## 5.3.3 Skills 分类

### 按领域分类

```
技术类 Skills：
├── 代码审查 (Code Review)
├── 代码生成 (Code Gen)
├── 文档生成 (Docs Gen)
├── 单元测试 (Unit Test)
├── Bug 修复 (Bug Fix)
└── 重构建议 (Refactor)

创意类 Skills：
├── 文案撰写 (Copywriting)
├── 故事创作 (Story Writing)
├── 营销策划 (Marketing)
├── PPT 制作 (Presentation)
└── 视频脚本 (Video Script)

效率类 Skills：
├── 日程管理 (Calendar)
├── 邮件处理 (Email)
├── 数据分析 (Data Analysis)
├── 报告生成 (Report Gen)
└── 会议纪要 (Meeting Notes)
```

### 按复杂度分类

```
简单 Skill：
- 单一工具调用
- 简单提示词
- 如：天气查询

复合 Skill：
- 多工具组合
- 复杂工作流
- 如：代码审查

高级 Skill：
- 多步骤推理
- 条件分支
- 如：完整开发流程
```

## 5.3.4 Skills 配置

### 启用 Skills

```json
{
  "skills": {
    "enabled": true,
    "auto_load": true,
    "paths": [
      "~/.openclaw/skills",
      "./skills"
    ]
  }
}
```

### 指定 Skill

```json
{
  "agent": {
    "skill": "code-review"
  }
}
```

### 多 Skill 组合

```json
{
  "agent": {
    "skills": ["code-review", "unit-test"]
  }
}
```

## 5.3.5 Skills 工作流

### 执行流程

```
用户：审查这个代码

═══════════════════════════════════════════════

Step 1: Skill 匹配
  → 识别需要 code-review Skill
  → 加载 Skill 配置

Step 2: 提示词组装
  → 加载 code-review prompt
  → 注入代码内容
  → 添加审查标准

Step 3: 工具准备
  → 准备 read 工具
  → 准备 grep 工具
  → 配置工具参数

Step 4: 执行推理
  → LLM 分析代码
  → 逐项检查
  → 生成建议

Step 5: 结果处理
  → 格式化输出
  → 添加修复建议
  → 返回审查报告

═══════════════════════════════════════════════

返回：
📋 代码审查报告
├── 代码规范：✅ 通过
├── 安全性：⚠️ 发现问题
├── 性能：✅ 良好
└── 建议：3 条
```

### 上下文传递

```
上下文包含：
├── 当前代码内容
├── 相关文件
├── 历史对话
└── 用户偏好

传递方式：
1. 自动注入
2. 按需加载
3. 手动指定
```

## 5.3.7 常见问题

### Q1: Skill 不生效

```
错误：Skill not found

解决：
1. 检查 Skill 名称是否正确
2. 确认 Skill 路径配置正确
3. 检查 Skill 文件格式
```

### Q2: Skill 冲突

```
问题：多个 Skill 功能重叠

解决：
1. 指定单一 Skill
2. 调整 Skill 优先级
3. 禁用冲突的 Skill
```

### Q3: Skill 执行慢

```
问题：Skill 响应慢

解决：
1. 检查 Skill 复杂度
2. 优化工具调用
3. 减少上下文
```
