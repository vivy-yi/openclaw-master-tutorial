# 5.6.1 浏览器自动化概述

本节讲解 OpenClaw 浏览器自动化的核心概念、架构设计和两种主要模式（openclaw vs chrome）。

---

## 5.6.1.1 什么是 OpenClaw 浏览器

OpenClaw 可以运行一个**专用的 Chrome/Brave/Edge/Chromium Profile**，由 Agent 控制。它与你的个人浏览器隔离，通过 Gateway（仅限 loopback）内的小型本地控制服务管理。

```
┌─────────────────────────────────────────────────────────────┐
│                    OpenClaw 浏览器架构                        │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐      CDP       ┌─────────────────────┐   │
│  │   Gateway   │ ──────────────→ │  OpenClaw-managed   │   │
│  │  (控制平面) │                 │   Chrome Profile    │   │
│  └─────────────┘                 │  (独立用户数据目录)   │   │
│                                  └─────────────────────┘   │
│                                                             │
│  ┌─────────────┐      CDP       ┌─────────────────────┐   │
│  │   Gateway   │ ──────────────→ │   你的个人 Chrome   │   │
│  │             │                 │  (通过扩展中继)     │   │
│  └─────────────┘                 └─────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 核心特性

- **独立的浏览器 Profile**：名为 `openclaw` 的独立浏览器（默认橙色主题）
- **确定性 Tab 控制**：列出/打开/聚焦/关闭 Tab
- **Agent 操作**：点击、输入、拖拽、选择、快照、截图、PDF
- **多 Profile 支持**：可选的 `openclaw`、`work`、`remote` 等

---

## 5.6.1.2 两种浏览器模式：openclaw vs chrome

### openclaw：OpenClaw 托管浏览器

- **完全托管**：无需安装扩展，OpenClaw 自动启动独立浏览器
- **隔离保证**：独立用户数据目录，不影响个人浏览器
- **开箱即用**：默认配置即可使用

```bash
# 启动 openclaw Profile
openclaw browser --browser-profile openclaw start

# 查看状态
openclaw browser --browser-profile openclaw status
```

### chrome：Chrome 扩展中继

- **控制现有 Chrome**：使用你已登录的 Chrome 账号
- **需要扩展**：需要安装 OpenClaw Browser Relay 扩展
- **手动附着**：点击扩展图标附着到目标 Tab

```bash
# 查看 chrome Profile 的 Tab
openclaw browser --browser-profile chrome tabs
```

### 选择建议

| 场景 | 推荐模式 |
|------|----------|
| 自动化任务、不需要登录态 | `openclaw` |
| 需要使用已登录的账号（如 Gmail、GitHub） | `chrome` |
| 需要多浏览器实例并行 | `openclaw` 多 Profile |
| 调试 Chrome 问题 | `chrome` |

## 5.6.1.3 快速开始

### 基本命令

```bash
# 查看浏览器状态
openclaw browser --browser-profile openclaw status

# 启动浏览器
openclaw browser --browser-profile openclaw start

# 打开网页
openclaw browser --browser-profile openclaw open https://example.com

# 获取页面快照
openclaw browser --browser-profile openclaw snapshot
```

### 启用浏览器

如果收到 "Browser disabled" 错误，需要在配置中启用：

```json5
{
  browser: {
    enabled: true,
    defaultProfile: "openclaw"
  }
}
```

然后重启 Gateway。

## 5.6.1.4 隔离保证

OpenClaw 浏览器提供以下隔离保证：

```
┌─────────────────────────────────────────────────────────────┐
│                      隔离保证                                 │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 独立用户数据目录                                          │
│     └── 从不访问你的个人浏览器 Profile                         │
│                                                             │
│  2. 独立端口                                                 │
│     └── 避免与开发工作流程的 9222 冲突                         │
│                                                             │
│  3. 确定性 Tab 控制                                          │
│     └── 通过 targetId 定位 Tab，而非"最后一个 Tab"            │
│                                                             │
│  4. Loopback 仅限                                           │
│     └── 浏览器控制服务仅绑定 loopback                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 5.6.1.5 浏览器选择

启动本地浏览器时，OpenClaw 按以下顺序自动检测：

1. **Chrome**
2. **Brave**
3. **Edge**
4. **Chromium**
5. **Chrome Canary**

可以手动指定浏览器路径：

```json5
{
  browser: {
    executablePath: "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser"
  }
}
```

### 各平台路径示例

```json5
// macOS
{
  browser: {
    executablePath: "/Applications/Brave Browser.app/Contents/MacOS/Brave Browser"
  }
}

// Windows
{
  browser: {
    executablePath: "C:\\Program Files\\BraveSoftware\\Brave-Browser\\Application\\brave.exe"
  }
}

// Linux
{
  browser: {
    executablePath: "/usr/bin/brave-browser"
  }
}
```

## 本节小结

1. **OpenClaw 浏览器**：Agent 专用的独立浏览器，与个人浏览器隔离
2. **两种模式**：`openclaw`（托管）和 `chrome`（扩展中继）
3. **隔离保证**：独立用户数据目录、独立端口、确定性 Tab 控制
4. **快速开始**：status → start → open → snapshot
