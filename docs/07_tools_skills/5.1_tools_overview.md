## 5.1.3 工具定义

### 工具结构

```json
{
  "name": "read",
  "description": "读取文件或目录内容",
  "parameters": {
    "type": "object",
    "properties": {
      "path": {
        "type": "string",
        "description": "文件或目录路径"
      },
      "limit": {
        "type": "number",
        "description": "读取行数限制"
      },
      "offset": {
        "type": "number",
        "description": "起始行号"
      }
    },
    "required": ["path"]
  }
}
```

### 工具元数据

```
工具元数据包含：

1. name - 工具名称
   - 唯一标识
   - 简短英文

2. description - 功能描述
   - 供 LLM 理解工具用途
   - 详细且准确

3. parameters - 参数定义
   - 类型、描述、必填
   - 供 LLM 生成参数

4. examples - 使用示例
   - 帮助 LLM 理解用法
   - 提高调用准确性
```

## 5.1.4 工具调用流程

### 完整流程

```
用户: "读取桌面上的 config.json"

══════════════════════════════════════════════════

Step 1: LLM 意图分析
  - 分析用户请求
  - 确定需要调用 read 工具
  - 提取参数：path = "~/Desktop/config.json"

Step 2: 参数验证
  - 检查 path 参数
  - 验证参数类型
  - 确认必填参数存在

Step 3: 安全检查
  - 权限验证
  - 路径安全检查
  - 危险操作拦截

Step 4: 工具执行
  - 调用 read 函数
  - 读取文件内容
  - 处理错误

Step 5: 结果处理
  - 格式化输出
  - 添加到上下文
  - 继续推理

══════════════════════════════════════════════════

返回: 文件内容
```

### 错误处理

```
工具调用可能失败：

1. 参数错误
   ├── 缺少必填参数
   ├── 参数类型错误
   └── 参数值无效

2. 执行错误
   ├── 文件不存在
   ├── 权限不足
   └── 命令执行失败

3. 超时错误
   ├── 执行时间过长
   └── 网络超时

处理方式：
- 返回错误信息
- 提供解决建议
- 询问用户重新输入
```

## 5.1.6 工具权限

### Profile 权限

```json
{
  "profiles": {
    "developer": {
      "tools": {
        "read": { "enabled": true },
        "write": { "enabled": true },
        "exec": {
          "enabled": true,
          "allowed_commands": ["npm", "git", "docker"]
        }
      }
    },
    "readonly": {
      "tools": {
        "read": { "enabled": true },
        "write": { "enabled": false },
        "exec": { "enabled": false }
      }
    }
  }
}
```

### 权限检查

```
权限检查流程：

1. Profile 级别
   └── 是否允许使用该工具

2. 命令级别（exec）
   └── 是否在白名单/黑名单

3. 路径级别（read/write）
   └── 是否在允许路径

4. 参数级别
   └── 参数是否合法
```

## 5.1.7 常见问题

### Q1: 工具调用失败

```
错误：Tool call failed

排查：
1. 检查参数是否正确
2. 检查权限是否足够
3. 检查目标文件/命令是否存在
4. 查看错误日志
```

### Q2: 权限不足

```
错误：Permission denied

解决：
1. 检查 Profile 配置
2. 确认工具已启用
3. 检查路径权限
```

### Q3: 超时问题

```
错误：Tool execution timeout

解决：
1. 增加 timeout 配置
2. 优化工具执行逻辑
3. 检查系统资源
```
