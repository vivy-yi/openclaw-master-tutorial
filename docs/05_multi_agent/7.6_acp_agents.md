# 7.6 ACP 代理协议

本节讲解 OpenClaw 的 ACP（Agent Client Protocol）代理功能，让你能够在 OpenClaw 中运行外部编码工具（如 Claude Code、Codex、Gemini CLI 等）。

---

## 7.6.1 什么是 ACP

ACP（Agent Client Protocol）是一种协议，允许 OpenClaw 运行外部编码工具作为会话代理。

```
┌─────────────────────────────────────────────────────────────┐
│                       ACP 架构                                │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  用户请求                                                    │
│       │                                                     │
│       ▼                                                     │
│  ┌─────────────┐                                           │
│  │   OpenClaw  │                                           │
│  │   Gateway   │                                           │
│  └──────┬──────┘                                           │
│         │                                                   │
│         │ ACP 协议                                          │
│         ▼                                                   │
│  ┌─────────────┐                                           │
│  │ ACP Backend │  ──→ Codex                                │
│  │  Plugin     │  ──→ Claude Code                          │
│  │             │  ──→ Gemini CLI                           │
│  │             │  ──→ OpenCode                             │
│  │             │  ──→ Pi                                   │
│  └─────────────┘                                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 支持的代理

- **Codex** - OpenAI 的编程代理
- **Claude Code** - Anthropic 的 CLI 代理
- **Gemini CLI** - Google 的 CLI 代理
- **OpenCode** - 开源的编码代理
- **Pi** - 另有编程代理

## 7.6.2 快速开始

### 基本命令流程

```bash
# 1. 创建会话
/acp spawn codex --mode persistent --thread auto

# 2. 在绑定的线程中工作
# (或显式指定会话 key)

# 3. 检查运行时状态
/acp status

# 4. 调整运行时选项
/acp model provider/model
/acp permissions <profile>
/acp timeout <seconds>

# 5. 引导活跃会话而不替换上下文
/acp steer tighten logging and continue

# 6. 停止工作
/acp cancel    # 停止当前回合
/acp close     # 关闭会话 + 移除绑定
```

### 自然语言请求示例

OpenClaw 可以理解自然语言请求并路由到 ACP 运行时：

```
"Start a persistent Codex session in a thread here and keep it focused."
"Run this as a one-shot Claude Code ACP session and summarize the result."
"Use Gemini CLI for this task in a thread, then keep follow-ups in that same thread."
```

## 7.6.3 ACP 命令详解

### 会话管理

```bash
# 创建新会话
/acp spawn <agent> --mode persistent --thread auto

# 查看状态
/acp status

# 关闭会话
/acp close

# 取消当前操作
/acp cancel
```

### 运行时配置

```bash
# 设置模型
/acp model anthropic/claude-sonnet-4-20250514

# 设置权限配置
/acp permissions browse-only

# 设置超时
/acp timeout 300
```

### 会话引导

```bash
# 引导活跃会话
/acp steer <instruction>

# 查看日志
/acp logs

# 获取输出
/acp output
```

---

## 7.6.4 会话模式

### 一次性模式 (One-shot)

执行单个任务并返回结果：

```
/acp spawn codex --mode one-shot
```

### 持久模式 (Persistent)

保持会话活跃，支持后续交互：

```
/acp spawn codex --mode persistent --thread auto
```

## 7.6.5 线程绑定

支持线程绑定的渠道（Discord、Telegram）可以将 ACP 会话绑定到特定线程：

```
┌─────────────────────────────────────────────────────────────┐
│                     线程绑定                                   │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Discord 频道                                               │
│  └── 绑定到特定频道                                         │
│                                                             │
│  Telegram 论坛                                              │
│  └── 绑定到特定主题                                         │
│                                                             │
│  后续消息自动路由到同一 ACP 会话                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 配置示例

```bash
# 绑定到 Discord 频道
/acp spawn codex --mode persistent --channel discord --thread 123456789

# 绑定到 Telegram 主题
/acp spawn codex --mode persistent --channel telegram --topic 123
```

## 7.6.6 权限配置

### 内置权限配置

```bash
# 仅浏览权限
/acp permissions browse-only

# 浏览 + 读取
/acp permissions read-only

# 浏览 + 读取 + 写入
/acp permissions read-write

# 完整权限
/acp permissions full
```

### 自定义权限

可以在配置中定义自定义权限：

```json5
{
  "acp": {
    "permissions": {
      "custom-profile": {
        "allowedTools": ["read", "glob", "grep"],
        "allowedChannels": ["discord", "telegram"]
      }
    }
  }
}
```

## 7.6.7 故障排除

### 常见问题

**Q: ACP 会话无法启动**

```
排查步骤：
1. 检查 ACP 插件是否安装
2. 验证目标代理是否可用
3. 检查权限配置
```

**Q: 线程消息未路由到 ACP 会话**

```
排查步骤：
1. 确认渠道支持线程
2. 检查会话是否绑定到线程
3. 验证会话是否处于活跃状态
```

**Q: 代理输出未返回**

```
排查步骤：
1. 检查 /acp output
2. 查看 /acp logs
3. 确认超时设置
```
