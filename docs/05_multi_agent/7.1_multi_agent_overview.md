# 7.1 多 Agent 概述

## 本节目标
- 理解多 Agent 协作的原理
- 掌握多 Agent 系统架构
- 了解适用场景

---

## 7.1.1 什么是多 Agent

### 单 Agent vs 多 Agent

```
┌─────────────────────────────────────────────┐
│         单 Agent vs 多 Agent                 │
├─────────────────────────────────────────────┤
│                                             │
│  单 Agent                                    │
│  ├── 一个 Agent 处理所有任务                  │
│  ├── 简单高效                               │
│  └── 适合单一职责任务                        │
│                                             │
│  多 Agent                                    │
│  ├── 多个专业 Agent 协作                     │
│  ├── 分工明确                               │
│  └── 适合复杂任务                           │
│                                             │
└─────────────────────────────────────────────┘
```

### 多 Agent 优势

```
优势：

1. 专业分工
   ├── 每个 Agent 专注领域
   └── 深度优化

2. 可扩展性
   ├── 轻松添加新 Agent
   └── 灵活扩展

3. 容错性
   ├── 单点故障不影响全局
   └── 可替换 Agent

4. 成本优化
   ├── 按需调用不同模型
   └── 复杂任务分配给强模型
```

```
┌─────────────────────────────────────────────┐
│           多 Agent 系统架构                   │
├─────────────────────────────────────────────┤
│                                             │
│              用户请求                         │
│                 │                            │
│                 ▼                            │
│  ┌───────────────────────────────────────┐  │
│  │         Agent Dispatcher              │  │
│  │         任务分发器                     │  │
│  └───────────────────┬───────────────────┘  │
│                      │                       │
│          ┌──────────┼──────────┐            │
│          ▼          ▼          ▼            │
│     ┌────────┐ ┌────────┐ ┌────────┐       │
│     │Agent A │ │Agent B │ │Agent C │       │
│     │  (UI)  │ │(Logic) │ │(Data)  │       │
│     └────┬───┘ └────┬───┘ └────┬───┘       │
│          │          │          │            │
│          └──────────┼──────────┘            │
│                     │                       │
│                     ▼                       │
│  ┌───────────────────────────────────────┐  │
│  │         Result Aggregator             │  │
│  │         结果聚合器                     │  │
│  └───────────────────┬───────────────────┘  │
│                      │                       │
│                      ▼                       │
│               返回结果                        │
│                                             │
└─────────────────────────────────────────────┘
```

### 组件说明

```
核心组件：

1. Dispatcher - 任务分发
   → 分析任务
   → 选择 Agent
   → 分配任务

2. Agent - 执行单元
   → 专业能力
   → 独立运行
   → 返回结果

3. Aggregator - 结果聚合
   → 合并结果
   → 格式统一
   → 返回用户
```

## 7.1.3 Agent 类型

### 按角色分类

```
Agent 角色类型：

1. 协调者 (Coordinator)
   ├── 任务分解
   ├── 流程控制
   └── 结果汇总
   例子：项目经理

2. 执行者 (Executor)
   ├── 具体执行
   ├── 工具调用
   └── 产出结果
   例子：开发人员

3. 审查者 (Reviewer)
   ├── 质量检查
   ├── 问题发现
   └── 改进建议
   例子：代码审查员

4. 规划者 (Planner)
   ├── 方案设计
   ├── 路径规划
   └── 策略制定
   例子：架构师
```

### 按模型分类

```
模型类型：

1. 强模型 Agent
   ├── GPT-4o / Claude 3.5
   ├── 复杂推理
   └── 重要决策

2. 平衡模型 Agent
   ├── GPT-4o-mini / Claude 3 Haiku
   ├── 日常任务
   └── 成本优化

3. 轻量模型 Agent
   ├── DeepSeek / GLM-4
   ├── 简单执行
   └── 批量处理
```

## 7.1.4 协作模式

### 模式分类

```
协作模式：

1. 串行模式
   A → B → C
   └── 顺序执行

2. 并行模式
   A → ┬ → D
   B ──┤
   C ──┘
   └── 同时执行，结果合并

3. 分支模式
   A → B (条件1)
   A → C (条件2)
   └── 根据条件选择

4. 循环模式
   A → B → A → B
   └── 迭代优化

5. 层级模式
   A → B → C
   └── 逐级传递
```

## 7.1.5 适用场景

### 场景示例

```
适合多 Agent：

1. 复杂项目开发
   ├── 需求分析 Agent
   ├── 开发 Agent
   ├── 测试 Agent
   └── 部署 Agent

2. 全方位客户服务
   ├── 接待 Agent
   ├── 问题分类 Agent
   ├── 处理 Agent
   └── 满意度回访

3. 内容创作
   ├── 选题 Agent
   ├── 写作 Agent
   ├── 配图 Agent
   └── 审核 Agent

4. 数据分析
   ├── 采集 Agent
   ├── 清洗 Agent
   ├── 分析 Agent
   └── 可视化 Agent

不适合多 Agent：
- 简单问答
- 单一任务
- 实时性要求极高
```

## 7.1.6 常见问题

### Q1: Agent 之间如何通信

```
解决：
1. 共享上下文
2. 结构化消息
3. 约定协议
```

### Q2: 结果冲突如何处理

```
解决：
1. 协调者仲裁
2. 优先级机制
3. 投票决定
```

### Q3: 如何避免无限循环

```
解决：
1. 设置最大迭代
2. 定义终止条件
3. 监控异常
```

## 本节小结

1. **多 Agent**：专业分工，协作完成复杂任务
2. **架构**：Dispatcher + Agent + Aggregator
3. **角色**：协调者、执行者、审查者、规划者
4. **模式**：串行、并行、分支、循环
