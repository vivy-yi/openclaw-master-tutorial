## 7.2.1 团队定义

### 配置结构

```json
{
  "teams": {
    "dev_team": {
      "name": "开发团队",
      "agents": [
        "architect",
        "developer",
        "reviewer",
        "tester"
      ],
      "coordinator": "architect"
    }
  }
}
```

### Agent 定义

```json
{
  "agents": {

## 7.2.2 角色配置

### 架构师角色

```yaml
architect:
  role: "技术架构师"
  responsibilities:
    - 系统设计
    - 技术选型
    - 代码审查
    - 性能优化

  skills:
    - system_design
    - tech_selection

  tools:
    - read
    - glob
    - exec

  output_format: |
    ## 架构设计
    - 系统结构
    - 技术栈
    - 接口设计
```

### 开发者角色

```yaml
developer:
  role: "全栈开发工程师"
  responsibilities:
    - 代码实现
    - Bug 修复
    - 单元测试
    - 文档编写

  skills:
    - code_generation
    - debug
    - unit_test

  tools:
    - read
    - write
    - edit
    - exec

  output_format: |
    ## 实现代码
    ```语言
    代码内容
    ```
```

### 审查者角色

```yaml
reviewer:
  role: "代码审查专家"
  responsibilities:
    - 代码规范检查
    - 安全漏洞扫描
    - 性能问题识别
    - 改进建议

  skills:
    - code_review
    - security_scan

  tools:
    - read
    - grep

  output_format: |
    ## 审查结果
    ### 问题
    - 问题1
    - 问题2
    ### 建议
    - 建议1
```

## 7.2.3 协作规则

### 定义规则

```json
{
  "teams": {
    "dev_team": {
      "rules": {
        "flow": "sequential",
        "max_iterations": 5,
        "timeout": 600,
        "error_handling": "retry_then_skip"
      },
      "sequence": [
        {
          "agent": "architect",
          "task": "design",
          "output": "architecture_doc"
        },
        {
          "agent": "developer",
          "task": "implement",
          "input": "architecture_doc",
          "output": "code"
        },
        {
          "agent": "reviewer",
          "task": "review",
          "input": "code",
          "output": "review_report"
        },
        {
          "agent": "developer",
          "task": "fix",
          "input": "review_report",
          "output": "fixed_code"
        },
        {
          "agent": "tester",
          "task": "test",
          "input": "fixed_code",
          "output": "test_report"
        }
      ]
    }
  }
}
```

### 条件分支

```json
{
  "rules": {
    "conditions": [
      {
        "if": "review_passed",
        "then": "tester",
        "else": "developer"
      },
      {
        "if": "test_passed",
        "then": "deploy",
        "else": "developer"
      }
    ]
  }
}
```

## 7.2.7 常见问题

### Q1: Agent 任务分配不均

```
问题：某些 Agent 负载过高

解决：
1. 优化任务分配
2. 增加 Agent 副本
3. 调整超时设置
```

### Q2: 通信失败

```
问题：Agent 之间无法通信

解决：
1. 检查共享存储
2. 验证消息格式
3. 检查网络连接
```

### Q3: 循环依赖

```
问题：Agent A 等待 B，B 等待 A

解决：
1. 重新设计流程
2. 设置超时
3. 添加协调者
```
