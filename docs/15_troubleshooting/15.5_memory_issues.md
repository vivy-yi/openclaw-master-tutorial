# 15.5 记忆与上下文异常排查

## 本节目标

掌握记忆丢失、上下文遗忘等问题的排查方法15.5.。

---

## 1 现象描述

常见现象：
- "模型遗忘刚刚提到的信息"
- "连续要求重复输入参数"
- "上下文不连续"

---

## 15.5.2 排查检查项

### 1. 会话归属检查

会话归属是否漂移（同一用户被分配到不同 sessionKey）：

```bash
# 查看会话列表
openclaw sessions list

# 查看会话详情
openclaw sessions info <session_id>
```

**诊断方法**：
用结构化日志按 `sessionKey` 和 `traceId` 回放链路

```bash
cat runtime.log | jq -c 'select(.type=="log") | .log | select(.sessionKey!="") | {ts,trace_id,sessionKey,event,err_type}' | tail
```

### 2. 上下文预算检查

上下文预算是否打满，导致近期信息被裁剪：

```bash
# 查看上下文状态
openclaw context status

# 查看上下文使用
openclaw context usage
```

**诊断方法**：
观察日志中的裁剪/压缩事件与占位符频率

```
[INFO] Context compaction triggered: budget 80000/100000
[WARN] Recent message discarded due to budget limit
```

### 3. 会话存储检查

会话存储文件是否可读写，是否发生写入失败：

```bash
# 检查会话文件
ls -la ~/.openclaw/agents/<agentId>/sessions/

# 检查文件权限
ls -la ~/.openclaw/memory/
```

**诊断手段**：
检查 `~/.openclaw/agents/<agentId>/sessions/sessions.json` 的更新时间与权限

## 15.5.3 常见记忆问题

### 问题 1：会话漂移

```
现象：每次对话都像新会话
```

**原因**：
- sessionKey 不一致
- 会话未正确保存

**排查**：
```bash
# 检查 sessionKey
openclaw sessions list | grep <user_id>

# 检查会话历史
openclaw sessions history <session_id>
```

### 问题 2：上下文被裁剪

```
现象：早期的信息丢失
```

**原因**：
- 上下文超出预算
- 自动压缩删除了信息

**排查**：
1. 检查压缩设置
2. 调整上下文预算
3. 手动保存重要信息

```bash
# 查看压缩日志
openclaw logs | grep -i compaction
```

### 问题 3：记忆未保存

```
现象：之前记住的信息丢失
```

**原因**：
- 写入失败
- 存储满
- 清理策略

**排查**：
```bash
# 检查记忆存储
openclaw memory list

# 检查存储空间
df -h ~/.openclaw
```

## 15.5.4 诊断命令

```bash
# 1. 查看深度状态
openclaw status --deep

# 2. 实时日志跟踪
openclaw logs --follow --json

# 3. 查看会话详情
openclaw sessions info <session_id>

# 4. 查看上下文使用
openclaw context detail
```

## 15.5.5 复验与完整流程

修复后检查项：

1. **同场景复验**：用相同问题测试是否记住
2. **回归对比**：回放关键 trace 做对比
3. **更新文档**：更新 Runbook、告警阈值
4. **记录处理**：记录原因、处理动作、回滚点
