# 15.1 启动失败排查

## 本节目标

掌握 OpenClaw 启动失败的常见原因及排查方法。

---

## 15.1.1 排查检查项

### 1. 运行时与依赖检查

确保 Node.js、Docker 等依赖满足要求：

```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version

# 检查 Docker
docker --version

# 检查 OpenClaw 安装
openclaw --version
```

**要求**：
- Node.js >= 18.0.0
- npm >= 8.0.0
- Docker >= 20.0.0（如果使用 Docker 部署）

### 2. 配置文件检查

配置文件路径与语法正确性（尤其是 JSON5 注释与尾逗号）：

```bash
# 检查配置文件位置
ls -la ~/.openclaw/openclaw.json

# 验证 JSON 语法
cat ~/.openclaw/openclaw.json | python3 -m json.tool
```

**典型报错**：

```
SyntaxError: JSON5: invalid character '}' at 12:4
```

**修复方法**：
- 检查是否有尾逗号（如 `key: value,}`）
- 检查注释格式（JSON5 支持 `//` 和 `/* */`，标准 JSON 不支持）
- 检查引号匹配

### 3. 端口占用检查

端口是否被占用：

```bash
# 检查 18789 端口占用
lsof -nP -iTCP:18789 -sTCP:LISTEN

# 或者
netstat -an | grep 18789
```

**典型报错**：

```
EADDRINUSE: address already in use :::18789
```

**修复方法**：
- 找到占用进程：`lsof -i :18789`
- 杀死进程：`kill -9 <PID>`
- 或修改配置使用其他端口

### 4. 密钥与令牌检查

密钥与令牌是否注入且有效：

```bash
# 检查环境变量
env | grep -i openclaw
env | grep -i api

# 检查 .env 文件
cat ~/.openclaw/.env
```

## 15.1.3 常见启动错误

### 错误 1：配置文件解析失败

```
Error: Failed to parse config file
Reason: Unexpected token ',' at line 15
```

**原因**：JSON 语法错误

**排查**：
1. 检查配置文件第 15 行附近
2. 确认没有尾逗号
3. 确认引号匹配

### 错误 2：端口被占用

```
Error: listen EADDRINUSE: address already in use :::18789
```

**原因**：18789 端口已被占用

**排查**：
1. 查找占用进程：`lsof -i :18789`
2. 确认是否需要杀掉该进程
3. 或修改配置使用其他端口

### 错误 3：缺少依赖

```
Error: Cannot find module 'openai'
```

**原因**：npm 依赖未安装

**排查**：
```bash
# 重新安装依赖
cd ~/.openclaw
npm install
```

### 错误 4：权限不足

```
Error: EACCES: permission denied
```

**原因**：文件或目录权限不足

**排查**：
```bash
# 修复权限
chmod -R 755 ~/.openclaw
```

## 15.1.4 环境验证脚本

创建完整的环境检查脚本：

```bash
#!/bin/bash

echo "=== OpenClaw 环境检查 ==="

echo -e "\n1. Node.js 版本："
node --version

echo -e "\n2. npm 版本："
npm --version

echo -e "\n3. OpenClaw 安装状态："
openclaw --version 2>&1

echo -e "\n4. 配置文件检查："
if [ -f ~/.openclaw/openclaw.json ]; then
    echo "✓ 配置文件存在"
    cat ~/.openclaw/openclaw.json | python3 -m json.tool > /dev/null 2>&1
    if [ $? -eq 0 ]; then
        echo "✓ JSON 语法正确"
    else
        echo "✗ JSON 语法错误"
    fi
else
    echo "✗ 配置文件不存在"
fi

echo -e "\n5. 端口占用检查："
lsof -nP -iTCP:18789 -sTCP:LISTEN | head -5

echo -e "\n6. 最近错误日志："
openclaw logs --limit 50 2>&1 | grep -i error | tail -10

echo -e "\n=== 检查完成 ==="
```
