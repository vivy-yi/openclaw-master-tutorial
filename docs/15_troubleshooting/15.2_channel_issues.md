# 15.2 渠道异常排查

## 本节目标

掌握渠道连接异常、消息收发失败等问题的排查方法。

---

## 15.2.1 排查检查项

### 1. 渠道令牌与权限检查

渠道令牌与权限是否有效：

```bash
# 查看渠道状态
openclaw channels status

# 测试渠道连接
openclaw channels test telegram
openclaw channels test feishu
```

**典型报错**：

```
[Telegram] Polling error: 401 Unauthorized
[Feishu] API error: invalid app_id
```

**原因**：
- Token 过期
- 权限不足
- App ID/Secret 错误

### 2. 配对关系检查

配对关系是否正确，是否存在未审批的配对请求：

```bash
# 查看配对列表
openclaw pairing list

# 审批配对请求
openclaw pairing approve <device_id>
```

**典型报错**：

```
[Gateway] Connection rejected: device 9f8a not paired
```

**原因**：
- 设备未配对
- 配对请求未批准

### 3. 路由配置检查

路由是否命中目标 Agent，会话归属是否稳定：

```bash
# 查看路由规则
openclaw routes list

# 查看当前会话
openclaw sessions list
```

### 4. 渠道日志检查

渠道状态与日志是否一致：

```bash
# 查看渠道详细日志
openclaw logs --channel telegram --limit 100

# 查看实时日志
openclaw logs --follow
```

## 15.2.2 常见渠道问题

### Telegram 问题

#### 问题 1：Bot 无法加入群组

```
错误：Bot is not a member of the group chat
```

**排查**：
1. 检查 Bot 是否已添加到群组
2. 检查 Bot 是否有管理员权限
3. 检查群组设置是否允许 Bot

#### 问题 2：消息发送失败

```
错误：Bad Request: chat not found
```

**排查**：
1. 确认 chat_id 正确
2. 检查 Bot 是否有发送消息权限
3. 确认 Bot 与用户有对话

#### 问题 3：Webhook 设置失败

```
错误：Webhook validation failed
```

**排查**：
1. 确认 Webhook URL 可访问
2. 检查 SSL 证书有效
3. 确认 Token 正确

### 飞书问题

#### 问题 1：消息收不到

```
错误：Webhook call failed: verification token mismatch
```

**排查**：
1. 检查 Verification Token 正确
2. 确认 Webhook URL 可公网访问
3. 检查事件订阅是否启用

#### 问题 2：权限不足

```
错误：permission denied
```

**排查**：
1. 检查应用权限配置
2. 确认已发布应用
3. 重新安装应用到群

## 15.2.3 诊断命令

```bash
# 1. 查看所有渠道状态
openclaw channels status

# 2. 查看特定渠道详情
openclaw channels info telegram

# 3. 测试渠道连接
openclaw channels test <channel_name>

# 4. 查看渠道日志
openclaw logs --channel <channel_name> --limit 200

# 5. 查看配对状态
openclaw pairing list
```
