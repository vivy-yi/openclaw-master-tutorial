## 2.5.1 为什么使用 Docker

### Docker 优势

```
┌─────────────────────────────────────────────────────────────┐
│                      Docker 优势                            │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  🛡️ 环境隔离                                                │
│     └── 独立运行环境，互不影响                                │
│                                                              │
│  🚀 快速部署                                                │
│     └── 一键启动，无需手动安装依赖                             │
│                                                              │
│  📦 易于迁移                                                │
│     └── 容器化打包，环境一致                                 │
│                                                              │
│  ⚙️ 资源控制                                                │
│     └── 可限制 CPU、内存使用                                 │
│                                                              │
│  🔄 易于更新                                                │
│     └── 拉取新镜像即可升级                                   │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Docker vs 直接安装

| 特性 | Docker | 直接安装 |
|-----|--------|---------|
| 环境依赖 | 自包含 | 需要手动安装 |
| 部署速度 | 快 | 依赖网络 |
| 隔离性 | 好 | 一般 |
| 资源占用 | 略高 | 低 |
| 调试难度 | 较难 | 容易 |

## 2.5.2 安装 Docker

### macOS

```bash
# 方法一：Homebrew（推荐）
brew install docker

# 方法二：官网下载
# https://www.docker.com/products/docker-desktop
```

### Linux

```bash
# Ubuntu/Debian
curl -fsSL https://get.docker.com | sh

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户加入 docker 组
sudo usermod -aG docker $USER
```

### 验证安装

```bash
# 检查版本
docker --version
docker-compose --version

# 运行测试
docker run hello-world
```

## 2.5.3 Docker 基础部署

### 步骤一：创建配置目录

```bash
# 创建配置目录
mkdir -p ~/.openclaw

# 创建配置文件
touch ~/.openclaw/openclaw.json
```

### 步骤二：编辑配置文件

```json
{
  "model": {
    "provider": "deepseek",
    "model": "deepseek-chat",
    "apiKey": "${DEEPSEEK_API_KEY}"
  },
  "gateway": {
    "port": 18789,
    "channels": {
      "cli": { "enabled": true }
    }
  }
}
```

### 步骤三：运行容器

```bash
# 基本运行
docker run -d \
  --name openclaw \
  -v ~/.openclaw:/root/.openclaw \
  -p 18789:18789 \
  openclaw/openclaw:latest

# 完整配置（推荐）
docker run -d \
  --name openclaw \
  --restart unless-stopped \
  -v ~/.openclaw:/root/.openclaw \
  -p 18789:18789 \
  -e TZ=Asia/Shanghai \
  -e DEEPSEEK_API_KEY=$DEEPSEEK_API_KEY \
  openclaw/openclaw:latest
```

### 参数说明

| 参数 | 说明 |
|-----|------|
| `-d` | 后台运行 |
| `--name` | 容器名称 |
| `--restart unless-stopped` | 开机自启 |
| `-v` | 目录映射 |
| `-p` | 端口映射 |
| `-e` | 环境变量 |

## 2.5.4 Docker Compose 部署（推荐）

### 创建 docker-compose.yml

```yaml
version: '3.8'

services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    ports:
      - "18789:18789"
    volumes:
      - ./openclaw.json:/root/.openclaw/openclaw.json:ro
      - openclaw-data:/root/.openclaw
    environment:
      - TZ=Asia/Shanghai
      - DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:18789/health"]
      interval: 30s
      timeout: 10s
      retries: 3

volumes:
  openclaw-data:
```

### 环境变量文件

创建 `.env` 文件：

```bash
# .env
DEEPSEEK_API_KEY=sk-你的API密钥
```

### 常用命令

```bash
# 启动
docker-compose up -d

# 查看状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 查看实时日志
docker-compose logs -f --tail=100

# 停止
docker-compose down

# 重启
docker-compose restart

# 更新镜像
docker-compose pull
docker-compose up -d
```

## 2.5.5 数据持久化

### 目录映射

```yaml
volumes:
  # 配置文件（只读）
  - ./openclaw.json:/root/.openclaw/openclaw.json:ro

  # 工作目录
  - ./workspace:/root/.openclaw/workspace

  # 记忆数据
  - ./memory:/root/.openclaw/memory

  # 日志
  - ./logs:/root/.openclaw/logs

  # Skills
  - ./skills:/root/.openclaw/skills
```

### 完整配置示例

```yaml
version: '3.8'

services:
  openclaw:
    image: openclaw/openclaw:latest
    container_name: openclaw
    ports:
      - "18789:18789"
    volumes:
      # 配置（只读，保护配置）
      - ./config/openclaw.json:/root/.openclaw/openclaw.json:ro

      # 数据卷
      - openclaw-workspace:/root/.openclaw/workspace
      - openclaw-memory:/root/.openclaw/memory
      - openclaw-skills:/root/.openclaw/skills
      - openclaw-logs:/root/.openclaw/logs
    environment:
      - TZ=Asia/Shanghai
      # 可以添加多个 API Key
      - DEEPSEEK_API_KEY=${DEEPSEEK_API_KEY}
      - OPENAI_API_KEY=${OPENAI_API_KEY}
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:18789/health"]
      interval: 30s
      timeout: 10s
      retries: 3
    networks:
      - openclaw-network

networks:
  openclaw-network:
    driver: bridge

volumes:
  openclaw-workspace:
  openclaw-memory:
  openclaw-skills:
  openclaw-logs:
```

## 2.5.6 资源限制

### 限制 CPU 和内存

```yaml
services:
  openclaw:
    image: openclaw/openclaw:latest
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 4G
        reservations:
          cpus: '1'
          memory: 2G
```

### 监控资源使用

```bash
# 查看容器资源使用
docker stats openclaw

# 实时监控
docker stats --no-stream openclaw

# 查看所有容器
docker stats
```

### 资源使用示例


## 2.5.8 更新与升级

### 方法一：重新拉取镜像

```bash
# 1. 停止容器
docker-compose down

# 2. 拉取新镜像
docker-compose pull

# 3. 重新启动
docker-compose up -d
```

### 方法二：使用 Watchtower（自动更新）

```yaml
# docker-compose.yml
services:
  openclaw:
    image: openclaw/openclaw:latest
    # ...

  watchtower:
    image: containrrr/watchtower
    container_name: watchtower
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock
    environment:
      - WATCHTOWER_CLEANUP=true
      - WATCHTOWER_SCHEDULE=0 0 3 * * *  # 每天凌晨3点检查
    command: --interval 86400 openclaw
```

## 2.5.9 备份与恢复

### 备份脚本

```bash
#!/bin/bash
# backup-openclaw.sh

BACKUP_DIR="./backups"
DATE=$(date +%Y%m%d_%H%M%S)

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份配置文件
cp ~/.openclaw/openclaw.json $BACKUP_DIR/config_$DATE.json

# 备份数据卷
docker run --rm \
  -v openclaw-memory:/data \
  -v $(pwd)/$BACKUP_DIR:/backup \
  alpine tar czf /backup/memory_$DATE.tar.gz -C /data .

echo "备份完成: $DATE"
```

### 恢复脚本

```bash
#!/bin/bash
# restore-openclaw.sh

BACKUP_FILE=$1

if [ -z "$BACKUP_FILE" ]; then
  echo "用法: ./restore-openclaw.sh <备份文件>"
  exit 1
fi

# 恢复数据
docker run --rm \
  -v openclaw-memory:/data \
  -v $(pwd)/backups:/backup \
  alpine tar xzf /backup/$BACKUP_FILE -C /data

echo "恢复完成"
```

## 2.5.10 常见问题

### 问题一：容器无法启动

```bash
# 查看错误日志
docker logs openclaw

# 检查配置文件
docker exec openclaw cat /root/.openclaw/openclaw.json | jq .

# 检查权限
ls -la ~/.openclaw/
```

### 问题二：端口冲突

```bash
# 检查端口占用
netstat -tuln | grep 18789

# 更改端口
# docker-compose.yml
ports:
  - "18889:18789"  # 外部18889映射到容器内18789
```

### 问题三：内存不足

```bash
# 查看 Docker 资源使用
docker stats

# 限制内存
# docker-compose.yml
deploy:
  resources:
    limits:
      memory: 2G
```

### 问题四：数据丢失

```bash
# 检查数据卷
docker volume ls

# 备份数据卷
docker run --rm -v openclaw-data:/data -v $(pwd)/backup:/backup alpine tar czf /backup/data.tar.gz /data

# 恢复数据卷
docker run --rm -v openclaw-data:/data -v $(pwd)/backup:/backup alpine tar xzf /backup/data.tar.gz -C /data
```

### 问题五：网络问题

```bash
# 检查网络
docker network ls
docker network inspect openclaw-network

# 重建网络
docker network rm openclaw-network
docker-compose up -d
```

## 2.5.6 v2026.3.24 新功能：容器内运行 OpenClaw（--container）

### 背景

v2026.3.24 新增 `--container` 和 `OPENCLAW_CONTAINER` 环境变量支持，允许 OpenClaw 在 Docker 或 Podman **容器内部**运行。

### 典型场景

```
┌─────────────────────────────────────────────────────────────┐
│                 容器内运行 vs 容器外运行                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  容器外运行（直接安装）：                                      │
│  └── 宿主机直接运行 openclaw，二进制在主机命名空间              │
│                                                              │
│  容器内运行（v2026.3.24+）：                                  │
│  └── openclaw 二进制打包进容器，容器内运行 openclaw           │
│      适用于：沙箱隔离、便携部署、统一环境                       │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### 使用方法

**方式一：CLI 参数**

```bash
openclaw --container
```

**方式二：环境变量**

```yaml
# docker-compose.yml
environment:
  OPENCLAW_CONTAINER: "true"
```

**方式三：配置文件**

```json
{
  "container": {
    "enabled": true
  }
}
```

### Podman 支持（v2026.3.24 PR #55388）

v2026.3.24 同时更新了 Podman 的安装文档。以下 Podman 配置与 Docker 几乎完全兼容：

```yaml
# podman-compose.yml
version: "3.8"

services:
  openclaw:
    image: docker.io/openclaw/openclaw:latest
    container_name: openclaw
    environment:
      - OPENCLAW_CONTAINER=true
      - OPENCLAW_GATEWAY_URL=http://localhost:18789
    volumes:
      - openclaw-data:/root/.openclaw
      - /var/run/podman/podman.sock:/var/run/docker.sock
    ports:
      - "18789:18789"
    restart: unless-stopped

volumes:
  openclaw-data:
```

### 与 Docker 直接部署的区别

| 维度 | 直接部署（docker run） | 容器内运行（--container） |
|------|----------------------|--------------------------|
| openclaw 进程位置 | 宿主机 | 容器内 |
| 适用场景 | 容器作为宿主机 wrapper | 沙箱隔离、便携打包 |
| 资源控制 | 由 docker 限制 | 由容器 runtime 限制 |
| 调试 | 需要进入容器 | 同直接部署 |

### 已知问题

> ⚠️ 在某些 Linux 发行版上，使用 `--container` 配合 Podman 时，systemd 服务注册可能存在兼容性问题。建议优先使用 Docker 或等待官方修复（跟踪 [PR #55388](https://github.com/openclaw/openclaw/issues/55388)）。
