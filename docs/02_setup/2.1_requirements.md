## 2.1.2 硬件要求

### 最低配置

| 项目 | 要求 | 说明 |
|-----|------|------|
| CPU | 任意现代处理器 | i5 以上或 M1 及以上 |
| 内存 | 4 GB | 低于 4GB 可能导致 OOM |
| 存储 | 1 GB 可用空间 | 推荐 10 GB+ |
| 网络 | 能访问互联网 | 稳定宽带 |

### 推荐配置

| 项目 | 推荐 | 说明 |
|-----|------|------|
| CPU | 4 核+ | 处理多任务更流畅 |
| 内存 | 8 GB+ | 运行浏览器工具需要更多内存 |
| 存储 | 10 GB+ SSD | 更快的数据读写 |
| 网络 | 稳定宽带 | 减少 API 调用延迟 |

> [!WARNING]
> **内存过低会导致严重问题**：
> - 服务器执行自动升级时可能 OOM
> - 运行浏览器工具时进程卡死
> - 长上下文任务无法完成
>
> **结论**：省下的费用远不抵后续排障成本

## 2.1.4 网络连通性验证

OpenClaw 系统重度依赖各种远端 API，网络状况直接决定可用性。

### 需要验证的网络

```
┌─────────────────────────────────────────────────────────┐
│                    网络连通性检查                        │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  1. 安装期网络                                          │
│     └─→ npm registry (npmjs.org)                       │
│                                                         │
│  2. 运行时网络                                          │
│     └─→ LLM 提供商 API                                  │
│        - api.openai.com (OpenAI)                       │
│        - api.deepseek.com (DeepSeek)                   │
│        - api.moonshot.cn (Kimi)                         │
│                                                         │
│  3. 渠道网络（可选）                                    │
│     └─→ Telegram / 飞书 / Discord 等                    │
│                                                         │
└─────────────────────────────────────────────────────────┘
```

### 验证命令

```bash
# 1. 验证 npm registry
curl -sS -m 5 -o /dev/null -w "npm registry: %{http_code}\n" https://registry.npmjs.org/

# 2. 验证 OpenAI API (需要设置环境变量)
if [ -n "${OPENAI_API_KEY:-}" ]; then
  curl -sS -m 10 -o /dev/null -w "OpenAI API: %{http_code}\n" \
    https://api.openai.com/v1/models \
    -H "Authorization: Bearer $OPENAI_API_KEY"
fi

# 3. 验证 DeepSeek API
if [ -n "${DEEPSEEK_API_KEY:-}" ]; then
  curl -sS -m 10 -o /dev/null -w "DeepSeek API: %{http_code}\n" \
    https://api.deepseek.com/v1/models \
    -H "Authorization: Bearer $DEEPSEEK_API_KEY"
fi
```

### 响应码说明

| 响应码 | 含义 |
|-------|------|
| 200 | 网络可达，鉴权通过 ✅ |
| 401/403 | 网络可达，鉴权失败（检查 API Key）⚠️ |
| 000/超时 | 网络不可达 ❌ |

## 2.1.6 环境自检脚本

你可以使用以下脚本快速检查环境：

```bash
#!/bin/bash
echo "=== OpenClaw 环境自检 ==="
echo ""

# 1. 检查操作系统
echo "📋 操作系统信息："
uname -a
echo ""

# 2. 检查 Node.js
echo -n "📦 Node.js: "
if command -v node &> /dev/null; then
    node --version
else
    echo "❌ 未安装"
fi

# 3. 检查 npm
echo -n "📦 npm: "
if command -v npm &> /dev/null; then
    npm --version
else
    echo "❌ 未安装"
fi

# 4. 检查 Docker
echo -n "🐳 Docker: "
if command -v docker &> /dev/null; then
    docker --version
else
    echo "❌ 未安装"
fi

echo ""
echo "🌐 网络连通性检查："

# 5. 检查 npm registry
echo -n "  - npm registry: "
RESULT=$(curl -s -m 5 -o /dev/null -w "%{http_code}" https://registry.npmjs.org/ 2>/dev/null)
if [ "$RESULT" = "200" ]; then
    echo "✅ 可达"
else
    echo "❌ 不可达 (HTTP $RESULT)"
fi

# 6. 检查 OpenAI API (如果配置了)
if [ -n "${OPENAI_API_KEY:-}" ]; then
    echo -n "  - OpenAI API: "
    RESULT=$(curl -s -m 10 -o /dev/null -w "%{http_code}" \
        https://api.openai.com/v1/models \
        -H "Authorization: Bearer $OPENAI_API_KEY" 2>/dev/null)
    if [ "$RESULT" = "200" ]; then
        echo "✅ 可达"
    else
        echo "⚠️  需检查 (HTTP $RESULT)"
    fi
fi

echo ""
echo "=== 自检完成 ==="
```

### 运行脚本

```bash
# 保存脚本
chmod +x check_env.sh

# 运行
./check_env.sh

# 或者直接运行
bash check_env.sh
```

## 2.1.7 系统时间与权限

### 系统时间

很多云厂商的 API 鉴权基于 JWT，有严格的有效期校验。

```bash
# 检查时间
timedatectl

# 设置时区
timedatectl set-timezone Asia/Shanghai

# 启用时间同步
sudo systemctl enable chronyd
```

### 运行权限

> [!CAUTION]
> **OpenClaw 绝对不需要 root 权限运行！**

```bash
# 创建一个专用用户（可选）
sudo useradd -m openclaw

# 给用户授权目录
sudo chown -R $USER ~/.openclaw
```

## 2.1.8 常见问题

### Q1: Node.js 版本过低

```bash
# 使用 nvm 安装最新版本
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
source ~/.zshrc  # 或 source ~/.bashrc
nvm install 22
nvm use 22

# 设为默认
nvm alias default 22
```

### Q2: 网络无法访问

**检查代理**：
```bash
# 查看代理
echo $HTTP_PROXY
echo $HTTPS_PROXY
```

**配置代理**：
```bash
export HTTP_PROXY=http://proxy:8080
export HTTPS_PROXY=http://proxy:8080
```

### Q3: 内存不足

```bash
# 查看内存使用
free -h  # Linux
top -l 1 | head -n 10  # macOS
```

**解决方案**：
1. 关闭其他占用内存的程序
2. 增加物理内存
3. 选择更高配置的云服务器
