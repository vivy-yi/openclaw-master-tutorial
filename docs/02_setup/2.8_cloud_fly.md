## 2.8.3 部署步骤

### 步骤一：克隆仓库

```bash
git clone https://github.com/openclaw/openclaw.git
cd openclaw
```

### 步骤二：创建应用

```bash
# 创建应用（选择唯一名称）
fly apps create my-openclaw

# 或指定名称
fly apps create my-openclaw --org personal
```

### 步骤三：创建存储卷

```bash
# 创建 1GB 存储卷
fly volumes create openclaw_data --size 1 --region iad
```

> **提示**：选择靠近你的区域
> - `iad`: 弗吉尼亚（美东）
> - `lhr`: 伦敦
> - `sjc`: 圣何塞（美西）
> - `hkg`: 香港
> - `nrt`: 东京

### 步骤四：配置 fly.toml

编辑 `fly.toml`：

```toml
app = "my-openclaw"
primary_region = "iad"

[build]
  dockerfile = "Dockerfile"

[env]
  NODE_ENV = "production"
  OPENCLAW_PREFER_PNPM = "1"
  OPENCLAW_STATE_DIR = "/data"
  NODE_OPTIONS = "--max-old-space-size=1536"

[processes]
  app = "node dist/index.js gateway --allow-unconfigured --port 3000 --bind lan"

[http_service]
  internal_port = 3000
  force_https = true
  auto_stop_machines = false
  auto_start_machines = true
  min_machines_running = 1
  processes = ["app"]

[[vm]]
  size = "shared-cpu-2x"
  memory = "2048mb"

[mounts]
  source = "openclaw_data"
  destination = "/data"
```

### 关键配置说明

| 配置 | 说明 | 推荐值 |
|-----|------|-------|
| `--bind lan` | 绑定到 0.0.0.0 | 必须 |
| `--allow-unconfigured` | 无配置启动 | 必须 |
| `internal_port` | 内部端口 | 3000 |
| `memory` | 内存 | 2048mb（512MB 太小）|

### 步骤五：设置密钥

```bash
# Gateway Token（必须）
fly secrets set OPENCLAW_GATEWAY_TOKEN=$(openssl rand -hex 32)

# 模型 API Keys
fly secrets set ANTHROPIC_API_KEY=sk-ant-...
fly secrets set OPENAI_API_KEY=sk-...
fly secrets set DEEPSEEK_API_KEY=sk-...

# 渠道 Tokens（可选）
fly secrets set DISCORD_BOT_TOKEN=MTQ...
fly secrets set TELEGRAM_BOT_TOKEN=123...
```

> **安全提示**：优先使用环境变量而非配置文件存储 API Keys

### 步骤六：部署

```bash
fly deploy
```

首次部署需要 2-3 分钟构建镜像，后续部署更快。

### 步骤七：验证部署

```bash
# 查看状态
fly status

# 查看日志
fly logs

# 访问应用
fly open
```

## 2.8.4 配置 OpenClaw

### SSH 进入容器

```bash
fly ssh console
```

### 创建配置文件

```bash
mkdir -p /data
vim /data/openclaw.json
```

### 配置文件示例

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "deepseek/deepseek-chat",
        "fallbacks": ["anthropic/claude-sonnet-4-5"]
      },
      "maxConcurrent": 4
    },
    "list": [
      {
        "id": "main",
        "default": true
      }
    ]
  },
  "auth": {
    "profiles": {
      "deepseek:default": { "mode": "token", "provider": "deepseek" },
      "anthropic:default": { "mode": "token", "provider": "anthropic" }
    }
  },
  "bindings": [
    {
      "agentId": "main",
      "match": { "channel": "discord" }
    }
  ],
  "channels": {
    "discord": {
      "enabled": true,
      "groupPolicy": "allowlist",
      "guilds": {
        "YOUR_GUILD_ID": {
          "channels": { "general": { "allow": true } },
          "requireMention": false
        }
      }
    }
  },
  "gateway": {
    "mode": "local",
    "bind": "auto"
  },
  "meta": {
    "lastTouchedVersion": "2026.3.15"
  }
}
```

### 重启应用

```bash
# 退出 SSH
exit

# 获取机器 ID
fly machines list

# 重启机器
fly machine restart <machine-id>
```

## 2.8.5 访问控制 UI

### 公开访问

```bash
# 打开浏览器
fly open

# 或直接访问
https://my-openclaw.fly.dev/
```

粘贴 `OPENCLAW_GATEWAY_TOKEN` 进行认证。

### 私有部署

如需隐藏公开访问，使用 `fly.private.toml`：

```bash
# 使用私有配置部署
fly deploy -c fly.private.toml

# 释放公网 IP
fly ips list
fly ips release <ipv4> -a my-openclaw
fly ips release <ipv6> -a my-openclaw
```

### 私有访问方式

```bash
# 本地代理（推荐）
fly proxy 3000:3000 -a my-openclaw

# 然后访问
http://localhost:3000

# 或使用 WireGuard
fly wireguard create
```

## 2.8.6 更新与维护

### 手动更新

```bash
# 拉取最新代码
git pull

# 重新部署
fly deploy
```

### 自动更新（GitHub Actions）

```yaml
# .github/workflows/fly-deploy.yml
name: Deploy to Fly.io

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: superfly/flyctl-actions/setup-flyctl@main
      - run: flyctl deploy --remote-only
        env:
          FLY_API_TOKEN: ${{ secrets.FLY_API_TOKEN }}
```

获取 Token：

```bash
fly tokens create deploy
```

## 2.8.8 常见问题

### 问题一：App 未监听预期地址

**症状**：`App is not listening on expected address`

**解决**：添加 `--bind lan` 到启动命令

```toml
[processes]
  app = "node dist/index.js gateway --allow-unconfigured --port 3000 --bind lan"
```

### 问题二：健康检查失败

**症状**：`connection refused`

**解决**：确保端口匹配

```toml
[http_service]
  internal_port = 3000  # 必须与 --port 3000 匹配
```

### 问题三：内存不足 OOM

**症状**：容器不断重启

**解决**：增加内存

```toml
[[vm]]
  memory = "2048mb"  # 至少 2GB
```

### 问题四：状态未持久化

**症状**：重启后丢失凭据

**解决**：确认 `OPENCLAW_STATE_DIR=/data` 配置

```toml
[env]
  OPENCLAW_STATE_DIR = "/data"
```

### 问题五：Gateway 锁定

**症状**：`already running` 错误

**解决**：删除锁文件

```bash
fly ssh console --command "rm -f /data/gateway.*.lock"
fly machine restart <machine-id>
```

## 2.8.9 成本估算

### 费用构成

| 项目 | 配置 | 价格 |
|-----|------|------|
| 共享 CPU | 2x | 免费 |
| 内存 | 2GB | $1.94/月 |
| 存储 | 1GB | $0.50/月 |
| 出站流量 | - | ~$0.15/月 |
| **总计** | | **~$2.60/月** |

### 节省技巧

1. **选择合适区域**：不同区域价格略有差异
2. **最小化存储**：1GB 通常够用
3. **监控流量**：避免意外流量费用
