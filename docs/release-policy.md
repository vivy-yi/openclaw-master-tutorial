## 一、策略概述

PR #95613 引入了 OpenClaw 官方的**月度发布、每日构建、稳定版**三位一体的发布策略。

---

## 二、发布节奏

| 类型 | 频率 | 说明 |
|------|------|------|
| **Monthly** | 每月一次 | 主要版本发布 |
| **Daily** | 每日构建 | 每日合并的构建 |
| **Stable** | 按需发布 | 稳定版随时发布 |

### 2.1 Monthly (月度发布)

- 每月发布一次主要版本
- 包含功能更新、重大修复
- 遵循语义化版本 (SemVer)

### 2.2 Daily (每日构建)

- 每日自动构建
- 包含当日合并的 PRs
- 用于快速验证和测试

### 2.3 Stable (稳定版)

- 基于 Monthly 或按需发布
- 经过验证的稳定版本
- 生产环境推荐使用

| 类型 | 格式 | 示例 |
|------|------|------|
| Monthly | vYYYY.MM | v2026.06 |
| Beta | vYYYY.MM-beta.N | v2026.6.10-beta.1 |
| Stable | vYYYY.MM | v2026.6.9 |

---

## 四、升级路径

```
Daily → Monthly → Stable
```

**推荐升级路径**：
1. 生产环境 → 使用 Stable
2. 测试环境 → 使用 Beta
3. 开发环境 → 使用 Daily

|------|----------|------|

| 功能测试 | Beta | 新功能 |

## 七、FAQ

### Q1: 何时升级到新版本？

**A**: 生产环境建议使用 Stable 版本，等待正式发布后升级。

### Q2: Daily 版本是否稳定？

**A**: Daily 版本包含最新代码，可能存在不稳定因素，不建议生产环境使用。

### Q3: 如何获取最新版本？

**A**: 
- 官方文档: https://docs.openclaw.ai
- GitHub: https://github.com/openclaw/openclaw/releases
