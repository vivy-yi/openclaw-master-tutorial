## 9.5.1 案例一：每日数据报告

### 需求

```
自动化需求：
- 每天早上 8:00 自动生成昨日数据报告
- 发送到指定邮箱
- 包含关键指标和趋势分析
```

### 配置

```json
{
  "scheduled_tasks": {
    "tasks": [
      {
        "id": "daily_report",
        "name": "每日数据报告",
        "cron": "0 8 * * *",
        "enabled": true,

        "action": {
          "type": "agent",
          "agent": "data_analyst",
          "prompt": "生成昨日数据报告"
        },

        "options": {
          "timeout": 300,
          "retry": {
            "enabled": true,
            "max_attempts": 2
          }
        },

        "notifications": {
          "on_success": {
            "email": {
              "to": ["team@example.com"],
              "subject": "每日数据报告 - {{date}}"
            }
          },
          "on_failure": {
            "channels": ["slack"]
          }
        }
      }
    ]
  }
}
```

## 9.5.2 案例二：代码审查自动化

### 需求

```
自动化需求：
- GitHub PR 创建时自动触发代码审查
- 检查代码规范
- 扫描安全漏洞
- 评论审查结果
```

### 配置

```json
{
  "webhook_triggers": {
    "triggers": [
      {
        "id": "pr_review",
        "name": "PR 自动审查",

        "event": "pull_request",
        "source": "github",
        "action": "opened",

        "filter": {
          "branches": ["main", "develop"]
        },

        "action": {
          "type": "workflow",
          "workflow": "code_review"
        },

        "response": {
          "on_success": {
            "github": {
              "comment": "代码审查完成\n\n{{review_summary}}"
            }
          }
        }
      }
    ]
  }
}
```

## 9.5.3 案例三：智能客服工单

### 需求

```
自动化需求：
- 用户提交工单后自动分类
- 根据类型分配处理 Agent
- 处理完成后通知用户
```

### 配置

```yaml
workflow: ticket_processing

triggers:
  - type: webhook
    event: ticket_created

stages:
  - name: classify
    agent: classifier

  - name: route
    type: condition
    conditions:
      - type: technical
        agent: technical_support
      - type: billing
        agent: billing_support
      - type: general
        agent: general_support

  - name: process
    parallel: true
    agents: [technical_support, billing_support, general_support]

  - name: notify
    agent: notifier
    input: process.result
```

## 9.5.4 案例四：定时数据同步

### 需求

```
自动化需求：
- 每小时同步外部 API 数据
- 增量更新
- 异常告警
```

### 配置

```json
{
  "scheduled_tasks": {
    "tasks": [
      {
        "id": "data_sync",
        "name": "外部数据同步",
        "cron": "0 */1 * * *",

        "action": {
          "type": "workflow",
          "workflow": "sync_external_data"
        },

        "options": {
          "timeout": 600,
          "retry": {
            "enabled": true,
            "max_attempts": 3,
            "interval": 300
          }
        }
      }
    ]
  }
}
```

### 工作流

```yaml
name: sync_external_data

stages:
  - name: fetch
    type: http
    url: https://api.example.com/data
    method: GET

  - name: process
    agent: data_processor

  - name: save
    tool: write
    path: "./data/sync_{{timestamp}}.json"

  - name: notify
    condition: has_errors
    agent: error_notifier
```

## 9.5.5 最佳实践

### 设计原则

```
自动化设计原则：

1. 明确目标
   └── 清楚自动化的目的

2. 最小化复杂
   └── 从简单开始，逐步扩展

3. 错误处理
   └── 每个任务都要有容错

4. 监控告警
   └── 及时发现和响应问题

5. 文档记录
   └── 记录配置和流程
```

### 常见陷阱

```
避免的问题：

1. 过度自动化
   └── 不要自动化所有东西

2. 忽视错误
   └── 必须有错误处理

3. 没有测试
   └── 先测试再部署

4. 监控缺失
   └── 不知道执行状态
```
