- 掌握 Agent 定时任务的配置

## 9.7.1 Cron 任务概述

### 什么是 Cron

```
┌─────────────────────────────────────────────────────────────┐
│                    Cron 定时任务                             │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  Cron = Gateway 内置调度器                                  │
│  ├── 定时执行任务                                            │
│  ├── 持久化存储                                              │
│  └── 支持多种交付方式                                        │
│                                                              │
│  存储位置：~/.openclaw/cron/                                │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Cron vs Heartbeat

| 特性 | Cron | Heartbeat |
|-----|------|-----------|
| 触发方式 | 定时 | 定时+消息 |
| 运行环境 | Gateway 内部 | Gateway 内部 |
| 持久化 | ✅ | ✅ |
| 消息触发 | ❌ | ✅ |

## 9.7.2 创建 Cron 任务

### CLI 基本命令

```bash
# 添加一次性提醒
openclaw cron add \
  --name "Reminder" \
  --at "2026-02-01T16:00:00Z" \
  --session main \
  --system-event "Reminder: check the docs" \
  --wake now \
  --delete-after-run
```

### 循环任务

```bash
# 每日早报
openclaw cron add \
  --name "Morning Brief" \
  --cron "0 7 * * *" \
  --tz "Asia/Shanghai" \
  --session isolated \
  --message "Summarize overnight updates." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

## 9.7.3 绑定到特定 Agent

### 指定 Agent 执行

```bash
# 绑定到特定 Agent
openclaw cron add \
  --name "Ops Check" \
  --cron "0 6 * * *" \
  --session isolated \
  --message "Check ops queue status." \
  --agent ops

# 绑定到 work Agent
openclaw cron add \
  --name "Work Report" \
  --cron "0 9 * * 1-5" \
  --session isolated \
  --message "Generate weekly work report." \
  --agent work
```

### 配置文件指定

```json
{
  "cron": {
    "jobs": [
      {
        "jobId": "morning-brief",
        "name": "Morning Brief",
        "agentId": "work",
        "schedule": {
          "kind": "cron",
          "expr": "0 7 * * *",
          "tz": "Asia/Shanghai"
        },
        "sessionTarget": "isolated",
        "payload": {
          "kind": "agentTurn",
          "message": "Summarize overnight updates."
        },
        "delivery": {
          "mode": "announce",
          "channel": "slack",
          "to": "channel:C1234567890"
        }
      }
    ]
  }
}
```

## 9.7.4 任务类型

### 主会话任务 (System Event)

```bash
# 主会话任务 - 使用心跳上下文
openclaw cron add \
  --name "Daily Standup" \
  --cron "0 9 * * 1-5" \
  --session main \
  --system-event "Daily standup: summarize progress." \
  --wake now
```

特点：
- 使用主会话上下文
- 通过 heartbeat 运行
- 适合需要完整上下文的场景

### 隔离任务 (Isolated)

```bash
# 隔离任务 - 独立会话
openclaw cron add \
  --name "Background Scan" \
  --cron "0 */4 * * *" \
  --session isolated \
  --message "Scan for updates." \
  --announce
```

特点：
- 独立会话，无历史
- 不干扰主对话
- 适合后台任务

### 当前会话任务

```bash
# 绑定到当前会话
openclaw cron add \
  --name "Project Update" \
  --cron "0 10 * * *" \
  --session current \
  --message "Update project status."
```

## 9.7.5 定时表达式

### 常用 Cron 表达式

| 表达式 | 含义 |
|-------|------|
| `0 * * * *` | 每小时 |
| `0 0 * * *` | 每天午夜 |
| `0 7 * * *` | 每天早上 7 点 |
| `0 9 * * 1-5` | 工作日早上 9 点 |
| `0 */2 * * *` | 每 2 小时 |
| `0 0 * * 0` | 每周日 |

### 相对时间

```bash
# 20 分钟后
--at "20m"

# 2 小时后
--at "2h"

# 明天
--at "tomorrow 9am"
```

## 9.7.6 交付模式

### 公告模式 (Announce)

```bash
# 发送结果到渠道
openclaw cron add \
  --name "Daily Report" \
  --cron "0 18 * * *" \
  --session isolated \
  --message "Generate daily report." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

### Webhook 模式

```bash
# 发送到 Webhook
openclaw cron add \
  --name "API Notify" \
  --cron "0 8 * * *" \
  --session isolated \
  --message "Check API status." \
  --webhook "https://your-server.com/webhook"
```

### 无交付

```bash
# 仅内部执行
openclaw cron add \
  --name "Internal Task" \
  --cron "0 */2 * * *" \
  --session isolated \
  --message "Internal check." \
  --no-announce
```

> 📌 **执行流程图请参考 [9.2 定时任务](../09_automation/9.2_scheduled_tasks.md) 章节**

## 9.7.7 管理 Cron 任务

### 常用命令

```bash
# 列出所有任务
openclaw cron list

# 查看任务详情
openclaw cron info <job-id>

# 手动运行任务
openclaw cron run <job-id>

# 查看运行历史
openclaw cron runs --id <job-id>

# 编辑任务
openclaw cron edit <job-id> --message "New message"

# 删除任务
openclaw cron remove <job-id>

# 禁用/启用任务
openclaw cron disable <job-id>
openclaw cron enable <job-id>
```

## 9.7.8 模型覆盖

### 指定模型

```bash
# 使用 Opus 模型
openclaw cron add \
  --name "Deep Analysis" \
  --cron "0 6 * * 1" \
  --session isolated \
  --message "Weekly deep analysis." \
  --model "opus" \
  --thinking high \
  --announce
```

### 轻量上下文

```bash
# 不加载工作空间上下文
openclaw cron add \
  --name "Quick Check" \
  --cron "0 */4 * * *" \
  --session isolated \
  --message "System health check." \
  --light-context
```
