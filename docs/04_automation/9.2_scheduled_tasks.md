## 9.2.1 定时任务概述

### 什么是定时任务

```
定时任务 = 定时自动执行的任务

场景示例：
- 每日数据报告
- 定时健康检查
- 周期性数据同步
- 定时提醒
```

## 9.2.2 Cron 表达式

### 基本格式

```
Cron 格式：
┌───────────── 分钟 (0 - 59)
│ ┌─────────── 小时 (0 - 23)
│ │ ┌───────── 日 (1 - 31)
│ │ │ ┌─────── 月 (1 - 12)
│ │ │ │ ┌───── 星期 (0 - 6, 0 = 周日)
│ │ │ │ │
* * * * *

特殊字符：
*     任意值
,     值列表  1,3,5
-     范围    1-5
/     步长    */5
```

### 常用示例

```
常用 Cron 表达式：

| 表达式 | 含义 |
|--------|------|
| * * * * * | 每分钟 |
| 0 * * * * | 每小时 |
| 0 0 * * * | 每天午夜 |
| 0 0 * * 0 | 每周日 |
| 0 0 1 * * | 每月1日 |
| 0 9 * * 1-5 | 工作日9点 |
| */15 * * * * | 每15分钟 |
| 0 0 * * * | 每天0点 |
| 0 */2 * * * | 每2小时 |
```

## 9.2.3 配置定时任务 (CLI 方式)

### CLI 基本命令

```bash
# 添加一次性提醒
openclaw cron add \
  --name "Reminder" \
  --at "2026-02-01T16:00:00Z" \
  --session main \
  --system-event "Reminder: check the cron docs draft" \
  --wake now \
  --delete-after-run

# 添加循环任务
openclaw cron add \
  --name "Morning brief" \
  --cron "0 7 * * *" \
  --tz "America/Los_Angeles" \
  --session isolated \
  --message "Summarize overnight updates." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

### 基础配置 (JSON 格式)

Job 存储在 `~/.openclaw/cron/jobs.json`，由 Gateway 管理。

```json5
{
  "jobs": [
    {
      "jobId": "daily_report",
      "name": "每日报告",
      "schedule": {
        "kind": "cron",
        "expr": "0 8 * * *",
        "tz": "Asia/Shanghai"
      },
      "sessionTarget": "isolated",
      "payload": {
        "kind": "agentTurn",
        "message": "生成昨日数据报告"
      },
      "delivery": {
        "mode": "announce",
        "channel": "slack",
        "to": "channel:C1234567890"
      },
      "enabled": true
    }
  ]
}
```

### 完整配置

```json5
{
  "cron": {
    "enabled": true,
    "store": "~/.openclaw/cron/jobs.json",
    "maxConcurrentRuns": 1,
    "retry": {
      "maxAttempts": 3,
      "backoffMs": [60000, 120000, 300000],
      "retryOn": ["rate_limit", "overloaded", "network", "server_error"]
    },
    "webhook": "https://example.invalid/legacy",
    "webhookToken": "replace-with-dedicated-webhook-token",
    "sessionRetention": "24h",
    "runLog": {
      "maxBytes": "2mb",
      "keepLines": 2000
    }
  }
}
```

## 9.2.4 Payload 类型

### System Event (主会话)

```json5
{
  "sessionTarget": "main",
  "payload": {
    "kind": "systemEvent",
    "text": "Reminder: check the calendar"
  },
  "wakeMode": "now"
}
```

### Agent Turn (独立会话)

```json5
{
  "sessionTarget": "isolated",
  "payload": {
    "kind": "agentTurn",
    "message": "分析昨日用户数据",
    "model": "opus",
    "thinking": "high",
    "lightContext": true
  },
  "delivery": {
    "mode": "announce",
    "channel": "slack",
    "to": "channel:C1234567890",
    "bestEffort": true
  }
}
```

### 模型与思考覆盖

isolated 任务可以覆盖模型和思考级别：

```bash
# 使用 Opus 模型，高思考级别
openclaw cron add \
  --name "Deep analysis" \
  --cron "0 6 * * 1" \
  --session isolated \
  --message "Weekly deep analysis" \
  --model "opus" \
  --thinking high \
  --announce
```

### 轻量级上下文

对于不需要工作区 bootstrap 的任务，使用 `--light-context`：

```bash
openclaw cron add \
  --name "Lightweight check" \
  --cron "0 * * * *" \
  --session isolated \
  --message "Quick system check" \
  --light-context \
  --no-deliver
```

## 9.2.5 任务管理

### 查看任务

```bash
# 列出所有定时任务
openclaw cron list

# 查看任务详情
openclaw cron status

# 查看执行历史
openclaw cron runs --id <job-id> --limit 50
```

### 管理命令

```bash
# 手动触发任务
openclaw cron run <job-id>

# 强制立即运行
openclaw cron run <job-id> --due

# 编辑任务
openclaw cron edit <job-id> --message "新提示词"

# 删除任务
openclaw cron remove <job-id>
```

## 9.2.6 Cron Job 核心概念

### Job 结构

Cron Job 由以下部分组成：

```
┌─────────────────────────────────────────────────────────────┐
│                     Cron Job 结构                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. Schedule (何时执行)                                      │
│     ├── at: 一次性时间戳                                     │
│     ├── every: 固定间隔                                      │
│     └── cron: Cron 表达式                                    │
│                                                             │
│  2. Payload (执行什么)                                       │
│     ├── systemEvent: 主会话系统事件                           │
│     └── agentTurn: 独立 Agent 回合                            │
│                                                             │
│  3. Session Target                                          │
│     ├── main: 下次 Heartbeat 时执行                           │
│     └── isolated: 独立会话执行                                │
│                                                             │
│  4. Delivery (输出方式)                                      │
│     ├── announce: 发送到渠道                                 │
│     ├── webhook: HTTP POST 到 URL                            │
│     └── none: 仅内部执行                                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### Schedule 类型

#### at：一次性任务

```bash
# 一次性提醒 (UTC 时间)
openclaw cron add \
  --name "Reminder" \
  --at "2026-02-01T16:00:00Z" \
  --session main \
  --system-event "Reminder: check the cron docs" \
  --wake now \
  --delete-after-run
```

```bash
# 相对时间 (20分钟后)
openclaw cron add \
  --name "Meeting reminder" \
  --at "20m" \
  --session main \
  --system-event "Meeting starts in 10 minutes" \
  --wake now
```

#### every：固定间隔

```bash
# 每小时执行
openclaw cron add \
  --name "Hourly check" \
  --every "1h" \
  --session isolated \
  --message "Run hourly system check" \
  --announce
```

#### cron：Cron 表达式

```bash
# 每天早上 7 点 (美国西部时间)
openclaw cron add \
  --name "Morning brief" \
  --cron "0 7 * * *" \
  --tz "America/Los_Angeles" \
  --session isolated \
  --message "Summarize overnight updates." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

### Session Target

#### Main Session (主会话)

- 在下次 Heartbeat 时执行
- 共享主会话上下文
- 适合需要上下文的提醒

```bash
openclaw cron add \
  --name "Calendar check" \
  --cron "0 9 * * *" \
  --session main \
  --system-event "Check today's calendar"
```

#### Isolated Session (独立会话)

- 独立 Agent 回合，会话 ID 为 `cron:<jobId>`
- 每次运行全新会话，无历史
- 适合频繁、不污染主会话的后台任务

```bash
openclaw cron add \
  --name "Background sync" \
  --cron "*/15 * * * *" \
  --session isolated \
  --message "Sync data from API" \
  --announce \
  --channel whatsapp \
  --to "+15551234567"
```

## 9.2.7 重试策略

### 错误分类

```
┌─────────────────────────────────────────────────────────────┐
│                      错误分类                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  瞬态错误 (可重试)                                           │
│  ├── 429 Rate Limit                                        │
│  ├── 提供商过载 (529 overloaded)                            │
│  ├── 网络错误 (超时、ECONNRESET)                            │
│  └── 5xx 服务器错误                                         │
│                                                             │
│  永久错误 (立即禁用)                                         │
│  ├── 认证失败 (无效 API Key)                                │
│  └── 配置错误                                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 重试行为

**一次性任务 (`at`)**：
- 瞬态错误：指数退避重试 3 次 (30s → 1m → 5m)
- 永久错误：立即禁用

**循环任务 (`cron`/`every`)**：
- 任何错误：指数退避 (30s → 1m → 5m → 15m → 60m)
- 下次成功运行后重置

## 9.2.8 常见问题

### Q1: 任务不执行

```
问题：定时任务没有触发

排查步骤：
1. 检查 cron 是否启用: cron.enabled
2. 确认 Gateway 持续运行
3. 检查时区配置 (--tz vs 主机时区)
4. 查看日志
```

### Q2: 执行时间不准

```
问题：执行时间有偏差

原因：
- 整点负载分散 (自动 5 分钟窗口)
- 主机时区差异

解决：
1. 使用 --exact 强制精确时间
2. 明确指定时区 --tz
3. 使用 --stagger 设置分散窗口
```

### Q3: Telegram 投递错误

```
问题：投递到错误位置

解决：
- 使用明确格式: -1001234567890:topic:123
- 避免歧义
```
