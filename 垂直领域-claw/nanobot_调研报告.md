## 一、面向开发者 👨‍💻

### 1.1 技术栈

```
┌─────────────────────────────────────────────────────────────┐
│                    nanobot 技术栈                            │
├─────────────────────────────────────────────────────────────┤
│  Core                                                       │
│  ├── Python 3.11+ · asyncio                                 │
│  ├── LiteLLM (统一多模型接口)                               │
│  └── MCP Client (Model Context Protocol)                   │
├─────────────────────────────────────────────────────────────┤
│  Memory                                                     │
│  └── Token-based Memory · LangSmith Integration            │
├─────────────────────────────────────────────────────────────┤
│  Channels (9+)                                              │
│  ├── Telegram · Discord · WhatsApp                         │
│  ├── Feishu · DingTalk · QQ · WeCom                        │
│  ├── Slack · Email · Matrix · Mochat                       │
│  └── WebSocket / HTTP 网关                                 │
└─────────────────────────────────────────────────────────────┘
```

### 1.2 核心特性

**超轻量**:
- 相比 OpenClaw 代码量减少 99%
- 运行内存 <100MB

**MCP 支持** (2026-02-14):
- 原生 Model Context Protocol 集成
- 可连接任意 MCP 服务器扩展能力

**Agent 社交网络**:
- 自动加入 Moltbook、ClawdChat 等平台
- 一句话指令完成注册和配置

### 1.3 快速开始

```bash
# 方式1: 源码安装
git clone https://github.com/HKUDS/nanobot.git
cd nanobot
pip install -e .

# 方式2: uv 安装 (推荐)
uv tool install nanobot-ai

# 方式3: pip 安装
pip install nanobot-ai

# 初始化
nanobot onboard
# 或使用交互式向导
nanobot onboard --wizard

# 配置 API Key (~/.nanobot/config.json)
{
  "providers": {
    "openrouter": { "apiKey": "sk-or-v1-xxx" }
  }
}

# 启动对话
nanobot agent
```

## 二、面向产品经理 📊

### 2.1 产品定位

**nanobot** 是面向技术用户的 **超轻量级个人 AI 助手**，主打"简洁、快速、可扩展"。

### 2.2 核心价值主张

| 用户群体 | 价值 |
|----------|------|
| **开发者** | 简洁代码，易于理解和二次开发 |
| **效率用户** | 2 分钟搭建工作助手 |
| **隐私用户** | 本地运行，数据自控 |
| **研究人员** | 干净的代码库，适合研究 |

### 2.3 差异化特性

1. **极致轻量**: 99% 代码减少，更快启动
2. **MCP 协议**: 最新开放标准支持
3. **社交网络**: 一键加入 Agent 社区
4. **快速迭代**: 社区驱动，响应迅速

### 2.4 使用场景

- **开发助手**: 代码审查、文档查询、脚本编写

- **自动化**: 定时任务、数据处理

## 五、总结与建议

### 优势
1. **极致简洁**: 代码量少，易于理解
2. **快速迭代**: 社区活跃，更新频繁
3. **MCP 支持**: 开放标准，生态丰富
4. **轻量高效**: 资源占用低

### 风险
1. **早期版本**: v0.1.x，API 可能变动
2. **社区支持**: 无商业支持承诺
3. **稳定性**: 快速迭代可能引入问题

### 适用场景
- 个人学习和研究
- 快速原型开发
- 技术团队内部工具
- 轻量级 IM 机器人

### 建议
- **立即使用**: 适合技术用户个人使用
- **谨慎生产**: 建议等 1.0 版本后
- **积极参与**: 反馈问题，贡献代码
