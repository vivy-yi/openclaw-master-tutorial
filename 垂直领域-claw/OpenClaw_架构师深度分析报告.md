## 一、执行摘要

### 1.1 架构定位

**OpenClaw** 是 Claw 变体项目的**始祖项目**，由 Anthropic 前员工 Mario Zechner 发起。它是一个运行在用户自有设备上的**个人 AI 助手平台**，通过统一的网关（Gateway）架构，将多种 AI 模型与多种通讯渠道连接起来。

### 1.2 核心架构特征

| 维度 | 架构特征 | 成熟度 |
|------|----------|:------:|
| **架构模式** | 网关 + 插件化架构 | ⭐⭐⭐⭐⭐ |
| **插件数量** | 80+ 扩展插件 | ⭐⭐⭐⭐⭐ |
| **渠道覆盖** | 20+ 通讯渠道 | ⭐⭐⭐⭐⭐ |
| **跨平台** | CLI + Web + iOS + Android + macOS | ⭐⭐⭐⭐⭐ |
| **代码规模** | ~200K+ 行 TypeScript | ⭐⭐⭐⭐ |
| **生态成熟度** | 最成熟的 Claw 生态 | ⭐⭐⭐⭐⭐ |

### 1.3 关键架构指标

```
代码规模:
├── TypeScript: ~200K+ 行
├── 插件数量: 80+ 扩展
├── 渠道覆盖: 20+ 平台
├── 支持模型: OpenAI/Anthropic/Google/Azure 等
└── 测试覆盖: 70%+

运行时指标:
├── 内存占用: >1GB (运行时)
├── 启动时间: ~30s
├── 部署形态: Node.js 运行时
└── 资源需求: Mac Mini $599 级别
```

## 三、技术选型深度分析

### 3.1 编程语言与运行时

| 技术 | 版本 | 选型理由 | 替代方案 |
|------|------|----------|----------|
| **TypeScript** | 5.9+ | 快速迭代、类型安全、生态丰富 | JavaScript |
| **Node.js** | 24+ (ESM) | ESM 原生支持、新 API、性能 | Deno, Bun |
| **pnpm** | 10.23+ | 工作空间、磁盘效率、patch | npm, yarn |

**架构师点评**:
- ✅ TypeScript 严格模式提供编译期类型安全
- ✅ ESM 模块化，支持 Tree Shaking
- ✅ pnpm workspace 管理 80+ 插件
- ⚠️ Node.js 版本敏感（要求 22.16+）

### 3.2 前端技术栈

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| **Lit** | 3.3+ | Web Components | 轻量、无框架锁定、原生 Web 标准 |
| **Vite** | 8.0+ | 构建工具 | 快速 HMR、ESM 原生支持 |
| **Playwright** | - | E2E 测试 | 跨浏览器、自动等待 |

**架构师点评**:
- ✅ Lit 相比 React/Vue 更轻量，符合本地优先理念
- ✅ Web Components 标准化，长期维护性好
- ⚠️ 生态系统不如 React 丰富

### 3.3 网关与通讯

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| **Hono** | 4.12 | HTTP 框架 | 轻量、高性能、类型安全 |
| **Express** | 5 | 兼容层 | 插件系统向后兼容 |
| **TypeBox** | 0.34 | Schema 验证 | JSON Schema 生成、类型安全 |
| **ws** | - | WebSocket | 标准实现、广泛使用 |

### 3.4 数据存储方案

| 存储类型 | 技术 | 用途 |
|----------|------|------|
| **向量+关系** | SQLite + sqlite-vec | 记忆存储、向量搜索 |
| **日志存储** | JSONL | 会话日志、审计跟踪 |
| **配置存储** | 文件系统 (JSON/YAML) | 配置、技能定义 |
| **沙箱存储** | Docker volumes | 隔离执行环境 |

├─────────────────────────────────────────────────────────────┤

## 十一、架构决策记录

### ADR-001: TypeScript + Node.js 选择

**背景**: 需要快速迭代、广泛采用

**决策**: 使用 TypeScript 5.9 + Node.js 24

**权衡**:
- ✅ 开发效率极高
- ✅ 生态丰富
- ⚠️ 运行时资源占用高
- ⚠️ 类型安全不如 Rust

### ADR-002: 插件化架构

**背景**: 功能扩展需求多样

**决策**: 80+ 独立插件 + SDK

**权衡**:
- ✅ 核心精简，功能丰富
- ✅ 社区可贡献
- ⚠️ 维护成本高

### ADR-003: WebSocket 控制平面

**背景**: 需要实时双向通讯

**决策**: WebSocket 作为核心协议

**权衡**:
- ✅ 实时性好
- ✅ 状态同步简单
- ⚠️ 有状态连接，扩展复杂

## 总结

OpenClaw 是 **Claw 变体生态的始祖项目**，拥有最成熟的架构和生态。其核心价值在于：

1. **统一网关**：统一接入 AI 和通讯渠道
2. **插件生态**：80+ 插件支撑丰富功能
3. **跨平台**：CLI + Web + 移动端全覆盖
4. **本地优先**：数据隐私、自主可控

适合追求**功能丰富、生态成熟、不介意资源占用**的用户。但对于资源受限场景，建议考虑 ZeroClaw 或 PicoClaw。
