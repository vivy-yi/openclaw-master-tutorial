# 三个科研 Agent 项目综合分析

## 概述

本文档分析 `/Volumes/waku/大模型/bolt/clawd/claw变体/科研claw/` 下的三个科研 Agent 项目：
1. **dr-claw** - 全栈研究工作空间
2. **ResearchClaw** - 基于 AgentScope 的学术研究助手
3. **NanoResearch** - 端到端自主科研引擎

---

## 一、项目定位对比

| 维度 | dr-claw | ResearchClaw | NanoResearch |
|------|---------|--------------|-------------|
| **核心理念** | "Vibe Researching" - 利用 AI 增强认知 | 本地运行的学术研究助手 | 端到端自主科研引擎 |
| **执行方式** | 多 Agent 后端 (Claude/Gemini/Codex) | 单 ReAct Agent | 多阶段流水线 |
| **实验执行** | ❌ 无 | ❌ 无 | ✅ GPU 集群训练 |
| **论文生成** | 基于研究任务生成内容 | 辅助写作 | 生成完整 LaTeX 论文 |
| **数据来源** | 研究任务产物 | 研究笔记/论文 | **真实实验结果** |

## 五、各项目优缺点分析

### dr-claw

**优点**：
- ✅ 完整的 Web UI
- ✅ 多 Agent 后端支持
- ✅ 100+ 研究 Skills
- ✅ Pipeline 任务管理

**缺点**：
- ❌ 不执行实际实验
- ❌ 基于 Claude Code UI，依赖重
- ❌ 需要云端 API

---

### ResearchClaw

**优点**：
- ✅ 完全本地运行，隐私保护
- ✅ 基于 AgentScope，架构清晰
- ✅ 丰富的文献管理工具
- ✅ 多渠道支持 (CLI/Web/Slack/Email)

**缺点**：
- ❌ 不执行实际实验
- ❌ 单 Agent 架构，能力有限
- ❌ 依赖模型质量

---

### NanoResearch (最接近目标)

**优点**：
- ✅ **真正执行 GPU 训练实验**
- ✅ **基于真实实验数据写论文**
- ✅ 完整的多阶段流水线
- ✅ 断点续跑
- ✅ 多模型路由
- ✅ 支持 Claude Code 模式

**缺点**：
- ⚠️ 主要针对"写论文"场景，不是"训练模型"
- ⚠️ 实验执行依赖 SLURM 集群或本地 GPU
- ⚠️ 论文质量需要人工审核

## 七、结论

| 项目 | 定位 | 实验执行 | 自主程度 | 适用场景 |
|------|------|---------|---------|---------|
| **dr-claw** | AI 研究助手 | ❌ | 辅助 | 论文写作辅助 |
| **ResearchClaw** | 本地研究工具 | ❌ | 辅助 | 文献管理 |
| **NanoResearch** | 端到端科研引擎 | ✅ | 全自主 | 论文自动化 |
| **AutoResearch** | 训练自动化 | ✅ (本地) | 全自主 | 模型训练 |
| **ClawTeam** | 多 Agent 协调 | N/A | 协调 | 多任务管理 |

**关键发现**：
1. **NanoResearch 是最接近"AI Scientist"目标的项目** - 它真正执行实验并基于结果写论文
2. **AutoResearch 专注于模型训练**，NanoResearch 专注于科研论文
3. **两者可以互补**：AutoResearch 负责训练，NanoResearch 负责把实验结果写成论文
4. **ClawTeam 可以协调多阶段训练**，但需要配合 AutoResearch 的动态提示词能力
