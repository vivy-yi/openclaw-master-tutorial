## 一、执行摘要

### 1.1 架构定位

CoPaw 是一个**个人 AI 助手运行时平台**，基于阿里巴巴开源的 AgentScope 框架构建。其定位是"Works for you, grows with you"的私有化部署个人助手，支持多工作区、多 Agent 的分布式个人助理系统。

### 1.2 核心架构特征

| 维度 | 架构特征 | 成熟度 |
|------|----------|:------:|
| **架构模式** | 多工作区 + Service Manager | ⭐⭐⭐⭐ |
| **扩展能力** | Entry Points 插件机制 | ⭐⭐⭐⭐ |
| **安全设计** | ToolGuard + SkillScanner | ⭐⭐⭐⭐ |
| **渠道覆盖** | 13+ 即时通讯渠道 | ⭐⭐⭐⭐⭐ |
| **热重载** | 零停机 Agent 重载 | ⭐⭐⭐⭐ |

### 1.3 关键架构指标

```
代码规模:
├── Python 代码: ~30K 行
├── 前端代码: ~10K 行 (React)
├── 依赖数量: 35+ 核心依赖
└── 支持模型: 50+ 内置模型

运行时指标:
├── 内存占用: ~200MB
├── 启动时间: 3-5 秒
├── 渠道覆盖: 13+ 平台
└── 内置工具: 15+
```

### 4.3 渠道网关架构

```python
class BaseChannel(ABC):
    """所有渠道的抽象基类"""
    
    @abstractmethod
```

### 4.4 MCP (Model Context Protocol) 集成

```python
class MCPClientManager:
    """MCP 客户端管理器"""
    
    async def init_from_config(self, config: Dict) -> None:
        """从配置初始化 MCP 客户端"""
        for name, cfg in config.items():
            if cfg['transport'] == 'stdio':
                client = await self._create_stdio_client(cfg)
            elif cfg['transport'] == 'http':
                client = await self._create_http_client(cfg)
            self.clients[name] = client
```

## 五、数据流与状态管理

### 5.1 请求处理流程

```
用户消息 → Channel → ChannelManager Queue → Worker Pool (4 workers) → 
┌─────────────────────────────────────────────────────────────┐
│                        AgentRunner                          │
│  1. 检查待审批状态                                          │
│  2. 检查系统命令 (CommandDispatcher)                         │
│  3. 构建 AgentRequest → 调用 CoPawAgent                      │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│                        CoPawAgent                           │
│  ReAct Loop:                                                │
│  while iteration < max_iters:                               │
│    1. _reasoning(): LLM 生成思考/工具调用                      │
│    2. ToolGuard 拦截检查                                     │
│    3. _acting(): 执行工具/返回结果                            │
│    4. 更新 Memory                                           │
└─────────────────────────────────────────────────────────────┘
```

### 5.2 状态一致性策略

| 状态类型 | 一致性策略 | 实现方式 |
|----------|-----------|----------|
| **配置状态** | 最终一致性 | 文件监听 + 热重载 |
| **Agent 状态** | 会话级隔离 | 每个 Workspace 独立 |
| **记忆状态** | 持久化存储 | ReMe 自动落盘 |
| **任务状态** | 持久化 + 内存 | JSON 文件 + 内存缓存 |

### 6.2 ToolGuard 实现

```python
class ToolGuardMixin:
    """工具调用安全拦截 Mixin"""
    
    async def _acting(self, *args, **kwargs):
        # 拦截工具调用，进行安全检查
        if self._is_dangerous_command(args):
            await self._request_approval()
        return await super()._acting(*args, **kwargs)
```

## 十一、架构决策记录

### ADR-001: 多工作区架构

**背景**: 需要支持用户创建多个不同用途的 Agent

**决策**: 采用 Workspace + MultiAgentManager 架构

**权衡**:
- ✅ 每个 Agent 完全隔离
- ✅ 支持热重载单个 Agent
- ⚠️ 内存占用随 Agent 数量增长

### ADR-002: 文件型存储 vs 数据库

**决策**: 使用 JSON 文件存储所有配置和状态

**理由**:
- ✅ 零外部依赖
- ✅ 便于版本控制
- ⚠️ 多机部署需要共享存储

## 十二、风险与建议

### 12.1 架构风险评估

| 风险 | 级别 | 说明 | 缓解措施 |
|------|------|------|----------|
| **单点故障** | 中 | 单机部署 | 用户自建备份 |
