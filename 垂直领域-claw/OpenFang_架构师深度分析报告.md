## 一、执行摘要

### 1.1 架构定位

**OpenFang** 是一个开源的 **Agent 操作系统（Agent OS）**，不是简单的聊天机器人框架或 Python LLM 包装器。它从头开始用 Rust 构建，旨在运行真正自主的代理——这些代理按时间表工作、全天候运行、构建知识图谱、监控目标、生成潜在客户，并向仪表板报告结果。

### 1.2 核心架构特征

| 维度 | 架构特征 | 成熟度 |
|------|----------|:------:|
| **语言** | Rust (137K+ LOC, 0 Clippy 警告) | ⭐⭐⭐⭐⭐ |
| **架构** | Cargo Workspace, 14 个 crate | ⭐⭐⭐⭐⭐ |
| **Hands 系统** | 7 个自主能力包 | ⭐⭐⭐⭐ |
| **渠道覆盖** | 40 个适配器 (业界最多) | ⭐⭐⭐⭐⭐ |
| **安全** | 16 层深度防御 | ⭐⭐⭐⭐⭐ |
| **桌面应用** | Tauri 2.0 跨平台 | ⭐⭐⭐⭐ |

### 1.3 关键架构指标

```
代码统计:
├── Rust 代码: ~137K 行
├── 测试: 1,767+ 个
├── 代码质量: 0 Clippy 警告
└── 架构: 14 个 crate

运行时指标:
├── 内存占用: ~40MB 空闲
├── 启动时间: ~180ms 冷启动
├── 二进制大小: ~32MB
└── 支持渠道: 40+ 平台
```

## 四、核心子系统架构

### 4.1 内核启动序列

```
1. 加载配置 (config.toml)
2. 创建数据目录
3. 初始化内存基底 (SQLite schema v5)
4. 初始化 LLM 驱动
5. 初始化模型目录 (51 内置模型, 20+ 提供商)
6. 初始化计量引擎
7. 初始化模型路由器
8. 初始化核心子系统
   - AgentRegistry (DashMap)
   - CapabilityManager
   - EventBus (async broadcast)
   - AgentScheduler
   - Supervisor
   - WorkflowEngine
   - TriggerEngine
   - BackgroundExecutor
   - WasmSandbox (Wasmtime)
9. 初始化 RBAC 权限管理
10. 初始化技能注册表 (60 内置技能)
11. 初始化 Web 工具上下文
12. 恢复持久化代理
13. 发布 KernelStarted 事件
14. 返回内核实例
```

**理由**:

**理由**:

- ✅ 原子事务
- ✅ 向量搜索
- ✅ 全文检索

### ADR-003: 16 层安全架构

**决策**: 深度防御，无单点故障
