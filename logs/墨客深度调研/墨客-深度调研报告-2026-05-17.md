**Bug #82751 — Telegram Polling 无限循环**:
```markdown
### 🔴 Bug #82751 — Telegram Polling 无限循环

**问题**：Telegram 频道在某些情况下进入无限循环，导致消息延迟 3-6 分钟。

**影响**：用户消息延迟，Agent 无法实时响应。

**根因**：v2026.5.3+ 引入的 regression，消息队列处理异常。

**Workaround**：
1. **降级版本**：使用 v2026.5.3-1 或 v2026.4.29
2. **重启 Gateway**：
```bash
openclaw gateway restart
```
3. **检查队列配置**：
```json
{
  "messages": {
    "queue": {
      "mode": "steer"
    }
  }
}
```

**状态**：🟠 OPEN | [Issue #82751](https://github.com/openclaw/openclaw/issues/82751)
```

**Bug #82722 — Heartbeat 打断 Streaming**:
```markdown
### 🔴 Bug #82722 — Heartbeat 打断 Streaming

**问题**：Heartbeat 消息打断 streaming 响应，导致输出不完整或截断。

**影响**：长文本输出被截断，Agent 回复不完整。

**根因**：Heartbeat 与 streaming 响应共享同一通道，优先级处理不当。

**Workaround**：
1. **关闭 Heartbeat**：在配置中暂时禁用
```json
{
  "gateway": {
    "heartbeat": {
      "enabled": false
    }
  }
}
```
2. **使用 Webhook 模式**：切换到 webhook delivery 模式

**状态**：🟠 OPEN | [Issue #82722](https://github.com/openclaw/openclaw/issues/82722)
```
