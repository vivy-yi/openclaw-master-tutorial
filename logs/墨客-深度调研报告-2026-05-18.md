### 🔴 Bug #81934 — 升级后部分功能失效

**问题**：从旧版本升级到 v2026.5.12+ 后，部分 Skills 和 Plugins 无法正常加载。

openclaw plugins list

2. **重新安装依赖**（如使用这些渠道）：
```bash
openclaw update --include-optional
```
3. **降级**（如不影响业务）：
```bash
openclaw update --to v2026.5.11
```

**Bug #81935 — 成本异常（7x 增长）**:
```markdown
### 🔴 Bug #81935 — 成本异常（7x 增长）

**问题**：v2026.5.12+ 版本中，部分对话产生异常高昂的 Token 消耗，增长约 7 倍。

**影响**：用户成本大幅增加，可能触发预算警报。

**根因**：疑似 streaming 模式下 Heartbeat 重复发送相同内容导致。

**Workaround**：
1. **关闭 Heartbeat**：
```json
{
  "gateway": {
    "heartbeat": {
      "enabled": false
    }
  }
}
```
2. **监控成本**：
```bash
openclaw gateway stats --cost
```
3. **回滚版本**：如成本问题严重，回滚至 v2026.5.11

**状态**：🟠 OPEN | [Issue #81935](https://github.com/openclaw/openclaw/issues/81935)
```

**Bug #80420 — Cron Idle Timeout 回归**:
```markdown
### 🟠 Bug #80420 — Cron Idle Timeout 回归问题

**问题**：Cron 任务在空闲超时时提前终止，长时间运行的任务被意外中断。

**影响**：需要长时间运行的自动化任务（如批量处理、数据同步）可能在中途失败。

**根因**：v2026.5.14 Session 管理器引入 regression。

**Workaround**：
1. 使用 `cron run --wait` 模式（v2026.5.16+）：
```bash
openclaw cron run <cron-id> --wait
```
2. 增加 idle timeout 配置：
```json
{
  "cron": {
    "idleTimeout": "24h"
  }
}
```

**状态**：🟡 修复中 | [Issue #80420](https://github.com/openclaw/openclaw/issues/80420)
```

## 9.7.7.1 cron run --wait 新用法 (v2026.5.16+)
